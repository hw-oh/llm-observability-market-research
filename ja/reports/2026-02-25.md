# 週刊 LLM Observability 市場調査レポート
**日付**: 2026-02-25 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-25

## 1. エグゼクティブ・サマリー

- W&B Weave は、ネイティブのオーディオモニターや動的なリーダーボードをリリースすることで機能のギャップを急速に埋め、トップティアのマルチモーダルプラットフォームとしての地位を確立しました。
- LangSmith は、エージェント形式の Observability において支配的な地位を維持しており、最近では LangGraph ユーザー向けに堅牢な Sandbox とカスタマイズ可能な Trace プレビュー機能を強化しました。
- Langfuse は、オープンソースのリーダーとして差別化を図っており、複雑な Trace 内で詳細な品質チェックを可能にする「Single Span Evals」を導入しました。
- Braintrust は、自動化された Scoring 作成のための「Loop」AI アシスタントや、セキュリティのための新しい AI Proxy を提供し、開発ライフサイクルを積極的にターゲットにしています。
- MLflow は、マルチワークスペースの Organization Support を追加し、プロンプト最適化のために DSPy と深く統合することで、エンタープライズでの有用性を固めました。
- Arize Phoenix は、埋め込み空間（UMAP）を可視化する独自の機能を維持しており、最近ではネイティブの Model Context Protocol (MCP) 統合を追加しました。

> **市場の洞察**: Weave は、軽量な Tracing ツールからトップティアのマルチモーダルプラットフォームへと急速に進化しており、W&B のトレーニングエコシステムを活用して、既存の特化型プラットフォームに挑んでいます。


## 2. 新機能 (直近30日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Trace analytics overviews**: リクエスト数、Latency のパーセンタイル、トークン使用量、コストを表示するプロジェクト概要機能。(2026-02-23, Analytics)
- **Trace comparison summaries**: 集約されたツール使用状況、Scoring、コストを伴う Trace 比較のためのフラット化されたビュー。(2026-02-23, Evaluation)
- **Audio monitors**: LLM ジャッジを使用してオーディオ入力を監視・判定するモニター作成のサポート。(2026-02-01, Evaluation)
- **Dynamic leaderboards**: 永続的なカスタマイズと CSV エクスポート機能を備えた、Eval から自動生成されるリーダーボード。(2026-01-29, Evaluation)

### [LangSmith](https://changelog.langchain.com)
- **Sandbox Exception Types & Plumbing**: エージェント Sandbox におけるエラーハンドリングを改善するため、Sandbox 例外タイプとクライアント配管を追加。(2026-02-21, Development Lifecycle)
- **Google Gen AI Wrapper Export**: Google Gen AI ラッパーのエクスポート機能と、非 otel ラッパーのサポートを追加。(2026-02-02, Integration & DX)
- **Customize Trace Previews**: LangSmith UI 内での Trace プレビューの表示方法をカスタマイズする機能。(2026-02-06, Core Tracing & Logging)

### [Langfuse](https://langfuse.com/changelog)
- **Single Span Evals**: 個別の Span に対して Eval を実行する機能（Beta）を導入し、品質チェックの粒度を向上。(2026-02-15, Evaluation & Quality)
- **LLM-as-a-Judge on Observations**: 特定の Observation をターゲットにした LLM-as-a-judge 機能を拡張し、より的を絞った自動フィードバックを可能に。(2026-02-10, Evaluation & Quality)
- **Events-based Trace Table**: パフォーマンスとフィルタリングを改善するため、イベントベースのアーキテクチャを利用するように Trace/Observation テーブルを最適化。(2026-02-05, Analytics & Dashboard)
- **Bloom Filter Indexes**: 大規模データセットでの検索を大幅に高速化するため、user_id および session_id クエリにブルームフィルタインデックスを追加。(2026-02-20, Infrastructure)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Public Span Name Property**: Trace の識別を容易にするため、Python SDK の Span インターフェースにパブリックな名前プロパティを追加。(2026-02-12, Integration & DX)
- **Python Thread Retrieval**: Python SDK 内でスレッドコンテキストを直接取得する新機能。(2026-02-12, Agent & RAG Specifics)
- **Classifications Field**: より豊かなデータラベリングを可能にするため、Python SDK で classifications フィールドのサポートを導入。(2026-01-31, Core Tracing & Logging)
- **Eval Cache Control**: 最新の結果を保証するため、Eval 中にキャッシュを明示的にオフにするオプションを追加。(2026-01-29, Evaluation & Quality)
- **Experiment Tags**: 整理を容易にするため、実験作成時にタグを渡すことが可能に。(2026-02-25, Development Lifecycle)

### [MLflow](https://mlflow.org/releases)
- **Organization Support in MLflow Tracking Server**: 実験とモデルの論理的な分離と整理を可能にする、マルチワークスペース環境をサポート。(2026-02-20, Enterprise & Infrastructure)
- **MLflow Assistant**: UI 内で直接問題を特定、診断、修正するための、Claude Code をバックエンドとした製品内チャットボット。(2026-01-29, Development Lifecycle)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Conciseness Classification Evaluator**: LLM 出力の簡潔さを評価するための新しいエバリュエーターを追加。(2026-02-20, Evaluation & Quality)
- **AWS Bedrock Cross-region Preference**: AWS Bedrock のクロスリージョン推論において、モデルプレフィックスの優先順位を設定する構成オプション。(2026-02-19, Integration & DX)
- **Model to Evaluator Details**: エバリュエーターの詳細ビューにモデル情報を直接追加し、可視性を向上。(2026-02-18, Evaluation & Quality)
- **Autocomplete in LLM Eval Prompt Editor**: Eval 設定を容易にするため、プロンプトエディタにオートコンプリート機能を追加。(2026-02-13, Evaluation & Quality)
- **Tool Response Handling Evaluator**: モデルがツールのレスポンスをどのように処理するかを評価するための新しいテンプレート。(2026-02-13, Agent & RAG Specifics)

## 4. エンタープライズ・シグナル

- MLflow は v3.10 で Organization Support を導入し、大規模なエンタープライズ導入に不可欠なマルチワークスペースの論理的隔離を可能にしました。
- Braintrust は、モデル呼び出しの上流でセキュリティ、キャッシュ、計測を処理するための専用 AI Proxy を発表しました。
- W&B Weave は、コアとなるトレーニングプラットフォームのコンプライアンス基準を反映し、エンタープライズグレードの Audit Logs と RBAC を追加しました。
- LangSmith は、本番環境でのエージェントコードの信頼性の高い実行をサポートするため、新しい例外タイプで Sandbox 環境を強化しました。
- Langfuse は、大量のエンタープライズ Trace データに対するクエリパフォーマンスを大幅に最適化するため、Bloom Filter Indexes を実装しました。
