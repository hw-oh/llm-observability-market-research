# 週刊 LLM Observability 市場調査レポート
**日付**: 2026-02-23 | **モデル**: google/gemini-3-pro-preview | **データ収集日**: 2026-02-23

## 1. エグゼクティブ・サマリー

- MLflow は、マルチワークスペース環境とリソースの分離を可能にする Organization Support (v2026-02-20) をリリースしました。
- Langfuse は、ガバナンス強化のため、シングルスパンの判定機能と Organization Audit Log Viewer (v2026-02-06) を導入しました。
- Weave は、音声エージェント用の新しいオーディオモニターと、Model Context Protocol (MCP) とのネイティブ統合をデプロイしました。
- Braintrust は、ハイブリッドデプロイモデルを維持しつつ、OpenAI Agents のサポートを Eval プラットフォームに追加しました。
- LangSmith は、複雑なエージェント実行グラフを可視化するために、フルセッションのリプレイと階層的な実行ツリーを提供しています。
- Arize Phoenix は、ネイティブの OpenTelemetry サポートを通じて、検索チャンクとエージェントの実行パスに特化した可視化を提供しています。

> **市場の洞察**: Weave は新しいオーディオモニターによってマルチモーダルな Tracing でリードを保っていますが、Langfuse と MLflow によるきめ細かなガバナンス機能のリリースは、エンタープライズ管理機能における競争圧力の高まりを示唆しています。


## 2. 新機能 (直近30日間)

### [Weave](https://app.getbeamer.com/wandb/en)
- **オーディオモニター**: テキストと並行してオーディオ出力を監視・判定するモニターの作成をサポートし、音声エージェントの Eval を可能にしました。(2026-02-01, Guardrails & Safety)
- **ダイナミック・リーダーボード**: 手動設定に代わり、フィルタリングやカスタマイズが可能な Eval 結果からのリーダーボード自動生成機能。(2026-01-29, Evaluation & Quality)

### [LangSmith](https://changelog.langchain.com)
- **Trace プレビューのカスタマイズ**: LangSmith UI で Trace プレビューをカスタマイズできる機能。(2026-02-06, Core Tracing & Logging)
- **Python async 用サンドボックスエンドポイント**: Python の非同期実行用のサンドボックスエンドポイントサポートを追加。(2026-02-05, Development Lifecycle)
- **非 OTEL Google ADK ラッパー**: Google ADK 用の非 OpenTelemetry ラッパーを追加。(2026-02-02, Integration & DX)
- **Google Gen AI ラッパーのエクスポート**: SDK で Google Gen AI ラッパーをエクスポート。(2026-01-31, Integration & DX)

### [Langfuse](https://langfuse.com/changelog)
- **シングルスパン Eval (オープンベータ)**: フル Trace だけでなく、個別のスパンに対して Eval を実行可能に。(2026-02-23, Evaluation & Quality)
- **Observation に対する LLM-as-a-Judge**: 特定の Observation に対して直接 LLM-as-a-judge エバリュエーターを実行する機能。(2026-02-23, Evaluation & Quality)
- **Reasoning/Thinking Trace のレンダリング**: 推論モデルなどのモデル出力における「思考 (thinking)」や推論部分をレンダリングするための UI サポート。(2026-02-06, Core Tracing & Logging)
- **Org Audit Log Viewer**: 組織レベルの監査ログを表示するための新しい UI。(2026-02-06, Enterprise & Infrastructure)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **実験タグ**: SDK を介して実験作成時にタグを渡すことが可能に。(2026-02-23, Development Lifecycle)
- **スレッド取得**: Python SDK でスレッドの詳細を取得する機能を追加。(2026-02-12, Core Tracing & Logging)
- **OpenAI Agents 統合**: OpenAI エージェント統合のために、すべてのスパンタイプを処理できるよう Python SDK を更新。(2026-02-05, Integration & DX)
- **Review スパンタイプ**: アノテーションワークフロー用と思われる、特定の「review」スパンタイプを SDK に追加。(2026-02-05, Core Tracing & Logging)
- **Classifications フィールド**: Python SDK に classifications フィールドを追加。(2026-01-31, Core Tracing & Logging)
- **Eval キャッシュ制御**: Eval 中にキャッシュをオフにするオプションを追加。(2026-01-29, Evaluation & Quality)

### [MLflow](https://mlflow.org/releases)
- **Organization Support**: 実験とリソースを論理的に分離するためのマルチワークスペース環境のサポート。(2026-02-20, Enterprise & Infrastructure)
- **MLflow Assistant**: UI 上で直接問題を診断・修正するのを支援する、Claude Code をベースとした製品内チャットボット。(2026-01-29, Integration & DX)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **簡潔性分類エバリュエーター**: 回答の簡潔さを測定するための新しいエバリュエーターを追加。(2026-02-20, Evaluation & Quality)
- **AWS Bedrock クロスリージョンサポート**: AWS Bedrock のクロスリージョン推論モデルプレフィックスの優先設定サポートを追加。(2026-02-19, Integration & DX)
- **Eval プロンプトのオートコンプリート**: LLM Eval プロンプトエディタにオートコンプリート機能を追加。(2026-02-13, Evaluation & Quality)
- **ツールレスポンス・エバリュエーター・テンプレート**: ツールレスポンスの処理に特化した新しいエバリュエーターテンプレートを導入。(2026-02-13, Evaluation & Quality)

## 4. エンタープライズ・シグナル

- MLflow は、リソース分離を改善するためのマルチワークスペース環境を可能にする Organization Support (v2026-02-20) を導入しました。
- Langfuse は、ガバナンスとコンプライアンスの可視性を高めるために Organization Audit Log Viewer (v2026-02-06) をリリースしました。
- Braintrust は、規制の厳しい業界向けに厳格なデータ主権を提供するため、ハイブリッドデプロイモデルを引き続き活用しています。
- Weave は、親プラットフォームから継承した SOC 2 Type II、HIPAA、および GDPR への準拠により、強力なエンタープライズ姿勢を維持しています。
