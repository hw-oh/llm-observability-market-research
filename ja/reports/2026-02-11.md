# W&B Weave — 週刊競合インテリジェンスレポート

## 1. エグゼクティブ・サマリー

- LangSmith は「LangGraph Cloud」によって、受動的なオブザーバビリティから能動的なランタイム管理へと積極的に移行しており、Weave がデータを取得する前にエージェントのライフサイクル全体を囲い込もうとしています。
- Braintrust は、IDE（Cursor）への直接統合や、現在 Weave が対応していないバックエンド言語（Java、Go、C#）のサポートにより、「エンタープライズ・エンジニアリング」の側面から効果的に側面攻撃を仕掛けています。
- Humanloop の買収と閉鎖は、行き場を失った顧客を獲得するための即時的な戦術的機会を生み出していますが、Langfuse の優れた Prompt CMS は、この移行における強力な競合となっています。
- Arize Phoenix と Langfuse は、強力な「ローカルファースト」およびセルフホスティングのストーリーでボトムアップの採用プロセスを圧迫しており、SaaS の導入を躊躇するセキュリティ意識の高いエンジニアに訴求しています。
- Logfire の Pydantic との深い統合により、「フルスタック」のトレース（データベース + API + LLM）を所有することが可能になり、LLM 以外のインフラコンポーネントに対する Weave の可視性の欠如を露呈させています。
- エージェントのオブザーバビリティが主要な戦場になりつつあります。LangSmith と Langfuse は専用のグラフ/推論ビジュアライゼーションをリリースしており、循環ワークフローにおいては現在の Weave の標準的なトレースビューを凌駕しています。

> **一行判定**: Weave はモデルのトレーニングから本番環境へのリネージ（系統管理）において優位性を維持していますが、エージェントのオーケストレーションでは LangSmith から、エンタープライズ開発者のワークフローでは Braintrust からの激しい圧力に直面しています。

### Weave の主な強み

- W&B Model Registry およびトレーニングパイプラインとの独占的な統合により、競合が模倣できない独自の「データフライホイール」を構築しています。
- Helicone や Logfire の硬直的な UI ベースのワークフローと比較して、プログラムによる評価において優れた柔軟性を備えています。
- 複数のステークホルダーによるレビューのための洗練されたコラボレーション UI は、実用本位な Arize Phoenix や Langfuse のインターフェースを凌駕しています。

### Weave の改善点

- Langfuse や旧 Humanloop の製品と比較して、成熟した非技術者向けの「Prompt CMS」およびプレイグラウンドが不足しています。
- SDK エコシステムが限定的（Python/JS のみ）であるため、エンタープライズのバックエンドチーム（Java/Go/C#）が Braintrust に流れるリスクがあります。
- 深層インフラ（データベースクエリ、API 内部）をトレースできないため、Logfire や Arize のような OTel ネイティブなツールに対して不利な状況にあります。

## 3. 今週の新機能

### LangSmith
- **トレースプレビューのカスタマイズ**: トレースの入力および出力のどの部分をトレーステーブルに直接表示するかをユーザーが設定できるようになり、カスタムデータ構造の使い勝手が向上しました。(2026-02-06, DevEx / 統合)
- **LangSmith セルフホスト v0.13**: セルフホスト顧客向けのメジャーリリース。「Insights」やパフォーマンス向上など、クラウド版との機能パリティ（同等性）を拡大。(2026-01-16, エンタープライズ & セキュリティ)
- **SDK v0.7.1**: Python/JS クライアントライブラリのアップデート。最新の LangChain バージョンとの互換性確保と安定性の向上。(2026-02-10, DevEx / 統合)

### Arize Phoenix
- **Claude Opus 4.6 サポート**: プレイグラウンドで Claude Opus 4.6 をサポートし、並行比較が可能になりました。(2026-02-09, 実験 / 改善ループ)
- **ツール選択エバリュエーター**: エージェントのツール選択ロジックの精度を測定するために特別に設計された新しいエバリュエーター。(2026-02-06, エージェント / RAG オブザーバビリティ)
- **FaithfulnessEvaluator**: よりきめ細かな RAG チェックのための新しい FaithfulnessEvaluator の導入（HallucinationEvaluator は非推奨）。(2026-02-02, 評価統合)
- **ツール呼び出し精度メトリクス**: エージェントが正しくツールを呼び出した頻度を追跡するための特定のメトリクスを追加。(2026-01-27, モニタリング & メトリクス)

### Braintrust
- **トレースの起点へのナビゲート**: トレースを、それを生成した特定のプロンプトバージョンやデータセットの行にリンクする機能。(2026-02, コア・オブザーバビリティ)
- **トレースレベルのスコーラー**: カスタムスコーラーが実行トレース全体にアクセスし、マルチステップのワークフローやエージェントの動作を評価できるようになりました。(2026-02, 評価統合)
- **LangSmith 統合**: LangSmith のトレースおよび評価コールを Braintrust に並行して、あるいは排他的にルーティングするためのラッパー。(2026-02, DevEx / 統合)
- **Cursor 統合**: 自然言語を介してログを照会し、実験結果を取得するための Cursor エディタ用 Braintrust 拡張機能。(2026-02, DevEx / 統合)
- **自動インストルメンテーション (Python/Ruby/Go)**: 主要言語のゼロコードトレースサポートにより、初期設定を簡素化。(2026-01, DevEx / 統合)

### Langfuse
- **トレースの修正済み出力**: ファインチューニング用データセットを構築するために、LLM 出力の改善されたバージョンをトレースビューで直接キャプチャ。(2026-01-14, 実験 / 改善ループ)
- **インラインコメント**: トレースの入力および出力内の特定のテキスト選択箇所にコメントを固定。(2026-01-07, DevEx / 統合)

### Logfire
- **マルチトークンプロジェクトの移行**: 複数のトークンを使用しているプロジェクトの移行をサポート。(2026-02-04, エンタープライズ & セキュリティ)
- **OTel Gen AI セマンティックコンベンション**: OpenTelemetry Gen AI セマンティックコンベンションのスカラー属性のサポートを追加。(2026-01-28, コア・オブザーバビリティ)
- **Pytest 統合**: pytest 実行をトレースするためのネイティブ統合。(2026-01-26, DevEx / 統合)
- **DSPy 統合**: DSPy フレームワークのインストルメンテーションサポートを追加。(2026-01-16, DevEx / 統合)
- **Claude SDK インストルメンテーション**: Anthropic Claude SDK 専用のインストルメンテーションを追加。(2026-01-12, DevEx / 統合)

## 4. ポジショニングの変化

| ベンダー | 現在 | 今後の方向性 | シグナル |
|---|---|---|---|
| **Weave** | LLM アプリケーションを構築する ML 中心のチーム向けの、最高の評価およびオブザーバビリティプラットフォーム。 | ランタイムアプリケーションをモデルトレーニングループに接続する、汎用 AI エンジニアリングプラットフォームへの拡大。 | 競合がデプロイメント（LangSmith）という「右」方向へ動く中、Weave はトレーニングデータという「左」への接続を活用する必要がある。 |
| LangSmith | オブザーバビリティ、評価、プロンプトエンジニアリングを単一プラットフォームに統合した、LangChain エコシステムのデフォルト「OS」。 | エージェントのライフサイクル全体を所有するため、デプロイメントおよび能動的なランタイム管理レイヤー（LangGraph Cloud）へ拡大。 | 「LangSmith Deployment」と「Polly」AI アシスタントのリリースは、受動的な監視から能動的な管理へのシフトを示唆。 |
| Arize Phoenix | ローカルファーストのデバッグと深い埋め込み分析を求めるエンジニアにとって、主要なオープンソースかつ OTLP ネイティブな選択肢。 | 純粋なオブザーバビリティから、LangSmith と直接競合する完全なエージェント評価・テストプラットフォームへの拡大。 | 2026年2月におけるエージェント専用エバリュエーター（ツール選択、忠実度）の迅速なリリースとプレイグラウンドの強化。 |
| Braintrust | 評価、プロキシ、オブザーバビリティを統合する「エンタープライズ AI エンジニアリングプラットフォーム」。 | IDE（Cursor）への深い統合と言語サポート（Java/Go/C#）の拡大により、エンタープライズスタック全体を捕捉する AI 開発の中心的なインフラレイヤーを目指す。 | エンタープライズ言語（Java、C#）用のネイティブ SDK の最近のリリースと、アーキテクチャにおける AI プロキシの中心的な役割。 |
| Langfuse | セルフホスティングとベンダーロックインの回避を重視するエンジニアリングチーム向けの、LangSmith に代わる主要なオープンソース。 | より深いエージェントサポートとエンタープライズコンプライアンス機能を備えた、完全な「AI エンジニアリングプラットフォーム」への拡大。 | 「エージェントグラフ」、「監査ログ」、大規模対応のための ClickHouse 統合の最近の追加は、アップマーケットへの進出を示唆。 |
| Humanloop | 廃止 / Anthropic により買収（2025年9月終了） | Anthropic の内部ツールに技術を吸収。 | 公式ドキュメントにより、2025年9月8日のプラットフォーム終了が確認されている。 |
| Logfire | Pydantic の普及を活用して、LLM 呼び出しだけでなくアプリケーションスタック全体（DB、API、LLM）を捕捉する「AI エンジニア向け APM」。 | 開発者優先、コード中心のフォーカスを維持しつつ、より広範なフレームワークサポート（DSPy、Pytest）とエンタープライズ対応（EU リージョン、新価格体系）へ移行。 | 最近のリリースは、評価機能よりも統合（DSPy、Pytest、Claude）と OTel 準拠に重点を置いている。 |
| Helicone | トラフィック管理の機能としてオブザーバビリティをバンドルした、主要なオープンソース「AI ゲートウェイ」。 | 純粋なプロキシ/ロギングから、プロンプト管理や評価などのより広範な開発者ワークフローへの拡大。 | ドキュメントのナビゲーションに「プロンプト管理」、「実験」、「ベクトル DB」統合が最近追加された。 |

## 5. エンタープライズ・シグナル

- LangSmith セルフホスト v0.13 がクラウド版と機能的に同等になったことで、規制の厳しい業界における主要な導入障壁が取り除かれました。
- Braintrust による Java および C# SDK のリリースは、伝統的なエンタープライズ（銀行/保険）の技術スタックに対する直接的な攻撃を意味します。
- Langfuse による ClickHouse の採用と監査ログの導入は、同社が高ボリュームのエンタープライズアカウントをターゲットにアップマーケットへ移動していることを示しています。

## 6. ウォッチリスト

- LangGraph Cloud を介した LangSmith の「デプロイメント」および能動的なランタイム管理への拡大。
- CI/CD ステージに到達する前の開発者を捕捉するための、Braintrust による IDE（Cursor/MCP）との統合深化。
- 閉鎖された Humanloop プラットフォームからの顧客の移行パターン。特に、Weave の評価の深さと Langfuse のプロンプト管理のどちらが優先されるか。
