## 1. エグゼクティブサマリー

- Weave は、Audio Monitors（2026年2月）のリリースにより、マルチモーダル・オブザーバビリティにおける先行者利益を確立しました。これにより、テキスト中心の競合である LangSmith や Langfuse との差別化を図っています。
- LangSmith は、複雑なエージェント・ワークフローにおける主要な脅威であり続けています。Weave の現在のトレースツリー表示ではまだ対応できない、循環グラフやステートマシンの優れた可視化を提供しています。
- Braintrust は、「データプレーン」アーキテクチャと幅広い SDK サポート（Java, Go, C#）を武器に、エンタープライズ側からの切り崩しに成功しており、Weave の Python/TS 中心の SDK ではカバーしきれないバックエンドエンジニアリングチームを獲得しています。
- 「ヒューマン・イン・ザ・ループ（Human-in-the-Loop）」の格差が広がっています。LangSmith、Langfuse、Braintrust はいずれも専用の「アノテーションキュー」やカンバンワークフローをリリースしていますが、Weave はレビューにおいて、より汎用的なボード/テーブルインターフェースに依存しています。
- MLflow (v3.9) は、「Judge Builder」と「MemAlign」によって評価機能の差を猛烈に縮めており、Databricks ネイティブなチームにおける Weave の評価主導型開発ループの優位性を脅かしています。
- Arize Phoenix による特化型「ツール選択エバリュエーター」のリリースは、市場が粒度の細かいエージェントコンポーネントテストへと移行していることを示唆しています。Weave はエージェントの信頼性において同等性を維持するために、この領域を優先すべきです。

> **一言判定**: Weave はマルチモーダル対応とトレーニングから推論への連続性においてリードしていますが、エージェント・ワークフローの可視化では LangSmith から、エンタープライズデータプライバシーアーキテクチャでは Braintrust から大きな圧力を受けています。

### Weave の主な強み

- マルチモーダル・オブザーバビリティ: Audio Monitors と再生機能のネイティブサポートにより、音声エージェント開発においてテキストのみの競合他社をリードしています。
- トレーニング統合: 本番環境のトレースを W&B Model Registry やファインチューニングジョブ（Serverless LoRAs）に直接リンクできる比類なき能力。
- ダイナミック・リーダーボード: 新しく自動生成される評価リーダーボードにより、MLflow で必要な手動設定なしに、即座にモデル比較が可能です。
- データ探索: エクスプレッション（式）ベースのボードシステムは、Langfuse や Arize の固定されたダッシュボードよりも柔軟なアドホック分析を提供します。

### Weave の改善領域

- 人手によるアノテーションワークフロー: 体系的な人手レビューのための専用の「アノテーションキュー」やカンバンインターフェースが不足しています。これは LangSmith や Langfuse では現在標準的な機能です。
- エージェントグラフの可視化: LangGraph エージェントにおける循環依存やステートマシンを、LangSmith ほど効果的に可視化できません。
- SDK 言語サポート: Python/TypeScript に限定されており、エンタープライズの Java/C#/Go チームを Braintrust や OpenTelemetry ベースのライバルに奪われています。
- プライバシーアーキテクチャ: SaaS UI を使用しながらトレースデータを完全に顧客のクラウド内に保持できる「データプレーン」相当の機能（Braintrust）が欠けています。

## 3. 今週の新機能

### [Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: 音声対応 LLM ジャッジを使用して、テキストと並行して音声出力を監視・判定するモニター作成のサポート。（2026-02-01, コア・オブザーバビリティ）
- **Dynamic Leaderboards**: モデル/評価フィルターに基づいて即座にデータが入力される、評価機能内の自動生成リーダーボード。（2026-01-29, 評価統合）
- **Custom LoRAs in Playground**: W&B Artifacts からカスタムファインチューニングされた LoRA 重みを直接 Weave Playground にロードしてテストする機能。（2026-01-16, 実験 / 改善ループ）

### [LangSmith](https://changelog.langchain.com)
- **Customize Trace Previews**: リストビューでのトレースデータのプレビュー方法をユーザーが設定できる UI アップデート。（2026-02-06, DevEx / 統合）
- **SDK v0.7.1**: LangSmith オブザーバビリティおよび評価プラットフォームに接続するためのクライアントライブラリのアップデート。（2026-02-10, DevEx / 統合）

### [Langfuse](https://langfuse.com/changelog)
- **Corrected Outputs for Traces**: ファインチューニング用データセットを構築するために、トレースビューで LLM 出力の改善版を直接キャプチャ。（2026-01-14, 実験 / 改善ループ）
- **Inline Comments on Observation I/O**: トレースの入力および出力内の特定のテキスト選択箇所にコメントを固定。（2026-01-07, DevEx / 統合）
- **Reasoning/Thinking Rendering**: トレース内での推論モデル（例：O1, R1）の「思考（thinking）」部分に特化した UI レンダリング。（2026-02-01, コア・オブザーバビリティ）
- **Org Audit Log Viewer**: セキュリティとコンプライアンスのための組織レベルの監査ログを表示する新しい UI。（2026-02-01, エンタープライズ & セキュリティ）
- **Single Observation Evals**: 完全なトレースではなく、個別のオブザベーションに対して評価を実行する機能。（2026-02-01, 評価統合）

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Trace-level Scorers**: カスタムコードスコーラーが実行トレース全体にアクセスし、マルチステップのワークフローやエージェントの動作を評価可能に。（2026-02, 評価統合）
- **LangSmith Integration**: トレースを LangSmith と Braintrust の両方にルーティング、またはトラフィックを完全に移行するためのラッパー。（2026-02, DevEx / 統合）
- **Cursor Integration**: Braintrust MCP サーバーを設定する拡張機能。Cursor IDE から直接ログのクエリや実験の取得が可能に。（2026-02, DevEx / 統合）
- **Auto-instrumentation (Python/Ruby/Go)**: Python、Ruby、Go SDK 向けのゼロコード・トレースサポートを追加。（2026-01, DevEx / 統合）
- **Temporal Integration**: Temporal のワークフローとアクティビティの自動トレース。ワーカー間の分散トレースをキャプチャ。（2026-01, エージェント / RAG 観測）

### [MLflow](https://mlflow.org/releases)
- **MLflow Assistant**: UI 内で問題を診断し修正を提案する、Claude Code 搭載の製品内チャットボット。（2026-01-29, DevEx / 統合）
- **Agent Performance Dashboards**: エージェントのレイテンシ、リクエスト数、品質スコアを監視するための構築済みチャート。（2026-01-29, モニタリング & メトリクス）
- **MemAlign Judge Optimizer**: 過去のフィードバックから評価ガイドラインを学習し、ジャッジの精度を向上させるアルゴリズム。（2026-01-29, 評価統合）
- **Judge Builder UI**: コードなしでカスタム LLM ジャッジプロンプトを作成、テスト、検証するためのビジュアルインターフェース。（2026-01-29, 評価統合）
- **Continuous Online Monitoring**: 入力される本番トレースに対して LLM ジャッジを自動実行し、品質問題をリアルタイムで検出。（2026-01-29, 評価統合）

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 Support**: Playground において Anthropic の Claude Opus 4.6 モデルのサポートを追加。正確なコスト追跡が可能。（2026-02-09, 実験 / 改善ループ）
- **Tool Selection & Invocation Evaluators**: エージェントが正しいツールを選択し、有効なパラメータで呼び出したかを評価する新しい特化型エバリュエーター。（2026-01-31, エージェント / RAG 観測）
- **Phoenix CLI Expansion**: ターミナルから直接プロンプト、データセット、実験を管理するための包括的な CLI コマンド。（2026-01-22, DevEx / 統合）
- **Trace-to-Dataset with Span Links**: リネージ（系統）のためにソーススパンへの双方向リンクを維持したまま、トレースからデータセットを作成する機能。（2026-01-21, 評価統合）
- **Export Annotations with Traces**: オフライン分析のために、トレースと共に人間および AI のアノテーションをエクスポートする CLI サポート。（2026-01-19, 評価統合）

## 4. ポジショニングの変遷

| ベンダー | 現在 | 今後の方向性 | シグナル |
|---|---|---|---|
| **Weave** | LLM を継続的な洗練が必要なソフトウェアコンポーネントとして扱うチーム向けの、最高の「モデル中心」オブザーバビリティプラットフォーム。 | テキストだけでなく、音声やリッチメディアの評価も処理するマルチモーダルな「GenAI オペレーティングシステム」。 | 2026年2月1日の Audio Monitors のリリースと、Playground へのカスタム LoRA の統合。 |
| LangSmith | LangChain および LangGraph で構築するチーム向けの、デフォルトの統合オブザーバビリティスタック。 | デバッグツールから、デプロイ、アノテーション、オンライン評価を含む完全な「Agent Ops」プラットフォームへの拡大。 | 「LangSmith Deployment」（旧 LangGraph Cloud）や「アノテーションキュー」の最近のリリースは、フルライフサイクル管理への移行を示唆。 |
| Langfuse | セルフホストとコスト管理を優先するチームに好まれる、LLM エンジニアリング向けの主要なオープンソースの選択肢。 | より深いデータセット管理とヒューマン・イン・ザ・ループ・ワークフローを備えた、フルサイクルの「LLM エンジニアリングプラットフォーム」への拡大。 | 最近のリリースは、受動的な監視だけでなく、「アノテーションキュー」、「修正済み出力」、「データセットのバージョニング」に重点を置いている。 |
| Braintrust | データプレーンアーキテクチャによるデータプライバシーと、プロンプト作成と評価の緊密なループを強調する、AI エンジニア向けの「エンタープライズグレード」スタック。 | 外部ツール（LangSmith, Cursor, Temporal）との統合と、エンジニアリング市場全体を取り込むための言語サポートの拡大により、ユニバーサルな開発者ハブを目指す。 | LangSmith ラッパーや Cursor IDE 統合の最近のリリースは、共存と開発者ワークフローの支配戦略を示唆。 |
| MLflow | Databricks との統合により、GenAI 領域へ支配力を拡大しているデフォルトの MLOps プラットフォーム。 | 従来の ML と GenAI の評価/オブザーバビリティを統合する、完全な「AI オペレーティングシステム」。 | エージェント、評価、オブザーバビリティを網羅する「オールインワン」のメッセージを掲げた v3.9 のリリース。 |
| Arize Phoenix | 独自の SaaS エコシステムよりも OpenTelemetry 標準とセルフホストを優先するエンジニアリングチーム向けの、主要なオープンソースかつローカルファーストの選択肢。 | エンタープライズニーズにスケールする前に技術ユーザーを囲い込むため、エージェント・ワークフロー（ツール評価）と開発者の生産性（CLI ツール）へ積極的に拡大。 | 2026年1月の CLI ツールと特定のエージェントツールエバリュエーターの急速なリリースは、複雑なエージェントシステムを構築する「パワーユーザー」開発者への注力を示唆。 |

## 5. エンタープライズ・シグナル

- Braintrust の「データプレーン」アーキテクチャは、UI とデータストレージを分離することで、セキュリティ審査において優位に立っています。
- LangSmith の「Deployment」（インフラ管理）への拡大は、完全なランタイム環境になることを目指しており、ベンダーロックインを強めています。
- MLflow の「MemAlign」機能は、LLM ジャッジを人間のポリシーガイドラインに自動的に合わせることで、エンタープライズガバナンスをターゲットにしています。
- Langfuse が「組織監査ログ」（2026年2月）に注力していることは、オープンソースの代替手段においてもコンプライアンス機能の需要が高まっていることを浮き彫りにしています。

## 6. ウォッチリスト

- LangSmith の「Deployment」の採用状況を監視すること。彼らがランタイムを制御すれば、オブザーバビリティ層はコモディティ化されます。
- Braintrust の IDE 統合（Cursor）に注目すること。開発者のエディタを所有することは、強力な上流の楔（くさび）となります。
- Arize Phoenix の「ツール選択エバリュエーター」の採用を追跡すること。これがエージェントの信頼性メトリクスの新しい標準になる可能性があります。
- Weave の Audio Monitors の影響を評価すること。競合他社は第2四半期にマルチモーダルサポートの追加を急ぐでしょうか？
