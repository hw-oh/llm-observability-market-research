# W&B Weave — 週刊競合インテリジェンスレポート
**日付**: 2026-02-11 | **モデル**: google/gemini-3-pro-preview | **データ収集日**: 2026-02-11

[詳細比較](../comparison) · [製品詳細](../competitor-detail)

## 1. エグゼクティブ・サマリー

- Weave は2月1日の Audio Monitors のリリースにより、マルチモーダル・オブザーバビリティにおける先行者利益を確立しました。これにより、急成長するボイスエージェント分野において、テキスト中心の競合である LangSmith や MLflow を引き離しています。
- LangSmith は、LangGraph Cloud を通じて純粋なオブザーバビリティからインフラのロックインへと積極的にピボットしており、トレースレイヤーだけでなくデプロイメントレイヤーを支配することで Weave を代替しようとしています。
- MLflow 3.9 の「Judge Builder」と「MemAlign」のリリースは、当社の評価ワークフローを直接コモディティ化するものです。エンタープライズ向けに自動化された QA を提供し、Weave が優先している手動検査ツールへの依存度を低下させます。
- Weave における成熟した「アノテーションキュー（Annotation Queues）」の欠如は、大規模なヒューマン・イン・ザ・ループのラベル付けチーム向けの標準ワークフローを持つ LangSmith や Langfuse に対する、依然として重大な販売上の障壁となっています。
- Braintrust は、ネイティブな Cursor IDE 統合をリリースすることで、当社の開発者体験（DX）戦略の先手を打ちました。開発者が Weave のダッシュボードに到達する前の「インナーループ」ワークフローを掌握しています。
- Weave Playground へのサーバーレス LoRA 推論の統合（1月16日）は、Arize Phoenix や Braintrust のようなスタンドアロンのプレーヤーでは技術的に再現不可能な、独自の「トレーニングから推論へ」のフライホイールを生み出しています。
- 要対応アクション：製品チームは第2四半期に OpenTelemetry (OTel) 互換性を優先する必要があります。MLflow と Arize Phoenix は、「ネイティブ OTel」サポートをより安全でベンダーニュートラルな選択肢として位置づけることで、エンタープライズのアーキテクチャ審査で勝利を収めています。

> **一行判定**: Weave はマルチモーダルおよびトレーニング統合ワークフローにおいて明確な技術的リードを保っていますが、LangSmith のインフラロックインと MLflow の自動化されたエンタープライズ QA 機能による存亡の危機に直面しています。

### Weave の主な強み

- トレーニング・リネージの統合: Weave は、本番環境のトレースを W&B のモデルアーティファクト、トレーニングラン、スイープにネイティブにリンクできる唯一のプラットフォームであり、真のデータフライホイールを実現します。
- マルチモーダル評価: 最近リリースされた Audio Monitors（2026年2月）は、ボイスエージェント開発者にとって、LangSmith や MLflow などのテキスト中心の競合他社に対する明確な優位性を提供します。
- インタラクティブなデバッグ: Weave の Playground は、静的なトレース表示に重点を置く MLflow や Arize Phoenix と比較して、迅速なイテレーションのための優れた「編集して実行」体験を提供します。
- フレームワーク非依存: Weave は LangSmith よりも軽量で特定の思想に縛られないため、LangChain エコシステム外でカスタムスタックを構築する開発者にとって魅力的です。

### Weave の改善点

- ヒューマン・イン・ザ・ループ・ワークフロー: LangSmith と Langfuse は、大規模な人的ラベル付けチームを管理するための、より成熟した「アノテーションキュー」を提供しています。
- エージェント状態の可視化: LangSmith の LangGraph との深い統合は、複雑なステートマシンや循環的なエージェントワークフローの優れた可視化を提供します。
- トラフィック管理: Weave には、Braintrust が提供しているような、レート制限、キャッシング、トラフィック制御のためのアクティブな AI プロキシ/ゲートウェイアーキテクチャが不足しています。
- OpenTelemetry 標準化: MLflow と Arize Phoenix は「ネイティブ OTel」アプローチを採用しており、Weave の SDK よりもオープンな標準を優先する企業にとって安全な選択肢となっています。

## 3. 新機能（過去30日間）

### [Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: テキストと並行して音声出力を監視・判定するモニターの作成をサポートし、ボイスエージェントの評価を可能にしました。（2026-02-01, コア・オブザーバビリティ）
- **Dynamic Leaderboards**: 評価から自動生成されるリーダーボード。永続的なカスタマイズと CSV エクスポート機能を備えています。（2026-01-29, 評価統合）
- **Custom LoRAs in Playground**: 比較のために、カスタムでファインチューニングされた LoRA の重みを Weave Playground に直接ロードしてテストする機能。（2026-01-16, 実験 / 改善ループ）

### [LangSmith](https://changelog.langchain.com)
- **Customize Trace Previews**: デバッグを高速化するために、トレースリストビューで表示するフィールドを設定する機能。（2026-02-06, DevEx / 統合）
- **Google Gen AI Wrapper**: OpenTelemetry を使用せずに Google の Generative AI モデルをネイティブにトレースするための新しい SDK ラッパー。（2026-01-31, DevEx / 統合）
- **LangSmith Self-Hosted v0.13**: パフォーマンスの向上と新しい設定オプションを備えたセルフホスト版のアップデート。（2026-01-16, エンタープライズ & セキュリティ）

### [Langfuse](https://langfuse.com/changelog)
- **Corrected Outputs for Traces**: ファインチューニング用データセットを構築するために、LLM 出力の修正版をトレースビューで直接キャプチャ。（2026-01-14, コア・オブザーバビリティ）
- **Python SDK v3.14.1**: Langfuse 機能にアクセスするためのクライアントライブラリのアップデート。（2026-02-09, DevEx / 統合）

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Trace-level Scorers**: カスタムコードスコーラーが実行トレース全体にアクセスできるようになり、マルチステップのワークフローやエージェントの動作を評価可能に。（2026-02, 評価統合）
- **LangSmith Integration**: LangSmith のトレースおよび評価コールを Braintrust にルーティングするラッパー。ツールの集約を可能にします。（2026-02, DevEx / 統合）
- **Cursor Integration**: Cursor エディタ用の拡張機能。Braintrust MCP サーバーを自動設定し、自然言語でログをクエリできます。（2026-02, DevEx / 統合）
- **Auto-instrumentation (Python/Ruby/Go)**: Python、Ruby、Go アプリケーション向けのコード改変不要なトレースサポートを追加。（2026-01, DevEx / 統合）
- **Temporal Integration**: Temporal のワークフローとアクティビティの自動トレース。親子関係のマッピングに対応。（2026-01, DevEx / 統合）

### [MLflow](https://mlflow.org/releases)
- **MLflow Assistant**: Claude Code を搭載した製品内チャットボット。UI からのコンテキストを使用して、問題の診断、テストの設定、コードの修正を行います。（2026-01-29, DevEx / 統合）
- **Agent Performance Dashboards**: 設定なしでレイテンシ、リクエスト数、品質スコアを表示する、GenAI 実験用の構築済み「概要」タブ。（2026-01-29, モニタリング & メトリクス）
- **MemAlign Judge Optimizer**: 過去のフィードバックから評価ガイドラインを学習し、LLM 判定の精度を自動的に向上させるアルゴリズム。（2026-01-29, 評価統合）
- **Judge Builder UI**: コードを書かずにカスタム LLM 判定を作成、テスト、検証するためのビジュアルインターフェース。（2026-01-29, 評価統合）
- **Continuous Online Monitoring**: 本番環境のトレースに対して LLM 判定を自動実行し、品質問題をリアルタイムで検出。（2026-01-29, モニタリング & メトリクス）

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 Support**: Playground において Anthropic の Claude Opus 4.6 モデルのサポートを追加。拡張思考パラメータに対応。（2026-02-09, DevEx / 統合）
- **FaithfulnessEvaluator**: 誠実性（Faithfulness）を測定するための新しいエバリュエーター。非推奨となった HallucinationEvaluator を置き換えます。（2026-02-02, 評価統合）
- **Tool Selection & Invocation Evaluators**: エージェントが正しいツールを選択し、有効なパラメータで呼び出したかを評価する専用エバリュエーター。（2026-01-31, エージェント / RAG オブザーバビリティ）
- **CLI for Prompts & Datasets**: ターミナルからプロンプト、データセット、実験を管理するための包括的な CLI コマンド。（2026-01-22, DevEx / 統合）
- **Trace-to-Dataset with Span Associations**: ソーススパンへの双方向リンクを維持したまま、本番トレースからデータセットを作成する機能。（2026-01-21, 評価統合）

## 5. エンタープライズ・シグナル

- MLflow 3.9 の「Judge Builder」と「MemAlign」のリリースは、エンタープライズ QA を自動化し、手動評価チームの必要性を減らす動きを示しています。
- LangSmith の LangGraph Cloud によるデプロイメントへの拡大は、インフラレイヤー全体を所有し、ベンダーロックインを強める戦略を示唆しています。
- Langfuse の ClickHouse バックエンドへの移行は、膨大なトレースデータに対してリアルタイム分析を必要とする、大規模でコスト意識の高い企業への注力を示しています。
- Braintrust の新しい Cursor 統合と LangSmith ラッパーは、IDE レベルで開発者のワークフローを掌握しようとする積極的な戦略を示しています。
