# W&B Weave — 週刊競合インテリジェンスレポート

## 1. エグゼクティブ・サマリー

- 市場の大きな転換: Humanloop は Anthropic による買収に伴い、2025年9月にサービスを終了します。これにより、安定したプロンプト管理および評価の代替手段を探しているエンタープライズ顧客を獲得する、緊急かつ優先度の高い機会が Weave に生まれています。
- 「プロキシ vs 非同期」の分断: Braintrust や Helicone などの競合は、コスト制御、キャッシング、レート制限で優位に立つためにゲートウェイ/プロキシアーキテクチャを強化しています。対照的に、Weave、LangSmith、Phoenix は開発者の利便性のために非同期ロギングを優先しており、市場に明確なアーキテクチャの分裂が生じています。
- 評価の成熟度曲線: 主要な競合（LangSmith、Braintrust）は、単純な「LLM-as-a-Judge」を超えて、高度な Human-in-the-Loop (HITL) ワークフローへと移行しており、手動レビューを運用化するための専用の「アノテーションキュー」や「ペア比較」UIを導入しています。
- SQLベースのオブザーバビリティの台頭: Logfire と Braintrust (BTQL) は、トレース用の SQL ライクなクエリ言語で差別化を図っており、固定されたダッシュボードよりも柔軟でコード駆動型の分析を必要とするパワーユーザーに訴求しています。
- エージェント特化型ツール: 市場は汎用的な LLM トレースから「エージェント」オブザーバビリティへとシフトしています。LangSmith、Langfuse、Logfire はすべて、ツール呼び出し、多段階の推論ループ、サブエージェントのネストに対応した専用の可視化機能をリリースしました。
- OpenTelemetry の標準化: Arize Phoenix、Langfuse、Logfire は、OpenTelemetry (OTel) と OpenInference を標準として積極的に推進しており、ベンダーニュートラルな計装を約束することで、独自の SDK に挑戦しています。
- 言語エコシステムの拡大: Braintrust は言語サポートにおいて競合を圧倒しており、Java、Go、Ruby、C# 用のネイティブ SDK をリリースし、多言語（ポリグロット）なエンタープライズ環境における第一の選択肢としての地位を固めています。

> **一行評決**: Weave はフルライフサイクルの ML チームにとって依然として最高の選択肢ですが、コスト制御におけるプロキシベースのゲートウェイや、標準化における OTel ネイティブツールからの圧力が高まっています。

### Weave の主な強み

- フルライフサイクルのリネージ: 本番環境のトレースをモデルのトレーニングランやアーティファクトに直接リンクできる独自の能力（W&B エコシステム統合）。
- プログラマブルなダッシュボード: Weave Boards は、LangSmith や Phoenix の固定された UI よりも優れた、コード駆動型で高度にカスタマイズ可能な可視化体験を提供します。
- 低摩擦な統合: 非同期ロギング手法により、Braintrust や Helicone が採用しているプロキシ/ゲートウェイアーキテクチャのようなレイテンシやセキュリティの複雑さを回避できます。
- データプライバシー: データのクリティカルパスに介在するプロキシミドルウェアを必要としないため、セキュリティを重視するチームにとって強い魅力となります。

### Weave の改善点

- ゲートウェイ機能の欠如: Helicone や Braintrust などの競合は、ネイティブキャッシング、レート制限、シークレット管理などのインフラ機能で先行しています。
- 人間によるアノテーションワークフロー: LangSmith と Braintrust は、大規模な人間によるアノテーションキューやペア比較のための、より成熟した専用 UI を提供しています。
- 言語サポート: Braintrust と Langfuse は、Python と TypeScript に重点を置く Weave と比較して、より幅広い SDK サポート（Java、Go、C#）を提供しています。
- 特化型 RAG 可視化: Arize Phoenix は、LlamaIndex との深い統合により、ベクトル検索の内部（埋め込みクラスター、チャンクランキング）の可視化においてリードしています。

## 3. 今週の新機能

### LangSmith
- **LangSmith Fetch CLI**: ターミナルから直接トレースを取得し、エージェントをデバッグするための CLI ツール。(2025-12, DevEx / 統合)
- **ペア比較アノテーションキュー**: 2つのエージェント出力を並べて比較し、勝者を選択するための構造化されたワークフロー。(2025-12, 評価統合)
- **統合コスト追跡**: LLM、ツール、検索ステップにわたるフルスタックのコスト監視。(2025-12, モニタリング & メトリクス)
- **Polly (Beta)**: エージェントのデバッグと分析を支援する LangSmith 内の AI アシスタント。(2025-12, DevEx / 統合)
- **セルフホスト v0.13**: Insights を含む、クラウド版と機能が同等になったセルフホストリリースの更新。(2026-01, エンタープライズ & セキュリティ)

### Arize Phoenix
- **Claude Opus 4.6 サポート**: Prompt Playground で Claude Opus 4.6 モデルのサポートを追加。(2026-02-09, 実験 / 改善ループ)
- **ツール選択エバリュエーター**: エージェントワークフローにおけるツール選択の精度を測定する新しいエバリュエーター。(2026-02-06, 評価統合)
- **忠実度（Faithfulness）エバリュエーター**: 精度向上のため、従来の HallucinationEvaluator を非推奨とし、FaithfulnessEvaluator を導入。(2026-02-02, 評価統合)
- **OAuth2 メール抽出**: OAuth2 属性を介してユーザーのメールアドレスを抽出するためのセキュリティ設定の強化。(2026-01-28, エンタープライズ & セキュリティ)

### Braintrust
- **トレースレベルのスコーラー**: カスタムコードスコーラーが実行トレース全体にアクセスし、多段階ワークフローを評価できるようになりました。(2026-02, 評価統合)
- **LangSmith 統合**: LangSmith のトレースを Braintrust、またはその両方に並行してルーティングするためのラッパー。(2026-02, DevEx / 統合)
- **自動計装 (Python/Ruby/Go)**: 主要言語に対するコード記述不要のトレースサポート。(2026-01, DevEx / 統合)
- **Temporal 統合**: Temporal のワークフローとアクティビティの自動トレース。(2026-01, DevEx / 統合)
- **Claude Code 統合**: 自然言語によるログクエリを可能にする、Anthropic のエージェント型コーディングツールとの統合。(2025-12, DevEx / 統合)

### Langfuse
- **トレースの修正済み出力**: ファインチューニング用データセットを構築するために、トレースビューで LLM 出力の改善版を直接キャプチャ。(2026-01-14, 実験 / 改善ループ)
- **オブザベーション I/O へのインラインコメント**: トレースの入力/出力内の特定のテキスト選択にコメントを固定。(2026-01-07, DevEx / 統合)
- **データセット項目のバージョニング**: 更新または削除時にデータセット項目のバージョンを自動生成。(2025-12-15, 実験 / 改善ループ)
- **ホスト型 MCP サーバー**: エージェントがプロンプトを直接取得/更新できるようにするネイティブの Model Context Protocol サーバー。(2025-11-20, DevEx / 統合)
- **Langfuse for Agents**: ツール呼び出しのレンダリングとエージェント固有の評価を備えた強化されたトレース機能。(2025-11-05, エージェント / RAG オブザーバビリティ)

### Humanloop
- **プラットフォームの終了**: Humanloop は Anthropic に買収され、2025年9月8日にプラットフォームを終了します。(2025-08-13, エンタープライズ & セキュリティ)
- **LLMs.txt サポート**: ドキュメントコンテキスト用の LLMs.txt 標準のサポートを追加。(2025-05-01, DevEx / 統合)

### Logfire
- **マルチトークンサポート**: プロジェクトの移行を容易にするための複数トークンのサポートを追加。(2026-02-04, エンタープライズ & セキュリティ)
- **Pytest 統合**: Pytest 実行内でのトレースとデバッグのためのネイティブ統合。(2026-01-26, DevEx / 統合)
- **DSPy 統合**: DSPy フレームワークの公式計装を追加。(2026-01-16, フレームワーク統合)
- **フィードバックアノテーション**: トレースに対するユーザーフィードバックを記録するための実験的関数。(2025-04-11, 評価統合)
- **OpenAI Agents 計装**: 新しい OpenAI Agents フレームワークをトレースするためのサポート。(2025-03-11, エージェント / RAG オブザーバビリティ)

### Helicone
- **Experiments**: プラットフォーム内で直接データセットに対してプロンプトをテストおよび評価する機能。(2024-05, 実験 / 改善ループ)
- **プロンプト管理**: コードから切り離されたプロンプトの作成、バージョニング、管理のための UI。(2024-01, 実験 / 改善ループ)
- **Sessions**: 複数のリクエストを単一のインタラクションフローにリンクするグルーピングメカニズム。(2023-11, コアオブザーバビリティ)

## 4. ポジショニングの変遷

| ベンダー | 現在 | 今後の方向性 | シグナル |
|---|---|---|---|
| **Weave** | モデルトレーニングと本番環境のオブザーバビリティのギャップを埋める、フルスタック AI エンジニアリングの開発者中心の標準。 | 深いコードレベルのデバッグとハイレベルな本番監視のバランスをとる、包括的な AI アプリケーションプラットフォーム。 | 競合が「インフラ/プロキシ」（Helicone、Braintrust）対「ワークフロー/デバッグ」（LangSmith、Logfire）に二分される中、Weave は後者でのリードを維持しつつ前者の課題に対応する必要がある。 |
| LangSmith | 深いフレームワーク統合を備えた LangChain エコシステムのデフォルトのオブザーバビリティプラットフォーム。 | プロンプトエンジニアリングから本番監視までのフルライフサイクルを管理する、汎用的なエンタープライズグレードの LLM Ops スイートへの拡大。 | 「LangSmith Fetch」や「Polly」などのフレームワークに依存しない機能のリリース、およびエンタープライズ向けセルフホスト版の機能拡充への多額の投資。 |
| Arize Phoenix | OpenTelemetry 上に構築された、オープンソースでローカルファーストな LLM オブザーバビリティの標準。 | エンタープライズチーム向けにオブザーバビリティ、評価、プロンプト管理を組み合わせた包括的な AI エンジニアリングプラットフォーム。 | コアとなるトレース製品と並行して、「プロンプト管理」および「実験」機能への多額の投資。 |
| Braintrust | プロキシベースの LLM オブザーバビリティと評価におけるエンタープライズ標準。 | 開発者の IDE ワークフロー（Cursor、Claude Code）への統合を深め、言語エコシステムのサポートを拡大。 | Java、Go、Ruby、C# 用のネイティブ SDK のリリース、および Cursor や Claude Code との統合。 |
| Langfuse | セルフホスト型のオブザーバビリティとプロンプト管理を必要とするチームにとっての主要なオープンソースの選択肢。 | 大規模なエージェント型ワークロードに対応する、ClickHouse を活用したスケーラブルでデータ集約型の「LLM Ops」プラットフォーム。 | ClickHouse バックエンドへの移行と、特化型エージェントトレース機能のリリース。 |
| Humanloop | クロスファンクショナルなチーム向けの主要なプロンプト管理および評価プラットフォーム（現在は終了フェーズ）。 | Anthropic による買収に伴うサービスの終了。 | プラットフォーム終了日（2025年9月8日）の公式発表と Anthropic による買収。 |
| Logfire | Python および Pydantic を多用する AI アプリケーションに最適化された、開発者優先の OTel ネイティブなオブザーバビリティツール。 | デバッグツールから、より広範なフレームワークをサポートする包括的な AI アプリケーションパフォーマンスモニタリング (APM) プラットフォームへの拡大。 | DSPy、Pytest、OpenAI Agents 統合の最近の追加、およびエンタープライズ向け価格設定ティアの導入は、フルスタックの本番対応への移行を示唆している。 |
| Helicone | コスト制御、キャッシング、ハイレベルなオブザーバビリティを必要とする開発者向けの、主要なオープンソース AI ゲートウェイおよびプロキシ。 | 純粋なインフラプロキシから、プロンプト管理や評価機能を備えたより広範な LLM 開発プラットフォームへの拡大。 | 「Experiments」や「Prompt Management」の最近のローンチは、アプリケーション開発レイヤーに向けたスタックの上位移行を示唆している。 |

## 5. エンタープライズ・シグナル

- Humanloop の終了により、エンタープライズ顧客は移行を余儀なくされており、期限付きの獲得機会が生まれています。
- LangSmith の「セルフホスト v0.13」のリリースは、厳格なエンタープライズコンプライアンスを満たすために、クラウドとオンプレミスの機能パリティを推進していることを示しています。
- Braintrust の Java/C# SDK への拡大は、Python AI スタック以外のレガシーなエンタープライズ環境を取り込むための戦略的な動きを示しています。
- 「アノテーションキュー」（LangSmith、Braintrust）の台頭は、企業が手動レビュープロセスをスケールさせており、そのための専用ツールを必要としていることを浮き彫りにしています。

## 6. ウォッチリスト

- Humanloop からの移行: 離脱した Humanloop 顧客がどこに移行しているかを監視し、「プロンプト管理」機能の比較で彼らをターゲットにします。
- OpenTelemetry の採用: Phoenix や Logfire の OTel ネイティブなアプローチが、大規模なエンジニアリング組織において独自の SDK を置き換え始めるかどうかを注視します。
- プロキシ機能: Braintrust/Helicone に対抗するために、LangSmith や Weave がゲートウェイのような機能（キャッシング/レート制限）を導入するかどうかを追跡します。
- エージェントフレームワーク: 競合が OpenAI Agents や PydanticAI などの新しいフレームワークにどのように適応するかを監視します（現在、Logfire がこの分野で先行しています）。
