## 1. エグゼクティブ・サマリー

- LangSmith と Langfuse は「エージェントの可視化」においてリードを築きつつあります。専用のグラフビューや状態デバッガー（例：LangGraph Studio）を提供しており、Weave の線形トレースツリーよりも、複雑なループ構造を持つエージェントの理解において優れています。
- Braintrust と Helicone は、「ゲートウェイ」機能（キャッシング、レート制限、プロキシ）によって差別化に成功しており、オブザーバビリティをインフラ層へと押し広げています。これは現在 Weave に欠けている機能です。
- Weave と W&B Training の統合は、依然として最も強力な競合優位性（モート）です。Arize Phoenix や LangSmith のような単独の競合他社がネイティブに複製できない、独自の「本番環境からファインチューニングへ」というワークフローを提供しています。
- Logfire は、「魔法のような」自動インスツルメンテーションによって Python/Pydantic 開発者セグメントを急速に獲得しています。これは、ベンダー独自の SDK よりも標準ライブラリの統合を好むコードファーストなエンジニアリングチームにおいて、Weave の採用を脅かしています。
- LangSmith と Braintrust は、「ヒューマン・イン・ザ・ループ（Human-in-the-Loop）」ワークフロー（アノテーションキュー、カンバン形式のレビュー）を大幅に成熟させており、手動評価やデータキュレーションのユースケースにおいて Weave との差を広げています。
- 市場は「インフラ/ゲートウェイ」プロバイダー（Helicone, Braintrust）と「ワークフロー/エージェント」プラットフォーム（LangSmith, Langfuse）に二分されつつあります。Weave は、板挟みになるのを避けるため、「モデル改善」プラットフォームとしての地位を明確にする必要があります。

> **一言評価**: Weave は W&B エコシステムを通じた**ファインチューニングのフィードバックループ**でリードしていますが、**エージェントのワークフロー可視化**では **LangSmith** から、**エンタープライズ向けゲートウェイインフラ**では **Braintrust** からの圧力にさらされています。

### Weave の主な強み

- **ネイティブなトレーニング統合:** 本番環境のトレースを W&B のモデルトレーニングランにシームレスにリンクできる唯一のプラットフォームであり、ファインチューニングのための真のデータフライホイールを実現します。
- **プログラムによる評価:** 強力な「eval-as-code（コードとしての評価）」の哲学により、硬直的な UI ベースの評価ツールよりも、データサイエンティストに好まれる複雑でカスタムなスコアリングロジックが可能です。
- **探索的キャンバス:** ノートブックのような UI パラダイムにより、Helicone や Datadog の静的なダッシュボードと比較して、より深くアドホックなデータの探索やフィルタリングが可能です。

### Weave の改善が必要な領域

- **エージェントの可視化:** 複雑でステートフルなエージェントループのための専用の「グラフビュー」やビジュアルデバッガーが不足しています。これは LangSmith (Studio) や Langfuse が優れている領域です。
- **ゲートウェイ/プロキシ層の欠如:** Braintrust や Helicone に見られるような、キャッシング、レート制限、フェイルオーバー・ルーティングなどのアクティブなトラフィック管理機能を提供していません。
- **人間によるアノテーションワークフロー:** LangSmith と比較して、大規模な人間によるレビューチームを管理するための成熟した UI 機能（例：割り当てキュー、コンセンサススコアリング）が不足しています。

## 3. 今週の新機能

### Weave
- **Stats Clickhouse クエリレイヤー**: 統計データを処理するためのバックエンドクエリレイヤーの改善。ダッシュボードのパフォーマンス向上が期待されます。(2026-01-20, プラットフォーム・パフォーマンス)
- **OTel エクスポートの改善**: 信頼性と標準準拠を向上させるため、OpenTelemetry のエクスポートを非同期インサートから分離しました。(2026-01-08, 相互運用性)

### LangSmith
- **トレースプレビューのカスタマイズ**: トレースの入力と出力のどの部分をトレーステーブルに直接表示するかをユーザーが設定できるようになり、カスタムデータ構造の使い勝手が向上しました。(2026-02-06, コア・オブザーバビリティ)
- **LangSmith セルフホスト v0.13**: セルフホスト顧客向けのメジャーリリース。クラウド版との機能パリティ、パフォーマンス、運用管理機能が向上しました。(2026-01-16, エンタープライズ & セキュリティ)

### Arize Phoenix
- **Claude Opus 4.6 サポート**: プレイグラウンドで Claude Opus 4.6 モデルのサポートを追加しました。(2026-02-09, 実験 / 改善ループ)
- **ツール選択エバリュエーター**: ツール選択ロジックを評価するための新しい専用エバリュエーター。(2026-02-06, 評価統合)
- **Faithfulness（忠実性）エバリュエーター**: RAG 評価を向上させるため、FaithfulnessEvaluator を追加し、HallucinationEvaluator を非推奨にしました。(2026-02-02, 評価統合)
- **ツール呼び出し精度メトリクス**: トレース内でのツール呼び出しの精度を追跡する新しいメトリクス。(2026-02-02, モニタリング & メトリクス)
- **カスタムメトリクス・カーソルルール**: カーソルルールを介して新しい組み込みメトリクス（LLM 分類）を作成できる UI アップデート。(2026-01-21, モニタリング & メトリクス)

### Braintrust
- **トレースレベル・スコアラー**: カスタムコードスコアラーが実行トレース全体にアクセスできるようになり、マルチステップのワークフローやエージェントの動作を評価できるようになりました。(2026-02, 評価統合)
- **LangSmith 統合**: LangSmith のトレースを Braintrust にルーティングするラッパー。並行利用や移行を容易にします。(2026-02, 開発者体験 / 統合)
- **Cursor & MCP 統合**: Cursor エディタおよび Model Context Protocol との統合により、IDE からログのクエリや評価の実行が可能になりました。(2026-02, 開発者体験 / 統合)
- **自動インスツルメンテーション (Python/Ruby/Go)**: 主要プロバイダー向けの Python、Ruby、Go SDK におけるゼロコード・トレース。(2026-01, 開発者体験 / 統合)
- **Temporal 統合**: 分散エージェントの可視化のため、Temporal ワークフローとアクティビティの自動トレースに対応しました。(2026-01, エージェント / RAG オブザーバビリティ)

### Langfuse
- **トレースの修正済み出力**: ファインチューニング用データセットを構築するため、LLM 出力の改善版をトレースビューで直接キャプチャできるようになりました。(2026-01-14, 実験 / 改善ループ)
- **単一オブザベーション評価**: トレース内の個別のオブザベーションに対して評価を実行できるようになりました。(2026-02-09, 評価統合)
- **思考 / 推論のレンダリング**: トレース詳細において、モデル出力の「思考（thinking）」や推論部分を視覚的にレンダリングします。(2026-02-09, コア・オブザーバビリティ)
- **組織監査ログビューアー**: セキュリティおよびアクセスイベントを追跡するための、組織レベルの新しい監査ログビューアー。(2026-02-09, エンタープライズ & セキュリティ)

### Logfire
- **プロジェクト移行のマルチトークン対応**: プロジェクト移行ワークフローにおける複数トークンの処理をサポートしました。(2026-02-04, エンタープライズ & セキュリティ)

## 5. エンタープライズ・シグナル

- LangSmith は「セルフホスト v0.13」（2026年1月）をリリースし、データレジデンシー要件への対応を強化しました。
- Braintrust は、エンタープライズコンプライアンスのためのコスト管理およびセキュリティレイヤーとして「AI プロキシ」を積極的にマーケティングしています。
- Langfuse は ClickHouse バックエンドへの移行を完了し、大規模なエンタープライズログの取り込みに対応できる準備が整ったことを示しました。

## 6. ウォッチリスト

- LangSmith の「デプロイメント」およびサービングインフラへの拡大。
- Braintrust の IDE 統合（Cursor/MCP）。これにより、オブザーバビリティがエディタ側（シフトレフト）へ移行する可能性があります。
- Arize の新しい「ツール選択」エバリュエーター。これはエージェントの信頼性における特定のペインポイントをターゲットにしています。
