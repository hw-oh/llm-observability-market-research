## 1. エグゼクティブ・サマリー

- 市場統合の機会: AnthropicによるHumanloopの買収と、それに伴うサービス終了（2025年9月）は、プロンプトエンジニアリングや評価の新たな拠点を求めるエンタープライズ顧客を獲得するための、緊急かつ優先度の高い機会を生み出しています。
- 「Agent Ops」へのシフト: 競合他社は、単純なトレーシングを超えて、フルライフサイクル管理へと積極的に移行しています。LangSmithによる「Agent Servers」と「Unified Cost Tracking（統合コスト追跡）」のリリースは、ホスティングと財務運用（FinOps）へのピボットを象徴しており、Weaveに対して本番環境へのデプロイメントストーリーを定義するよう迫っています。
- 可視化の軍拡競争: 循環型/グラフベースのエージェントワークフローに特化した可視化への明確なトレンドがあります。LangSmith (LangGraph)、Langfuse (Agent Graphs)、Arize Phoenixは専用のグラフビューで差別化を図っており、複雑なエージェントに対して線形なトレースリストは時代遅れに感じられ始めています。
- OpenTelemetryとMCPによる標準化: Arize Phoenix、Langfuse、Logfireは、標準規格としてOpenTelemetry (OTel) と Model Context Protocol (MCP) に結集しつつあります。この圧力は、独自のSDKに対する「ベンダーロックインなし」と相互運用性を主要なセールスポイントとして強調しています。
- 評価機能の成熟: 市場は「Managed Evaluators（管理型エバリュエーター）」と「Annotation Queues（アノテーションキュー）」へと移行しました。LangSmithとLangfuseは現在、ヒューマン・イン・ザ・ループによるレビューのための専用ワークフローを提供しており、評価を単なる開発者のユーティリティではなく、チームスポーツとして位置づけています。
- 多言語展開: Braintrustは、Java、Go、Ruby、C#へのネイティブSDKサポートを拡大することで、エンタープライズのバックエンド市場を積極的にターゲットにしており、多くのオブザーバビリティツールにおけるPython/JSの優位性に挑戦しています。
- 差別化要因としてのSQL: LogfireとBraintrustは、トレースに対するSQLベースのクエリ機能をパワーユーザーに提供し、定義済みのダッシュボードを超えた、きめ細かくアドホックなデータ分析機能を求めるエンジニアのニーズに応えています。

> **一行判定**: Weaveは「トレーニングから本番まで」の系統管理（リネージ）においてリードを保っていますが、エージェントのデプロイに関してはLangSmithから、オープン標準とコストの可視化に関してはLangfuseやArize Phoenixから激しい圧力を受けています。

### Weave の主な強み

- 深いトレーニングリネージ: 本番環境のトレースをトレーニングランやモデルアーティファクト（W&Bエコシステム）に紐付ける独自の能力。これは単体のオブザーバビリティツールには真似できません。
- ホワイトボックスの可視化: 任意のPythonコードや内部ロジックをトレースする優れた能力。プロキシベースの競合（Helicone）はAPIの入出力しか見ることができません。
- 柔軟な「Board」UI: 高度にカスタマイズ可能なノートブック形式のダッシュボードにより、LangSmithやLangfuseの固定された定義済みダッシュボードよりも深い探索的分析が可能です。
- モデルレジストリ統合: 真のモデルレジストリとのネイティブな統合により、メタデータ文字列だけでなく、モデルバイナリの厳密なバージョン管理が可能です。

### Weave の改善が必要な領域

- プロンプト管理CMS: LangSmith (Hub)、Langfuse、および旧Humanloopなどの競合他社は、コードを変更せずにプロンプトのバージョニングや編集を行うための、より優れた非技術者向けUIを提供しています。
- 財務運用（FinOps）とコストの粒度: HeliconeとLangfuseは、カスタム価格設定ティアや予算アラートなど、本番運用に不可欠な大幅に高度なコスト追跡機能を提供しています。
- 人的アノテーションワークフロー: Weaveには、人間のラベラーを管理するための専用の「アノテーションキュー」ワークフローが欠けています。これは現在、LangSmithやLangfuseでは標準的な機能です。
- エージェントグラフの可視化: 競合他社は、循環的なエージェントワークフロー（例：LangGraph統合）に特化した可視化をより迅速に市場投入していますが、Weaveのトレースビューは主に線形です。

## 3. 今週の新機能

### LangSmith
- **トレースプレビューのカスタマイズ**: トレーステーブルビューに表示される入力/出力を制御するための設定オプション。(2026-02-06, DevEx / 統合)
- **LangSmith Fetch CLI**: コマンドラインから直接トレースにアクセスし、デバッグするためのターミナルベースのツール。(2025-12-10, DevEx / 統合)
- **Polly AI アシスタント**: エージェントのデバッグやトレースの分析を行うためのプラットフォーム内AIアシスタント（ベータ版）。(2025-12-10, エージェント / RAG 観測性)
- **統合コスト追跡**: LLM、ツール、リトリーバルステップにわたるフルスタックのコストモニタリング。(2025-12-09, モニタリング & メトリクス)
- **ペアワイズ・アノテーションキュー**: 人間のレビュー担当者が2つのエージェント出力を並べて比較するための構造化されたワークフロー。(2025-12-17, 評価統合)

### Arize Phoenix
- **FaithfulnessEvaluator**: RAGの精度測定を向上させるため、HallucinationEvaluatorに代わる新しいエバリュエーター。(2026-02-02, 評価統合)
- **ツール呼び出し精度メトリクス**: エージェントのツール呼び出しの成功率を測定するために特別に設計された新しいメトリクス。(2026-01-27, エージェント / RAG 観測性)
- **Claude Opus 4.6 サポート**: プレイグラウンドにおける最新のClaudeモデルのサポートを追加。(2026-02-09, 実験 / 改善ループ)
- **メトリクス用 Cursor ルール**: Cursorルールを使用してカスタム分類エバリュエーターを作成する機能を強化。(2026-01-21, モニタリング & メトリクス)

### Braintrust
- **トレースレベル・スコアラー**: カスタムコードスコアラーが実行トレース全体にアクセスし、マルチステップのワークフローやエージェントの動作を評価できるようになりました。(2026-02, 評価統合)
- **SDKサポートの拡大 (Java, Go, Ruby, C#)**: Java、Go、Ruby、C#/.NET向けの自動インスツルメンテーションを備えたネイティブSDK。(2026-01, DevEx / 統合)
- **SQL構文のサポート**: トレースや実験のクエリ用に、標準的なSQL構文（HAVINGや暗黙的なエイリアスを含む）をサポート。(2026-01, モニタリング & メトリクス)
- **Temporal 統合**: Temporalのワークフローとアクティビティを自動的にトレースし、ワーカー間の分散トレースをキャプチャ。(2026-01, DevEx / 統合)
- **Claude Code & Cursor 統合**: AIコーディングアシスタントが自然言語を介してログや実験結果を照会できるようにする深い統合。(2025-12, DevEx / 統合)

### Langfuse
- **トレースの修正済み出力**: LLM出力の改善されたバージョンをトレースビューで直接キャプチャし、ファインチューニング用のデータセットを構築。(2026-01-14, 実験 / 改善ループ)
- **データセット項目のバージョニング**: データセット項目の追加、更新、削除ごとに自動的にバージョニングを実行。(2025-12-15, 実験 / 改善ループ)
- **ホスト型 MCP サーバー**: AIエージェントがプロンプトを直接取得・更新できるようにするネイティブの Model Context Protocol (MCP) サーバー。(2025-11-20, DevEx / 統合)
- **LLM-as-a-Judge 実行トレース**: すべてのエバリュエーターの実行において、プロンプトや評価用のトークン使用量を確認するためのトレースを作成。(2025-10-16, 評価統合)
- **実験ランナー SDK**: 自動トレースと並列実行を備えた、データセット上での実験実行用ハイレベルSDK。(2025-09-17, 実験 / 改善ループ)

### Humanloop
- **プラットフォームの終了**: HumanloopはAnthropicに買収され、2025年9月8日にサービスを終了します。(2025-08, 全般)

### Logfire
- **Pytest 統合**: pytestの実行をスパンとしてログに記録し、テストの観測性を可能にする直接統合。(2026-01-26, DevEx / 統合)
- **DSPy 統合**: LLMプログラミング用フレームワークであるDSPyの自動インスツルメンテーション。(2026-01-16, フレームワーク統合)
- **MCP サポート**: コンテキスト交換を標準化するための Model Context Protocol (MCP) のサポート。(2025-03-31, エージェント / RAG 観測性)
- **OpenAI Agents フレームワーク**: OpenAIの新しいAgents SDKのインスツルメンテーション。(2025-03-11, エージェント / RAG 観測性)
- **SQLベースのメトリクス**: トレースに対して任意のSQLクエリを実行し、カスタムメトリクスを生成する機能。(2025-09-05, モニタリング & メトリクス)

### Helicone
- **実験 (Experiments)**: データセットに対してプロンプトをテストし、出力とコストを比較するためのUI。(2024-10, 実験 / 改善ループ)
- **プロンプト管理 (Assembly)**: コードから切り離されたプロンプトのバージョニングと管理のための強化されたUI。(2024-09, 実験 / 改善ループ)
- **セッショングループ化**: チャット履歴の追跡のために、複数のリクエストを単一のセッションにグループ化する機能。(2024-08, コア観測性)

## 5. エンタープライズ・シグナル

- Humanloop の終了: Humanloopのエンタープライズ顧客は2025年9月までに移行を余儀なくされており、特定の「リプレイス」営業の動きを生み出しています。
- LangSmith オンプレミス: LangSmithは、堅牢なセルフホストオプションとSSO/RBACの更新により、規制の厳しい業界を直接ターゲットにし、エンタープライズ向けの姿勢を固めています。
- Braintrust の多言語SDK: BraintrustによるJavaとC#のサポート追加は、PythonベースのAI研究チームだけでなく、伝統的なエンタープライズのバックエンドチームを取り込む動きを示しています。
- SOC2/コンプライアンスへの注力: 全般的（Langfuse, Arize, LangSmith）に、PII（個人情報）マスキングと監査ログが重視されており、「エンタープライズ機能の最小要件」の基準が引き上げられています。

## 6. ウォッチリスト

- Humanloop からの移行: どの競合他社が専用の「Humanloop移行」ツールやキャンペーンを開始するかを監視。
- エージェント・デプロイメント・ツール: LangSmithの「Agent Servers」の採用状況を注視。成功すれば、購買決定権がR&DからOps（運用）へと移る可能性があります。
- MCP の採用: Model Context Protocol (MCP) のサポートが、エージェント観測性の標準要件になる速さを追跡。
- ビジュアル・デバッグ: 非線形なエージェント実行を可視化する、競合他社からの新しい「グラフビュー」のリリースを注視。
