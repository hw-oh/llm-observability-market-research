# LLM Observability 市場調査週報
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-12

## 1. エグゼクティブ・サマリー

- LangSmith は、エンタープライズのコンプライアンスとデータ主権の要件に対応するため、Self-Hosted v0.13 をリリースし、EU データレジデンシーのサポートを開始しました。
- Langfuse は、セキュリティガバナンスのための新しい Organization Audit Log Viewer をリリースし、Agent Graphs 内での reasoning ステップのレンダリング機能を追加しました。
- MLflow は、マルチワークスペース管理を可能にする Organization Support を導入し、tool calling のレスポンスや retriever クエリをキャプチャするための auto-instrumentation を拡張しました。
- Braintrust は、サブエージェントのネストを明示的にサポートするために、プラットフォームの用語を「agents」から「workflows」に更新し、Python の trace scoring に関するアップデートをリリースしました。
- Arize Phoenix は、オープンソースライブラリに特定の Faithfulness および Tool Selection の Eval を追加し、設定可能な OAuth2 メール抽出機能を実装しました。
- W&B Weave は、Audio Monitors と Dynamic Leaderboards で Eval スイートを拡張し、安全なデプロイのための SDK レベルの PII 編集（redaction）のサポートを継続しています。

**AI による市場インサイト**:

> LangSmith による Self-Hosted v0.13 のリリースと EU データレジデンシーへの対応は、規制の厳しい業界に対して同等のインフラ制御を提供することで、Weave のエンタープライズにおける優位性に直接挑戦するものです。
> Langfuse がワークフローのトポロジーを可視化する Agent Graphs を導入したことは、エージェントのステップはキャプチャするものの専用の DAG 可視化を欠いている Weave の UI 上のギャップを浮き彫りにしています。
> Arize Phoenix が専門的な Faithfulness および Tool Selection メトリクスを追加したことで、Weave に対して、エージェント特化型のプリセット Eval ライブラリを拡充すべきという競争圧力が高まっています。

## 3. 新機能 (直近30日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: LLM judge を使用して、テキストと並行してオーディオ出力（MP3/WAV）を監視・判定するモニターのサポート。(2026-02-01, Monitoring & Metrics)
- **Dynamic Leaderboards**: フィルタリング、カスタマイズ、CSV エクスポート機能を備えた、Eval から自動生成されるリーダーボード。(2026-01-29, Evaluation Integration)
- **Playground でのカスタム LoRA**: Serverless LoRA Inference により、Weave Playground で直接カスタム Fine-tuning された LoRA 重みを使用可能に。(2026-01-16, Experiment / Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **Trace プレビューのカスタマイズ**: LangSmith UI での Trace プレビュー方法をユーザーがカスタマイズ可能に。(2026-02-06, Core Observability)
- **Non-Otel Google Vertex AI (ADK) ラッパー**: Python SDK に Google Vertex AI (ADK) 用の非 OpenTelemetry ラッパーを追加。(2026-02-02, DevEx / Integration)
- **LangSmith Self-Hosted v0.13**: セルフホスト型プラットフォームのバージョン 0.13 をリリース。(2026-01-16, Enterprise & Security)

### [Langfuse](https://langfuse.com/changelog)
- **Single Observation Evals**: 単一の observation に対して Eval を実行するサポートを追加。(2026-02-09, Evaluation Integration)
- **Reasoning/Thinking のレンダリング**: Trace 詳細において、思考/推論プロセス（reasoning モデルなど）のレンダリングをサポート。(2026-02-06, Core Observability)
- **Org Audit Log Viewer**: 組織レベルの監査ログを表示するための新しい UI。(2026-02-06, Enterprise & Security)
- **インライン Trace コメント**: Trace 内の入出力データの一部に対してインラインでコメントを追加可能に。(2026-01-27, DevEx / Integration)
- **Trace プレビューでの修正**: Trace および observation のプレビューで直接修正を表示・管理する機能を追加。(2026-01-13, Experiment / Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Claude Agent のサブエージェント・ネスト**: 階層的な Observability を向上させるため、Claude Agent SDK ラッパー内でのサブエージェントのネストをサポート。(2026-02-05, Agent / RAG Observability)
- **Review Span Type**: Trace における手動または自動のレビュー・ステップをサポートするため、新しい 'review' span type を SDK に導入。(2026-02-05, Core Observability)
- **Classifications フィールド**: データラベリングとカテゴリ化機能を強化するため、SDK に classifications フィールドを追加。(2026-01-31, Evaluation Integration)
- **Trace Scoring 候補**: 自動 Eval ロジックを洗練させるため、Python の trace scoring 機能をアップデート。(2026-01-21, Evaluation Integration)
- **Workflows への名称変更**: より広範なオーケストレーション機能を反映させるため、SDK 内の 'agents' を 'workflows' に変更。(2026-01-15, DevEx / Integration)

### [MLflow](https://mlflow.org/releases)
- **MLflow Tracking Server での Organization サポート**: マルチワークスペース環境をサポートし、異なるワークスペース間での実験やリソースの整理が可能に。(2026-02-12, Enterprise & Security)
- **MLflow Assistant**: MLflow UI 内で直接問題を特定、診断、修正するのを支援する、Claude Code をベースとした製品内チャットボット。(2026-01-29, DevEx / Integration)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 サポート**: Playground に Claude Opus 4.6 モデルのサポートを追加。(2026-02-09, DevEx / Integration)
- **Tool Selection Evaluator**: ライブラリに不足していた tool_selection Eval を追加。(2026-02-06, Evaluation Integration)
- **Faithfulness Evaluator**: FaithfulnessEvaluator を追加し、HallucinationEvaluator を非推奨化。(2026-02-02, Evaluation Integration)
- **Tool Invocation Accuracy メトリクス**: ツール呼び出しの精度を追跡するための新しいメトリクスを追加。(2026-02-02, Monitoring & Metrics)
- **設定可能な OAuth2 メール抽出**: OAuth2 における設定可能なメール抽出のための EMAIL_ATTRIBUTE_PATH を追加。(2026-01-28, Enterprise & Security)
- **組み込みメトリクス用 Cursor ルール**: 新しい組み込みメトリクス（LLM 分類 Eval）を作成するための Cursor ルールを追加。(2026-01-21, Evaluation Integration)

|---|---|---|---|

| Langfuse | セルフホストとエンジニアリング・ワークフローに重点を置いた、オープンソースで開発者中心の Observability および Eval プラットフォーム。 | コラボレーションの強化と、複雑な reasoning モデルおよびエージェント・ワークフローへのより深いサポート。 | reasoning モデル用の 'Thinking' レンダリング、チームコラボレーション用のインラインコメント、きめ細かな単一 observation Eval の追加。 |

| MLflow | 実験管理とモデルレジストリ機能を GenAI ライフサイクル全体に拡張することに成功した、主要なオープンソース MLOps プラットフォーム。 | エンタープライズグレードの組織管理の強化と、開発を効率化するための AI 支援型デバッグ・ワークフローの統合。 | マルチワークスペース対応の Organization Support と、Claude 搭載の MLflow Assistant のリリース。 |
| Arize Phoenix | RAG やエージェント・ワークフローの OpenInference 標準に深く統合された、オープンソース・ファーストの Observability および Eval プラットフォーム。 | エージェントの挙動（ツール選択、誠実性）に関する特定のメトリクスによる Eval 精度の向上と、モデルサポートの拡大。 | 最近のリリースは、新しい Eval（Faithfulness, Tool Selection）と精度メトリクスに重点を置いている。 |

## 5. エンタープライズ・シグナル

- LangSmith が Self-Hosted v0.13 をリリースし、エンタープライズ・インフラ制御へのコミットメントを強化。
- MLflow が Organization Support を導入し、大規模チーム向けのマルチワークスペース管理を可能に。
- Langfuse が新しい Organization Audit Log Viewer をリリースし、セキュリティとコンプライアンスの可視性を向上。
- Braintrust は、データが顧客の VPC 内に留まることを保証するセルフホスト型のデータプレーン・アーキテクチャで差別化を継続。
- Arize Phoenix が設定可能な OAuth2 メール抽出機能を追加し、エンタープライズ ID プロバイダーとの統合を改善。
