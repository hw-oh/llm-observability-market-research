# 週刊 LLM Observability 市場調査レポート
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-12

## 1. エグゼクティブ・サマリー

- LangSmith は Self-Hosted v0.13 をリリースし、オンプレミスおよび VPC 環境のデプロイメントオプションを拡張して、データ主権の要件への対応を強化しました。
- Langfuse は Org Audit Log Viewer を導入し、エンタープライズ層のユーザーに必要なセキュリティの可視性とコンプライアンス追跡機能を追加しました。
- MLflow は、Databricks エコシステム内の大規模なエンジニアリングチームを対象に、マルチワークスペース管理を容易にする Organization Support を開始しました。
- W&B Weave は、SOC 2 および HIPAA への準拠姿勢を強調しつつ、マルチモーダルな Tracing と Eval のための GUI ベースの Judge Wizard を提供しています。
- Arize Phoenix は、DSPy サポートや、リトリーバルチャンクおよび関数呼び出し（function calls）の専用可視化機能により、RAG Eval における地位を固めています。
- Braintrust は、コントロールプレーンと顧客 VPC 内のデータを分離するハイブリッドデプロイメントモデルを引き続き活用し、セキュリティを重視する組織にアピールしています。

**AI による市場インサイト**:

> LangGraph を介したエージェント実行グラフの LangSmith による市場をリードする可視化は、複雑なエージェントワークフローのデバッグにおいて、階層ツリー構造に依存する Weave に対する直接的な挑戦となっています。
> LangSmith Self-Hosted v0.13 と Langfuse の Org Audit Log Viewer のリリースは、Weave に対し、独自の専用クラウドとコンプライアンス機能を積極的にマーケティングするよう競争圧力を高めています。
> Arize Phoenix によるオフライン Eval のための DSPy 統合や、MLflow のノーコード Judge Wizard は、Weave がマルチモーダル Tracing 機能をさらに活用しない限り、Eval ワークフローにおける Weave の差別化を損なう恐れがあります。

## 3. 新機能 (過去30日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: オーディオ対応 LLM を使用して、テキストと並行してオーディオ出力を監視および判定するモニターの作成をサポート。(2026-02-01, Evaluation & Quality)
- **Dynamic Leaderboards**: 豊富なカスタマイズ、フィルタリング、CSV エクスポート機能を備えた、Eval 内の自動生成リーダーボード。(2026-01-29, Evaluation & Quality)
- **Custom LoRAs in Playground**: Weave Playground で、カスタム Fine-tuning された LoRA の重みを直接テストおよび比較する機能をサポート。(2026-01-16, Development Lifecycle)

### [LangSmith](https://changelog.langchain.com)
- **Customize trace previews**: LangSmith UI で Trace プレビューをカスタマイズする機能。(2026-02-06, Integration & DX)
- **Google Gen AI Wrapper**: Python および JS SDK における Google Gen AI (Gemini) の新しいラッパーサポート。(2026-02-02, Integration & DX)
- **LangSmith Self-Hosted v0.13**: LangSmith プラットフォームのセルフホスト版のアップデート。(2026-01-16, Enterprise & Infrastructure)

### [Langfuse](https://langfuse.com/changelog)
- **Single Observation Evals**: フル Trace だけでなく、個別の Observation に対して Eval を実行する機能をサポート。(2026-02-12, Evaluation & Quality)
- **Reasoning Trace Rendering**: Trace 詳細において、思考/推論プロセス（推論モデルなど）をレンダリングする新しい UI 機能。(2026-02-12, Core Tracing & Logging)
- **Org Audit Log Viewer**: Dashboard 内の組織レベルの監査ログ専用ビューアー。(2026-02-12, Enterprise & Infrastructure)
- **Inline Trace Comments**: コラボレーションのため、Trace 内の入出力データの一部にインラインでコメントを追加する機能。(2026-02-12, Integration & DX)
- **Trace Corrections**: Trace および Observation のプレビューに修正を追加し、データセットのキュレーションを強化するワークフロー。(2026-02-12, Evaluation & Quality)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Sub-agent nesting for Claude Agent**: Claude Agent SDK ラッパー内でのサブエージェントのネストサポートを追加。(2026-02-05, Agent & RAG Specifics)
- **Classifications field**: 分類を容易にするため、Trace/Span に新しい分類フィールドを追加。(2026-01-31, Core Tracing & Logging)
- **Evaluation Cache Control**: Eval 実行中にキャッシュをオフにするオプションを追加。(2026-01-29, Evaluation & Quality)
- **Trace Scoring Candidate**: Python SDK において Trace の Scoring を行うための候補（candidate）機能を導入。(2026-01-21, Evaluation & Quality)
- **Playground Trace Scorer**: Playground 内の JS Trace Scorer 機能を修正し有効化。(2026-01-21, Evaluation & Quality)
- **Facet Typespecs**: データ処理を改善するための新しい Facet 型仕様を導入。(2026-01-15, Core Tracing & Logging)

### [MLflow](https://mlflow.org/releases)
- **Organization Support**: 異なるワークスペース間で実験やリソースを整理できるマルチワークスペース環境のサポート。(2026-02-12, Enterprise & Infrastructure)
- **MLflow Assistant**: UI 内で直接問題を特定、診断、修正するのを支援する、Claude Code を活用した製品内チャットボット。(2026-01-29, Integration & DX)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 Support**: Playground に Claude Opus 4.6 モデルのサポートを追加。(2026-02-09, Development Lifecycle)
- **Tool Selection Evaluator**: 両方のライブラリに不足していた tool_selection エバリュエーターを追加。(2026-02-06, Evaluation & Quality)
- **Faithfulness Evaluator**: FaithfulnessEvaluator を追加し、HallucinationEvaluator を非推奨化。(2026-02-02, Evaluation & Quality)
- **Tool Invocation Accuracy Metric**: ツール呼び出しの精度を追跡するメトリクスを追加。(2026-02-02, Analytics & Dashboard)
- **OAuth2 Email Configuration**: OAuth2 における構成可能なメール抽出のための EMAIL_ATTRIBUTE_PATH を追加。(2026-01-28, Enterprise & Infrastructure)
- **LLM Classification Evaluators**: 新しい組み込みメトリクス（LLM 分類エバリュエーター）を作成するための cursor rule を追加。(2026-01-21, Evaluation & Quality)

| Arize Phoenix | LLM アプリケーション（特に RAG とエージェント）の Tracing と Eval のための、開発者優先のオープンソース Observability 標準。 | エージェントワークフロー（ツール選択、誠実性）の Eval 機能を深め、Playground でのモデルサポートを拡張。 | 新しいエバリュエーター（Faithfulness、Tool Selection）とツール呼び出し精度のメトリクスに重点を置いた最近のリリース。 |

## 5. エンタープライズ・シグナル

- LangSmith は Self-Hosted v0.13 をリリースし、オンプレミスおよび VPC デプロイメントオプションに対する市場の需要を再確認しました。
- MLflow は Organization Support を導入し、大規模なエンタープライズチーム向けのマルチワークスペース管理を可能にしました。
- Langfuse は Org Audit Log Viewer を追加し、エンタープライズのコンプライアンスとセキュリティの可視性要件に直接対応しました。
- Braintrust は、セキュリティを重視する組織にアピールするため、ハイブリッドデプロイメントモデル（クラウドにコントロールプレーン、VPC にデータ）を引き続き活用しています。
- W&B Weave は SOC 2、HIPAA、GDPR への準拠を強調し、規制の厳しい業界向けの安全な選択肢として自社を位置づけています。
