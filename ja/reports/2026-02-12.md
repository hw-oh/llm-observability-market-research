# 週刊 LLM Observability 市場調査レポート
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-12

## 1. エグゼクティブ・サマリー

- MLflow は、サービス間のコンテキスト伝播のための分散 Tracing 機能と、Eval 設定を簡素化するビジュアルな Judge Builder UI をリリースしました。
- LangSmith は、プラットフォームをセルフホスト版 v0.13 にアップデートし、LangGraph との深い統合を通じてエージェントの可視性を強化しました。
- W&B Weave は、マルチモーダル Eval のための Audio Monitors と、コストとパフォーマンスのメトリクスを可視化する Dynamic Leaderboards を導入しました。
- Braintrust は、長時間実行されるワークフローを Tracing するための Temporal との統合を追加し、Trace 内のメディア添付ファイルのレンダリングを可能にしました。
- Langfuse は、エンタープライズガバナンス向けに Organization Audit Log Viewer をローンチし、reasoning/thinking ステップの専用レンダリングサポートを追加しました。
- Arize Phoenix は、サーバーサイドでの自動実験のための Dataset Evaluators と、Tool Selection の信頼性を評価するための専門的な Evaluator をリリースしました。
- MLflow は、レイテンシとツールの効率を監視するための専用の Agent Performance Dashboards をデプロイし、Langfuse は詳細なツールコールフィルタリングを追加しました。

**AI による市場インサイト**:

> MLflow のビジュアルな Judge Builder と Arize Phoenix の Dataset Evaluators は、自動化されたメトリクスの設定を簡素化することで、Weave の Eval における差別化を脅かしています。
> LangSmith の LangGraph 可視化と Langfuse の reasoning ステップのレンダリングは、Weave のエージェントワークフローの Tracing およびツールコール分析と直接競合しています。
> Langfuse の Audit Log Viewer と LangSmith のセルフホスト版 v0.13 は、Weave の現在の RBAC やデプロイ仕様を超える、厳格なエンタープライズガバナンスへの市場のシフトを強調しています。

## 3. 新機能 (直近30日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: オーディオ対応の LLM Judge を使用して、テキストと共にオーディオ出力 (MP3/WAV) を監視および判定するモニター。(2026-02-01, Evaluation Integration)
- **Dynamic Leaderboards**: カスタマイズ、フィルタリング、CSV エクスポートが可能な、Eval から自動生成されるリーダーボード。(2026-01-29, Evaluation Integration)
- **Playground でのカスタム LoRA**: W&B Artifacts からのカスタム Fine-tuning 済み LoRA 重みを、Weave Playground で直接推論や Eval に使用できる機能。(2026-01-16, DevEx / Integration)

### [LangSmith](https://changelog.langchain.com)
- **Trace プレビューのカスタマイズ**: UI での Trace プレビューの表示方法をカスタマイズできる機能。(2026-02-06, Core Observability)
- **Google Gen AI Wrapper**: SDK における Google Gen AI の新しいラッパーサポート。(2026-01-31, DevEx / Integration)
- **LangSmith セルフホスト版 v0.13**: アップデートされたセルフホスト版のリリース。(2026-01-16, Enterprise & Security)

### [Langfuse](https://langfuse.com/changelog)
- **バージョン管理された Dataset での実験実行**: 特定のタイムスタンプで Dataset を取得し、再現性のために過去のバージョンで実験を実行。(2026-02-11, Experiment / Improvement Loop)
- **Single Observation Evals**: Trace 内の単一の Observation に対して Eval を追加する機能のサポート。(2026-02-09, Evaluation Integration)
- **イベントベースの Trace テーブル**: イベント/Observation に基づく新しい Trace テーブルビュー。(2026-02-09, Core Observability)
- **Reasoning/Thinking Trace レンダリング**: Trace 詳細における思考および推論プロセスの視覚的レンダリング。(2026-02-05, Agent / RAG Observability)
- **Org Audit Log Viewer**: 組織レベルの監査ログを表示するための UI。(2026-02-05, Enterprise & Security)
- **Corrected Outputs**: Fine-tuning 用 Dataset を構築するために、Trace ビューで LLM 出力の改善版を直接キャプチャ。(2026-01-14, Experiment / Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Trace レベルの Scoring**: カスタムコードによる Scoring で、マルチステップのワークフローを評価するために実行 Trace 全体にアクセス可能に。(2026-02-01, Evaluation Integration)
- **LangSmith 統合**: LangSmith の Trace を Braintrust にルーティングするための実験的ラッパー。(2026-02-01, DevEx / Integration)
- **Cursor 統合**: Cursor がログを照会し、実験を取得できるようにする MCP サーバー統合。(2026-02-01, DevEx / Integration)
- **添付ファイルのレンダリング**: カスタム Trace ビューで、署名付き URL から画像、動画、オーディオをレンダリング可能に。(2026-02-01, Core Observability)
- **Trace オリジンへのナビゲーション**: Trace から元のプロンプトや Dataset の行へ直接移動。(2026-02-01, Evaluation Integration)
- **集計を伴う単一 Span フィルター**: 単一 Span フィルターと GROUP BY を組み合わせて、集計された Trace 分析を実行。(2026-02-01, Monitoring & Metrics)
- **自動インストルメンテーション (Python, Ruby, Go)**: 主要言語に対するコード記述不要の Tracing サポート。(2026-01-21, DevEx / Integration)
- **Temporal 統合**: Temporal のワークフローとアクティビティの自動 Tracing。(2026-01-21, Agent / RAG Observability)
- **TrueFoundry 統合**: OpenTelemetry を介して TrueFoundry AI Gateway から LLM Trace をエクスポート。(2026-01-21, DevEx / Integration)
- **レビュー用カンバンレイアウト**: フラグが立てられた Span やレビューを管理するためのドラッグ＆ドロップインターフェース。(2026-01-21, Evaluation Integration)

### [MLflow](https://mlflow.org/releases)
- **組織サポート**: 実験やリソースを整理するためのマルチワークスペース環境のサポート。(2026-02-12, Enterprise & Security)
- **MLflow Assistant**: 問題の診断と修正を支援する、Claude Code を搭載した製品内チャットボット。(2026-01-29, DevEx / Integration)
- **Agent Performance Dashboards**: エージェントのレイテンシ、リクエスト数、品質スコアを監視するための構築済みチャート。(2026-01-29, Monitoring & Metrics)
- **MemAlign Judge Optimizer**: 過去のフィードバックから Eval ガイドラインを学習し、Judge の精度を向上させるアルゴリズム。(2026-01-29, Evaluation Integration)
- **Judge Builder UI**: カスタム LLM Judge プロンプトを作成、テスト、エクスポートするためのビジュアルインターフェース。(2026-01-29, Evaluation Integration)
- **継続的オンラインモニタリング**: 本番環境の入力 Trace に対して LLM Judge を自動的に実行。(2026-01-29, Monitoring & Metrics)
- **分散 Tracing**: コンテキスト伝播により、複数のサービスにわたるリクエストを追跡。(2026-01-29, Core Observability)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **OpenAI Responses API タイプのサポート**: Playground およびカスタムプロバイダーでの OpenAI API タイプ (Chat Completions vs Responses) の選択をサポート。(2026-02-12, DevEx / Integration)
- **Dataset Evaluators**: Evaluator を Dataset に直接アタッチし、実験中にサーバーサイドで自動実行。(2026-02-12, Evaluation Integration)
- **Playground 用カスタムプロバイダー**: Playground とプロンプト間で再利用可能なカスタムモデルプロバイダーの集中設定。(2026-02-11, DevEx / Integration)
- **Claude Opus 4.6 サポート**: 拡張された thinking パラメータを備えた Anthropic の Claude Opus 4.6 モデルのサポート。(2026-02-09, Core Observability)
- **Tool Selection & Invocation Evaluators**: エージェントのツール選択の正確性とパラメータ形式を評価するための専門的な Evaluator。(2026-01-31, Agent / RAG Observability)
- **プロンプト/Dataset 用 CLI コマンド**: ターミナルからプロンプト、Dataset、実験を管理するための新しい CLI コマンド。(2026-01-22, DevEx / Integration)
- **Trace からの Dataset 作成**: Span の関連付けを維持したまま、本番 Trace を Dataset に変換。(2026-01-21, Evaluation Integration)
- **Trace と共にアノテーションをエクスポート**: オフライン分析のために、Trace とそのアノテーションを併せてエクスポートするための CLI サポート。(2026-01-19, DevEx / Integration)

## 4. ポジショニングの変化

| 製品 | 現在 | 今後の方向性 | シグナル |
|---|---|---|---|
| W&B Weave | ML ライフサイクルに深く統合された、開発者中心の Observability および Eval プラットフォーム。 | テキストを超えてマルチモーダル Eval (Audio) や自動化された動的レポートへと拡大。 | Audio Monitors と Dynamic Leaderboards のリリースは、複雑なマルチモーダルエージェント対応への推進を示唆。 |
| LangSmith | LangChain エコシステムに深く根ざした、フレームワークに依存しない主要な Observability および Eval プラットフォーム。 | デプロイ (Agent Servers) やノーコード構築 (Agent Builder) へと拡大し、AI ライフサイクルのオールインワン・プラットフォームへ。 | Agent Servers、Agent Builder、AI アシスタント「Polly」のローンチは、単なる Observability を超えたシフトを示唆。 |
| Langfuse | 完全なデータ制御と深い Observability を必要とするチーム向けの、主要なオープンソース LLM エンジニアリングプラットフォーム。 | ClickHouse を活用した分析と高度な Eval ワークフローにより、大規模なエンタープライズ向けエージェントワークロードをサポート。 | スケールのための ClickHouse 統合、高度なエージェント Tracing (reasoning ステップ) のリリース、監査ログなどのエンタープライズ機能。 |
| Braintrust | 「計測・評価・改善」のループに焦点を当てた、開発者ファーストの Eval および Observability プラットフォーム。 | 複雑なエージェントワークフロー (Temporal, Workflows) のサポートを深め、開発環境 (Cursor, MCP) に直接統合。 | Temporal 統合、Cursor/MCP サポートの最近のリリース、および「Agents」から「Workflows」への名称変更は、複雑なオーケストレーションサポートへのシフトを示唆。 |
| MLflow | 主要なオープンソース MLOps プラットフォームから、包括的な GenAI およびエージェント Observability スイートへと完全に拡張。 | 実験と本番監視のギャップを埋める、AI エージェントのためのデフォルトの「オールインワン」OS へ。 | v3.9.0 における分散 Tracing、MemAlign、継続的オンラインモニタリングなどの専門的なエージェント機能のリリース。 |
| Arize Phoenix | オフラインの実験と本番監視を橋渡しする、包括的なオープンソースファーストの Observability および Eval プラットフォーム。 | CLI ツールや高度なエージェント Eval 機能を通じて、開発者ワークフローへのより深い統合へ。 | 専門的な Tool Evaluator、広範な CLI コマンド、Dataset Evaluator の最近のリリースは、エージェントの信頼性と開発効率への注力を示唆。 |

## 5. エンタープライズ・シグナル

- Langfuse は Organization Audit Log Viewer を導入し、エンタープライズチーム向けのガバナンスを強化しました。
- MLflow は、マルチワークスペース環境と認証を処理するための Organization Support をリリースしました。
- LangSmith はセルフホスト版 (v0.13) をアップデートし、オンプレミスのエンタープライズニーズへのコミットメントを強化しました。
- Braintrust は Temporal との統合を追加し、複雑なエンタープライズグレードのワークフローオーケストレーションのサポートを表明しました。
