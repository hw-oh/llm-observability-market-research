# 週刊 LLM Observability 市場調査レポート
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-12

## 1. エグゼクティブ・サマリー

- LangSmith は、より厳格な地域のコンプライアンス要求に応えるため、Self-Hosted v0.13 をリリースし、EU データレジデンシーのサポートを導入しました。
- Langfuse は Organization Audit Log Viewer をローンチし、ガバナンスを重視するチーム向けに完全なセルフホスティングのサポートを継続しています。
- MLflow は、Tracking Server 内でマルチワークスペース管理と PII マスキングを可能にする Organization Support を追加しました。
- Braintrust は、サブエージェントのネスト構造のサポートと OpenAI エージェントとの深い統合により、エージェントの Observability を拡張しました。
- Arize Phoenix は、Tool Selection（ツール選択）と Faithfulness（忠実性）メトリクスに特化した、構築済み Eval ライブラリを導入しました。
- W&B Weave は、深いネスト構造を持つ関数呼び出しの Tracing と、LLM-as-a-judge による Scoring のための動的リーダーボードで差別化を続けています。
- 市場全体としてエージェント対応への広範なシフトが見られ、Weave、LangSmith、Braintrust はいずれも検索ステップやツール呼び出しを明示的に追跡しています。

**AI による市場インサイト**:

> LangSmith の Self-Hosted v0.13 リリースと Langfuse の新しい Audit Log Viewer は、安全で自己管理型のエンタープライズ導入における Weave の競争優位性に直接挑戦するものです。
> Arize Phoenix による専門的な Tool Selection Eval の追加は、現在 Weave が深いトレース可視化で差別化を図っているエージェント・ワークフロー分析のユースケースをターゲットにしています。
> Braintrust のサブエージェント・ネストと OpenAI 統合の新しいサポートは、複雑で階層的なエージェント・アーキテクチャをデバッグするための主要ツールとしての Weave のポジショニングを脅かしています。

## 3. 新機能 (直近30日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: 会話の質を評価するために LLM judge を使用し、オーディオ出力 (MP3/WAV) を Eval するためのサポート。(2026-02-01, Monitoring & Metrics)
- **Dynamic Leaderboards**: Eval から自動生成されるリーダーボード。永続的なカスタマイズと CSV エクスポートが可能。(2026-01-29, Evaluation Integration)
- **Custom LoRAs in Playground**: 推論と比較のために、Weave Playground でカスタム Fine-tuning された LoRA ウェイトを直接使用するサポート。(2026-01-16, Experiment / Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **Client SDK v0.7.1**: Python および JS クライアントライブラリのアップデート。OIDC の修正と依存関係の更新を含む。(2026-02-10, DevEx / Integration)
- **Customize trace previews**: UI での Tracing プレビューのレンダリング方法をカスタマイズする新機能。(2026-02-06, Core Observability)
- **LangSmith Self-Hosted v0.13**: セルフホスト導入オプションの新しいバージョンリリース。(2026-01-16, Security & Governance)

### [Langfuse](https://langfuse.com/changelog)
- **Single Observation Evals**: 単一の Observation に対して Eval を実行するサポートを追加。(2026-02-12, Evaluation Integration)
- **Events-based Trace Table**: イベント/Observation に基づく Tracing の新しい UI ビュー。(2026-02-12, Core Observability)
- **Reasoning Trace Rendering**: Tracing 詳細における思考/推論プロセス（推論モデルなど）のレンダリングをサポート。(2026-02-12, Core Observability)
- **Org Audit Log Viewer**: 組織レベルの監査ログ用の新しいビューアー。(2026-02-12, Security & Governance)
- **Inline Trace Comments**: Tracing 内の IO データの断片に対してインラインでコメントを追加する機能。(2026-02-12, DevEx / Integration)
- **Corrections in Trace Preview**: Tracing および Observation のプレビューで修正を直接表示・管理する機能を追加。(2026-02-12, Experiment / Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **OpenAI Agents Integration**: OpenAI エージェント統合のすべてのスパンタイプを処理するように SDK を更新。(2026-02-05, DevEx / Integration)
- **Sub-agent Nesting**: Claude Agent SDK ラッパー専用のサブエージェント・ネストのサポートを追加。(2026-02-05, Agent / RAG Observability)
- **Review Span Type**: Tracing 内で直接ヒューマンレビュー・ワークフローをサポートするための、新しい 'review' スパンタイプを導入。(2026-02-05, Evaluation Integration)
- **Classifications Field**: SDK に classifications フィールドを追加し、構造化データのキャプチャを強化。(2026-01-31, Core Observability)
- **Eval Cache Control**: Eval 中およびスパンエクスポート後のキャッシュをオフにするオプションを追加。(2026-01-29, Evaluation Integration)
- **Python Trace Scoring**: Python SDK 内での Tracing Scoring の候補サポートを導入。(2026-01-21, Evaluation Integration)
- **Workflow Renaming**: 一般的な実行グラフをより適切に反映させるため、SDK 内で 'agents' を 'workflows' に名称変更。(2026-01-15, DevEx / Integration)

### [MLflow](https://mlflow.org/releases)
- **Organization Support**: MLflow Tracking Server におけるマルチワークスペース環境のサポート。異なるワークスペース間での実験やリソースの整理が可能に。(2026-02-12, Security & Governance)
- **MLflow Assistant**: MLflow UI 内で直接問題を特定、診断、修正するのを支援する、Claude Code を活用した製品内チャットボット。(2026-01-29, DevEx / Integration)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 Support**: Playground に Claude Opus 4.6 モデルを追加。(2026-02-09, DevEx / Integration)
- **Tool Selection Evaluator**: ライブラリに不足していた tool_selection Eval を追加。(2026-02-06, Evaluation Integration)
- **Faithfulness Evaluator**: FaithfulnessEvaluator を追加し、HallucinationEvaluator を非推奨化。(2026-02-02, Evaluation Integration)
- **Tool Invocation Accuracy Metric**: ツール呼び出しの精度を追跡するための特定のメトリクスを追加。(2026-01-27, Monitoring & Metrics)
- **LLM Classification Evaluators**: LLM 分類のための新しい組み込みメトリクスを作成するための cursor rule を追加。(2026-01-21, Evaluation Integration)

## 4. ポジショニングの変遷

| 製品 | 現状 | 今後の方向性 | シグナル |
|---|---|---|---|
| W&B Weave | W&B エコシステム内で構成可能な AI アプリケーションを構築・評価するための、コードファーストで開発者向けのツールキット。 | プロダクション・グレードのエージェント・システムをサポートするため、マルチモーダル Eval と自動化されたガバナンスへのサポートを拡大。 | 音声エージェント向けの Audio Monitors と、自動モデルランキングのための Dynamic Leaderboards の最近のリリース。 |
| LangSmith | LangChain エコシステムをリードする Observability および Eval プラットフォーム。深い Tracing の可視性とデータセット駆動の開発に焦点を当てている。 | 複雑なエージェント・ワークフローをデバッグするための開発者体験を洗練させつつ、エンタープライズ対応とデプロイの柔軟性を拡大。 | Self-Hosted v0.13 の最近のリリースと、広範な統合サポートのための SDK の継続的なアップデート。 |
| Langfuse | LLM アプリケーションのための、開発者ファーストでオープンソースの Observability および Eval プラットフォーム。 | エンタープライズ・グレードのガバナンスと、高度な Human-in-the-loop ワークフロー。 | Organization Audit Logs、インラインコメント、修正ワークフローの最近の追加。 |
| Braintrust | プロダクションの Tracing とゴールデンデータセットの間のループを閉じることに焦点を当てた、エンタープライズ・グレードの Eval および Observability プラットフォーム。 | 複雑なエージェント階層と自動化されたワークフローへのサポートを深化。 | サブエージェントのネスト、OpenAI エージェント統合の追加、および 'agents' から 'workflows' への名称変更を含む最近の SDK アップデート。 |
| MLflow | オープンソースの実験管理とモデルライフサイクル管理の業界標準。現在は GenAI 向けに大幅に最適化されている。 | AI 支援によるデバッグと組織階層を通じて、エンタープライズ・グレードのマルチテナンシーと開発者の生産性を向上。 | Organization Support (v3.10) と MLflow Assistant (v3.9) のリリース。 |
| Arize Phoenix | OpenTelemetry や Python フレームワークと深く統合された、オープンソース・ファーストでコード中心の Observability プラットフォーム。 | エージェントの Eval 機能を深化させ、包括的な RAG およびエージェント監視のためのモデルサポートを拡大。 | ツール選択 Eval、Faithfulness メトリクス、ツール呼び出し精度に重点を置いた最近のリリース。 |

## 5. エンタープライズ・シグナル

- LangSmith は、デプロイの柔軟性を高めるために Self-Hosted ソリューションの v0.13 をリリースしました。
- Langfuse は、コンプライアンス追跡を改善するために Organization Audit Log Viewer を導入しました。
- MLflow は、Tracking Server でのマルチワークスペース管理を可能にする Organization Support を追加しました。
- Braintrust は、機密データを顧客の VPC 内に保持するセルフホスト型のデータプレーンで差別化を続けています。
