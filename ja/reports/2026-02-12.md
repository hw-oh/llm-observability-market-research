# 週刊 LLM Observability 市場調査レポート
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-12

## 1. エグゼクティブ・サマリー

- LangSmith は、オンプレミス展開をサポートする Self-Hosted v0.13 をリリースし、回帰テスト用のペアワイズ・アノテーション・キューを含むプラットフォームのアップデートを行いました。
- MLflow はバージョン 3.10 で Organization Support を導入し、マルチワークスペース環境を可能にするとともに、Eval 用のビジュアルな Judge Builder UI を追加しました。
- Langfuse は、セキュリティの可視性を高めるための Organization Audit Log Viewer を追加し、Tracing の詳細内で推論（reasoning/thinking）ステップを直接レンダリングできるようになりました。
- Arize Phoenix は、Viewer ロールを含む RBAC を実装し、ツールの選択エラーやパラメータの問題を特定するための専用エバリュエーターをリリースしました。
- Braintrust は、カンバンレイアウトによるヒューマンレビュー機能を拡張し、6つの SDK 言語にわたって o1 などの推論モデルの Tracing サポートを確定しました。
- W&B Weave は、マルチモーダルな Judge を備えたダイナミック・リーダーボードをデプロイし、特にオーディオ Eval 用の新しいモニターを追加しました。

**AI による市場インサイト**:

> Weave のエージェント Observability は、Langfuse や Arize Phoenix が推論ステップのレンダリングやツール選択エラーの検出のための専用機能をリリースしたことで、圧力にさらされています。
> LangSmith と MLflow が堅牢なセルフホスト機能やマルチワークスペース機能をリリースする一方で、Weave には現時点で明示的な PII マスキングの詳細が欠けており、エンタープライズ対応の格差が広がっています。
> Weave は新しいオーディオモニターによってマルチモーダル Eval における差別化の優位性を維持していますが、Braintrust などの競合他社は主にテキストベースの Scoring とヒューマンレビューのワークフローに焦点を当てています。

## 3. 新機能 (直近30日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **オーディオモニター**: LLM Judge を使用して、テキストと並行してオーディオ出力を監視および評価するモニター。(2026-02-01, Evaluation Integration)
- **ダイナミック・リーダーボード**: Eval から自動生成されるリーダーボード。永続的なカスタマイズと CSV エクスポートが可能。(2026-01-29, Evaluation Integration)
- **Playground でのカスタム LoRA**: Weave Playground で直接、カスタム Fine-tuning された LoRA の重みをテストおよび評価する機能をサポート。(2026-01-16, Experiment / Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **Trace プレビューのカスタマイズ**: UI での Trace プレビュー方法をカスタマイズする機能。(2026-02-06, DevEx / Integration)
- **Google Gen AI Wrapper エクスポート**: SDK における Google Gen AI ラッパーのエクスポート機能。(2026-01-31, DevEx / Integration)
- **LangSmith Self-Hosted v0.13**: アップデートされたセルフホスト版のリリース。(2026-01-16, Enterprise & Security)

### [Langfuse](https://langfuse.com/changelog)
- **バージョン管理されたデータセットでの実験実行**: 特定のタイムスタンプでデータセットを取得し、再現性のために過去のバージョンで実験を実行。(2026-02-11, Experiment / Improvement Loop)
- **Trace の修正済み出力**: Fine-tuning 用データセットを構築するために、Trace ビューで LLM 出力の改善版を直接キャプチャ。(2026-01-14, Experiment / Improvement Loop)
- **単一 Observation の Eval**: 単一の Observation に対して Eval を実行する機能をサポート。(2026-02-05, Evaluation Integration)
- **イベントベースの Observation テーブル**: イベントに基づいた Trace/Observation の新しいテーブルビュー。(2026-02-05, Core Observability)
- **推論（Reasoning/Thinking）の Trace レンダリング**: Trace の詳細における思考/推論部分のビジュアルレンダリング。(2026-01-30, Agent / RAG Observability)
- **Org Audit Log Viewer**: 組織の監査ログを表示するための UI。(2026-01-30, Enterprise & Security)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Trace レベルの Scorer**: カスタムコード Scorer が実行 Trace 全体にアクセスし、マルチステップのワークフローやエージェントの動作を評価可能に。(2026-02-01, Evaluation Integration)
- **カスタムビューでの添付ファイルレンダリング**: カスタム Trace ビューで画像、動画、音声を直接レンダリングする機能をサポート。(2026-02-01, Core Observability)
- **LangSmith 統合**: Trace を LangSmith と Braintrust の両方に送信、または Braintrust のみにルーティングする実験的なラッパー。(2026-02-01, DevEx / Integration)
- **Cursor 統合**: MCP サーバーを介した Cursor エディタとの統合により、ログの照会や実験の取得が可能に。(2026-02-01, DevEx / Integration)
- **集計を伴う単一 Span フィルタ**: 単一 Span フィルタを GROUP BY と組み合わせて、集計された Trace 分析を行う機能。(2026-02-01, Monitoring & Metrics)
- **自動インスツルメンテーション (Python, Ruby, Go)**: Python、Ruby、Go アプリケーションのコードレス Tracing サポート。(2026-01-29, DevEx / Integration)
- **Temporal 統合**: 親子関係を持つ Temporal ワークフローとアクティビティの自動 Tracing。(2026-01-21, DevEx / Integration)
- **レビュー用カンバンレイアウト**: フラグが立てられた Span やレビューワークフローを管理するための新しいカンバンビュー。(2026-01-20, Evaluation Integration)
- **Trace ページ上の Loop**: 分析のために、個別の Trace ページで AI アシスタント「Loop」が直接利用可能に。(2026-01-20, Core Observability)
- **TrueFoundry 統合**: OpenTelemetry を介した TrueFoundry AI Gateway との統合。(2026-01-20, DevEx / Integration)

### [MLflow](https://mlflow.org/releases)
- **Organization Support**: 実験やリソースを整理するためのマルチワークスペース環境のサポート。(2026-02-12, Enterprise & Security)
- **MLflow Assistant**: アプリやエージェントのデバッグ、問題解決を支援する Claude Code ベースの製品内チャットボット。(2026-01-29, DevEx / Integration)
- **エージェント・パフォーマンス・Dashboard**: レイテンシ、リクエスト数、品質 Scoring を監視するための構築済みチャート。(2026-01-29, Monitoring & Metrics)
- **MemAlign Judge Optimizer**: フィードバックから評価ガイドラインを学習し、Judge の精度を向上させるアルゴリズム。(2026-01-29, Evaluation Integration)
- **Judge Builder UI**: コードなしでカスタム LLM Judge プロンプトを作成およびテストするためのビジュアルインターフェース。(2026-01-29, Evaluation Integration)
- **継続的オンラインモニタリング**: リアルタイムの品質評価のために、流入する Trace に対して LLM Judge を自動的に実行。(2026-01-29, Evaluation Integration)
- **分散 Tracing**: コンテキスト伝播を使用して、複数のサービスにわたるリクエストを追跡。(2026-01-29, Core Observability)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **OpenAI Responses API タイプのサポート**: Playground で OpenAI および Azure OpenAI の Chat Completions と Responses API タイプを選択可能に。(2026-02-12, DevEx / Integration)
- **データセット・エバリュエーター**: データセットにエバリュエーターを直接アタッチし、実験中にサーバー側で自動実行。(2026-02-12, Evaluation Integration)
- **Playground およびプロンプト用カスタムプロバイダー**: Playground やプロンプトバージョン間で再利用可能なカスタム AI プロバイダーの集中設定。(2026-02-11, DevEx / Integration)
- **Claude Opus 4.6 モデルサポート**: Playground での Anthropic Claude Opus 4.6 モデルのサポートと自動 Cost Tracking。(2026-02-09, Core Observability)
- **ツール選択およびツール呼び出しエバリュエーター**: エージェントのツール選択の正確性と呼び出しパラメータの正しさを判定する専用エバリュエーター。(2026-01-31, Agent / RAG Observability)
- **OAuth2 用の構成可能なメール抽出**: Azure AD などの OAuth2 プロバイダー向けのカスタムメール抽出パス（例：preferred_username）のサポート。(2026-01-28, Enterprise & Security)
- **プロンプト、データセット、実験用 CLI コマンド**: ターミナルからプロンプトやデータセットを管理し、実験を実行するための新しい CLI コマンド。(2026-01-22, DevEx / Integration)
- **Span 関連付けを伴う Trace からのデータセット作成**: ソース Span への双方向リンクを保持したまま、Trace をデータセットに変換。(2026-01-21, Evaluation Integration)
- **Trace と共のアノテーション・エクスポート**: オフライン分析のために Trace と共にアノテーションをエクスポートする CLI サポート。(2026-01-19, DevEx / Integration)
- **AI コーディングアシスタント用 CLI ターミナルアクセス**: AI コーディングアシスタント（Cursor, Windsurf）が Phoenix インスタンスを照会できるようにするための CLI 拡張。(2026-01-17, DevEx / Integration)

## 5. エンタープライズ・シグナル

- MLflow は、マルチワークスペース環境と詳細な権限設定を可能にする「Organization Support」(v3.10) を導入しました。
- LangSmith は Self-Hosted v0.13 をリリースし、オンプレミスおよびハイブリッドなエンタープライズ展開へのコミットメントを強化しました。
- Langfuse は Organization Audit Log Viewer を追加し、セルフホストおよびクラウドユーザーのセキュリティ可視性を向上させました。
- Arize Phoenix は RBAC と Viewer ロールを実装し、より優れたチームガバナンスに向けて前進しました。
- Braintrust はレビュー用のカンバンレイアウトを追加し、大規模なヒューマンアノテーション・ワークフローを合理化しました。
