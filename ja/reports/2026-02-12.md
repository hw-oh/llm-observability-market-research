# 週刊 LLM Observability 市場調査レポート
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-12

## 1. エグゼクティブ・サマリー

- LangSmith は Self-Hosted v0.13 をリリースし、きめ細かな RBAC と監査ログによってエンタープライズ導入向けのオンプレミス機能を強化しました。
- Langfuse は Organization Audit Log Viewers を導入し、オープンソースプラットフォーム内でのガバナンス機能を拡張して、より大規模なエンジニアリングチームをサポートします。
- MLflow は Tracking Server を Organization Support でアップデートし、MLOps 実践者向けにマルチワークスペース環境の集中管理を可能にしました。
- Arize Phoenix は、ツール選択と Faithfulness（誠実性）に特化した Eval をデプロイしました。深い OpenTelemetry 統合を活用し、エージェントのワークフローのデバッグをターゲットとしています。
- Braintrust は、SSO や SOC 2 コンプライアンスを提供しつつ、顧客の VPC 内にデータを保持するハイブリッドセルフホスティングモデルを通じて、インフラの差別化を継続しています。
- W&B Weave は、SOC 2 コンプライアンスとエクスポート可能な監査ログを通じてエンタープライズ対応を確認し、デコレータによるネストされたスパンの Tracing に対する堅牢なサポートを維持しています。

**AI による市場インサイト**:

> Arize Phoenix がツール選択に特化した Eval を導入したことは、サブエージェントの意思決定に関する詳細なメトリクスを提供することで、Weave のエージェント Observability における差別化に挑戦しています。
> LangSmith の Self-Hosted v0.13 のリリースと Langfuse の新しい Audit Log Viewers は、ガバナンスとデプロイの柔軟性の差を埋めることで、Weave のエンタープライズ層への競争圧力を高めています。
> MLflow が Tracking Server に Organization Support を追加したことは、すでに Databricks エコシステムに定着しているチームにとって、Weave に代わる統合 MLOps の選択肢としての地位を強化します。

## 3. 新機能 (直近30日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: LLM ジャッジを使用して、テキストと共にオーディオ出力（MP3/WAV）を監視および判定するオンライン Eval モニター。(2026-02-01, Evaluation & Quality)
- **Dynamic Leaderboards**: フィルタ、永続的なカスタマイズ、CSV エクスポート機能を備えた、Eval から自動生成されるリーダーボード。(2026-01-29, Evaluation & Quality)
- **Playground でのカスタム LoRA**: 推論と比較のために、Fine-tuning された LoRA の重みを Playground に持ち込むためのサポート。(2026-01-16, Experiment & Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **Client SDK アップデート (v0.7.x)**: バグ修正と依存関係の更新を含む Python および JS クライアントライブラリのアップデート。(2026-02-10, Developer Experience & Integration)
- **Trace プレビューのカスタマイズ**: LangSmith UI 内での Trace のプレビュー方法をカスタマイズする新機能。(2026-02-06, Core Tracing & Logging)
- **LangSmith Self-Hosted v0.13**: セルフホスト型のエンタープライズデプロイオプションの新しいバージョンリリース。(2026-01-16, Infrastructure & Enterprise)

### [Langfuse](https://langfuse.com/changelog)
- **Single Observation Evals**: 単一の Observation に対して Eval を実行するサポートを追加。(2026-02-12, Evaluation & Quality)
- **イベントベースの Trace テーブル**: 視認性向上のため、Trace および Observation テーブルをイベントベースにリファクタリング。(2026-02-12, Core Tracing & Logging)
- **Reasoning/Thinking Trace のレンダリング**: Trace 詳細における思考および推論部分（推論モデルなど）のレンダリングサポートを追加。(2026-02-12, Agent & RAG Observability)
- **Org Audit Log Viewer**: 組織レベルの監査ログ用の新しいビューアー。(2026-02-12, Infrastructure & Enterprise)
- **インライン Trace コメント**: Trace 内の IO データの断片に対してインラインでコメントを追加可能に。(2026-02-12, Developer Experience & Integration)
- **Trace プレビューでの修正**: Trace および Observation プレビューで修正を直接表示および管理するサポートを追加。(2026-02-12, Experiment & Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Claude Agent SDK のサブエージェントネスト**: Claude Agent SDK ラッパー内でのネストされたサブエージェント Tracing のサポートを追加。(2026-02-12, Agent & RAG Observability)
- **Review スパンタイプ**: 人間によるレビューワークフローをサポートするため、SDK に新しい「Review」スパンタイプを導入。(2026-02-12, Evaluation & Quality)
- **Classifications フィールド**: スパンの構造化されたカテゴリ化をサポートするため、Trace に「Classifications」フィールドを追加。(2026-01-31, Core Tracing & Logging)
- **Eval 時のキャッシュ無効化**: 最新の結果を保証するために、Eval 実行中にキャッシュをオフにする新しいオプション。(2026-01-29, Evaluation & Quality)
- **ワークフロー用語の変更**: より広範なユースケースを反映させるため、SDK および UI で「agents」を「workflows」に名称変更。(2026-01-15, Developer Experience & Integration)

### [MLflow](https://mlflow.org/releases)
- **MLflow Tracking Server での Organization Support**: 異なるワークスペース間で実験やリソースを整理できるマルチワークスペース環境のサポート。(2026-02-12, Infrastructure & Enterprise)
- **MLflow Assistant**: UI 内で直接問題を特定、診断、修正するのを支援する、Claude Code を活用した製品内チャットボット。(2026-01-29, Developer Experience & Integration)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Playground での Claude Opus 4.6**: Playground 環境内での Claude Opus 4.6 モデルのサポートを追加。(2026-02-09, Experiment & Improvement Loop)
- **Tool Selection Evaluator**: エージェントにおけるツール選択の正確性を評価するための新しい Eval を追加。(2026-02-06, Evaluation & Quality)
- **Faithfulness Evaluator**: 回答の根拠を確認するための FaithfulnessEvaluator を導入し、HallucinationEvaluator を非推奨化。(2026-02-02, Evaluation & Quality)
- **Tool Invocation Accuracy メトリクス**: ツール呼び出しの精度を測定するための新しいメトリクス。(2026-02-02, Evaluation & Quality)
- **設定可能な OAuth2 メール抽出**: OAuth2 のメール抽出を設定可能にするための EMAIL_ATTRIBUTE_PATH を追加。(2026-01-28, Infrastructure & Enterprise)
- **メトリクス作成用の Cursor ルール**: 新しい組み込み LLM 分類 Eval の作成を容易にするための Cursor ルールを追加。(2026-01-21, Evaluation & Quality)

| LangSmith | 複雑な LLM アプリケーション、特に LangChain を使用する開発者向けの主要な Observability および Eval プラットフォーム。 | LangChain 以外の幅広い LLM ワークフローをサポートするため、エンタープライズ対応とエコシステムの互換性を拡張。 | セルフホスト版の最近のアップデートや、LangChain 以外の SDK ラッパーは、より広範なエンタープライズおよびエコシステムへの普及を狙っている。 |
| Langfuse | 開発者向けの詳細な Tracing と Eval に重点を置いた、主要なオープンソース LLM エンジニアリングプラットフォーム。 | エンタープライズガバナンスの強化と、高度な推論モデルのサポートへの移行。 | 組織監査ログの最近のリリースと、推論モデルの「思考」部分のレンダリングサポート。 |

- Braintrust は、顧客の VPC 内にデータを保持するハイブリッドセルフホスティングモデルで差別化を続けています。
