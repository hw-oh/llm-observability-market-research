# Weekly LLM Observability 市場調査レポート
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-12

## 1. エグゼクティブ・サマリー

- LangSmith は、オンプレミス要件をサポートするために Self-Hosted v0.13 をリリースし、自動回帰検知のためのペアワイズ・アノテーション・キューを導入しました。
- MLflow は、マルチワークスペースの組織サポートによりエンタープライズ機能セットを拡張し、ビジュアルな Judge Builder と MemAlign オプティマイザーを追加しました。
- Langfuse は、コンプライアンス監視を強化するために Organization Audit Log Viewer をローンチし、バッチデータセット実行のためのマネージド LLM-as-a-judge 機能を有効にしました。
- W&B Weave は、マルチモーダルな LLM-as-a-judge ワークフローをサポートするために、ダイナミック・リーダーボードと新しいオーディオモニターをデプロイしました。
- Braintrust は、Temporal ワークフローとトレースレベルのスコアラーによる深い階層型 Tracing をサポートすることで、コードファーストのポジショニングを固めました。
- Arize Phoenix は、エージェントシステムにおけるツール選択の正確性とパラメータ呼び出しを判定するために特別に設計された、専用の Eval 機能を導入しました。

**AI による市場インサイト**:

> LangSmith と Langfuse は、エージェントの推論のための専用ワークフローグラフ可視化を提供しており、メモリや状態グラフの特定のビューを欠いている Weave の現在のエージェント Observability ツールキットとの差を浮き彫りにしています。
> LangSmith の v0.13 セルフホスト版と Langfuse の Audit Log Viewer のリリースは、Weave に対し、エンタープライズ・コンプライアンスのための PII マスキングや監査機能の公開ドキュメント化を求める圧力を強めています。
> MLflow の新しいビジュアル Judge Builder と MemAlign オプティマイザーは、Weave のコード中心の Eval ワークフローに対してローコードの代替案を提供することで、Eval 統合における Weave の優位性に挑戦しています。

## 3. 新機能 (直近 30 日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **オーディオモニター**: LLM judge を使用してオーディオ出力 (MP3/WAV) を観察および判定するモニターを作成する機能。 (2026-02-01, Eval 統合)
- **ダイナミック・リーダーボード**: 永続的なカスタマイズと CSV エクスポート機能を備えた、Eval から自動生成されるリーダーボード。 (2026-01-29, Eval 統合)
- **Playground でのカスタム LoRA**: W&B Artifacts からのカスタム Fine-tuning 済み LoRA 重みを Weave Playground で直接使用する機能。 (2026-01-16, 実験 / 改善ループ)

### [LangSmith](https://changelog.langchain.com)
- **クライアントライブラリ v0.7.1**: プラットフォームに接続するための Python および JS クライアントライブラリのアップデート。 (2026-02-10, DevEx / 統合)
- **トレースプレビューのカスタマイズ**: UI でのトレースプレビューの表示方法をカスタマイズする新機能。 (2026-02-06, コア Observability)
- **Google Gen AI ラッパー**: SDK における Google Gen AI ラッパーのサポートとエクスポート。 (2026-01-31, DevEx / 統合)
- **Self-Hosted v0.13**: セルフホスト・デプロイオプションの新しいバージョンリリース。 (2026-01-16, エンタープライズ & セキュリティ)

### [Langfuse](https://langfuse.com/changelog)
- **バージョン付きデータセットでの実験実行**: 特定のタイムスタンプでデータセットを取得し、再現性のために過去のバージョンで実験を実行。 (2026-02-11, 実験 / 改善ループ)
- **単一 Observation の Eval**: 単一の Observation に直接 Eval を追加する機能。 (2026-02-05, Eval 統合)
- **思考 / 推論部分のレンダリング**: トレース詳細における Chain-of-thought/推論ステップの可視化。 (2026-01-30, エージェント / RAG Observability)
- **組織監査ログビューア**: 組織レベルの監査ログを表示するための UI。 (2026-01-30, エンタープライズ & セキュリティ)
- **トレースの修正済み出力**: Fine-tuning データセットを構築するために、トレースビューで改善されたバージョンの LLM 出力をキャプチャ。 (2026-01-14, 実験 / 改善ループ)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **トレースレベルのスコアラー**: カスタムコードスコアラーが実行トレース全体にアクセスし、多段階のワークフローやエージェントの動作を評価できるようになりました。 (2026-02-01, Eval 統合)
- **LangSmith 統合**: Tracing および Eval のコールを LangSmith と Braintrust の両方に送信するか、Braintrust のみにルーティングするラッパー。 (2026-02-01, DevEx / 統合)
- **Cursor 統合**: MCP サーバーを介した Cursor IDE との統合により、エディタ内で直接ログの照会や実験結果の取得が可能。 (2026-02-01, DevEx / 統合)
- **カスタムビューでの添付ファイルレンダリング**: カスタムトレースビューで画像、ビデオ、オーディオを直接レンダリングするサポート。 (2026-02-01, コア Observability)
- **自動インスツルメンテーション**: Python、Ruby、Go アプリケーションのゼロコード Tracing サポート。 (2026-01-29, DevEx / 統合)
- **Temporal 統合**: 親子関係のマッピングを伴う Temporal ワークフローとアクティビティの自動 Tracing。 (2026-01-21, DevEx / 統合)
- **TrueFoundry 統合**: OpenTelemetry を介して TrueFoundry AI Gateway から LLM トレースをエクスポートする統合。 (2026-01-21, DevEx / 統合)
- **レビュー用カンバンレイアウト**: フラグが立てられたスパンとレビュー・ステータスを管理するための新しいカンバンボードビュー。 (2026-01-21, Eval 統合)
- **トレースページ上の Loop**: AI アシスタント「Loop」が、分析とデバッグのために個別のトレースビューで直接利用可能になりました。 (2026-01-21, コア Observability)
- **生のトレースデータの表示**: 個別のスパンまたはトレースの完全な JSON 表現を表示および検索する機能。 (2026-01-21, コア Observability)

### [MLflow](https://mlflow.org/releases)
- **組織サポート**: 実験とリソースを整理するためのマルチワークスペース環境のサポート。 (2026-02-12, エンタープライズ & セキュリティ)
- **MLflow Assistant**: コンテキストを認識して問題をデバッグおよび修正するための、Claude Code を搭載した製品内チャットボット。 (2026-01-29, DevEx / 統合)
- **エージェントパフォーマンス Dashboard**: レイテンシ、リクエスト数、品質スコア、ツール使用率を監視するための構築済みチャート。 (2026-01-29, モニタリング & メトリクス)
- **MemAlign Judge オプティマイザー**: フィードバックから評価ガイドラインを学習し、Judge の精度を向上させるアルゴリズム。 (2026-01-29, Eval 統合)
- **Judge Builder UI**: コードなしでカスタム LLM judge プロンプトを作成、テスト、エクスポートするためのビジュアルインターフェース。 (2026-01-29, Eval 統合)
- **継続的なオンラインモニタリング**: リアルタイムの品質評価のために、本番環境の着信トレースに対して LLM judge を自動的に実行。 (2026-01-29, Eval 統合)
- **分散 Tracing**: エンドツーエンドの可視化のために、コンテキスト伝播を使用して複数のサービスにわたるリクエストを追跡。 (2026-01-29, コア Observability)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **OpenAI Responses API タイプサポート**: Playground で Chat Completions と Responses API タイプの選択をサポート。 (2026-02-12, DevEx / 統合)
- **データセット Eval 機能**: データセットに Eval 機能を直接アタッチし、実験中にサーバー側で自動的に実行。 (2026-02-12, Eval 統合)
- **Playground 用カスタムプロバイダー**: Playground 全体で再利用可能なカスタム AI プロバイダー (OpenAI、Azure、Anthropic など) の集中設定。 (2026-02-11, DevEx / 統合)
- **Claude Opus 4.6 サポート**: 拡張思考パラメータと Cost Tracking を備えた Claude Opus 4.6 の Playground サポート。 (2026-02-09, コア Observability)
- **ツール選択 & 呼び出し Eval 機能**: エージェントのツール選択精度とパラメータ呼び出しの正しさを判定する専用の Eval 機能。 (2026-01-31, エージェント / RAG Observability)
- **Phoenix CLI コマンド**: ターミナルからプロンプト、データセット、実験を管理するための新しい CLI コマンド。 (2026-01-22, DevEx / 統合)
- **スパン ID を伴うトレースからデータセットへの変換**: ソーススパンへの双方向リンクを保持したまま、トレースをデータセットに変換。 (2026-01-21, Eval 統合)
- **トレースとアノテーションのエクスポート**: 手動および自動アノテーションとともにトレースをエクスポートするための CLI サポート。 (2026-01-19, Eval 統合)
- **CLI ターミナルアクセス**: AI コーディングアシスタントがターミナルコマンドを介して Phoenix データに直接照会できるようにします。 (2026-01-17, DevEx / 統合)

## 5. エンタープライズ・シグナル

- LangSmith は Self-Hosted v0.13 をリリースし、オンプレミス・デプロイオプションの需要を再確認しました。
- MLflow は、エンタープライズリソースとアクセスをより適切に管理するために、マルチワークスペースの組織サポートを導入しました。
- Langfuse は Organization Audit Log Viewer を追加し、コンプライアンスとセキュリティの監視機能を強化しました。
- Braintrust と MLflow は、高セキュリティ環境に対応するため、セルフホスティングと VPC デプロイを強調しました。
