# LLM Observability 市場調査週報
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-12

## 1. エグゼクティブ・サマリー

- Weave は2月1日の Audio Monitors のリリースにより、ボイスエージェントのオブザーバビリティにおいて先行者利益を確立しました。これは、マルチモーダル評価の競争において、Langfuse や Arize Phoenix といったテキスト中心の競合他社を上回る動きです。
- Braintrust は、Java、Go、C# の SDK を追加することで、レガシーなエンタープライズ・スタックを積極的にターゲットにしており、Python/TS 以外の環境における Weave の参入障壁を大きく高めています。
- Weave の「データ・フライホイール」による差別化が強化されています。Playground でのカスタム LoRA のホットスワップ機能（1月16日）は、ネイティブなトレーニング・レジストリを持たない MLflow や LangSmith では再現できない実験ループを提供します。
- ガバナンスへの圧力が高まっています。MLflow が「Organization Support（組織サポート）」（2026年2月）をリリースしたことで、マルチチーム・テナンシーの新たな基準が設定されました。Weave が大規模なプラットフォーム契約を獲得するには、これに匹敵する機能が必要です。
- Weave はコントロールプレーンにおいて依然として脆弱です。キャッシングやレート制限のためのネイティブな AI Proxy が欠如しているため、本番環境のコスト管理が主要な KPI となるアーキテクチャ決定において、Braintrust に敗北するリスクがあります。
- Dynamic Leaderboards（1月29日）のローンチにより、評価レポート作成における手動の摩擦を解消することに成功し、これまで LangSmith のランキング表示が持っていたユーザビリティ上の優位性を中和しました。
- 戦略的注視事項：エンジニアリング部門は「自動評価者（Judge）最適化」機能を優先すべきです。MLflow の新しい「Judge Builder」は評価のチューニングを自動化しており、Weave の手動による評価設定が時代遅れに見えるリスクがあります。

> **一言評価**: Weave はマルチモーダルおよびトレーニングループのオブザーバビリティで市場をリードしていますが、Braintrust の優れたプロキシ・アーキテクチャと幅広い SDK サポートにより、高価値なエンタープライズ・プラットフォーム契約を失うリスクがあります。

### Weave の主な強み

- **マルチモーダル・オブザーバビリティ:** 最近リリースされた Audio Monitors（2026年2月）により、Langfuse などのテキスト中心の競合他社と比較して、ボイスエージェントの評価において明確な優位性を持っています。
- **トレーニングから推論へのリネージ:** Arize Phoenix や LangSmith とは異なり、Weave は本番環境のトレースを W&B Registry 内のトレーニング実行やアーティファクトにネイティブにリンクさせ、真の反復的な改善ループを可能にします。
- **サーバーレス LoRA 統合:** Playground（2026年1月）でカスタム微調整済みアダプターをホットスワップできる機能は、MLflow や Braintrust には欠けている迅速な実験能力を提供します。

### Weave の改善が必要な領域

- **AI Proxy の欠如:** Weave にはキャッシング、レート制限、キー管理のための仲介プロキシ層がありません。これは現在 Braintrust がリードしている主要な価値提案です。
- **人間によるアノテーション・ワークフロー:** LangSmith と Braintrust は、手動データラベル付けのための優れた専用の「アノテーション・キュー」やカンバン形式のビューを提供しています。Weave の UI は大規模な人間によるレビューには最適化されていません。
- **自動評価者（Judge）の最適化:** MLflow の新しい「MemAlign」および「Judge Builder」機能は LLM 評価者のチューニングを自動化しますが、Weave の評価信頼性のためのセットアッププロセスはより手動的です。
- **エンタープライズ SDK の幅:** Braintrust が Java、Go、Ruby、C# をサポートしていることは、Python/TypeScript 専用ではない伝統的なエンタープライズ環境において、Weave の参入障壁となっています。

## 3. 新機能（過去30日間）

### [Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: LLM 評価者を使用して音声の入力/出力を監視・判定するモニター作成をサポートし、ボイスエージェントの評価を可能にしました。(2026-02-01, エージェント観測)
- **Dynamic Leaderboards**: 評価からリーダーボードを自動生成し、永続的なカスタマイズと CSV エクスポートが可能に。手動設定が不要になりました。(2026-01-29, 評価統合)
- **Playground でのカスタム LoRA**: W&B Artifacts からカスタム LoRA の重みを直接 Weave Playground にロードしてテストできる統合機能。(2026-01-16, 実験 / 改善ループ)

### [LangSmith](https://changelog.langchain.com)
- **トレースプレビューのカスタマイズ**: トレースプレビューテーブルに表示される列とデータをカスタマイズする機能。(2026-02-06, DevEx / 統合)
- **Google Gen AI ラッパー**: Google Gen AI モデルのトレースを容易にする新しいラッパーのエクスポート。(2026-01-31, DevEx / 統合)
- **セルフホスト v0.13**: 安定性が向上したセルフホスト版のアップデート。(2026-01-16, エンタープライズ & セキュリティ)

### [Langfuse](https://langfuse.com/changelog)
- **トレースの修正済み出力**: トレースビューで LLM 出力の改善版を直接キャプチャし、微調整用データセットを構築。(2026-01-14, 実験 / 改善ループ)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **LangSmith 統合**: LangSmith のトレースを Braintrust にルーティングする実験的ラッパー。並行利用や移行を可能にします。(2026-02, DevEx / 統合)
- **トレースレベルのスコアラー**: カスタムコードスコアラーが実行トレース全体にアクセスし、マルチステップのワークフローを評価できるようになりました。(2026-02, 評価統合)
- **自動インストルメンテーション (Py/Ruby/Go)**: Python、Ruby、Go アプリケーション向けのコード不要のトレースサポートを追加。(2026-01, DevEx / 統合)
- **Temporal 統合**: Temporal のワークフローとアクティビティを自動トレースし、親子関係をマッピング。(2026-01, エージェント / RAG 観測)
- **レビュー用カンバンレイアウト**: フラグが立てられたスパンや人間によるレビューキューを管理するための新しいドラッグ＆ドロップ・インターフェース。(2026-01, 評価統合)

### [MLflow](https://mlflow.org/releases)
- **Organization Support**: マルチワークスペース環境と組織レベルのリソース管理をサポート。(2026-02-12, エンタープライズ & セキュリティ)
- **MLflow Assistant**: Claude Code を搭載した製品内 AI チャットボット。トレースのデバッグ、テスト作成、エージェント最適化を支援。(2026-01-29, DevEx / 統合)
- **MemAlign Judge Optimizer**: フィードバックから学習し、LLM 評価者のプロンプトと基準を自動的に最適化するアルゴリズム。(2026-01-29, 評価統合)
- **Judge Builder UI**: カスタム LLM 評価者を作成、テスト、検証するためのノーコード・ビジュアルインターフェース。(2026-01-29, 評価統合)
- **エージェント・パフォーマンス・ダッシュボード**: 実験 UI におけるレイテンシ、リクエスト数、品質スコアの事前構築済み可視化。(2026-01-29, モニタリング & メトリクス)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **データセット評価者**: 評価者をデータセットに直接アタッチし、実験中にサーバー側で自動実行。(2026-02-12, 評価統合)
- **Playground 用カスタムプロバイダー**: Playground におけるカスタムモデルプロバイダーとルーティングの集中設定。(2026-02-11, DevEx / 統合)
- **ツール選択 & 呼び出し評価者**: エージェントが適切なツールを選択し、正しいパラメータで呼び出したかを判定する専用評価者。(2026-01-31, エージェント / RAG 観測)
- **プロンプト/データセット用 CLI コマンド**: プロンプトやデータセットのリスト表示、閲覧、他のツールへのパイプ処理を行うターミナルコマンド。(2026-01-22, DevEx / 統合)
- **スパン関連付けを伴うトレースからデータセットへの変換**: 元のソーススパンへの双方向リンクを保持したまま、トレースからデータセットを作成。(2026-01-21, 評価統合)

- MLflow の「Organization Support」（2026年2月）のリリースは、マルチチームのガバナンスを直接ターゲットにしており、テナンシー管理の基準を引き上げています。
