# 週刊 LLM Observability 市場調査レポート
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-12

## 1. エグゼクティブ・サマリー

- LangSmith はセルフホストプラットフォームの v0.13 をリリースし、マルチステップの推論を可視化する LangGraph との深い統合により、オンプレミス機能を強化しました。
- MLflow は、マルチワークスペース管理のための Organization Support と、自動化されたエバリュエーターの作成を効率化するビジュアルな Judge Builder を導入しました。
- W&B Weave は、ダイナミックなリーダーボードとカスタム Scoring 機能を展開し、あわせてマルチモーダルな Eval ワークフロー向けに設計された独自のオーディオモニターをリリースしました。
- Braintrust は、設定可能な画像レンダリングのセキュリティコントロールを実装し、ツール呼び出しの Tracing のための新しい Temporal 統合によりエージェントの Observability を拡張しました。
- Langfuse は、コンプライアンスの可視性を高める Organization Audit Log Viewer をローンチし、複雑なチェーンのレイテンシをデバッグするためのタイムラインビューを追加しました。
- Arize Phoenix は、ツール選択の正確性のための専用メトリクスをリリースし、セルフホストデプロイメントでの RBAC サポートを確定させることで、エージェントの Eval に注力しました。

**AIによる市場インサイト**:

> LangSmith と Arize Phoenix は、特定のツール選択エバリュエーターと推論 Tracing を備えたエージェントワークフローを積極的にターゲットにしており、Weave に対して基本的なツール呼び出し Tracing 以上のエージェント可視化を拡張するよう圧力をかけています。
> Weave は強力なオンプレミスオプションを提供していますが、Langfuse の新しい Audit Log Viewer や LangSmith の v0.13 セルフホストリリースは、PII（個人識別情報）マスキングやコンプライアンスログに関する Weave の明示的なドキュメントの不足を浮き彫りにしています。
> Braintrust の深い IDE 統合と MLflow のビジュアルな Judge Builder は、コーディング環境と Eval ロジックをより緊密に結合させることで、Weave のデベロッパーファーストなポジショニングに挑戦しています。

## 3. 新機能 (直近30日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: LLM Judge を使用して、テキストとともにオーディオ出力を監視および判定するモニター。(2026-02-01, Monitoring & Metrics)
- **Dynamic Leaderboards**: 永続的なカスタマイズと CSV エクスポート機能を備えた、Eval から自動生成されるリーダーボード。(2026-01-29, Evaluation Integration)
- **Custom LoRAs in Playground**: Weave Playground でカスタム Fine-tuning された LoRA の重みを直接テストおよび Eval する機能。(2026-01-16, Experiment / Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **Python SDK v0.7.1**: LangSmith プラットフォームに接続するためのクライアントライブラリのアップデート。(2026-02-10, DevEx / Integration)
- **Customize trace previews**: UI での Tracing プレビュー方法をカスタマイズする機能。(2026-02-06, Core Observability)
- **Google Gen AI Wrapper Export**: SDK における Google Gen AI ラッパーのエクスポート機能。(2026-01-31, DevEx / Integration)
- **LangSmith Self-Hosted v0.13**: セルフホストプラットフォームの新しいバージョンリリース。(2026-01-16, Enterprise & Security)

### [Langfuse](https://langfuse.com/changelog)
- **Org Audit Log Viewer**: 組織の監査ログを表示するための新しい UI。(2026-02-09, Enterprise & Security)
- **Render Thinking/Reasoning Parts**: Tracing の詳細において、思考および推論の Tracing（推論モデルなど）をレンダリングする機能のサポート。(2026-02-09, Core Observability)
- **Single Observation Evals**: 単一の Observation に Eval を追加する機能。(2026-02-09, Evaluation Integration)
- **Events-based Trace Table**: Observation/Tracing テーブル用の新しいイベントベースの表示モード。(2026-02-09, Core Observability)
- **Corrected Outputs for Traces**: Fine-tuning 用のデータセットを構築するために、Tracing ビューで LLM 出力の改善版を直接キャプチャ。(2026-01-14, Experiment / Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Render attachments in custom views**: カスタム Tracing ビューで、画像、ビデオ、オーディオ、その他の添付ファイルを直接レンダリングできるようになりました。(2026-02, Core Observability)
- **Navigate to trace origins**: ログ内の Tracing から、その発生元となったプロンプトまたはデータセットの行に移動。(2026-02, Experiment / Improvement Loop)
- **Trace-level scorers**: カスタムコードによる Scoring 機能が、実行 Tracing 全体にアクセスしてマルチステップのワークフローを Eval できるようになりました。(2026-02, Evaluation Integration)
- **LangSmith integration**: Tracing および Eval の呼び出しを LangSmith と Braintrust の両方に送信するか、Braintrust のみにルーティングするためのラッパー。(2026-02, DevEx / Integration)
- **Cursor integration**: Cursor からログや実験をクエリするために Braintrust MCP サーバーを自動構成する拡張機能。(2026-02, DevEx / Integration)
- **Image rendering security controls**: 画像 URL からの機密データ漏洩を防ぐための、設定可能なモード（自動読み込み、クリックして読み込み、ブロック）。(2026-02, Enterprise & Security)
- **Single span filters with aggregations**: 単一スパンのフィルターを GROUP BY と組み合わせて、スパンレベルの条件に基づいて Tracing を集計。(2026-02, Monitoring & Metrics)
- **Auto-instrumentation (Python, Ruby, Go)**: Python、Ruby、Go アプリケーション向けのコード不要の Tracing サポート。(2026-01, DevEx / Integration)
- **Temporal integration**: 分散 Tracing サポートにより、Temporal のワークフローとアクティビティを自動的に Tracing。(2026-01, DevEx / Integration)
- **Kanban layout for reviews**: フラグが立てられたスパンとレビュー・ステータスを管理するためのドラッグ・アンド・ドロップ・インターフェース。(2026-01, Evaluation Integration)

### [MLflow](https://mlflow.org/releases)
- **Organization Support**: 実験やリソースを整理するためのマルチワークスペース環境のサポート。(2026-02-12, Enterprise & Security)
- **MLflow Assistant**: エージェントのデバッグや問題解決を支援する、Claude Code を搭載した製品内チャットボット。(2026-01-29, DevEx / Integration)
- **Agent Performance Dashboards**: エージェントのレイテンシ、リクエスト数、品質スコアを監視するための構築済みチャート。(2026-01-29, Monitoring & Metrics)
- **MemAlign Judge Optimizer**: フィードバックから Eval ガイドラインを学習し、判定の精度を向上させるアルゴリズム。(2026-01-29, Evaluation Integration)
- **Judge Builder UI**: コードなしで LLM Judge プロンプトを作成、テスト、反復するためのビジュアルインターフェース。(2026-01-29, Evaluation Integration)
- **Continuous Online Monitoring**: 本番環境の入力 Tracing に対して LLM Judge を自動的に実行。(2026-01-29, Monitoring & Metrics)
- **Distributed Tracing**: コンテキスト伝播により、複数のサービスにわたるリクエストを追跡。(2026-01-29, Core Observability)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Dataset Evaluators**: 実験中にサーバー側で自動実行するために、エバリュエーターをデータセットに直接アタッチ。(2026-02-12, Experiment / Improvement Loop)
- **OpenAI Responses API Type Support**: OpenAI モデルの Chat Completions と Responses API タイプの選択をサポート。(2026-02-12, DevEx / Integration)

- **Tool Selection & Invocation Evaluators**: エージェントのツール選択の正確性とパラメータのフォーマットを評価するための新しいエバリュエーター。(2026-01-31, Agent / RAG Observability)
- **Configurable Email Extraction**: JMESPath を使用した OAuth2 プロバイダーからのカスタムメール抽出のサポート。(2026-01-28, Enterprise & Security)
- **Phoenix CLI Commands**: ターミナルからプロンプト、データセット、実験を管理するための CLI コマンド。(2026-01-22, DevEx / Integration)
- **Trace to Dataset with Span Associations**: ソーススパンへの双方向リンクを維持しながら、Tracing をデータセットに変換。(2026-01-21, Evaluation Integration)

## 5. エンタープライズ・シグナル

- LangSmith はセルフホストプラットフォームの v0.13 をリリースし、オンプレミスのエンタープライズデプロイメントへのコミットメントを強化しました。

- Langfuse は、コンプライアンスとセキュリティの可視性を高めるために、新しい Organization Audit Log Viewer をローンチしました。
- Braintrust は、添付ファイルを介したデータ漏洩を防ぐために、設定可能な画像レンダリングのセキュリティコントロールを実装しました。
- Arize Phoenix は、Docker および Kubernetes のセルフホストオプションと並んで、RBAC および LDAP 統合のサポートを確定しました。
