# 週刊 LLM Observability 市場調査レポート
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview | **収集データ**: 2026-02-12

## 1. エグゼクティブ・サマリー

- Langfuse は、エンタープライズのコンプライアンスとガバナンス要件を明確なターゲットとした、Organization Audit Logs（組織監査ログ）と RBAC ロールを導入しました。
- LangSmith は Self-Hosted v0.13 をリリースし、オンプレミスやプライベートクラウドのインフラを必要とするチーム向けにデプロイの選択肢を拡大しました。
- MLflow は、マルチワークスペース環境全体での管理を容易にするため、Tracking Server の Organization Support（組織サポート）をリリースしました。
- Braintrust は、顧客クラウド内でのデータレジデンシーのためのハイブリッドアーキテクチャを維持しつつ、サブエージェントのネストをサポートするようにエージェントの Observability 機能をアップデートしました。
- Arize Phoenix は、ワンラインのインストルメンターと、Faithfulness（誠実性）およびツール選択のためのプリセット Eval を提供することで、OpenTelemetry ネイティブとしてのポジショニングを固めました。
- W&B Weave は、W&B エコシステムとの深い統合、ネイティブな OpenTelemetry サポート、およびマルチターン・エージェントの推論のためのインタラクティブなトレースツリーを提供することで、差別化を継続しています。

**AI による市場インサイト**:

> Langfuse が RBAC と Audit Logs を追加したことは、Weave が成熟した W&B プラットフォームのガバナンスを継承することで保持していた主要なエンタープライズ差別化ポイントを中和するものです。
> LangSmith Self-Hosted v0.13 のリリースと Braintrust のハイブリッドモデルは、Weave に対して、特定のデータレジデンシーやデプロイ上の優位性を明確にするよう競争圧力を高めています。
> Arize Phoenix のツール選択に特化したプリセット Eval は、Weave の広範で柔軟なカスタム Scorer フレームワークにおける特定の機能ギャップを浮き彫りにしています。

## 3. 新機能 (直近30日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: LLM Judge を使用して、テキストと並行して音声出力を監視・判定できるオンライン Eval モニター。(2026-02-01, Evaluation & Quality)
- **Dynamic Leaderboards**: フィルター、メトリクス、表示設定の永続的なカスタマイズが可能な自動生成リーダーボード。(2026-01-29, Evaluation & Quality)
- **Playground でのカスタム LoRA**: Weave Playground 内でカスタム LoRA ウェイトを直接テストおよび比較するためのサポート。(2026-01-16, Experiment & Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **トレースプレビューのカスタマイズ**: LangSmith インターフェースでトレースプレビューをカスタマイズする機能。(2026-02-06, Developer Experience & Integration)
- **LangSmith Self-Hosted v0.13**: プラットフォームのセルフホスト版のアップデート。(2026-01-16, Infrastructure & Enterprise)

### [Langfuse](https://langfuse.com/changelog)
- **Single Observation Evals**: 完全なトレースだけでなく、単一の Observation に対して Eval を実行するためのサポート。(2026-02, Evaluation & Quality)
- **イベントベースのトレーステーブル**: より詳細な粒度のために、イベントに基づいたトレースと Observation の新しいテーブルビュー。(2026-02, Core Tracing & Logging)
- **思考/推論トレースのレンダリング**: トレース詳細における「思考（Thinking）」または推論部分の視覚的レンダリング（例：DeepSeek R1 用）。(2026-01, Agent & RAG Observability)
- **組織監査ログビューア**: ユーザーのアクションやセキュリティイベントを追跡するための、組織レベルの監査ログ用新しいビューア。(2026-01, Infrastructure & Enterprise)
- **インライントレースコメント**: コラボレーションのために、トレース内の IO データの一部にインラインでコメントを追加する機能。(2026-01, Developer Experience & Integration)
- **トレース修正**: ヒューマンフィードバックループのために、トレースと Observation のプレビューに修正を提供する機能を追加。(2026-01, Evaluation & Quality)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **サブエージェントのネスト**: Claude Agent SDK ラッパーにおけるサブエージェントのネストのサポートを追加。(2026-02-12, Agent & RAG Observability)
- **Classifications フィールド**: データ分類を改善するための SDK への新しい Classifications フィールドの追加。(2026-01-31, Core Tracing & Logging)
- **Eval キャッシュ制御**: 新鮮な実行を保証するために、Eval 中にキャッシュをオフにする新しいオプション。(2026-01-29, Evaluation & Quality)
- **トレース Scoring 候補**: Python トレース Scoring 機能のアップデート。(2026-01-21, Evaluation & Quality)
- **Workflows への名称変更**: より広範なユースケースを反映させるため、SDK 内で「agents」を「workflows」に名称変更。(2026-01-15, Agent & RAG Observability)

### [MLflow](https://mlflow.org/releases)
- **Tracking Server での組織サポート**: 異なるワークスペース間で実験やリソースを整理できる、マルチワークスペース環境のサポート。(2026-02-12, Infrastructure & Enterprise)
- **MLflow Assistant**: UI 内で直接問題を特定、診断、修正するのを支援する、Claude Code をバックエンドとした製品内チャットボット。(2026-01-29, Developer Experience & Integration)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Playground での Claude Opus 4.6**: Playground インターフェース内での Claude Opus 4.6 モデルのサポート追加。(2026-02-09, Experiment & Improvement Loop)
- **ツール選択 Eval**: エージェント評価を改善するため、ライブラリに欠落していた tool_selection Eval を追加。(2026-02-06, Evaluation & Quality)
- **Faithfulness Eval**: FaithfulnessEvaluator を導入し、HallucinationEvaluator を非推奨化。(2026-02-02, Evaluation & Quality)
- **ツール呼び出し精度メトリクス**: ツール呼び出しの精度を測定するための新しいメトリクスを追加。(2026-02-02, Evaluation & Quality)
- **設定可能な OAuth2 メール抽出**: OAuth2 における設定可能なメール抽出のための EMAIL_ATTRIBUTE_PATH を追加。(2026-01-28, Infrastructure & Enterprise)
- **メトリクス用 Cursor ルール**: 新しい組み込みメトリクス（LLM 分類 Eval）を作成するための Cursor ルールを追加。(2026-01-21, Monitoring & Analytics)

## 5. エンタープライズ・シグナル

- Langfuse は Organization Audit Logs と RBAC ロールを導入し、エンタープライズのコンプライアンスニーズに直接対応しました。
- MLflow は、マルチワークスペース環境をより適切に管理するために、Tracking Server の Organization Support をリリースしました。
- LangSmith は Self-Hosted v0.13 をリリースし、オンプレミスおよびプライベートクラウドへのデプロイに対するコミットメントを強化しました。
- Braintrust は、SOC 2 コンプライアンスを維持しながら、機密データを顧客の VPC 内に保持するハイブリッドアーキテクチャで差別化を続けています。
