# 週間 LLM Observability 市場調査レポート
**日付**: 2026-02-13 | **モデル**: google/gemini-3-pro-preview | **データ収集日**: 2026-02-13

## 1. エグゼクティブ・サマリー

- LangSmith が Self-Hosted v0.13 をリリースし、LLM、ツール、検索（retrieval）コンポーネントを網羅する統合的なコスト追跡機能を導入しました。
- Langfuse は、コンプライアンス監視のための Organization Audit Log Viewer をローンチし、エージェントの可視性を向上させる推論ステップ（reasoning step）のレンダリング機能を追加しました。
- MLflow は、Tracking Server 内でマルチワークスペース管理を可能にする Organization Support を導入し、ガバナンス機能を専門的な LLM ツールに近い水準まで引き上げました。
- Braintrust は、再現可能なエンタープライズ実験と回帰テストをサポートするため、Eval 用のきめ細かなキャッシュ制御機能を追加しました。
- Arize Phoenix は、ツールの選択と忠実性（faithfulness）に特化した新しいエバリュエーターをリリースし、あわせて LLM Judge 設定用の GUI ベースのウィザードを公開しました。
- W&B Weave は、マルチモーダル入力の Tracing へのネイティブ対応を維持しつつ、チームベースでの Eval 比較を容易にする動的リーダーボードをローンチしました。

> **市場の洞察**: Langfuse と MLflow が監査ログと組織レベルの管理機能を同時にリリースしたことは、管理ガバナンスの格差を埋めるものであり、Weave のエンタープライズ対応というナラティブに対する挑戦となります。 | Arize Phoenix による LLM Judge 用 GUI ウィザードの導入は、Weave の既存のノーコード Eval 作成ワークフローと直接競合し、Judge 設定の使いやすさをコモディティ化させています。 | LangSmith の高度なエージェントサンドボックスや Langfuse の推論ステップレンダリングは、複雑なエージェントワークフローに対する高レベルな実行グラフが不足している Weave のエージェント Observability における特定のギャップを浮き彫りにしています。


## 2. 新機能 (直近 30 日間)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: オーディオ出力用のオンライン Eval モニター。LLM Judge がテキストと並行して MP3/WAV ファイルを評価可能に。（2026-02-01, Evaluation & Quality）
- **Dynamic Leaderboards**: フィルターやメトリクスのカスタマイズを保持できる、Eval 内の自動生成リーダーボード。（2026-01-29, Evaluation & Quality）
- **Custom LoRAs in Playground**: Weave Playground 内で、カスタム Fine-tuning された LoRA 重みを直接テスト・比較する機能をサポート。（2026-01-16, Development Lifecycle）

### [LangSmith](https://changelog.langchain.com)
- **Customize trace previews**: LangSmith UI での Trace プレビューのレンダリング方法をカスタマイズする機能。（2026-02-06, Integration & DX）
- **LangSmith Self-Hosted v0.13**: セルフホスト型のエンタープライズデプロイメントオプションの新しいバージョンをリリース。（2026-01-16, Enterprise & Infrastructure）

### [Langfuse](https://langfuse.com/changelog)
- **LLM-as-a-Judge on Observations**: 個別の Observation に対して直接 LLM-as-a-judge Eval を実行する機能を追加。（2026-02-13, Evaluation & Quality）
- **Single Observation Evals**: 単一の Observation に対する Eval ワークフローを有効化。（2026-02-10, Evaluation & Quality）
- **Thinking/Reasoning Rendering**: Trace 詳細に思考・推論部分のレンダリングを追加し、Chain-of-Thought の可視性を向上。（2026-01-28, Core Tracing & Logging）
- **Org Audit Log Viewer**: 組織レベルの監査ログ用ビューアーを導入。（2026-01-28, Enterprise & Infrastructure）
- **Inline Trace Comments**: Trace 内の入出力データの一部に対してインラインでコメントを追加可能に。（2026-01-20, Integration & DX）

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Sub-agent nesting for Claude Agent**: Claude Agent SDK ラッパーにサブエージェントのネストサポートを追加し、複雑なエージェントワークフローの Tracing を改善。（2026-02-12, Agent & RAG Specifics）
- **Classifications Field**: SDK に新しい「Classifications」フィールドを導入。メタデータのタグ付けや Trace のカテゴリ化を強化。（2026-01-31, Core Tracing & Logging）
- **Eval Cache Control**: Eval 中およびスパンエクスポート後のキャッシュをオフにするオプションを追加し、実験の再現性をより細かく制御可能に。（2026-01-29, Evaluation & Quality）
- **Python Trace Scoring**: Python での Trace Scoring の新しい実装候補を追加し、プログラムによる Eval 機能を強化。（2026-01-21, Evaluation & Quality）
- **Review Span Type**: SDK に特定の「review」スパンタイプを追加し、Trace 内の人間によるレビューステップのカテゴリ化を容易に。（2026-01-15, Evaluation & Quality）

### [MLflow](https://mlflow.org/releases)
- **Organization Support**: MLflow Tracking Server でのマルチワークスペース環境をサポートし、異なるワークスペース間での実験の整理を可能に。（2026-02-12, Enterprise & Infrastructure）
- **MLflow Assistant**: Claude Code を活用した製品内チャットボット。MLflow UI 内で直接問題を特定、診断、修正することを支援。（2026-01-29, Integration & DX）

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 Support**: Playground で Claude Opus 4.6 モデルのサポートを追加。（2026-02-09, Development Lifecycle）
- **Tool Selection Evaluator**: ライブラリに不足していた tool_selection エバリュエーターを追加。（2026-02-06, Evaluation & Quality）
- **Faithfulness Evaluator**: FaithfulnessEvaluator を導入し、HallucinationEvaluator を非推奨化。（2026-02-02, Evaluation & Quality）
- **Tool Invocation Accuracy Metric**: ツール呼び出しの精度を追跡するための新しいメトリクスを追加。（2026-01-27, Analytics & Dashboard）
- **Cursor Rule for Metrics**: 新しい組み込みメトリクス（LLM 分類エバリュエーター）を作成するための Cursor ルールを追加。（2026-01-21, Evaluation & Quality）

## 4. エンタープライズ・シグナル

- Langfuse は、コンプライアンスとセキュリティ監視を強化するために Organization Audit Log Viewer をリリースしました。
- MLflow は、Tracking Server 内でマルチワークスペース管理を可能にする Organization Support を導入しました。
- LangSmith は Self-Hosted v0.13 をリリースし、オンプレミスのエンタープライズデプロイメントへの継続的な取り組みを強化しました。
- Braintrust は、再現可能なエンタープライズ実験をサポートするため、Eval 用のきめ細かなキャッシュ制御を追加しました。
- W&B Weave は、チームベースの Eval と比較ワークフローを促進するために動的リーダーボードをローンチしました。
