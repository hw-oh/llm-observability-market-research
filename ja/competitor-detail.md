# LLM Observability — 製品詳細
**日付**: 2026-02-16 | **モデル**: google/gemini-3-pro-preview

### W&B Weave

**概要**: W&B Weaveは、広範な Weights & Biases エコシステムと深く統合された、開発者中心の LLM Observability および Eval プラットフォームです。厳格なコードベースの Eval、マルチモーダルな Tracing（オーディオ/ビデオを含む）、およびエンタープライズグレードのデプロイメントオプションに優れていますが、プロキシベースのインターセプトではなく SDK 統合に依存しています。

**強み**:
- 統合された ML ライフサイクルのための W&B Experiments および Models との深い統合。
- 業界をリードするマルチモーダル Tracing サポート（オーディオ、ビデオ、画像）。
- カスタム Scoring と新しい GUI ウィザードを備えた、柔軟なコードファーストの Eval。
- エンタープライズグレードのセキュリティとデプロイメントオプション（セルフホスト、HIPAA、SOC2）。

**弱み**:
- ゼロコード統合のための Gateway/Proxy モードがない。
- アノテーションワークフローに、大規模チーム向けの高度なキュー管理機能が不足している。
- エージェントの可視化が一部の競合他社と比較して専門化されていない（専用のグラフ表示がない）。

**最新のアップデート**:
- Audio Monitors: オンライン Eval モニターがオーディオ入出力をサポートし、LLM Judge が音声エージェントの会話を評価できるようになりました。(2026-02-01)
- Dynamic Leaderboards: フィルタリングおよびカスタマイズオプションを備えた、Eval 結果からのリーダーボード自動生成機能。(2026-01-29)

### LangSmith

**概要**: LangSmith は、LLM アプリケーションのための包括的な DevOps プラットフォームであり、特に LangChain エコシステム内での深い Tracing、Eval、およびライフサイクル管理に特化しています。階層的な可視化による複雑なエージェントワークフローのデバッグに優れており、データセットのキュレーション、回帰テスト、およびオンラインの本番環境モニタリングのための堅牢なツールを提供します。

**強み**:
- 階層的なエージェントデバッグのための LangChain および LangGraph との深い統合。
- カスタム Scoring とアノテーションキューを備えた堅牢な Eval フレームワーク。
- セルフホストや BYOC を含む柔軟なエンタープライズデプロイメントオプション。
- 包括的なプロンプト管理およびバージョン管理システム。

**弱み**:
- ネイティブなリアルタイム・ブロッキング Guardrails の欠如（検出のみ）。
- LLM-as-a-Judge Eval 設定用の「ノーコード」ウィザードがない。
- Go SDK およびゼロコードの Proxy/Gateway 統合モードがない。
- 特化型の FinOps ツールと比較して、コスト配分機能が限定的。

**最新のアップデート**:
- Trace プレビューのカスタマイズ: LangSmith UI で Trace プレビューをカスタマイズする機能。(2026-02-06)
- Google Gen AI / Gemini ラッパー: Python および JS SDK における Google Gen AI と Gemini の新しいラッパー。(2026-02-02)
- Python Async Sandbox エンドポイント: サンドボックスでの Python 非同期サポート用エンドポイントの追加。(2026-02-05)

### Langfuse

**概要**: Langfuse は、Observability、Eval、およびプロンプト管理を統合した、包括的なオープンソースの LLM エンジニアリングプラットフォームです。OpenTelemetry 標準に基づいた詳細な Tracing に優れ、LLM-as-a-Judge やデータセットキュレーションのための堅牢なワークフローを提供します。個人開発者からセルフホストを必要とするエンタープライズチームまで適しています。

**強み**:
- 完全にオープンソースでセルフホスト可能であり、高いデータ主権を提供。
- 統合された LLM-as-a-Judge とデータセット管理による強力な Eval 機能。
- Chain-of-Thought 推論の最近の追加を含む、深い Tracing サポート。
- バージョン管理とプレイグラウンドを備えた包括的なプロンプト管理システム。
- カスタマイズ可能な Dashboard とコスト配分を備えた柔軟な Analytics。

**弱み**:
- マルチモーダル Tracing 機能（画像/オーディオ）の欠如。
- 公式の Go SDK がなく、Go ベースのスタックでの統合が制限される。
- 高度なエージェント実行グラフの可視化がない。
- 組み込みの Fine-tuning パイプライン統合がない。
- ゼロコード Tracing のための Proxy/Gateway モードがない。

**最新のアップデート**:
- Observation 上の LLM-as-a-Judge: 特定の Observation に対して直接 LLM-as-a-judge Eval を実行するサポートを追加。(2026-02-16)
- 単一 Observation Eval: 単一の Observation に対する Eval 作成を有効化。(2026-02-10)
- イベントベースの Trace テーブル: Observation および Trace テーブルにイベントベースのビューを導入。(2026-02-05)
- 思考/推論のレンダリング: Trace 詳細における思考および推論部分（CoT）のレンダリングサポートを追加。(2026-02-01)
- 組織監査ログビューアー: 組織レベルの監査ログ用の新しいビューアー。(2026-02-01)

### Braintrust

**概要**: Braintrust は、開発と本番のギャップを埋めるエンタープライズグレードの AI Eval および Observability プラットフォームであり、「LLM-as-a-Judge」ワークフローと回帰テストに重点を置いています。プロンプトエンジニアリング、ロギング、データセット管理のための統合環境を提供し、堅牢な SDK とハイブリッドデプロイオプションによってサポートされています。コア Tracing と Eval には優れていますが、複雑なエージェント向けの高度な視覚的デバッグツールや、専門的なセーフティプラットフォームに見られる専用のセキュリティ Guardrails は現在不足しています。

**強み**:
- 回帰テストを備えた包括的な「LLM-as-a-Judge」Eval フレームワーク。
- プレイグラウンド、プロンプト CMS、実験を組み合わせた統合開発ワークフロー。
- ハイブリッドデプロイと RBAC による強力なエンタープライズ対応。
- 主要言語に対する堅牢な自動インスツルメンテーションと SDK サポート。

**弱み**:
- 複雑なエージェントデバッグのための視覚的な実行グラフの欠如。
- 専用のセキュリティ Guardrails（ジェイルブレイク/トピックブロッキング）がない。
- ネイティブのデータウェアハウスエクスポート機能がない。
- クローズドソースの独自モデルであるため、コミュニティによる拡張性が制限される。

**最新のアップデート**:
- Thread Retrieval API: Python SDK でプログラムによってスレッドを取得する機能を追加。(2026-02-12)
- Review Span Type: 人間によるレビューワークフローをサポートするための新しい「review」スパンタイプを導入。(2026-02-05)
- OpenAI Agents 統合: OpenAI エージェント統合のすべてのスパンタイプを処理できるように SDK を強化。(2026-02-05)
- Classifications フィールド: Trace に分類フィールドのサポートを追加。(2026-01-31)
- Eval Cache Control: Eval 中にキャッシュをオフにするオプションを追加。(2026-01-29)
- Python Trace Scoring: Python SDK に Trace Scoring 候補機能を追加。(2026-01-21)

### MLflow

**概要**: MLflow は、GenAI の Observability 分野に積極的に進出している主要なオープンソース MLOps プラットフォームであり、堅牢な Tracing、Eval、およびライフサイクル管理を提供しています。業界標準の実験トラッキングと、急速に成熟している LLM-as-a-Judge、プロンプト管理、エージェントモニタリング機能を組み合わせ、研究から本番へのスケーリングに適した多才な選択肢となっています。

**強み**:
- 比類のない実験トラッキングとバージョン管理の伝統。
- LLM-as-a-Judge やプロンプト管理を含む、急速に成熟している GenAI 機能セット。
- 広範な自動インスツルメンテーションをサポートする巨大なオープンソースエコシステム。
- セルフホストからマネージドなエンタープライズ SaaS まで、柔軟なデプロイオプション。

**弱み**:
- 迅速なプロンプトプロトタイピングのためのネイティブな対話型 Playground/Sandbox の欠如。
- 限定的な Policy-as-Code またはプロアクティブな Guardrails 強制機能。
- 複雑なエージェント実行グラフ（DAG）の高度な可視化機能がない。

**最新のアップデート**:
- 組織サポート: MLflow Tracking Server におけるマルチワークスペース環境のサポートにより、リソースの組織化が向上。(2026-02-12)
- MLflow Assistant: アプリやエージェントの問題の特定、診断、修正を支援する、Claude Code を活用した製品内チャットボット。(2026-01-29)

### Arize Phoenix

**概要**: Arize Phoenix は、RAG およびエージェントワークフローに重点を置いて特別に設計された、オープンソースでコードファーストの Observability および Eval プラットフォームです。OpenTelemetry ネイティブな Tracing、プログラムによる Eval、および実験トラッキングに優れており、ローカルファーストの開発者ツールとして機能し、広範な Arize プラットフォームを介してエンタープライズデプロイメントまでスケールします。

**強み**:
- ネイティブな OpenTelemetry サポートにより、ベンダーニュートラルな Tracing と広範な統合互換性を確保。
- ツール呼び出しや取得に関する特定のメトリクスを含む、エージェントワークフローの評価のための強力な機能。
- 技術チームや複雑なロジックに最適な、柔軟でコードファーストなカスタム Eval システム。
- 完全にオープンソースのコアにより、ローカル開発と完全なデータ主権が可能。

**弱み**:
- 非技術的なステークホルダー向けのノーコード/ローコードインターフェース（例：GUI ベースの Judge ビルダー）がない。
- 組み込みのコスト配分および財務運用（FinOps）メトリクスがない。
- コア製品に監査ログや詳細な RBAC などのエンタープライズガバナンス機能が限定的。
- Fine-tuning パイプラインやプロンプトのバージョン管理に対するネイティブサポートがない。

**最新のアップデート**:
- LLM Eval プロンプトエディタのオートコンプリート: LLM-as-a-judge プロンプトの定義に使用されるエディタにオートコンプリート機能を追加。(2026-02-13)
- Tool Response Handling Evaluator: エージェントがツールのレスポンスをどのように処理するかを評価するための新しい Eval テンプレート。(2026-02-13)
- Claude Opus 4.6 サポート: プレイグラウンド環境内での Claude Opus 4.6 モデルのサポートを追加。(2026-02-09)
- Tool Selection Evaluator: エージェントワークフローにおけるツール選択の正確性を評価するための Eval を追加。(2026-02-06)
- Faithfulness Evaluator: 精度向上のため、非推奨となった HallucinationEvaluator に代わる FaithfulnessEvaluator を導入。(2026-02-02)
