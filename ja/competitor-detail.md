# W&B Weave — 製品詳細
**日付**: 2026-02-11 | **モデル**: google/gemini-3-pro-preview

[詳細な比較はこちら](./comparison)

### Weave

**概要**: Weave は、LLM アプリケーション開発とモデルトレーニングのライフサイクルのギャップを独自に埋める、軽量で Pythonic なオブザーバビリティ・ソリューションとして位置付けられています。LangSmith のフレームワーク中心のアプローチや MLflow の重厚なインフラとは異なり、Weave は広範な W&B エコシステムを活用し、本番環境のトレースからファインチューニングやモデルの反復へとシームレスにつなげるループを提供します。

**主な強み**:
- トレーニング・リネージの統合: Weave は、本番環境のトレースを W&B のモデルアーティファクト、トレーニング実行、スイープにネイティブにリンクできる唯一のプラットフォームであり、真のデータフライホイールを実現します。
- マルチモーダル評価: 最近リリースされた Audio Monitors（2026年2月）は、音声エージェントの開発者にとって、LangSmith や MLflow などのテキスト中心の競合他社に対する明確なアドバンテージとなります。
- インタラクティブなデバッグ: Weave の Playground は、静的なトレース表示に重点を置く MLflow や Arize Phoenix と比較して、迅速な反復のための優れた「編集して実行」体験を提供します。
- フレームワークに依存しない設計: Weave は LangSmith よりも軽量で特定の意見に固執しないため、LangChain エコシステム外でカスタムスタックを構築する開発者にとって魅力的です。

**改善が必要な領域**:
- ヒューマン・イン・ザ・ループ（Human-in-the-Loop）ワークフロー: LangSmith や Langfuse は、大規模な人間によるラベル付けチームを管理するための、より成熟した「アノテーション・キュー」を提供しています。
- エージェントの状態の可視化: LangSmith と LangGraph の深い統合により、複雑なステートマシンや循環的なエージェントワークフローの優れた可視化が提供されています。
- トラフィック管理: Weave には、Braintrust が提供しているような、レート制限、キャッシュ、トラフィック制御のためのアクティブな AI プロキシ/ゲートウェイ・アーキテクチャが欠けています。
- OpenTelemetry の標準化: MLflow と Arize Phoenix は「ネイティブ OTel」アプローチを採用しており、Weave の SDK よりもオープンな標準を優先する企業にとって安全な選択肢となっています。

**最新のアップデート**:
- Audio Monitors: テキストと並行して音声出力を監視・判定するモニターの作成をサポートし、音声エージェントの評価を可能にしました。(2026-02-01)
- ダイナミック・リーダーボード: 評価から自動生成されるリーダーボード。永続的なカスタマイズと CSV エクスポート機能を備えています。(2026-01-29)
- Playground でのカスタム LoRA: Weave Playground で直接、カスタムのファインチューニング済み LoRA 重みをロードしてテストし、比較できるようになりました。(2026-01-16)

### LangSmith

**概要**: LangSmith は、LLM アプリケーションのための包括的な DevOps プラットフォームであり、LangChain および LangGraph エコシステムと深く統合された、クラス最高のオブザーバビリティ、評価、プロンプトエンジニアリングツールを提供します。フレームワークに依存しないと主張していますが、主な強みは複雑なエージェントワークフローのシームレスな処理、ヒューマン・イン・ザ・ループのアノテーション・キュー、そして LangGraph Cloud を介したデプロイへの最近の拡張にあります。

**Weave に対する強み**:
- LangGraph とのネイティブな統合により、複雑なエージェント・ステートマシンの優れた可視化を提供。
- 成熟した「アノテーション・キュー」ワークフローは、ヒューマン・イン・ザ・ループのデータキュレーションにおいて Weave を大きく上回る。
- 統合された Prompt Hub により、プラットフォームを離れることなくプロンプトのバージョン管理とテストが可能。
- エンドツーエンドのライフサイクル管理にデプロイ（LangGraph Cloud）が含まれるようになり、外部のインフラツールの必要性が減少。

**Weave に対する弱み**:
- LangChain フレームワークへの強力なロックインという印象がある一方、Weave はより軽量で Pythonic であると見なされている。
- モデルのトレーニング/ファインチューニング・パイプライン（W&B の核心的な強み）との深い統合が欠けている。
- トレース/シート数に基づく価格モデルは、Weave と比較して、大量のトラフィックがあるコンシューマー向けアプリケーションでは非常に高価になる可能性がある。
- エージェント固有の機能が密集しているため、単純なユースケースでは UI が圧倒的に感じられることがある。

**最新のアップデート**:
- トレース・プレビューのカスタマイズ: トレース・リスト・ビューに表示するフィールドを設定可能にし、デバッグを高速化。(2026-02-06)
- Google Gen AI ラッパー: OpenTelemetry を使用せずに Google の生成 AI モデルをネイティブにトレースするための新しい SDK ラッパー。(2026-01-31)
- LangSmith セルフホスト v0.13: パフォーマンスの向上と新しい設定オプションを含む、セルフホスト版のアップデート。(2026-01-16)

### Langfuse

**概要**: Langfuse は、オブザーバビリティ、プロンプト管理、評価を統合した開発者体験を提供する、主要なオープンソース LLM エンジニアリングプラットフォームです。最近、分析エンジンを強化するために ClickHouse と提携（または買収）し、エージェントワークフローやヒューマン・イン・ザ・ループのアノテーションを深くサポートする、SaaS 専用ソリューションに代わる高性能でセルフホスト可能な選択肢として位置付けられています。

**Weave に対する強み**:
- オープンソース & セルフホスティング: MIT ライセンスにより、ベンダーロックインなしで完全なオンプレミス展開が可能であり、規制の厳しい業界にとって魅力的。
- プロンプト CMS: コードから独立してプロンプトを管理、バージョン管理、デプロイするための、非エンジニア向けの専用 UI。
- アノテーション・キュー: 人間によるラベル付けチームのための構造化された組み込みワークフロー。アドホックなフィードバックメカニズムよりも優れている。
- ClickHouse バックエンド: 高カーディナリティのメトリクスや、膨大なトレースボリュームに対するリアルタイム分析において、優れたクエリパフォーマンスを提供。

**Weave に対する弱み**:
- トレーニング統合の欠如: Weave が W&B から継承しているような、モデルトレーニング、スイープ、アーティファクトのバージョン管理への深いリネージを持っていない。
- オブジェクト中心のロギング: 任意の Python オブジェクト/型をネイティブにログに記録し可視化する Weave の能力と比較して、柔軟性が低い。
- データサイエンス・ワークフロー: Weave が Boards で実現している探索的な「データサイエンス」ワークフローよりも、「エンジニアリング/DevOps」（レイテンシ、コスト、トレース）に重点を置いている。

**最新のアップデート**:
- トレースの修正済み出力: トレースビューで LLM 出力の改善版を直接キャプチャし、ファインチューニング用データセットを構築可能に。(2026-01-14)
- Python SDK v3.14.1: Langfuse 機能にアクセスするためのクライアントライブラリのアップデート。(2026-02-09)

### Braintrust

**概要**: Braintrust は、堅牢な「評価ファースト」のワークフローとトラフィック管理用の独自の AI プロキシで差別化を図る、エンタープライズグレードの AI オブザーバビリティおよび評価プラットフォームです。6 つの言語（Java、Go、C# を含む）にわたる広範な SDK サポートを提供し、Cursor や VS Code などのツールを介して開発ループに深く統合されており、単なるパッシブなトラッカーではなく、AI 開発のための完全な「オペレーティングシステム」として位置付けられています。

**Weave に対する強み**:
- より広範な SDK エコシステム: Java、Go、Ruby、C# のネイティブサポートにより、Braintrust は Weave（Python/TS 中心）がカバーしていないエンタープライズのバックエンドチームを取り込むことができます。
- AI プロキシ・アーキテクチャ: 組み込みのゲートウェイがアクティブなトラフィック管理（キャッシュ、レート制限）を提供します。これは Weave のパッシブなオブザーバビリティモデルでは完全には再現できません。
- 統合された Prompt Playground: データセットや実験と密接に結合された、非常に成熟した「評価ファースト」の Playground。プロンプトエンジニアにとってよりスムーズな反復ループを提供します。

**Weave に対する弱み**:
- トレーニング統合の欠如: Braintrust はモデルトレーニング/ファインチューニング層から切り離されていますが、Weave はフルライフサイクルの ML Ops のために大規模な W&B エコシステムの恩恵を受けています。
- 可視化の柔軟性: Braintrust にはチャート用の「Loop」がありますが、Weave のカスタム Boards と式言語は、よりきめ細かくエンジニア中心のデータ探索機能を提供します。
- 価格モデル: Braintrust のシートベース + 使用量ベースの価格設定は、W&B の多くの場合バンドルされている、またはボリュームベースのエンタープライズ契約と比較して、摩擦点になる可能性があります。

**最新のアップデート**:
- トレースレベルのスコアラー: カスタムコードスコアラーが実行トレース全体にアクセスできるようになり、マルチステップのワークフローやエージェントの動作を評価可能に。(2026-02)
- LangSmith 統合: LangSmith のトレースおよび評価コールを Braintrust にルーティングするラッパー。ツールの統合を可能にします。(2026-02)
- Cursor 統合: Cursor エディタ用の拡張機能。Braintrust MCP サーバーを自動的に設定し、自然言語でログをクエリ可能に。(2026-02)
- 自動インストルメンテーション (Python/Ruby/Go): Python、Ruby、Go アプリケーション向けのコード不要のトレースサポートを追加。(2026-01)
- Temporal 統合: 親子関係のマッピングを備えた Temporal ワークフローおよびアクティビティの自動トレース。(2026-01)

### MLflow

**概要**: MLflow は 3.x シリーズで生成 AI へと積極的に舵を切り、ネイティブな OpenTelemetry サポートに裏打ちされた包括的な LLM ライフサイクルプラットフォームへと、従来の MLOps ツールから変貌を遂げました。MLflow 3.9（2026年1月）のリリースにより、高度な「Judge Builder」ワークフロー、継続的なオンライン評価、エージェント専用のパフォーマンスダッシュボードを提供し、その膨大な既存のインストールベースと Databricks との統合を通じて、特化型のオブザーバビリティツールにとって大きな脅威となっています。

**Weave に対する強み**:
- ネイティブな OpenTelemetry サポートにより、Weave の SDK と比較してベンダーロックインの懸念を軽減。
- Databricks エコシステム（Unity Catalog、Model Serving）との深い統合により、データチームに本番環境へのシームレスなパスを提供。
- MemAlign と Judge Builder UI により、Weave には現在欠けている評価の高度な「自動最適化」を提供。
- 大規模なオープンソースコミュニティとプラグインエコシステム（30 以上のフレームワーク）。

**Weave に対する弱み**:
- UI/UX は依然として「実験トラッキング」に根ざしており、Weave のモダンでインタラクティブな「Playground」中心の設計よりも重く、使いにくく感じられる。
- 再生/デバッグループの流動性が低い。Weave はトレースから即座に「編集して実行」が可能ですが、MLflow は実行を不変の記録として扱う傾向があります。
- スタック全体（トラッキングサーバー + UI + アーティファクト）をセルフホストするのは、Weave の軽量な開始方法よりも運用負荷が高い。
- Weave の専用アノテーション機能と比較して、「ヒューマン・イン・ザ・ループ」のアノテーション・ワークフローへの特化が不十分。

**最新のアップデート**:
- MLflow Assistant: Claude Code を搭載した製品内チャットボット。UI からのコンテキストを使用して、問題の診断、テストの設定、コードの修正を支援。(2026-01-29)
- エージェント・パフォーマンス・ダッシュボード: 設定なしでレイテンシ、リクエスト数、品質スコアを表示する、生成 AI 実験用の構築済み「概要」タブ。(2026-01-29)
- MemAlign Judge Optimizer: 過去のフィードバックから評価ガイドラインを学習し、LLM 判定の精度を自動的に向上させるアルゴリズム。(2026-01-29)
- Judge Builder UI: コードを書かずにカスタム LLM 判定を作成、テスト、検証するためのビジュアルインターフェース。(2026-01-29)
- 継続的なオンラインモニタリング: 本番環境のトレースに対して LLM 判定を自動的に実行し、品質の問題をリアルタイムで検出。(2026-01-29)

### Arize Phoenix

**概要**: Arize Phoenix は、OpenTelemetry ベースの OpenInference 標準に基づいて構築された、堅牢なオープンソース LLM オブザーバビリティおよび評価プラットフォームです。強力なローカルデバッグツール、ノートブック統合、成熟した評価ライブラリを備え、エンジニアリングの「インナーループ」に優れています。また、本番環境のモニタリングのために Arize エンタープライズプラットフォームへのシームレスなアップグレードパスを提供します。

**Weave に対する強み**:
- OpenTelemetry ネイティブ: OpenInference に基づいて構築されており、ベンダーニュートラルなインストルメンテーション標準を求めるチームにとって魅力的。
- ローカルファーストのデバッグ: 優れたノートブック統合とローカル UI により、クラウド同期を強制されることなく迅速な反復が可能。
- 特化型エージェント評価: RAG（リトリーバル）およびエージェント（ツールの選択/呼び出し）のための、成熟した専用エバリュエーターライブラリ。

**Weave に対する弱み**:
- 広範な ML エコシステム: W&B のような完全なトレーニング/モデルレジストリ・プラットフォームとのネイティブな統合が欠けている。
- セットアップの複雑さ: OTLP インストルメンテーションは、Weave の「一行」の自動インストルメンテーションよりも設定が冗長になる場合があります。
- UI の洗練度: インターフェースは機能的で実用的ですが、Weave のデザインと比較してモダンさや流動性に欠けると見なされることがよくあります。

**最新のアップデート**:
- Claude Opus 4.6 サポート: Playground で Anthropic の Claude Opus 4.6 モデルのサポートを追加し、拡張思考パラメータをサポート。(2026-02-09)
- FaithfulnessEvaluator: 非推奨となった HallucinationEvaluator に代わる、忠実度（Faithfulness）を測定するための新しいエバリュエーター。(2026-02-02)
- ツール選択 & 呼び出しエバリュエーター: エージェントが正しいツールを選択し、有効なパラメータで呼び出したかどうかを評価するための専用エバリュエーター。(2026-01-31)
- プロンプト & データセット用 CLI: ターミナルからプロンプト、データセット、実験を管理するための包括的な CLI コマンド。(2026-01-22)
- スパン関連付けを備えたトレースからデータセットへの変換: ソーススパンへの双方向リンクを維持しながら、本番環境のトレースからデータセットを作成する機能。(2026-01-21)
