# LLM Observability — 製品詳細
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview

### W&B Weave

**概要**: Weights & Biases MLプラットフォームと深く統合された、生成AIアプリケーションの構築、デバッグ、Evalのための包括的なツールキットです。実験ループとプロダクション監視の接続に優れており、マルチモーダルEvalや動的リーダーボードなどの高度な機能を提供します。

**強み**:
- W&Bのトレーニングおよびモデルレジストリエコシステムとの深い統合。
- 動的リーダーボードやAudio Monitorsを含む高度なEval機能。
- カスタムFine-tuningモデル（LoRA）のテストをシームレスにサポート。
- すべてのアーティファクト（プロンプト、モデル、データセット）に対する堅牢なバージョニング。

**弱み**:
- 提供された資料内では、明示的なPIIマスキングやコンプライアンス機能が不足している。
- フルスタックAPMと比較して、インフラレベルのメトリクスへの重点が低い。
- メモリTracingや複雑なエージェントグラフの可視化に関する詳細は限定的。

**最新のアップデート**:
- Audio Monitors: LLM Judgeを使用して、テキストと共にオーディオ出力を監視・判定するモニター。(2026-02-01)
- Dynamic Leaderboards: Evalから自動生成されるリーダーボード。永続的なカスタマイズとCSVエクスポートが可能。(2026-01-29)
- PlaygroundでのカスタムLoRA: Weave PlaygroundでカスタムFine-tuningされたLoRAの重みを直接テスト・Evalできる機能。(2026-01-16)

### LangSmith

**概要**: LangSmithは、開発、デバッグ、テスト、デプロイをカバーする、LLMアプリケーションのライフサイクル全体のための包括的かつフレームワークに依存しないプラットフォームです。LangChainやLangGraphと深く統合されていますが、様々なスタック（OpenAI, Vercel, Pydantic AI）をサポートし、セルフホスティングやコンプライアンス認証などの堅牢なエンタープライズ機能を提供します。

**強み**:
- LangChainおよびLangGraphとの深いネイティブ統合によるシームレスなエージェントTracing。
- ペアワイズ比較や人間によるアノテーションキューを含む包括的なEvalスイート。
- セルフホスティング、SSO、コンプライアンス認証（SOC 2, HIPAA）を備えた堅牢なエンタープライズ提供。
- プロダクションのTracingをデータセットやテストに直接接続する強力な「ループ」機能。

**弱み**:
- フレームワークに依存しない能力があるにもかかわらず、「LangChain専用」という認識が他のフレームワークのユーザーを遠ざける可能性がある。
- Trace数やシート数に基づく料金モデルは、大規模運用において複雑または高価になる可能性がある。
- 広範な機能セットは、新規ユーザーにとって学習曲線が急になる可能性がある。

**最新のアップデート**:
- Python SDK v0.7.1: LangSmithプラットフォームに接続するためのクライアントライブラリのアップデート。(2026-02-10)
- Traceプレビューのカスタマイズ: UIでのTraceのプレビュー方法をカスタマイズできる機能。(2026-02-06)
- Google Gen AI Wrapperエクスポート: SDKにおけるGoogle Gen AIラッパーのエクスポート機能。(2026-01-31)
- LangSmith Self-Hosted v0.13: セルフホスティングプラットフォームの新しいバージョンリリース。(2026-01-16)

### Langfuse

**概要**: Langfuseは、Observability、プロンプト管理、Evalを単一のワークフローに統合するオープンソースのLLMエンジニアリングプラットフォームです。強力なセルフホスティング機能と、グラフ表示や推論Traceを含むエージェントワークフローへの深いサポートが特徴です。

**強み**:
- 完全にオープンソースでセルフホスト可能であり、ベンダーロックインを防止。
- バージョニングとデプロイ機能を備えた統合プロンプト管理。
- LLM-as-a-Judgeや人間によるアノテーションを含む包括的なEvalスイート。
- エージェントワークフロー（グラフ、推論Trace）の強力なサポート。
- 支出アラートを備えた粒度の高いコストおよびトークントラッキング。

**弱み**:
- RLHF/Fine-tuningトレーニングジョブのネイティブなオーケストレーションが不足（データ準備に重点）。
- モデルのバージョニングが、完全なモデルの重みのレジストリではなく、設定/プロンプトに限定されている。
- 高度な機能やセルフホスティングのセットアップが複雑になる場合がある。

**最新のアップデート**:
- Org Audit Log Viewer: 組織の監査ログを表示するための新しいUI。(2026-02-09)
- 思考/推論プロセスのレンダリング: Trace詳細において、思考および推論Trace（推論モデルなど）のレンダリングをサポート。(2026-02-09)
- 単一ObservationのEval: 個別のObservationに対してEvalを追加できる機能。(2026-02-09)
- イベントベースのTraceテーブル: Observation/Traceテーブル用の新しいイベントベース表示モード。(2026-02-09)
- Traceの修正出力: Fine-tuningデータセットを構築するために、TraceビューでLLM出力の改善版を直接キャプチャ。(2026-01-14)

### Braintrust

**概要**: Braintrustは、Observability、Eval、プロンプト管理を組み合わせ、開発者体験に重点を置いた包括的なAIエンジニアリングプラットフォームです。複数の言語（Go, Java, C#を含む）にわたる堅牢なSDKサポート、CursorなどのIDEとの深い統合、およびオンラインScoringとデータセットを介した継続的改善のための厳格な「ループ」ワークフローで差別化を図っています。

**強み**:
- 市場で最も広範なSDKサポート（Python, TS, Go, Java, Ruby, C#）。
- 開発者ツール（Cursor, VS Code, MCP）との深い統合。
- データセット、実験、プロダクション監視を接続する統合ワークフロー。
- セルフホスティングやRBACを含む強力なエンタープライズ機能。
- カスタム分析のための柔軟なクエリ言語（BTQL）。

**弱み**:
- 汎用的なスパンと比較して、RAG検索チャンクに特化した可視化が不足。
- Fine-tuning APIへデータをエクスポートするための直接的な統合がない。
- PIIマスキング機能が、一部の競合他社と比較して目立たない、または自動化されていない。
- ツールの成功率分析には、標準のウィジェットではなくカスタムクエリが必要。

**最新のアップデート**:
- カスタムビューでの添付ファイルレンダリング: カスタムTraceビューで画像、動画、音声、その他の添付ファイルを直接レンダリング可能に。(2026-02)
- Traceの起点へのナビゲート: ログ内のTraceから、その起点となったプロンプトやデータセットの行へ戻るナビゲーション。(2026-02)
- TraceレベルのScorer: カスタムコードScorerが実行Trace全体にアクセスし、マルチステップのワークフローをEval可能に。(2026-02)
- LangSmith統合: TracingおよびEvalの呼び出しをLangSmithとBraintrustの両方に送信、またはBraintrustのみにルーティングするラッパー。(2026-02)
- Cursor統合: Cursorからログや実験をクエリするためのBraintrust MCPサーバーを自動設定する拡張機能。(2026-02)
- 画像レンダリングのセキュリティ制御: 画像URLからの機密データ漏洩を防ぐための設定可能なモード（自動読み込み、クリックで読み込み、ブロック）。(2026-02)
- 集計を伴う単一スパンフィルタ: 単一スパンフィルタとGROUP BYを組み合わせ、スパンレベルの条件に基づいてTraceを集計。(2026-02)
- 自動Instrumentation (Python, Ruby, Go): Python、Ruby、Goアプリケーション向けのコード改変不要なTracingサポート。(2026-01)
- Temporal統合: 分散Tracingサポートにより、Temporalのワークフローとアクティビティを自動的にTracing。(2026-01)
- レビュー用カンバンレイアウト: フラグが立てられたスパンやレビュー状態を管理するためのドラッグ＆ドロップインターフェース。(2026-01)

### MLflow

**概要**: 堅牢なObservability、Eval、エージェント管理機能を備え、GenAIへと大幅に拡張された包括的なオープンソースMLOpsプラットフォームです。実験やプロンプトエンジニアリングからプロダクション監視、デプロイまで、ライフサイクル全体をカバーするオールインワンソリューションを、完全なOpenTelemetry互換性と共に提供します。

**強み**:
- トラッキング、レジストリ、Eval、デプロイをカバーする包括的な「オールインワン」プラットフォーム。
- 完全なOpenTelemetry互換性を備えたベンダーニュートラルな設計。
- Judge BuilderやMemAlignオプティマイザーを含む高度なEval機能。
- PythonおよびTypeScript SDKによる強力なエコシステムサポート。
- 組織サポートやRBACなどのエンタープライズグレードの機能。

**弱み**:
- セルフホスティングには、インフラやデータベースバックエンドの管理が必要。
- 幅広い機能セットは、軽量でニッチなツールと比較して圧倒的に感じられる場合がある。
- UIのテレメトリ収集（オプトアウト可能）は、プライバシーに敏感なユーザーにとって懸念事項となる可能性がある。

**最新のアップデート**:
- 組織サポート: 実験やリソースを整理するためのマルチワークスペース環境のサポート。(2026-02-12)
- MLflow Assistant: エージェントのデバッグや問題解決を支援する、Claude Codeを搭載した製品内チャットボット。(2026-01-29)
- エージェントパフォーマンスDashboard: エージェントのレイテンシ、リクエスト数、品質スコアを監視するための構築済みチャート。(2026-01-29)
- MemAlign Judge Optimizer: フィードバックからEvalガイドラインを学習し、Judgeの精度を向上させるアルゴリズム。(2026-01-29)
- Judge Builder UI: コードなしでLLM Judgeプロンプトを作成、テスト、反復するためのビジュアルインターフェース。(2026-01-29)
- 継続的オンライン監視: プロダクションの入力Traceに対してLLM Judgeを自動的に実行。(2026-01-29)
- 分散Tracing: コンテキスト伝播により、複数のサービスにわたるリクエストを追跡。(2026-01-29)

### Arize Phoenix

**概要**: Arize Phoenixは、実験とプロダクション監視のギャップを埋めるオープンソースのAI ObservabilityおよびEvalプラットフォームです。OpenTelemetryによる堅牢なTracing、ツールの使用メトリクスを含む高度なエージェントEval機能、およびプロダクションのTraceをEvalデータセットに変換するシームレスなワークフローを提供します。

**強み**:
- 堅牢なセルフホスティングオプション（Docker/Kubernetes）を備えた強力なオープンソース基盤。
- 特化したツール選択および呼び出しメトリクスを含む、高度なエージェントEval機能。
- プロダクションのTraceをキュレートされたEvalデータセットに変換するシームレスなワークフロー。
- 機能豊富なCLI、SDK、Prompt Playgroundを備えた包括的な開発者体験。
- 主要なLLMフレームワーク（LangChain, LlamaIndex）およびプロバイダーとの深い統合。

**弱み**:
- 提供されたドキュメント内では、明示的なPIIマスキングおよびデータ秘匿化機能が不足。
- 監査ログ機能の詳細が明示されていない。
- データエクスポート以外の強化学習（RL）ワークフローに対するネイティブサポートが限定的。
- メモリ状態の可視化は、ツールや検索のTracingと比較してあまり強調されていない。

**最新のアップデート**:
- データセットEvaluator: 実験中にサーバー側で自動実行するために、Evaluatorをデータセットに直接アタッチ。(2026-02-12)
- OpenAI Responses APIタイプのサポート: OpenAIモデルにおいてChat CompletionsとResponses APIタイプの選択をサポート。(2026-02-12)
- Playground用のカスタムプロバイダー: Playgroundとプロンプト間で再利用可能なカスタムプロバイダーの集中設定。(2026-02-11)
- Claude Opus 4.6モデルのサポート: 拡張された思考パラメータをサポートするAnthropicのClaude Opus 4.6をサポート。(2026-02-09)
- ツール選択および呼び出しEvaluator: エージェントのツール選択の正確性とパラメータ形式を評価する新しいEvaluator。(2026-01-31)
- 設定可能なメール抽出: JMESPathを使用したOAuth2プロバイダーからのカスタムメール抽出をサポート。(2026-01-28)
- Phoenix CLIコマンド: ターミナルからプロンプト、データセット、実験を管理するためのCLIコマンド。(2026-01-22)
