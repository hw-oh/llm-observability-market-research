# W&B Weave — 製品詳細

[← ホーム](./) · [詳細比較](./comparison)

### Weave

**概要**: Weaveは、モデル構築（W&B）とアプリケーション開発の間の不可欠な架け橋として位置付けられており、評価とオブザーバビリティ（可観測性）に対してコードファーストのアプローチを提供します。アプリケーションレイヤーのみに焦点を当てた競合他社とは異なり、Weaveはその実績を活かして、本番環境のインサイトをファインチューニングやトレーニングデータのループにフィードバックします。

**主な強み**:
- W&B Model RegistryおよびTrainingパイプラインとの独占的な統合により、競合他社が模倣できない独自の「データフライホイール」を構築。
- HeliconeやLogfireのような硬直したUIベースのワークフローと比較して、プログラムによる評価における優れた柔軟性。
- Arize PhoenixやLangfuseの実用本位なインターフェースを凌駕する、複数のステークホルダーによるレビューのための洗練されたコラボレーションUI。

**改善が必要な領域**:
- Langfuseや旧Humanloopが提供していたような、非技術者向けの成熟した「プロンプトCMS」およびプレイグラウンドの欠如。
- SDKエコシステムの限定（Python/JSのみ）により、エンタープライズのバックエンドチーム（Java/Go/C#）がBraintrustに流れるリスク。
- 深いインフラストラクチャ（データベースクエリ、API内部）のトレースができないため、LogfireやArizeのようなOTelネイティブなツールに対して不利。

**最新のアップデート**:

### LangSmith

**概要**: LangSmithは、LLMアプリケーションのための包括的なDevOpsプラットフォームであり、LangChainおよびLangGraphエコシステムとの深い統合を提供します。エージェントのオブザーバビリティ、ヒューマン・イン・ザ・ループの評価ワークフロー（アノテーションキュー）、プロンプト管理に優れており、複雑なエージェントを構築するチームにとってのデフォルトの選択肢となっています。

**Weaveに対する強み**:
- LangChain/LangGraphエコシステムにおける圧倒的なシェアにより、それらのフレームワークのユーザーにとってデフォルトの選択肢となっている。
- 非技術者のレビュアー向けに専用のアノテーションキューを備えた、優れた「ヒューマン・イン・ザ・ループ」ワークフロー。
- 統合されたプロンプトハブにより、アプリを離れることなくシームレスなプロンプトのバージョニングとプレイグラウンドでのテストが可能。
- 複雑なエージェントワークフロー（LangGraph）のビジュアルデバッグは、現在クラス最高。

**Weaveに対する弱み**:
- モデルのトレーニング/ファインチューニングのライフサイクルとの深い統合（W&Bの核心的な強み）が欠けている。
- LangChainフレームワークを使用していない開発者にとっては、「重い」または過度に複雑に感じられることがある。
- トレース量やシート数に基づく価格モデルは、W&Bの柔軟なエンタープライズ条件と比較してコストが高くなる可能性がある。
- LLM以外の汎用的な機械学習実験トラッキングへの焦点が薄い。

**最新のアップデート**:
- トレースプレビューのカスタマイズ：トレースの入力と出力のどの部分をトレーステーブルに直接表示するかを構成できるようになり、カスタムデータ構造の使い勝手が向上。（2026-02-06）
- LangSmith セルフホスト v0.13：セルフホスト顧客向けのメジャーリリース。「インサイト」機能やパフォーマンス向上など、クラウド版との機能パリティを拡大。（2026-01-16）
- SDK v0.7.1：Python/JSクライアントライブラリのアップデート。最新のLangChainバージョンとの互換性確保と安定性の向上。（2026-02-10）

### Arize Phoenix

**概要**: Arize Phoenixは、オープンソースファーストかつOpenTelemetryネイティブなオブザーバビリティプラットフォームであり、ローカルデバッグとエンベディングレベルの分析に優れています。本番前の実験と本番モニタリングの架け橋として位置付けられ、広範なArize AIプラットフォームを活用してエンタープライズグレードの異常検知とベクトル可視化を提供します。

**Weaveに対する強み**:
- ネイティブなOpenTelemetry (OTLP) 基盤は、すでにDatadog/Honeycombを使用しているプラットフォームエンジニアにとって魅力的。
- 検索/RAGの失敗をデバッグするための優れたエンベディング可視化 (UMAP)。
- データをクラウドに送信せずにフルUIを使用できる、強力な「ローカルファースト」体験（OSS版）。
- LlamaIndexエコシステムとの深い統合。

**Weaveに対する弱み**:
- トレーニングレイヤー（W&B Runs/Artifacts）との統合が欠けており、MLトレーニングとLLMアプリ開発の間にサイロが生じている。
- UIがより技術的/実用本位であり、Weaveよりも非技術的なPMにとってアクセスしにくい可能性がある。
- フル機能を大規模にセルフホストするには、Weave SaaSと比較して多大なインフラ管理が必要。
- コラボレーション機能（アノテーションキュー、チームコメント）が比較的未成熟。

**最新のアップデート**:
- Claude Opus 4.6 サポート：PlaygroundにClaude Opus 4.6のサポートを追加し、サイドバイサイドの比較が可能に。（2026-02-09）
- ツール選択エバリュエーター：エージェントのツール選択ロジックの精度を測定するために特別に設計された新しいエバリュエーター。（2026-02-06）
- FaithfulnessEvaluator：よりきめ細かなRAGチェックのために、新しいFaithfulnessEvaluatorを導入（HallucinationEvaluatorは非推奨に）。（2026-02-02）
- ツール呼び出し精度メトリクス：エージェントがツールを正しく呼び出す頻度を追跡するための特定のメトリクスを追加。（2026-01-27）

### Braintrust

**概要**: Braintrustは、評価、プロンプトエンジニアリング、オブザーバビリティを密接に統合したエンタープライズグレードのAIエンジニアリングプラットフォームです。インフラ制御（キャッシュ、レート制限）のための堅牢なAI Proxyで差別化を図っており、最近では幅広いSDKサポート（Go, Java, C#）やIDE統合（Cursor, MCP）により開発者ワークフローへ積極的に拡大しています。

**Weaveに対する強み**:
- 幅広いSDKエコシステム：Go, Java, Ruby, C#のネイティブサポートにより、WeaveのPython/JS中心の焦点ではカバーしきれないエンタープライズのバックエンドチームを取り込んでいる。
- 統合されたAI Proxy：キャッシュ、レート制限、キー管理のための組み込みゲートウェイにより、即時のインフラユーティリティを提供。
- 高度な分析 (BTQL)：SQLベースのクエリにより、標準的なUIフィルターよりも深く柔軟なトレースデータの随時分析が可能。
- IDE統合：CursorやMCPとの直接統合により、Braintrustをコーディングワークフローに組み込み、コンテキストスイッチを削減。

**Weaveに対する弱み**:
- トレーニング/ファインチューニングの断絶：W&Bのモデルトレーニングパイプラインとのネイティブな統合が欠けており、フルライフサイクルのMLチームにとっての定着性が低い。
- 視覚的なカスタマイズ：ダッシュボードは機能的だが、W&Bのパネルシステムに見られるような高度なカスタマイズ性に欠ける。
- コミュニティの規模：ML研究コミュニティにおけるW&Bの巨大な足跡は、Braintrustがまだ到達していないネットワーク効果を提供している。

**最新のアップデート**:
- トレース起点へのナビゲート：トレースを、それを生成した特定のプロンプトバージョンやデータセットの行にリンクする機能。（2026-02）
- トレースレベルのスコアラー：カスタムスコアラーが実行トレース全体にアクセスして、マルチステップのワークフローやエージェントの動作を評価できるようになった。（2026-02）
- LangSmith 統合：LangSmithのトレースおよび評価コールを、並行してまたは排他的にBraintrustにルーティングするためのラッパー。（2026-02）
- Cursor 統合：自然言語を介してログを照会し、実験結果を取得するためのCursorエディタ用Braintrust拡張機能。（2026-02）
- 自動インストルメンテーション (Python/Ruby/Go)：主要言語のコードレス・トレース・サポートにより、初期セットアップを簡素化。（2026-01）

### Langfuse

**概要**: Langfuseは、オブザーバビリティ、プロンプト管理、評価を統合された開発者体験に結びつけた、主要なオープンソースLLMエンジニアリングプラットフォームです。「LangSmithのオープンソース代替」というポジションを積極的に狙っており、スケーラビリティのためにClickHouseバックエンドを活用し、管理型クラウド層と並んで堅牢なセルフホスト機能を提供しています。

**Weaveに対する強み**:
- オープンソース & セルフホスト：複雑なエンタープライズ契約なしにネイティブにセルフホストできる能力は、セキュリティを重視するチームにとって魅力的。
- プロンプト管理CMS：Weaveのコード中心のアプローチと比較して、非技術ユーザーがプロンプトを管理、バージョン管理、デプロイするためのよりフル機能なUI。
- エージェントの可視化：専用の「エージェントグラフ」ビューにより、標準的なトレースツリーよりも複雑なエージェントワークフローを直感的に把握できる。

**Weaveに対する弱み**:
- トレーニングの統合：WeaveがW&Bエコシステムから継承している、モデルトレーニング、ファインチューニング、アーティファクトのバージョニングとの深い接続が欠けている。
- UIの洗練度：機能的ではあるが、UIはWeaveの洗練されたデザインと比較して、より実用本位でまとまりに欠けると感じられることがある。
- 管理されたスケール：クラウド層はあるものの、W&Bの巨大なスケールのインフラとの関連性は、大企業にとっての信頼の拠り所となっている。

**最新のアップデート**:
- トレースの修正出力：ファインチューニングデータセットを構築するために、LLM出力の改善されたバージョンをトレースビューで直接取得。（2026-01-14）
- オブザベーション I/O へのインラインコメント：トレースの入力および出力内の特定のテキスト選択にコメントを固定。（2026-01-07）

### Logfire

**概要**: Logfireは、OpenTelemetry上に構築されたオブザーバビリティプラットフォームであり、Pydanticエコシステムとの深い統合を活用して、PythonアプリケーションやAIエージェントのシームレスなトレースを提供します。評価や実験ループに焦点を当てるWeaveとは異なり、Logfireは従来のAPMに近い位置付けで、LLM呼び出しと並んで本番トレース、バリデーションエラー、データベースの相互作用のデバッグに優れています。

**Weaveに対する強み**:
- シームレスなPydantic統合により、優れたバリデーションエラー追跡とデータ構造の可視化が可能。
- OpenTelemetryを介したフルスタックのオブザーバビリティ（SQL, Redis, FastAPI）により、LLMのみのトレースよりも優れたコンテキストを提供。
- トレースデータのSQLベースのクエリにより、開発者がカスタムビューを作成するための高い柔軟性を提供。
- Python向けのゼロ構成の自動インストルメンテーションは、非常に摩擦が少ない。

**Weaveに対する弱み**:
- 包括的な評価フレームワーク（データセット、スコアラー、リーダーボード）が欠けている。
- プロンプトエンジニアリングループのためのプロンプトプレイグラウンドや実験トラッキング機能がない。
- 非技術的なステークホルダー向けのコラボレーション機能（例：人間によるアノテーションキュー）がない。
- Python中心であり、Weaveはより広範な言語やフレームワークのエコシステムをサポートしている。

### Helicone

**概要**: Heliconeは、オープンソースのAIゲートウェイおよびオブザーバビリティプラットフォームであり、主にアプリケーションとLLMプロバイダーの間のプロキシとして機能します。運用制御（キャッシュ、レート制限、コスト追跡）に優れており、深いコードのインストルメンテーションを必要とせずにプロンプトとレスポンスのハイレベルなロギングを提供します。

**Weaveに対する強み**:
- ゼロコード統合：ユーザーは `base_url` を変更するだけでロギングを開始できる（WeaveはSDKのインストルメンテーションが必要）。
- ゲートウェイ機能：Weaveが提供していないキャッシュ、レート制限、スマートルーティング（フォールバック/ロードバランシング）を含む。
- コスト管理：コストの追跡と、ユーザー/キーごとの予算制限の強制において優れた粒度を持つ。

**Weaveに対する弱み**:
- 浅いトレース：Weaveのコードベースのトレースのように、アプリケーションロジックの内部（ループ、LLM以外の関数）を見ることはできない。
- 限定的な評価ワークフロー：Weaveのような豊富な比較評価ビューや回帰テスト機能が欠けている。
- トレーニングとのリンクなし：W&Bが支配するモデルトレーニング/ファインチューニングのライフサイクルから切り離されている。
