# LLM Observability — 製品詳細
**日付**: 2026-02-13 | **モデル**: google/gemini-3-pro-preview

### W&B Weave

**概要**: W&B Weaveは、Weights & Biasesのエコシステムに深く統合された、開発者中心のObservabilityおよびEvalツールキットであり、厳格なLLMアプリケーション構築のために設計されています。コードファーストのEvalワークフロー、複雑なエージェントのTracing（ネイティブなMCPサポートを含む）、エンタープライズグレードのインフラストラクチャに優れており、オフラインの実験とオンラインのプロダクションモニタリングのギャップを埋めます。

**強み**:
- ライフサイクル全体の管理を可能にするW&Bエコシステム（Experiments、Models）との深い統合。
- ネイティブなMCP統合や複雑なトレースツリーを含む、エージェントパターンの強力なサポート。
- コードベースのScoringとGUIウィザードの両方を備えた包括的なEval機能。
- 堅牢なデプロイオプション（SaaS、VPC、セルフホスト）とコンプライアンスを備えたエンタープライズ対応。

**弱み**:
- 非技術的なステークホルダー向けのノーコード「Prompt CMS」の欠如（プログラムによる作成に依存）。
- コード変更なしでインターセプトするためのネイティブなGateway/Proxyモードがない。
- 大規模チーム向けのアノテーションワークフローにおける高度なキュー管理の不足。

**最新のアップデート**:
- Audio Monitors: テキストと並行して音声出力を監視・判定するモニターを作成し、ボイスエージェントのEvalを可能に。(2026-02-01)
- Dynamic Leaderboards: フィルタリングやカスタマイズが可能な、Eval結果からのリーダーボード自動生成機能。手動設定を代替。(2026-01-29)
- PlaygroundでのカスタムLoRA: Weave Playgroundで直接、カスタムFine-tuningされたLoRA重みのテストとEvalをサポート。(2026-01-16)

### LangSmith

**概要**: LangSmithは、LLMアプリケーションのための包括的なDevOpsプラットフォームであり、他のフレームワークをサポートしつつ、LangChainエコシステムの事実上のObservability標準として機能しています。複雑なエージェントワークフローの深いTracing、Evalパイプライン、およびセルフホストを含むエンタープライズグレードのデプロイオプションに優れています。

**強み**:
- LangChainおよびエージェントワークフロー（ネストされたトレース、スレッドの再生）との深いネイティブ統合。
- オンライン/オフラインScoringとアノテーションキューを備えた堅牢なEvalフレームワーク。
- マルチテナント、専用環境、セルフホストKubernetesを含む柔軟なエンタープライズデプロイオプション。
- ターミナルベースの「Fetch」やカスタマイズ可能なトレースビューを含む包括的なデバッグツール。

**弱み**:
- リアルタイム防止のためのネイティブなアクティブGuardrailsの欠如（パッシブなロギングに重点）。
- ゼロコードのGateway/Proxyモードがなく、SDKによる実装が必要。
- テキスト機能と比較して、テキスト以外のマルチモーダルトレース可視化のサポートが限定的。
- コアプラットフォームがクローズドソースであるため、インフラ自体へのコミュニティ貢献が制限される。

**最新のアップデート**:
- トレースプレビューのカスタマイズ: LangSmith UIでのトレースのプレビュー方法をカスタマイズする機能。(2026-02-06)
- 非OTEL版 Google ADKラッパー: OpenTelemetryへの依存なしでGoogle ADK統合を可能にする新しいラッパー。(2026-02-02)
- Google Gen AIラッパー: Google Generative AI統合用のエクスポートされたラッパー。(2026-01-31)
- Gemini TSラッパー: Geminiモデル用のベータ版TypeScriptラッパー。(2026-01-26)
- LangSmith Self-Hosted v0.13: プラットフォームのセルフホスト版のアップデート。(2026-01-16)

### Langfuse

**概要**: Langfuseは、堅牢なObservabilityと包括的なEval、プロンプト管理機能を組み合わせたオープンソースのLLMエンジニアリングプラットフォームです。複雑なエージェントワークフローやRAGパイプラインのTracingに優れ、LangChainやLlamaIndexなどのフレームワークとの強力な統合を提供すると同時に、セルフホストソリューションを含む柔軟なデプロイオプションを提供します。

**強み**:
- フルセルフホストが可能なオープンソース（MITライセンス）。
- 主要なフレームワーク（LangChain、LlamaIndex）との強力な統合。
- 包括的なプロンプト管理およびバージョニングシステム（CMS）。
- 堅牢なLLM-as-a-JudgeおよびオンラインEval機能。
- 詳細なコストおよびトークン使用量のAnalytics。

**弱み**:
- ネイティブなマルチモーダルTracingサポートの欠如。
- PIIマスキングや機密データ検出機能が組み込まれていない。
- エージェントグラフ用の視覚的デバッグツールが限定的（DAGビューなし）。
- プロンプト出力のサイドバイサイド比較ビューがない。
- Fine-tuningパイプラインの統合が欠落している。

**最新のアップデート**:
- Observation単位のLLM-as-a-Judge: よりきめ細かな品質管理のために、特定のObservationに対して直接LLM-as-a-Judge Evalを実行するサポートを追加。(2026-02-13)
- Thinking/Reasoningトレースレンダリング: DeepSeekのようなChain-of-Thoughtモデルをサポートする、「思考」および「推論」部分の新しいトレース詳細レンダリング。(2026-02-05)
- インライントレースコメント: トレース内のIOデータの一部にインラインでコメントを追加できるようになり、コラボレーションが向上。(2026-01-25)
- 単一ObservationのEval: フル・トレースだけでなく、単一のObservationに対してEvalを実行可能に。(2026-02-08)

### Braintrust

**概要**: Braintrustは、Eval、プロンプト管理、Observabilityを継続的な改善ループに密接に統合したエンタープライズグレードのAI開発プラットフォームです。回帰テスト、カスタムScoring、データセット管理に優れており、厳格なエンジニアリング標準を重視するチームにとって非常に効果的です。強力なTracingとSDKサポートを提供していますが、現在は実行グラフやセッション再生のような視覚的デバッグツールよりも、定量的なEvalとリストベースのロギングを優先しています。

**強み**:
- プロンプトエンジニアリング、Eval、ロギングを接続する統合ワークフロー。
- 堅牢な回帰テストと実験管理機能。
- RBAC、SSO、ハイブリッドデプロイを含む強力なエンタープライズセキュリティ機能。
- 優れた自動実装サポートを備えた高品質なSDK。

**弱み**:
- エージェント実行グラフのような高度な視覚的デバッグツールの欠如。
- 複数ターンの会話を確認するためのネイティブなセッション再生UIがない。
- 特化型のセキュリティツールと比較して、標準のGuardrails（PII、ジェイルブレイク）が限定的。
- 完全なデータ主権のためのフルセルフホストやオープンソースのオプションがない。

**最新のアップデート**:
- Thread Retrieval API: Python SDKでプログラムによってスレッドを取得する機能を追加。(2026-02-12)
- サブエージェントのネスト: Claude Agent SDKラッパーでサブエージェントのネストをサポート。(2026-02-12)
- Review Span Type: 人間によるレビューワークフローをサポートするための特定の「Review」スパンタイプを導入。(2026-02-05)
- Classificationsフィールド: メタデータタグ付けを強化するためにSDKに分類フィールドを追加。(2026-01-31)
- Trace Scoring Candidate: Python SDK内で直接トレースをScoringする新機能。(2026-01-21)

### MLflow

**概要**: MLflowは、成熟したオープンソースのMLOpsプラットフォームであり、堅牢なTracing、Eval、ライフサイクル管理機能を備えたGenAI Observabilityへと正常に拡張されました。コアな実験管理、カスタムEval（LLM-as-a-Judge）、Autologによる広範なライブラリサポートに優れていますが、特化型のエージェント可視化やインタラクティブなプロンプトエンジニアリングワークフローでは、ニッチな競合他社に遅れをとっています。

**強み**:
- 業界標準の実験管理とモデルレジストリ。
- UIビルダーを備えた強力なLLM-as-a-Judge機能。
- LangChainなどのライブラリに対する広範な自動実装。
- 柔軟なオープンソースおよびセルフホストのデプロイオプション。
- 堅牢なカスタムメトリクスとDashboardシステム。

**弱み**:

### Arize Phoenix

**概要**: Arize Phoenixは、OpenTelemetry標準に基づいて構築された主要なオープンソースLLM ObservabilityおよびEvalプラットフォームであり、主にAIエンジニアが複雑なエージェントおよびRAGワークフローをTracing、デバッグ、Evalするために設計されています。堅牢なTracing可視化と包括的なEvalライブラリを組み合わせており、LangChainやLlamaIndexなどの主要フレームワークとのシームレスな統合を提供しますが、非技術的なプロンプト管理よりもコード中心のワークフローに重点を置いています。

**強み**:
- ネイティブなOpenTelemetryアーキテクチャにより、標準準拠とベンダーロックインの回避を保証。
- LangChain、LlamaIndex、およびその他の主要フレームワークに対するクラス最高の自動実装。
- 検索の可視化やセッション再生を含む、強力なRAGおよびエージェントデバッグツール。
- カスタムScorerと回帰テストをサポートする堅牢なコードベースのEvalフレームワーク。
- 完全なオープンソースのセルフホストからマネージドSaaSまで、柔軟なデプロイオプション。

**弱み**:
- コード外でプロンプトのバージョニングや管理を行うための非技術者向けPrompt CMSの欠如。
- ネイティブなGo SDKがなく、Goベースのインフラでの採用が制限される。
- コスト配分や予算追跡などのFinancial Operations機能の欠落。
- マルチモーダルTracing（画像/音声）や埋め込み空間の可視化をサポートしていない。
- GatewayやProxyモードがなく、すべてのTracingに直接的なSDK統合が必要。

**最新のアップデート**:
- Claude Opus 4.6 サポート: PlaygroundにClaude Opus 4.6モデルのサポートを追加。(2026-02-09)
- Tool Selection Evaluator: エージェントワークフローにおけるツール選択の品質を評価するための新しいEvaluatorを追加。(2026-02-06)
- Faithfulness Evaluator: 根拠性をチェックするためのFaithfulnessEvaluatorを導入（HallucinationEvaluatorは非推奨）。(2026-02-02)
- Tool Invocation Accuracy Metric: ツール呼び出しの精度を追跡するための新しいメトリクス。(2026-02-02)
- 設定可能なメール抽出: OAuth2での設定可能なメール抽出のためにEMAIL_ATTRIBUTE_PATHを追加。(2026-01-28)
- メトリクス用Cursorルール: 新しい組み込みメトリクス（LLM分類Evaluator）を作成するためのCursorルールを追加。(2026-01-21)
