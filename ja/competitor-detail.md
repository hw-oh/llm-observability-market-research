# W&B Weave — 製品詳細
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview

### Weave

**概要**: Weaveは「データ・フライホイール」エンジンを自負しており、W&Bの深いモデル・リネージ（系統管理）機能を活用することで、実験と本番環境のギャップを独自の手法で埋めています。Weaveはトレーニング成果物と推論トレースの接続に長けている一方で、成熟したアノテーション・ワークフローを持つ LangSmith や、本番デプロイメントに対して強力なガバナンスを提供する Braintrust のコントロールプレーン機能（AI Proxy）からの激しい追随に直面しています。

**主な強み**:
- **マルチモーダルなオブザーバビリティ:** 最近リリースされた Audio Monitors（2026年2月）により、Langfuse のようなテキスト中心の競合他社と比較して、音声エージェントの評価において明確な優位性を備えています。
- **トレーニングから推論までのリネージ:** Arize Phoenix や LangSmith とは異なり、Weaveは本番環境のトレースを W&B Registry 内のトレーニング実行や成果物にネイティブにリンクさせ、真の反復的な改善ループを可能にします。
- **サーバーレス LoRA 統合:** Playground（2026年1月）でカスタム・ファインチューニングされたアダプターをホットスワップできる機能は、MLflow や Braintrust には欠けている迅速な実験能力を提供します。

**改善が必要な領域**:
- **AI Proxy の欠如:** Weaveには、キャッシュ、レート制限、キー管理のための仲介プロキシ層がありません。これは Braintrust が現在リードしている主要な価値提案です。
- **人間によるアノテーション・ワークフロー:** LangSmith と Braintrust は、手動のデータ・ラベリング用に優れた専用の「アノテーション・キュー」やカンバン形式のビューを提供しています。WeaveのUIは、大規模な人間によるレビューにはあまり最適化されていません。
- **自動化された Judge（評価モデル）の最適化:** MLflow の新しい「MemAlign」および「Judge Builder」機能は、LLM Judge のチューニングを自動化しますが、Weaveは評価の信頼性を高めるためのセットアップ・プロセスがより手動的です。
- **エンタープライズ SDK の幅:** Braintrust が Java、Go、Ruby、C# をサポートしていることは、Python/TypeScript 専用ではない従来のエンタープライズ環境において、Weaveにとっての参入障壁となっています。

**最新のアップデート**:
- Audio Monitors: LLM Judge を使用して音声の入出力を監視・評価するモニターの作成をサポートし、音声エージェントの評価を可能にしました。(2026-02-01)
- Dynamic Leaderboards: 永続的なカスタマイズと CSV エクスポート機能を備えた、評価からのリーダーボード自動生成機能。手動セットアップが不要になりました。(2026-01-29)
- Playground でのカスタム LoRA: W&B Artifacts からカスタム LoRA 重みを読み込み、Weave Playground で直接テストできる統合機能。(2026-01-16)

### LangSmith

**概要**: LangSmith は LangChain エコシステムのためのネイティブなオブザーバビリティおよびエンジニアリング・プラットフォームであり、深いトレース、プロンプト管理、評価ワークフローを提供します。ゼロ構成の統合により LangChain/LangGraph アプリケーションのデバッグに優れていますが、フレームワークに依存しないワークロードのサポートも積極的に拡大しており、プロトタイプから本番まで LLM エンジニアリングの完全なライフサイクル・ツールとして位置づけられています。

**Weave に対する強み**:
- ネイティブな LangChain/LangGraph 統合: 膨大なユーザーベース向けのゼロ構成セットアップ。
- アノテーション・キュー: ヒューマン・イン・ザ・ループのレビューとデータ・ラベリングのための優れたワークフロー。
- Prompt Hub: Playground と直接統合された、プロンプトのバージョニングとコラボレーションのための成熟した専用 UI。
- オンライン評価: 本番トラフィック・ストリーム上で自動評価を実行するための、より強力な標準サポート。

**Weave に対する弱み**:
- トレーニングの統合: Weaveが W&B から継承している、モデル・トレーニングやファインチューニング・スイープとのシームレスな接続が欠けています。
- フレームワークの認識: 「LangChain 専用」と見なされることが多く、カスタム・スタックやライバル・フレームワーク（LlamaIndex/DSPy など）を使用しているチームを遠ざける可能性があります。
- カスタム可視化: LangSmith のやや硬直的なダッシュボード・ウィジェットと比較して、Weaveはより柔軟で高次元なカスタム・プロット機能を提供しています。

**最新のアップデート**:
- トレース・プレビューのカスタマイズ: トレース・プレビュー・テーブルに表示される列とデータをカスタマイズする機能。(2026-02-06)
- Google Gen AI ラッパー: Google Gen AI モデルのトレースを容易にする新しいラッパー・エクスポート。(2026-01-31)
- セルフホスト v0.13: 安定性が向上したセルフホスト版のアップデート。(2026-01-16)

### Langfuse

**概要**: Langfuse は、オブザーバビリティ、プロンプト管理、評価を緊密に統合した、オープンソースで開発者優先の LLM エンジニアリング・プラットフォームです。堅牢なセルフホスティング・オプション、大量のエージェント・トレースに対応する ClickHouse ベースのアーキテクチャ、およびデータセットやテストを含むアプリケーション・ライフサイクル全体のための包括的なツール・スイートによって差別化を図っています。

**Weave に対する強み**:
- オープンソース & セルフホスティング: ローカル/VPC デプロイメントのネイティブ・サポートは、セキュリティを重視するチームにとって魅力的です。
- プロンプト管理 CMS: Weaveの現在の提供内容と比較して、プロンプトのバージョニングとデプロイのためのより成熟した専用インターフェース。
- コスト & 請求の粒度: 複雑な価格設定ティアと支出アラートのための高度なモデリング。

**Weave に対する弱み**:
- トレーニング/ファインチューニング統合: Weaveが W&B エコシステムから継承している、モデル・トレーニング実行や成果物のバージョニングとの深い接続が欠けています。
- モデル・レジストリ: 重みのバージョニングのための真のモデル・レジストリは提供しておらず、プロンプト/構成のバージョニングのみに焦点を当てています。
- 視覚的分析: テーブルやグラフには強いものの、非構造化データの探索のための Weave の Board/Canvas 機能の方が柔軟性が高い場合が多いです。

**最新のアップデート**:
- トレースの出力修正: トレース・ビューで LLM 出力の改善バージョンを直接キャプチャし、ファインチューニング用データセットを構築する機能。(2026-01-14)

### Braintrust

**概要**: Braintrust は、独自の「AI Proxy」アーキテクチャを備え、オブザーバビリティ、評価、プロンプト管理を統合した、成熟したエンタープライズ向けの AI エンジニアリング・プラットフォームです。広範な SDK サポート（Java、Go、Ruby、C# を含む）と本番モニタリング用の洗練された「Online Evals」によって差別化されており、AI アプリケーションの包括的なオペレーティング・システムとして位置づけられています。

**Weave に対する強み**:
- **AI Proxy アーキテクチャ:** 組み込みプロキシにより、Weaveにはないキャッシュ、レート制限、キー管理機能を提供します。
- **エンタープライズ SDK の幅:** Java、Go、Ruby、C# のネイティブ・サポートは、Python/TS 中心の Weave よりも従来のエンタープライズ・バックエンドに適しています。
- **本番環境のアノテーション:** カンバン・ワークフローを備えた専用の「Review」ページは、Weaveの現在の UI よりもヒューマン・イン・ザ・ループの運用に特化しています。
- **プロンプト管理:** バージョニングと「ログからの実行」機能を備えた成熟した Playground は、プラットフォームの主要な柱です。

**Weave に対する弱み**:
- **トレーニングの統合:** ファインチューニング/トレーニング・フェーズにおける W&B の Model Registry や Sweeps との深い接続が欠けています。
- **価格の認識:** Weaveの開発者フレンドリーなエントリーポイントと比較して、プレミアムで高価なエンタープライズ・ツールと見なされることが多いです。
- **UI の複雑さ:** インターフェース（Loop、BTQL、Playgrounds、Datasets）は密度が高く、Weaveの軽量な UX よりも圧倒される可能性があります。
- **コミュニティの勢い:** Weaveは巨大な W&B コミュニティの恩恵を受けていますが、Braintrust はゼロからコミュニティを構築しています。

**最新のアップデート**:
- LangSmith 統合: LangSmith のトレースを Braintrust にルーティングするための実験的なラッパー。並行利用や移行を可能にします。(2026-02)
- トレース・レベルのスコアラー: カスタム・コード・スコアラーが実行トレース全体にアクセスし、多段階のワークフローを評価できるようになりました。(2026-02)
- 自動インストルメンテーション (Py/Ruby/Go): Python、Ruby、Go アプリケーション向けのコード不要のトレース・サポートが追加されました。(2026-01)
- Temporal 統合: 親子関係のマッピングを備えた Temporal ワークフローとアクティビティの自動トレース。(2026-01)
- レビュー用カンバン・レイアウト: フラグが立てられたスパンや人間によるレビュー・キューを管理するための新しいドラッグ・アンド・ドロップ・インターフェース。(2026-01)

### MLflow

**概要**: MLflow は成熟したオープンソースの MLOps プラットフォームであり、3.x シリーズで GenAI へと積極的に転換し、トラッキング、レジストリ、オブザーバビリティのための統合スタックを提供しています。2026年初頭の MLflow 3.9.0 のリリースにより、OpenTelemetry ネイティブのトレース、高度な「LLM-as-a-Judge」ツール、本番モニタリング・ダッシュボードを含む包括的なスイートを提供し、その膨大な導入ベースを活かして特化型の LLM ツールと直接競合しています。

**Weave に対する強み**:
- 統合プラットフォーム: 従来の ML（レジストリ、デプロイメント）と GenAI オブザーバビリティをシームレスに接続し、ツールの乱立を抑えます。
- 高度な評価ツール: 「MemAlign」と「Judge Builder」は、Judge の信頼性を向上させるための洗練された自動ワークフローを提供します。
- OpenTelemetry ネイティブ: 完全な OTel 互換性により、ベンダーロックインを回避し、既存の APM スタックとの統合を容易にします。
- エンタープライズへの浸透: すでに数千の企業で（多くの場合 Databricks 経由で）導入されており、データ・チームにとって導入の摩擦がありません。

**Weave に対する弱み**:
- インフラの複雑さ: トラッキング・サーバーとデータベースの管理が必要であり、Weaveと比較して個々の開発者にとっての摩擦が大きくなります。
- UI の密度: インターフェースは複雑で「重く」、アプリ開発者が好む迅速でインタラクティブな反復よりも、MLOps エンジニア向けに設計されています。
- レガシーの負債: 「実験/実行（Experiment/Run）」のパラダイムは、Weaveのトレース優先の設計と比較して、動的でマルチターンのエージェント・トレースに適用すると不自然に感じられることがあります。

**最新のアップデート**:
- 組織サポート: マルチワークスペース環境と組織レベルのリソース管理のサポート。(2026-02-12)
- MLflow Assistant: トレースのデバッグ、テストの作成、エージェントの最適化を支援する、Claude Code 搭載の製品内 AI チャットボット。(2026-01-29)
- MemAlign Judge Optimizer: フィードバックから学習し、LLM Judge のプロンプトと基準を自動的に最適化するアルゴリズム。(2026-01-29)
- Judge Builder UI: カスタム LLM Judge の作成、テスト、検証のためのノーコード・ビジュアル・インターフェース。(2026-01-29)
- エージェント・パフォーマンス・ダッシュボード: 実験 UI におけるレイテンシ、リクエスト数、品質スコアのための構築済み可視化機能。(2026-01-29)

### Arize Phoenix

**概要**: Arize Phoenix は、LLM アプリケーションのトレース、デバッグ、テストというエンジニアリングの「インナーループ」向けに設計された、オープンソースで OpenTelemetry ネイティブのオブザーバビリティおよび評価プラットフォームです。スタンドアロンのローカル開発ツールとして機能し、本番モニタリングのために商用の Arize AX プラットフォームへデータをシームレスに昇格させることができ、エージェント・ワークフローにおける標準準拠の選択肢として位置づけられています。

**Weave に対する強み**:
- OpenTelemetry ネイティブ: 完全に OTel 標準（OpenInference）に基づいて構築されており、独自の SDK よりも広範なエコシステム互換性を提供します。
- ローカル優先 & CLI: 堅牢なローカル・ホスティングと、プロンプト管理のための新しいターミナル・ツールにより、優れた「インナーループ」体験を提供します。
- 特化型エージェント評価器: 特定のエージェントの失敗（例：ツールの選択 vs 呼び出し）に対する標準のメトリクスにより、セットアップ時間を短縮します。

**Weave に対する弱み**:
- モデル・リネージのギャップ: トレーニング成果物やモデル・バージョニング（W&B）との深い統合が欠けており、モデルを主に API 文字列として扱います。
- 断片化されたプラットフォーム: Phoenix（開発）と Arize AX（本番）の分離は、Weaveの統合プラットフォームと比較して、一貫性のない体験を生み出します。
- 状態管理: メッセージのトレースは行いますが、時間の経過に伴う複雑なエージェントの状態変化を追跡するための、Weaveのような洗練されたオブジェクト・バージョニングが欠けています。

**最新のアップデート**:
- データセット評価器: 評価器をデータセットに直接アタッチし、実験中にサーバー側で自動的に実行する機能。(2026-02-12)
- Playground 用カスタム・プロバイダー: Playground におけるカスタム・モデル・プロバイダーとルーティングのための中央集中型構成。(2026-02-11)
- ツール選択 & 呼び出し評価器: エージェントが正しいツールを選択し、正しいパラメータで呼び出したかどうかを判定する専用の評価器。(2026-01-31)
- プロンプト/データセット用 CLI コマンド: プロンプト/データセットのリスト表示、閲覧、および他のツールへのパイプ渡しを行うためのターミナル・コマンド。(2026-01-22)
- スパン関連付けを備えたトレースからデータセットへの変換: 元のソース・スパンへの双方向リンクを保持したまま、トレースからデータセットを作成する機能。(2026-01-21)
