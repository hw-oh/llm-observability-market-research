# LLM Observability — 製品詳細
**日付**: 2026-02-23 | **モデル**: google/gemini-3-pro-preview

### W&B Weave

**概要**: W&B Weaveは、Weights & Biases MLOpsプラットフォームに深く統合された、開発者中心のObservabilityおよびEvaluationツールキットです。エージェントやマルチモーダルアプリケーションのプログラムによるTracingに優れており、堅牢なエンタープライズインフラを提供すると同時に、ダイナミックなリーダーボードやJudgeウィザードなどのローコードEvaluationツールへと拡張しています。

**強み**:
- エンドツーエンドのMLOpsを実現するW&Bエコシステム（Experiments、Models）との深い統合
- マルチモーダルTracingおよびEvaluation（Audio、Image、Video）のネイティブサポート
- 堅牢なエンタープライズコンプライアンス（SOC2、HIPAA、GDPR）と柔軟なデプロイメントオプション
- Model Context Protocol (MCP) Tracingのファーストクラスサポート
- 新しいノーコードウィザードオプションによる強力なプログラムによるEvaluation機能

**弱み**:
- キューやレビュー担当者の割り当てといった専用のアノテーションワークフロー機能の欠如
- ネイティブなプロンプト最適化やDSPy統合がない
- 競合他社と比較して限定的な「ノーコード」プロンプト管理（CMS）
- クラスター分析のためのEmbedding空間の可視化がない

**最新のアップデート**:
- Audio Monitors: テキストと並行してAudio出力を監視・判定するモニターの作成をサポートし、ボイスエージェントのEvaluationを可能にしました。(2026-02-01)
- Dynamic Leaderboards: フィルタリングやカスタマイズが可能なEvaluation結果からのリーダーボード自動生成機能。手動設定を不要にしました。(2026-01-29)

### LangSmith

**概要**: LangSmithは、LangChainエコシステム内での深いObservability、Evaluation、プロンプトエンジニアリングに特化した、LLMアプリケーション向けの包括的なDevOpsプラットフォームです。階層的なトレースを通じて複雑なエージェントワークフローを可視化することに優れており、データセット管理、回帰テスト、ヒューマンレビューのための堅牢なツールを提供します。開発とモニタリングには強い一方で、一部の競合他社に見られるネイティブなランタイムGuardrailsやプロキシベースのセキュリティ機能は現在欠けています。

**強み**:
- 複雑なエージェントワークフローやRAGをTracingするためのLangChainとの深い統合
- LLM-as-a-judgeやアノテーションキューを含む包括的なEvaluationスイート
- ハイブリッドおよびセルフホストVPCを含む、包括的なエンタープライズデプロイメントオプション
- Run Treesによるネストされたチェーンやツール呼び出しの階層的な可視化

**弱み**:
- プロキシベースの競合他社と比較して、ネイティブなランタイムGuardrails（PII、ジェイルブレイク、トピック）の欠如
- 現在のところマルチモーダルTracing（画像/音声）のサポートが見当たらない
- 手動のインストルメンテーションなしでは、LangChain以外のフレームワークに対するネイティブサポートが限定的
- RAG分析のためのEmbedding空間の可視化がない

**最新のアップデート**:
- Traceプレビューのカスタマイズ: LangSmith UIでTraceプレビューをカスタマイズする機能。(2026-02-06)
- Python async用サンドボックスエンドポイント: Pythonの非同期実行のためのサンドボックスエンドポイントサポートを追加。(2026-02-05)
- 非OTEL Google ADKラッパー: Google ADK用の非OpenTelemetryラッパーを追加。(2026-02-02)
- Google Gen AIラッパーのエクスポート: SDKでGoogle Gen AIラッパーをエクスポート。(2026-01-31)

### Langfuse

**概要**: Langfuseは、OpenTelemetryベースのTracingと、堅牢なEvaluationおよびプロンプト管理ツールを組み合わせた、主要なオープンソースLLMエンジニアリングプラットフォームです。複雑なRAGやエージェントワークフローにきめ細かな可視性を提供することに優れており、マルチテナントSaaSから完全にセルフホストされたエンタープライズ環境まで、柔軟なデプロイメントオプションを提供します。

**強み**:
- 完全にオープンソースでセルフホスト可能であり、完全なデータ主権を提供
- Evaluationワークフロー（LLM-as-a-judge）をTracingデータと直接強力に統合
- 非エンジニアとのコラボレーションのための堅牢なプロンプト管理およびバージョン管理システム（CMS）
- 属性別の包括的なコストおよびトークン使用量Analytics
- 標準準拠のObservabilityを保証するネイティブなOpenTelemetryサポート

**弱み**:
- ネイティブなマルチモーダルTracing機能の欠如
- 中央集権的なトラフィック管理のための組み込みAI Gatewayやプロキシモードがない
- セキュリティ重視の競合他社と比較して、自動化されたGuardrails（PII、ジェイルブレイク）が限定的
- 公式のGo SDKがなく、Goベースのバックエンドサービスのサポートが制限されている
- Fine-tuningパイプラインの統合が現在欠落している

**最新のアップデート**:
- Single Span Evals (Open Beta): トレース全体だけでなく、個々のスパンに対してEvaluationを実行可能に。(2026-02-23)
- Observations上のLLM-as-a-Judge: 特定のObservationに対して直接LLM-as-a-judgeエバリュエーターを実行するサポート。(2026-02-23)
- Reasoning/Thinking Traceのレンダリング: モデル出力の「思考（thinking）」や推論部分をレンダリングするためのUIサポート（例：推論モデル用）。(2026-02-06)
- 組織監査ログビューアー: 組織レベルの監査ログを表示するための新しいUI。(2026-02-06)

### Braintrust

**概要**: Braintrustは、Evaluation主導の開発ライフサイクルを強調する、開発者向けのAI ObservabilityおよびEvaluationプラットフォームです。堅牢なコアTracingおよび実験管理と、エンタープライズグレードのセキュリティおよびデプロイオプションを組み合わせており、本番用LLMアプリケーションを構築するエンジニアリングチームに非常に適しています。オフラインEvaluationとプロンプト管理に優れる一方で、リアルタイムのプロキシベースGuardrailsや広範なプリセットのエージェント可視化ツールは現在欠けています。

**強み**:
- カスタムScoring機能と回帰テストを備えた強力なEvaluationフレームワーク
- 堅牢なプロンプト管理とPlaygroundの統合
- ハイブリッドデプロイとコンプライアンス（HIPAA/SOC2）を備えたエンタープライズ対応
- Python、TypeScript、Go向けの高品質なSDK

**弱み**:
- Guardrailsのためのリアルタイムプロキシ/Gatewayの欠如
- ヒューマンレビューワークフローのための専用アノテーションキューUIがない
- エージェントグラフやEmbeddingのための標準の可視化機能が限定的
- 組み込みのFine-tuningパイプライン統合がない

**最新のアップデート**:
- Experiment Tags: SDK経由で実験作成時にタグを渡すことが可能に。(2026-02-23)
- Thread Retrieval: Python SDKにスレッドの詳細を取得する機能を追加。(2026-02-12)
- OpenAI Agents統合: OpenAIエージェント統合のために、すべてのスパンタイプを処理できるようPython SDKを更新。(2026-02-05)
- Review Span Type: アノテーションワークフロー用と思われる、特定の「review」スパンタイプをSDKに追加。(2026-02-05)
- Classificationsフィールド: Python SDKにclassificationsフィールドを追加。(2026-01-31)
- Eval Cache Control: Evaluation中にキャッシュをオフにするオプションを追加。(2026-01-29)

### MLflow

**概要**: 主要なオープンソース機械学習ライフサイクルプラットフォームであり、堅牢なTracing、Evaluation、実験管理により、GenAI Observabilityへと大幅に拡張されました。コード中心のワークフロー、自動インストルメンテーション、主要フレームワークとの統合に優れる一方で、特化したエージェントの可視化やネイティブなセーフティGuardrailsは、ニッチな競合他社と比較して現在欠けています。

**強み**:
- 標準を確立した実験管理を備えた、巨大なオープンソースエコシステム
- 主要なGenAIフレームワーク向けの1行の自動インストルメンテーション
- 「LLM-as-a-Judge」と回帰テストを備えた強力なEvaluationフレームワーク
- セルフホストおよびマネージドクラウド環境にわたる柔軟なデプロイメントオプション

**弱み**:
- 限定的なネイティブGuardrailsおよびセーフティ機能
- 実行グラフのような高度なエージェント可視化の欠如
- 非エンジニア向けの組み込みノーコードPlaygroundがない
- ネイティブなデータウェアハウスエクスポート機能の不在

**最新のアップデート**:
- 組織サポート: 実験とリソースを論理的に分離するためのマルチワークスペース環境のサポート。(2026-02-20)
- MLflow Assistant: UI内で直接問題を診断・修正するのを支援する、Claude Codeをバックエンドとした製品内チャットボット。(2026-01-29)

### Arize Phoenix

**概要**: Arize Phoenixは、LLMアプリケーションエンジニアリング専用に設計された、オープンソースでOpenTelemetryネイティブなObservabilityおよびEvaluationプラットフォームです。RAGの検索やツール実行を含む複雑なエージェントワークフローのTracingに優れており、堅牢なコードファーストのEvaluation機能を提供しますが、商用の競合他社に見られる非エンジニア向けのコラボレーション機能の一部は欠けています。

**強み**:
- ネイティブなOpenTelemetry (OTLP) サポートにより、ベンダーに依存しないTracingと容易な統合を実現
- 検索チャンクや実行グラフを含む、エージェント型RAG向けの高度な可視化
- 柔軟なセルフホストデプロイオプションを備えた強力なオープンソース基盤
- カスタムScoringをサポートする包括的なコードファーストのEvaluationライブラリ

**弱み**:
- アノテーションキューやノーコードウィザードのような、非エンジニア向けのコラボレーション機能の欠如
- コア製品におけるエンタープライズガバナンス機能（RBAC、SSO、監査ログ）の欠落
- 組み込みのコスト属性や財務Analyticsがない
- 専用のGo SDKの不在

**最新のアップデート**:
- 簡潔さ分類エバリュエーター: 回答の簡潔さを測定するための新しいエバリュエーターを追加。(2026-02-20)
- AWS Bedrockクロスリージョンサポート: AWS Bedrockクロスリージョン推論モデルプレフィックスの優先設定サポートを追加。(2026-02-19)
- Evalプロンプトオートコンプリート: LLM Evaluationプロンプトエディタにオートコンプリート機能を追加。(2026-02-13)
- ツールレスポンスエバリュエーターテンプレート: ツールレスポンスの処理に特化した新しいエバリュエーターテンプレートを導入。(2026-02-13)
