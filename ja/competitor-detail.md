# LLM Observability — 製品詳細
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview

### LangSmith

**概要**: プロトタイピングから本番環境まで、LLMアプリケーション開発のライフサイクル全体を対象に設計された、フレームワークに依存しない包括的なプラットフォームです。複雑なエージェントワークフローのディープトレーシング、人間によるアノテーションキューを備えた高度な評価、およびエンタープライズグレードのモニタリングに優れています。

**強み**:
- LangChainおよびLangGraphとの深い統合（フレームワークに依存しない柔軟性も維持）
- ペアワイズ比較や人間によるアノテーションキューを備えた包括的な評価スイート
- セルフホストオプションとSOC 2コンプライアンスによる強力なエンタープライズ対応
- LLM、ツール、検索コンポーネントにわたる統合されたコスト追跡
- ターミナルベースの「Fetch」やトレースリプレイなどの高度なデバッグツール

**弱み**:
- トレース数に基づく料金モデルは、大量のアプリケーションでは高額になる可能性がある
- 機能が豊富なため、単純なユースケースでは学習曲線が急になる場合がある
- メモリ状態の可視化は、ツールや検索のトレースと比較して明示的ではない

**最新のアップデート**:
- Python SDK v0.7.1: LangSmith Observability用のクライアントライブラリの更新。(2026-02-10)
- トレースプレビューのカスタマイズ: UIでのトレースプレビューの表示方法をカスタマイズする機能。(2026-02-06)
- Python SDK v0.6.9: pre-commitフックの追加とサンドボックスの非同期エンドポイントの修正。(2026-02-05)
- Python SDK v0.6.8: 非otelのGoogle ADKラッパーと、オプションのchat/completions名前パラメータの追加。(2026-02-02)
- Python SDK v0.6.7: Google Gen AIラッパーのエクスポート。(2026-01-31)
- JS SDK v0.6.6: JS SDK 0.4.10のリリースとサンドボックスのデフォルトエンドポイントの修正。(2026-01-27)
- LangSmith セルフホスト v0.13: プラットフォームのセルフホスト版の更新リリース。(2026-01-16)

### Langfuse

**概要**: Langfuseは、LLMアプリケーションの包括的なオブザーバビリティ、評価、およびプロンプト管理を提供するオープンソースのLLMエンジニアリングプラットフォームです。複雑なエージェントワークフローのトレース、評価用データセットの管理、およびエンタープライズセキュリティのための堅牢なセルフホスト機能に優れています。

**強み**:
- 完全にオープンソースでセルフホスト可能であり、完全なデータ主権を提供
- 自動化されたLLM-as-a-judgeと人間によるアノテーションキューを統合した包括的な評価スイート
- プロンプト管理とトレース、実験の深い統合
- 複雑なエージェントワークフロー（グラフ、ツールフィルター、推論の可視化）の強力なサポート

**弱み**:
- リプレイ機能は、トレースからのワンクリック自動回帰テストではなく、Playgroundを介した手動操作である
- クラウドリージョンのサポートは、セルフホストの柔軟性と比較して詳細が明示されていない
- ツールの成功率は、構築済みのダッシュボードウィジェットではなく、派生メトリクスとして扱われる

**最新のアップデート**:
- 単一オブザベーション評価: 単一のオブザベーションに対して直接評価を実行する機能。(2026-02-01)
- 組織監査ログビューア: セキュリティとアクセスイベントを追跡するための組織レベルの監査ログ用新ビューア。(2026-01-28)
- 推論/思考トレースのレンダリング: トレース詳細におけるモデル出力の「思考」または推論部分の可視化サポート。(2026-01-28)
- 修正済み出力: データセット作成のために、トレースビューでLLM出力の改善版を直接キャプチャ。(2026-01-14)
- イベントベースのトレーステーブル: 新しいイベントベースのオブザベーションおよびトレーステーブルビュー。(2026-01-29)

### Braintrust

**概要**: Braintrustは、厳格な「評価ファースト（evals-first）」のワークフローを強調し、CursorやVS Codeなどの開発者ツールと深く統合された、包括的なAIオブザーバビリティおよび評価プラットフォームです。本番環境のモニタリングと強力な実験フレームワークを組み合わせ、チームが実際のデータと自動スコアラーを使用してプロンプトやモデルを反復改善できるようにします。幅広いSDKをサポートし、セルフホストやきめ細かなアクセス制御などのエンタープライズ向け機能を提供します。

**強み**:
- 評価（Playground/データセット）と本番モニタリングの統合ワークフロー
- Python、TS、Go、Java、Ruby、C#をカバーする広範なSDKエコシステム
- カスタム分析のためのBTQLおよびSQLによる強力なクエリ機能
- Cursor、VS Code、Temporalなどの開発者ツールとの深い統合
- セルフホストや詳細なRBACを含む強力なエンタープライズ機能

**弱み**:
- ファインチューニングやRLHFトレーニングジョブのための組み込みオーケストレーションの欠如
- RAGの可視化はスパンベースであり、専用の埋め込み空間インスペクターは提供されていない
- 直接的なPIIテキストマスキング機能は、一般的なセキュリティ制御と比較して詳細が明示されていない

**最新のアップデート**:
- トレースレベルのスコアラー: カスタムコードスコアラーが実行トレース全体にアクセスし、多段階のワークフローやエージェントの動作を評価できるようになりました。(2026-02-01)
- LangSmith 統合: トレースおよび評価の呼び出しを LangSmith と Braintrust の両方に送信、または Braintrust のみにルーティングするラッパー。(2026-02-01)
- Cursor 統合: Cursorからログのクエリや実験の実行を行うための Braintrust MCPサーバーを自動構成する拡張機能。(2026-02-01)
- 画像レンダリングのセキュリティ制御: 画像URLからの機密データ漏洩を防ぐための構成可能なモード（自動ロード、クリックロード、ブロック）。(2026-02-01)
- 集計付き単一スパンフィルター: 単一スパンフィルターとGROUP BYを組み合わせ、スパンレベルの条件に基づいてトレースを集計。(2026-02-01)
- 自動インストルメンテーション (Python, Ruby, Go): Python、Ruby、Goアプリケーションのコードレス・トレース・サポート。(2026-01-20)
- Temporal 統合: 親子関係マッピングを備えた Temporal ワークフローおよびアクティビティの自動トレース。(2026-01-20)
- レビュー用カンバンレイアウト: フラグ付きスパンや人間によるレビューワークフローを管理するためのドラッグ＆ドロップインターフェース。(2026-01-20)
- 合理化されたオンラインスコアリング設定: 自動入力機能により、ログから直接オンラインスコアリングルールを作成。(2026-01-20)
- トレースページのLoop: AIアシスタント「Loop」が個別のトレースページで要約やエラー分析に利用可能になりました。(2026-01-20)

### MLflow

**概要**: MLflowは、実験から本番まで、GenAIのライフサイクル全体を管理する包括的なオープンソースプラットフォームです。OpenTelemetry互換のトレースによる堅牢なオブザーバビリティ、Judge Builderや継続的モニタリングを含む高度な評価機能、および強力なエージェント管理ツールを備えています。

**強み**:
- トラッキング、トレース、評価、デプロイをカバーする包括的なエンドツーエンドプラットフォーム
- ベンダーニュートラルかつOpenTelemetry互換で、ロックインを防止
- Judge BuilderやMemAlignオプティマイザーを含む高度な評価機能
- 30以上のフレームワークやライブラリをサポートする強力な統合エコシステム
- 新しいマルチワークスペース組織サポートによる堅牢なエンタープライズ機能

**弱み**:
- SaaS専用の代替案と比較して、セルフホストにはインフラ（DB、アーティファクトストア）の管理が必要
- MemAlignのような高度な機能には、特定のモデルプロバイダー構成が必要な場合がある
- 最近のバージョンで導入されたUIテレメトリ収集（オプトアウト方式）は、プライバシーを重視するユーザーにとって懸念事項となる可能性がある
- リプレイ機能は、即時のトレース再実行よりも構成の再現性に重点を置いている

**最新のアップデート**:
- 組織サポート: 実験やリソースを整理するためのマルチワークスペース環境のサポート。(2026-02-12)
- MLflow Assistant: アプリやエージェントのデバッグを支援する、Claude Codeを搭載した製品内チャットボット。(2026-01-29)
- エージェントパフォーマンスダッシュボード: エージェントのレイテンシ、リクエスト数、品質スコアを監視するための構築済みチャート。(2026-01-29)
- MemAlign Judge Optimizer: 過去のフィードバックから評価ガイドラインを学習し、ジャッジの精度を向上させるアルゴリズム。(2026-01-29)
- Judge Builder UI: カスタムLLMジャッジプロンプトを作成、テスト、エクスポートするためのビジュアルインターフェース。(2026-01-29)
- 継続的オンラインモニタリング: 本番環境の着信トレースに対してLLMジャッジを自動的に実行。(2026-01-29)
- 分散トレース: コンテキスト伝播により、複数のサービスにわたるリクエストを追跡。(2026-01-29)

### Arize Phoenix

**概要**: Arize Phoenixは、エンジニアリングチームがLLMアプリケーションをデバッグ、評価、反復するために設計された、オープンソースのAIオブザーバビリティおよび評価プラットフォームです。OpenTelemetryベースのトレースと、堅牢な評価スイート、プロンプト管理、実験ツールを組み合わせ、PythonとTypeScriptの両方のエコシステムをサポートしています。

**強み**:
- 包括的な開発者体験（CLI、SDK、VS Codeフレンドリー）
- 堅牢な評価フレームワーク（LLM-as-Judge、ツール評価、データセット管理）
- 自動インストルメンテーションを備えた深い OpenTelemetry 統合
- 強力なセルフホストおよびエンタープライズ機能（RBAC、OAuth2）
- プロンプトエンジニアリングとサイドバイサイド比較のための統合Playground

**弱み**:
- ドキュメント内に明示的なPIIマスキングやデータ秘匿化機能の記載がない
- 複雑なエージェントDAGのための専用の「ワークフローグラフ」可視化がない（トレースタイムラインに依存）
- メモリトレースは会話履歴に限定されており、状態検査までは行えない
- 監査ログ機能の詳細が明示されていない

**最新のアップデート**:
- OpenAI Responses API タイプのサポート: PlaygroundおよびカスタムプロバイダーでのOpenAI APIタイプ（Chat Completions vs Responses）の選択をサポート。(2026-02-12)
- データセットエバリュエーター: 実験中にサーバー側で自動実行するために、エバリュエーターをデータセットに直接アタッチ。(2026-02-12)
- Playground用カスタムプロバイダー: Playgroundやプロンプトで再利用可能なカスタムAIプロバイダー（OpenAI、Azure、Anthropicなど）の集中構成。(2026-02-11)
- Claude Opus 4.6 サポート: 拡張思考パラメータとコスト追跡を備えた Anthropic の Claude Opus 4.6 モデルをサポート。(2026-02-09)
- ツール選択 & 呼び出しエバリュエーター: エージェントのツール選択の正確性とパラメータ呼び出しの正しさを判定する専用エバリュエーター。(2026-01-31)
- 構成可能なメール抽出: JMESPathを使用したOAuth2プロバイダーからのカスタムメール抽出をサポート。(2026-01-28)
- CLI コマンド: ターミナルから直接プロンプト、データセット、実験を管理するための新しいCLIコマンド。(2026-01-22)
- スパンリンク付きトレース・ツー・データセット: ソーススパンへの双方向リンクを保持したまま、トレースをデータセットに変換。(2026-01-21)
- アノテーションのエクスポート: オフライン分析のためにトレースと共にアノテーションをエクスポートするCLIサポート。(2026-01-19)
- CLI ターミナルアクセス: AIコーディングアシスタントがCLI経由で Phoenix データにクエリを実行可能に。(2026-01-17)

### W&B Weave

**概要**: W&B Weaveは、Weights & BiasesのML実験追跡における深いルーツを活かした、LLMアプリケーションの構築、評価、モニタリングのための包括的なツールキットです。ネストされた操作のトレース、プロンプトとデータセットの管理、LLMジャッジやダイナミックリーダーボードを使用した厳格な評価のための堅牢な機能を提供します。

**強み**:
- W&BのMLエコシステム（トレーニング、モデル、RL）との強力な統合
- ダイナミックリーダーボードやオーディオモニターを含む高度な評価機能
- プロンプト、モデル、データセットの堅牢なバージョニング
- PlaygroundでのカスタムLoRAやファインチューニング済みモデルのサポート
- オンプレミス/顧客管理型を含む柔軟なデプロイオプション

**弱み**:
- ドキュメント内に明示的なPIIマスキングや高度なデータプライバシー機能の記載がない
- トレースリプレイ機能に関する具体的な言及がない
- ストリーミングトレースのサポート詳細が明示されていない
- エージェントのメモリトレースが強調されていない

**最新のアップデート**:
- オーディオモニター: LLMジャッジを使用して、テキストと共にオーディオ出力（MP3/WAV）を監視・判定するモニター。(2026-02-01)
- ダイナミックリーダーボード: 永続的なカスタマイズとCSVエクスポート機能を備えた、評価から自動生成されるリーダーボード。(2026-01-29)
- PlaygroundでのカスタムLoRA: Weave PlaygroundでカスタムLoRAの重みを直接テストおよび比較する機能のサポート。(2026-01-16)
