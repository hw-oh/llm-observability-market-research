# LLM Observability — 製品詳細
**日付**: 2026-02-12 | **モデル**: google/gemini-3-pro-preview

### W&B Weave

**概要**: W&B Weaveは、開発者中心のObservabilityおよびEvalプラットフォームであり、Weights & Biasesのエコシステムと深く統合して、プロンプト、モデル、データセットの完全なリネージを追跡します。反復的な改善ループに優れており、TracingベースのEval、Dynamic Leaderboards、マルチモーダルモニタリングのための堅牢なツールを提供し、柔軟なエンタープライズデプロイメントオプションによってサポートされています。

**強み**:
- モデルとデータセットの完全なリネージ追跡のためのW&Bエコシステムとの深い統合
- LLM-as-a-judgeや新しいDynamic Leaderboardsを含む強力なEvalワークフロー
- セキュアなオンプレミスおよびVPC環境を含む柔軟なデプロイメントオプション
- 最近追加されたAudio Monitorsによるマルチモーダルサポート
- 主要なフレームワーク向けの自動パッチ適用を備えた包括的なSDKサポート

**弱み**:
- 複雑なエージェントフローのための視覚的なDAGやワークフローグラフの可視化の欠如
- 会話のメモリストレージの状態をTracingするための明示的なサポートがない
- 手動のEval実行以外の自動回帰検出に関する詳細が限定的
- Weave固有のInfrastructure-as-codeおよびCLIツールが十分に文書化されていない

**最新のアップデート**:
- Audio Monitors: LLM judgeを使用して、テキストと並行してオーディオ出力（MP3/WAV）を監視および判定するモニターの作成をサポート。(2026-02-01)
- Dynamic Leaderboards: フィルタリング、カスタマイズ、CSVエクスポート機能を備えた、Evalから自動生成されるリーダーボード。(2026-01-29)
- PlaygroundでのカスタムLoRA: Serverless LoRA Inferenceにより、Weave PlaygroundでカスタムFine-tuningされたLoRAの重みを直接使用可能。(2026-01-16)

### LangSmith

**概要**: LangSmithは、LangChainおよびLangGraphエコシステムと深く統合された包括的なObservabilityおよびEvalプラットフォームであり、複雑なエージェントワークフローのTracingに優れています。データセット作成、LLM-as-a-judge Eval、プロンプトバージョニングのための堅牢なツールを提供することで、プロトタイプと本番環境のギャップを埋めます。

**強み**:
- 複雑な多段階エージェントワークフローをTracingするためのLangGraphとの深い統合。
- LLM-as-a-judgeとヒューマンフィードバックループをサポートする包括的なEvalスイート。
- 反復的な改善を容易にする強力なプロンプトおよびデータセットバージョニング機能。
- エンタープライズコンプライアンスのための堅牢なセルフホスティングオプションとデータレジデンシサポート。
- 本番環境のトレースをEvalデータセットへシームレスに変換。

**弱み**:
- 提供されたドキュメントに明示的なCLIツールやInfrastructure-as-codeのサポートがない。
- エンタープライズガバナンスのためのRBACおよび監査ログ機能に関するドキュメントの不足。
- 成功したトレースフローと比較して、UI内での失敗状態の可視化が限定的。
- プロンプトやチェーンのバージョニングと比較して、モデルのバージョニングへの重点が低い。
- ツールの成功率やカスタムメトリクスのDashboardは、コアとなるコスト/レイテンシメトリクスほど開発されていない。

**最新のアップデート**:
- Customize Trace Previews: LangSmith UIでのトレースのプレビュー方法をユーザーがカスタマイズ可能に。(2026-02-06)
- Non-Otel Google Vertex AI (ADK) Wrapper: Python SDKにGoogle Vertex AI (ADK)用の非OpenTelemetryラッパーを追加。(2026-02-02)
- LangSmith Self-Hosted v0.13: セルフホストプラットフォームのバージョン0.13をリリース。(2026-01-16)

### Langfuse

**概要**: Langfuseは、深いObservabilityと堅牢なEvalおよびプロンプト管理ワークフローを組み合わせたオープンソースのLLMエンジニアリングプラットフォームです。複雑なエージェントの動作やRAGパイプラインのTracingに優れており、エンタープライズのセキュリティとコンプライアンスのための強力なセルフホスティング機能を提供します。

**強み**:
- エンタープライズコンプライアンスに適した強力なセルフホスティングとデータプライバシー機能。
- エージェントフレームワーク（LangGraph, LlamaIndex）との深い統合と複雑なトレースの可視化。
- LLM-as-a-judge、ヒューマンアノテーション、データセット管理を組み合わせた統合Evalスイート。
- 堅牢なプロンプト管理およびバージョニングシステム。

**弱み**:
- プロンプトバージョニングとは別の、明示的なモデルバージョンレジストリの欠如。
- ネイティブなインフラ管理ツール（CLI/IaC）が限定的。
- 非常に大規模なデータセットや複雑なレイテンシクエリにおける潜在的なパフォーマンスのボトルネック。

**最新のアップデート**:
- Single Observation Evals: 単一のObservationに対してEvalを実行するサポートを追加。(2026-02-09)
- Reasoning/Thinking Rendering: トレース詳細において、思考/推論部分（例：推論モデル用）のレンダリングサポートを追加。(2026-02-06)
- Org Audit Log Viewer: 組織レベルの監査ログを表示するための新しいUI。(2026-02-06)
- Inline Trace Comments: トレース内のIOデータの一部に対してインラインでコメントを追加可能に。(2026-01-27)
- Corrections in Trace Preview: トレースおよびObservationのプレビューで修正を直接表示・管理する機能を追加。(2026-01-13)

### Braintrust

**概要**: Braintrustは、本番環境のObservabilityとEval主導の開発を密接に統合するエンタープライズグレードのLLMopsプラットフォームです。セキュリティのためのセルフホストデータプレーンアーキテクチャと、エージェントおよびRAGワークフローの複雑でネストされたTracingへの深いサポートで差別化を図っています。

**強み**:
- 本番トレースを回帰テスト用のデータセットに直接接続する強力な「evals-first」ワークフロー。
- セルフホストデータプレーン（AWS/Terraform）を備えたエンタープライズセキュリティモデルにより、データがユーザーのVPC内に留まることを保証。
- 複雑なエージェントやRAGに適した、ネストされた階層的なTracingへの深いサポート。
- LangChain、LlamaIndex、Vercel AI SDKを含む幅広いフレームワーク統合。

**弱み**:
- LangSmithのような競合他社と比較して、視覚的なワークフローグラフビルダーやトポロジビューが欠如している。
- 長時間実行される会話型エージェントのための明示的なメモリや状態追跡機能がない。
- モデル改善のためのFine-tuningやRLHFパイプラインとの直接的な統合が限定的。

**最新のアップデート**:
- Claude Agentのサブエージェントネスト: 階層的なObservabilityを向上させるため、Claude Agent SDKラッパー内でのサブエージェントネストのサポートを追加。(2026-02-05)
- Review Span Type: トレース内での手動または自動のレビューステップをサポートするため、新しい「review」スパンタイプをSDKに導入。(2026-02-05)
- Classifications Field: SDKに分類フィールドを追加し、データのラベル付けとカテゴリ化機能を強化。(2026-01-31)
- Trace Scoring Candidate: 自動Evalロジックを洗練させるためのPythonトレースScoring機能のアップデート。(2026-01-21)
- Workflows Renaming: 幅広いオーケストレーション機能をより適切に反映させるため、SDK内で「agents」を「workflows」に名称変更。(2026-01-15)

### MLflow

**概要**: MLflowは、成熟したオープンソースのMLOpsプラットフォームであり、堅牢なTracing、Eval、プロンプトエンジニアリング機能を備えてGenAI分野への拡張に成功しています。LLMアーティファクトの実験追跡とバージョン管理に優れていますが、一部の高度な本番モニタリングやコスト分析機能については、現在は統合やカスタム実装に依存しています。

**強み**:
- プロンプト、モデル、データセットのための包括的なバージョニングシステム。
- LangChainやLlamaIndexなどの人気のあるGenAIフレームワークとの深い統合。
- LLM-as-a-Judgeやヒューマンフィードバックループを含む堅牢なEvalスイート。
- セルフホストやエアギャップ環境を含む柔軟なデプロイメントオプション。
- 強力なコミュニティサポートと、カスタム統合のための広範なAPIサーフェス。

**弱み**:
- ネイティブで組み込みのコスト管理およびDashboard機能の欠如。
- 複雑なエージェントワークフローグラフ（DAG）の可視化機能が限定的。
- レイテンシモニタリングのための詳細なパーセンタイルベースのアラートがない。
- RBACなどのエンタープライズ機能は、マネージドサービスや外部統合を必要とすることが多い。
- 現在のドキュメントに詳細が記載されているStreamingレスポンスのTracingに対する明示的なサポートがない。

**最新のアップデート**:
- MLflow Tracking Serverでの組織サポート: マルチワークスペース環境のサポートにより、異なるワークスペース間での実験やリソースの整理が可能に。(2026-02-12)
- MLflow Assistant: MLflow UI内で直接問題を特定、診断、修正するのを支援する、Claude Codeをバックエンドとした製品内チャットボット。(2026-01-29)

### Arize Phoenix

**概要**: Arize Phoenixは、OpenInference標準に基づいて構築されたオープンソースのObservabilityおよびEvalプラットフォームであり、LLMアプリケーション、特にRAGやエージェントワークフローへの深い可視性を提供するように設計されています。堅牢な階層的Tracingと強力な「LLM-as-a-judge」Evalフレームワークを組み合わせ、SaaSから完全なセルフホストVPC環境まで柔軟なデプロイメントオプションを提供します。

**強み**:
- エージェントフレームワーク（LangGraph, LlamaIndex）およびOpenInference標準との深い統合。
- 事前構築済みおよびカスタムエバリュエーターを備えた強力な「LLM-as-a-Judge」Evalフレームワーク。
- 機能制限のない完全なセルフホスティング/VPCを含む柔軟なデプロイメントオプション。
- 階層的なスパンやトークン使用量を含む包括的なトレース可視化。

**弱み**:
- ネイティブなモデルバージョニングおよびモデルのサイドバイサイド比較UIの欠如。
- PIIマスキングが、自動化された標準ソリューションではなく手動の正規表現設定に依存している。
- 会話メモリの読み書きをTracingするための明示的なサポートがない。
- ドキュメントにプログラムによるAPIアクセスやCLIツールに関する具体的な詳細が不足している。

**最新のアップデート**:
- Claude Opus 4.6 サポート: PlaygroundにClaude Opus 4.6モデルのサポートを追加。(2026-02-09)
- Tool Selection Evaluator: ライブラリに不足していた tool_selection エバリュエーターを追加。(2026-02-06)
- Faithfulness Evaluator: FaithfulnessEvaluatorを追加し、HallucinationEvaluatorを非推奨化。(2026-02-02)
- Tool Invocation Accuracy Metric: ツールの呼び出し精度を追跡するための新しいメトリクスを追加。(2026-02-02)
- 設定可能なOAuth2メール抽出: OAuth2での設定可能なメール抽出のために EMAIL_ATTRIBUTE_PATH を追加。(2026-01-28)
- 組み込みメトリクス用Cursorルール: 新しい組み込みメトリクス（LLM分類エバリュエーター）を作成するためのCursorルールを追加。(2026-01-21)
