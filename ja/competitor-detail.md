# W&B Weave — 製品詳細
**日付**: 2026-02-11 | **モデル**: google/gemini-3-pro-preview

[← ホーム](./) · [詳細比較](./comparison)

### Weave

**概要**: Weaveは自らを「AIエンジニアのワークベンチ」と位置づけ、モデルのトレーニングライフサイクル（W&B）と深く統合された、コードファーストかつホワイトボックスなオブザーバビリティ（観測性）アプローチを優先しています。プロキシベースの競合（Helicone）やCMS中心のツール（Humanloop）とは異なり、Weaveは任意のコードのトレースや、推論データとモデルアーティファクトの紐付けに長けています。そのため、単にAPIを消費するだけでなく、複雑でカスタムなLLMアプリケーションを構築するチームにとって優れた選択肢となります。

**主な強み**:
- 深いトレーニングリネージ：本番環境のトレースをトレーニングランやモデルアーティファクト（W&Bエコシステム）にリンクさせる独自の能力があり、これは単独のオブザーバビリティツールでは不可能です。
- ホワイトボックスの可視性：プロキシベースの競合（Helicone）がAPIの入出力しか見られないのに対し、任意のPythonコードや内部ロジックをトレースする優れた能力を備えています。
- 柔軟な「Board」UI：高度にカスタマイズ可能なノートブック形式のダッシュボードにより、LangSmithやLangfuseの固定された定義済みダッシュボードと比較して、より深い探索的分析が可能です。
- モデルレジストリ統合：真のモデルレジストリとのネイティブな統合により、メタデータの文字列だけでなく、モデルバイナリの厳密なバージョン管理が可能です。

**改善が期待される点**:
- プロンプト管理CMS：LangSmith (Hub)、Langfuse、および従来のHumanloopは、コードを変更せずにプロンプトのバージョン管理や編集を行うための、非技術者向けの優れたUIを提供しています。
- Financial Opsとコストの粒度：HeliconeやLangfuseは、カスタム価格設定ティアや予算アラートなど、本番運用に不可欠な、より高度なコスト追跡機能を提供しています。
- 人手によるアノテーションワークフロー：Weaveには、LangSmithやLangfuseでは標準機能となっている、人間のラベラーを管理するための専用の「アノテーションキュー」ワークフローが不足しています。
- エージェントグラフの可視化：競合他社は、循環的なエージェントワークフロー（例：LangGraph統合）専用の可視化機能をいち早く市場に投入していますが、Weaveのトレースビューは主に線形です。

### LangSmith

**概要**: LangSmithは、LLMアプリケーションのための包括的なDevOpsプラットフォームであり、エンドツーエンドのオブザーバビリティ、評価、プロンプトエンジニアリングツールを提供します。フレームワークに依存しないと謳っていますが、LangChainおよびLangGraphエコシステムに深く最適化されており、複雑なエージェントワークフローやリトリーバルパイプラインに対してクラス最高の可視化を提供します。最近ではデプロイメント（Agent Servers）やエンタープライズグレードのモニタリングにも拡張し、完全なライフサイクル管理ソリューションとして位置づけられています。

**Weaveに対する強み**:
- 複雑で循環的なエージェントの挙動を可視化するためのLangGraphとのネイティブな統合。
- プロンプトの管理とバージョン管理のためのCMSとして機能する専用の「Prompt Hub」。
- 人間による評価（Human-in-the-loop）のための優れたワークフローを提供するペアワイズ・アノテーションキュー。
- トレースから即座に「編集して実行」できるデバッグを可能にするプレイグラウンド機能。

**Weaveに対する弱み**:
- モデルのトレーニング/ファインチューニングパイプラインとの深い統合（W&Bの核心的な強み）が欠けている。
- 他のフレームワークのユーザーにとっては、UIが煩雑で「LangChain中心」に感じられることがある。
- カスタムメトリクスのロギングが、Weaveの汎用的なオブジェクトロギング機能ほど柔軟ではない。
- モデルのバージョン管理が、完全なモデルレジストリではなくメタデータ文字列に限定されている。

**最新のアップデート**:
- トレースプレビューのカスタマイズ：トレーステーブルビューに表示される入出力を制御するための設定オプション。（2026-02-06）
- LangSmith Fetch CLI：コマンドラインから直接トレースにアクセスし、デバッグするためのターミナルベースのツール。（2025-12-10）
- Polly AIアシスタント：エージェントのデバッグやトレースの分析を行うためのプラットフォーム内AIアシスタント（ベータ版）。（2025-12-10）
- 統合コスト追跡：LLM、ツール、リトリーバルステップにわたるフルスタックのコストモニタリング。（2025-12-09）
- ペアワイズ・アノテーションキュー：人間のレビュー担当者が2つのエージェントの出力を並べて比較するための構造化されたワークフロー。（2025-12-17）

### Arize Phoenix

**概要**: Arize Phoenixは、OpenTelemetryネイティブなオブザーバビリティおよび評価プラットフォームであり、ローカル開発と本番モニタリングのギャップを埋めます。エージェントワークフローのトレースに優れ、オープンソースのローカルデバッグからArizeプラットフォームを通じたエンタープライズグレードのモニタリングへのシームレスなパスを提供します。

**Weaveに対する強み**:
- OpenTelemetryネイティブ：完全にOTLP標準に基づいて構築されており、ベンダーロックインなしに広範なインフラ（Grafana、Jaeger）との容易な統合を保証します。
- ローカルファーストの体験：OSS版は、完全にオフラインで動作する優れたローカルノートブック体験を提供し、プライバシーを重視する開発者にアピールします。
- 本番環境での実績：ドリフト検出や本番モニタリングにおいて、W&Bの開発中心のフォーカスよりも歴史的に強力なArizeの成熟したプラットフォームを活用しています。

**Weaveに対する弱み**:
- トレーニング統合：オブザーバビリティをモデルトレーニングランやアーティファクトのバージョン管理に結びつけるW&Bのシームレスなリネージが欠けています。
- カスタム可視化：UIは、高度にカスタマイズされたインタラクティブなダッシュボードを作成するためのWeave Boardsほど柔軟ではありません。
- エコシステムの広さ：W&Bが提供するフルスイートのMLOpsツール（Sweeps、Artifacts、Model Registry）を提供していないため、ライフサイクル全体をカバーするには他のツールとの統合が必要です。

**最新のアップデート**:
- FaithfulnessEvaluator：RAGの精度測定を向上させるため、HallucinationEvaluatorに代わる新しいエバリュエーター。（2026-02-02）
- ツール呼び出し精度メトリクス：エージェントのツール呼び出しの成功率を測定するために特別に設計された新しいメトリクス。（2026-01-27）
- Claude Opus 4.6 サポート：プレイグラウンドに最新のClaudeモデルのサポートを追加。（2026-02-09）
- メトリクスのためのCursorルール：Cursorルールを使用してカスタム分類エバリュエーターを作成する機能を強化。（2026-01-21）

### Braintrust

**概要**: Braintrustは、AI製品を構築するための「オペレーティングシステム」と自らを位置づける、エンタープライズグレードのAIオブザーバビリティおよび評価プラットフォームです。プロンプトエンジニアリング（AIRプレイグラウンド経由）、評価（LLM-as-a-judge）、および本番モニタリングを単一のワークフローに密接に統合し、キャッシュとレート制限のための堅牢なAI Proxyによって支えられています。このプラットフォームは、強力なクエリ機能（BTQL/SQL）と広範な多言語SDKサポートで際立っています。

**Weaveに対する強み**:
- 統合されたプロンプトプレイグラウンド（AIR）は、プロンプトを第一級のバージョン管理された資産として扱います。
- AI Proxyは、組み込みのキャッシュ、レート制限、およびキー管理を提供します。
- Python/TSだけでなく、広範な多言語サポート（Java, Go, C#, Ruby）を提供します。
- アドホックなデータ分析のための、UI内の強力なクエリエンジン（SQL/BTQL）。

**Weaveに対する弱み**:
- モデルトレーニングパイプラインやアーティファクトリネージとの深い統合が欠けています。
- 可視化機能は、WeaveのインタラクティブなBoardと比較すると、より実用的（ユーティリティ重視）なものです。
- 大規模なトレースを行う場合、オープンソースフレンドリーな選択肢と比較して、価格モデルが制限的になる可能性があります。

**最新のアップデート**:
- トレースレベルのスコーラー：カスタムコードスコーラーが実行トレース全体にアクセスできるようになり、マルチステップのワークフローやエージェントの挙動を評価可能に。（2026-02）
- SDKサポートの拡大（Java, Go, Ruby, C#）：Java, Go, Ruby, C#/.NET向けの自動インスツルメンテーションを備えたネイティブSDK。（2026-01）
- SQL構文のサポート：トレースや実験をクエリするための、標準SQL構文（HAVINGや暗黙的なエイリアスを含む）のサポート。（2026-01）
- Temporal統合：Temporalのワークフローとアクティビティの自動トレースにより、ワーカー間の分散トレースをキャプチャ。（2026-01）
- Claude Code & Cursor統合：AIコーディングアシスタントが自然言語を介してログや実験結果を照会できる深い統合。（2025-12）

### Langfuse

**概要**: Langfuseは、オブザーバビリティ、プロンプト管理、評価を組み合わせた、包括的なオープンソースのLLMエンジニアリングプラットフォームです。強力なセルフホスト機能、OpenTelemetryネイティブなアーキテクチャ、およびHuman-in-the-loopのアノテーションとコスト追跡のための成熟したツールスイートで差別化を図っています。

**Weaveに対する強み**:
- Weaveのコードファーストなアプローチに対し、成熟したプロンプト管理CMS（UIベースのバージョン管理、デプロイラベル）を提供。
- セキュリティを重視するチームに魅力的な、強力なセルフホストストーリー（オープンソース/Docker）。
- 人間のラベリングチームを管理するための専用の「アノテーションキュー」ワークフロー。
- 粒度の細かいコストモデリング（コンテキスト依存の価格ティア）と支出アラート。

**Weaveに対する弱み**:
- モデルのトレーニング/ファインチューニングパイプラインとの深い統合（W&Bの核心的な強み）が欠けている。
- UIはダッシュボード中心であり、Weave Boardsのような探索的な「ノートブック」感に欠ける。
- プロンプトとデータセット以外のアーティファクト管理への重点が低い。

**最新のアップデート**:
- トレースの修正済み出力：ファインチューニングデータセットを構築するために、トレースビューでLLM出力の改善版を直接キャプチャ。（2026-01-14）
- データセット項目のバージョン管理：データセット項目の追加、更新、削除ごとに自動的にバージョン管理。（2025-12-15）
- ホスト型MCPサーバー：AIエージェントがプロンプトを直接取得・更新できるようにするネイティブなModel Context Protocol (MCP) サーバー。（2025-11-20）
- LLM-as-a-Judge 実行トレース：評価のためのプロンプトやトークン使用量を確認するため、すべてのエバリュエーター実行でトレースを作成。（2025-10-16）
- Experiment Runner SDK：自動トレースと並列実行を備えた、データセット上での実験実行のためのハイレベルSDK。（2025-09-17）

### Humanloop

**概要**: かつてプロンプトエンジニアリング、評価、オブザーバビリティの主要なプラットフォームであったHumanloopは、2025年半ばにAnthropicによって買収されました。プラットフォームは現在サンセット（終了）フェーズにあり、2025年9月8日に完全にシャットダウンされる予定で、新しい機能開発は行われていません。既存のユーザーは代替ソリューションへの移行を積極的に進めています。

**Weaveに対する強み**:
- プロンプト管理CMS：非技術ユーザーがプロンプトのバージョン管理や編集を行うための優れたUI。
- 人間によるアノテーションワークフロー：Human-in-the-loopレビューのための高度に洗練されたインターフェース。
- A/Bテスト：複数の設定バージョンのデプロイと勝率の追跡をネイティブにサポート。

**Weaveに対する弱み**:
- サービス終了：プラットフォームがシャットダウンされるため、すべての顧客は移行を余儀なくされています。
- コードトレースの柔軟性：LLM呼び出し以外の任意のPythonコード/関数のトレースにおいて、Weaveほどの能力はありません。
- データ探索：Weaveほど強力なインタラクティブなデータ探索やカスタムボード作成機能はありません。

**最新のアップデート**:
- プラットフォームのサンセット：HumanloopはAnthropicに買収され、2025年9月8日にシャットダウンされます。（2025-08）

### Logfire

**概要**: LogfireはPydanticチームによって作成されたオブザーバビリティプラットフォームで、PythonおよびAIアプリケーション向けの、特定の意見に固執しないオープンソースベース（OpenTelemetry）のソリューションとして設計されています。深いPydantic/FastAPI統合、トレース用のSQLベースのクエリエンジン、およびLLMインタラクションを標準的なアプリケーションログとして扱う開発者中心のワークフローで差別化を図っています。

**Weaveに対する強み**:
- SQLベースのクエリ：標準SQLを使用して、トレースデータの非常に柔軟でアドホックな分析が可能です。
- Pydanticエコシステム：すでにPydanticやFastAPIを使用しているユーザーにとって、極めて摩擦の少ない導入が可能です。
- OpenTelemetryネイティブ：完全にOTelに基づいて構築されており、データ収集におけるベンダーロックインがありません。
- 価格モデル：寛大な無料ティアとシンプルな従量課金制により、個人の開発者やスタートアップにアピールします。

**Weaveに対する弱み**:
- 評価プラットフォームの欠如：データセットの管理、評価の実行、モデルバージョンの並列比較のための専用UIがありません。
- プロンプト管理なし：プロンプトのバージョン管理、管理、またはプレイグラウンドのためのツールを提供していません。
- 限定的な非技術者向けUI：インターフェースはデータ量が多く開発者向けであり、Weaveのようなコラボレーション向けの「Board」機能が欠けています。
- 実験ループの欠如：トレースとデータセット作成/ファインチューニングの間のループを閉じる機能がありません。

**最新のアップデート**:
- Pytest統合：pytestの実行をスパンとしてログに記録し、テストのオブザーバビリティを可能にする直接的な統合。（2026-01-26）
- DSPy統合：LLMプログラミング用フレームワークであるDSPyの自動インスツルメンテーション。（2026-01-16）
- MCPサポート：コンテキスト交換を標準化するためのModel Context Protocol (MCP) のサポート。（2025-03-31）
- OpenAI Agents Framework：OpenAIの新しいAgents SDKのインスツルメンテーション。（2025-03-11）
- SQLベースのメトリクス：トレースに対して任意のSQLクエリを実行し、カスタムメトリクスを生成する機能。（2025-09-05）

|---|---|---|

| エージェント / RAG オブザーバビリティ | Weaveがリード | PydanticAIやOpenAIのSDKで構築されたエージェントワークフローを強力にサポートしていますが、RAGやメモリを特殊なエンティティではなく標準的なスパンとして扱います。 |

| モニタリング & メトリクス | 同等 | SQLに慣れた開発者にとって強力なモニタリングの候補です。コスト、レイテンシ、エラーに対する粒度の細かい可視性を提供します。 |

| エンタープライズ & セキュリティ | Weaveがリード | ほとんどのスタートアップやスケールアップに適した標準的なエンタープライズ機能を提供しており、コンプライアンスや地域的なデータレジデンシへのサポートも拡大しています。 |

### Helicone

**概要**: Heliconeは、主にアプリケーションとLLMプロバイダー間のプロキシとして機能する、オープンソースのAIゲートウェイおよびオブザーバビリティプラットフォームです。APIトラフィックを傍受することで堅牢なロギング、キャッシュ、コスト分析を提供しますが、Weaveが提供するような深いコードレベルのインスツルメンテーションや、内部エージェントロジックの階層的なトレース可視化は欠けています。

**Weaveに対する強み**:
- ゼロコード統合（プロキシ/ゲートウェイモデル）：SDKのインスツルメンテーション変更は不要で、URLを差し替えるだけです。
- 組み込みゲートウェイ機能：Weaveにはないキャッシュ、レート制限、スマートルーティング（フォールバック）を提供します。
- 膨大な数のプロバイダーにわたる、標準で優れたコスト追跡と分析。
- 完全にオープンソースであり、厳格なデータプライバシー要件のために容易にセルフホスト可能です。

**Weaveに対する弱み**:
- 深いコードの可視性の欠如：内部の関数呼び出し、リトリーバルステップ、ツールの実行ロジックをトレースできません（ブラックボックス vs ホワイトボックス）。
- 評価フレームワークの弱さ：Weaveのようなプログラムによる「LLM-as-a-Judge」の深さやデータセットのバージョン管理が欠けています。
- 複雑なエージェントワークフローをデバッグするための階層的なフレームグラフがありません。
- 可視化がトレース/オペレーション中心ではなく、リクエスト中心です。

**最新のアップデート**:
- Experiments：データセットに対してプロンプトをテストし、出力とコストを比較するためのUI。（2024-10）
- プロンプト管理 (Assembly)：コードから切り離してプロンプトをバージョン管理・管理するための強化されたUI。（2024-09）
- セッショングループ化：チャット履歴の追跡のために、複数のリクエストを単一のセッションにグループ化する機能。（2024-08）
