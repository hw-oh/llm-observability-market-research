# W&B Weave — 製品詳細
**日付**: 2026-02-11 | **モデル**: google/gemini-3-pro-preview

[← ホーム](./) · [詳細比較](./comparison)

### Weave

**概要**: Weaveは、AIライフサイクル全体を接続する唯一の「System of Record（記録システム）」として位置付けられており、本番環境のLLMトレースを、W&Bで管理されているトレーニングやファインチューニングのデータに紐付けます。モニタリングやプロンプトエンジニアリングだけに焦点を当てた単機能のソリューションとは異なり、Weaveはプログラム可能な「Boards」UIを活用し、高度にカスタマイズ可能なコードファーストのエンジニアリングプラットフォームを提供します。

**主な強み**:
- ライフサイクル全体の系統（Lineage）: 本番環境のトレースをモデルのトレーニングランやアーティファクトに直接リンクできる独自の能力（W&Bエコシステムとの統合）。
- プログラム可能なダッシュボード: Weave Boardsは、LangSmithやPhoenixの硬直したUIよりも優れた、コード駆動型で高度にカスタマイズ可能な可視化体験を提供します。
- 低摩擦な統合: 非同期ロギング手法により、BraintrustやHeliconeが採用しているプロキシ/ゲートウェイアーキテクチャのようなレイテンシやセキュリティの複雑さを回避します。
- データプライバシー: データフローのクリティカルパスに介在するプロキシミドルウェアを必要としないため、セキュリティを重視するチームにとって強い魅力があります。

**改善の余地**:
- ゲートウェイ機能の欠如: HeliconeやBraintrustなどの競合他社は、ネイティブキャッシュ、レート制限、シークレット管理などのインフラ機能で先行しています。
- 人手によるアノテーションワークフロー: LangSmithやBraintrustは、大規模な人手によるアノテーションキューやペア比較のための、より成熟した専用UIを提供しています。
- 言語サポート: Weaveが主にPythonとTypeScriptに焦点を当てているのに対し、BraintrustやLangfuseはより幅広いSDKサポート（Java, Go, C#）を提供しています。
- 特化型RAG可視化: Arize Phoenixは、深いLlamaIndex統合を通じて、ベクトル検索の内部（埋め込みクラスター、チャンクランキング）の可視化をリードしています。

**最新のアップデート**:

### LangSmith

**概要**: LangSmithは、LLMアプリケーションのための包括的なDevOpsプラットフォームであり、エンドツーエンドのオブザーバビリティ、評価、デプロイ機能を提供します。LangChainおよびLangGraphエコシステムと深く統合されていますが、セルフホスティングや高度な人手によるアノテーションワークフローなどの強力なエンタープライズ機能を備えた、フレームワークに依存しないソリューションへと進化しています。

**Weaveと比較した強み**:
- LangChainおよびLangGraphアプリケーション向けのネイティブで設定不要なトレース。
- 統合されたPrompt Hubにより、同じUI内でシームレスなプロンプトのバージョニングとプレイグラウンドテストが可能。
- 高度な「Annotation Queues」により、ヒューマンインザループの評価やRLHFデータ収集のための優れたワークフローを提供。
- クラウド版と同等の機能を備えた成熟したセルフホスティングオプション。

**Weaveと比較した弱み**:
- Weaveのクリーンなインターフェースと比較して、機能の幅広さ（Hub, Deploy, Monitor, Trace）ゆえにUIが複雑で圧倒される場合があります。
- トレース量に基づいた価格モデルは、Weaveのモデルと比較して、大規模運用時に予測が難しく高価になる可能性があります。
- WeaveがコアとなるW&Bプラットフォームから継承している、モデルトレーニングやアーティファクト管理との深い統合が欠けています。

**最新のアップデート**:
- LangSmith Fetch CLI: ターミナルから直接トレースを取得し、エージェントをデバッグするためのCLIツール。(2025-12)
- Pairwise Annotation Queues: 2つのエージェント出力を並べて比較し、勝者を選択するための構造化されたワークフロー。(2025-12)
- 統合コストトラッキング: LLM、ツール、検索ステップにわたるフルスタックのコストモニタリング。(2025-12)
- Polly (Beta): エージェントのデバッグと分析を支援するLangSmith内のAIアシスタント。(2025-12)
- Self-Hosted v0.13: Insightsを含むクラウド版と同等の機能をもたらす、更新されたセルフホストリリース。(2026-01)

### Arize Phoenix

**概要**: Arize Phoenixは、OpenInference標準とOpenTelemetry統合を重視する、主要なオープンソースLLMオブザーバビリティおよび評価プラットフォームです。ローカルからクラウドへのシームレスなワークフローを提供し、RAGのトラブルシューティング、トレースの可視化、LLM-as-a-judge評価に優れており、広範なArize AI MLオブザーバビリティスイートのLLM特化型カウンターパートとして機能します。

**Weaveと比較した強み**:
- ネイティブのOpenTelemetry/OpenInferenceサポートにより、幅広い相互運用性と標準化を保証。
- ローカルファーストのアーキテクチャにより、インターネットやクラウド同期なしで即座にUIを起動可能（`px.launch_app()`）。
- LlamaIndexとの深い統合による、特化型のRAG検索可視化。
- 論文に裏打ちされた構築済みのエバリュエーター（例：Faithfulness, QA）の強力なライブラリ。

**Weaveと比較した弱み**:
- モデルのトレーニング/ファインチューニングパイプラインとのネイティブな統合が欠けている（W&Bエコシステムとは対照的）。
- カスタムダッシュボードを作成するためのUIのカスタマイズ性が、Weaveの「Boards」ほど高くありません。
- モデルのバージョニングが、完全なモデルレジストリではなくメタデータ文字列に限定されています。
- 履歴データの分析には、多くの場合、別途有料のArizeプラットフォームへのアップグレードが必要です。

**最新のアップデート**:
- Claude Opus 4.6 サポート: Prompt PlaygroundでClaude Opus 4.6モデルのサポートを追加。(2026-02-09)
- Tool Selection Evaluator: エージェントワークフローにおけるツール選択の精度を測定する新しいエバリュエーター。(2026-02-06)
- Faithfulness Evaluator: 精度向上のため、従来のHallucinationEvaluatorを非推奨とし、FaithfulnessEvaluatorを導入。(2026-02-02)
- OAuth2 Email Extraction: OAuth2属性を介してユーザーのメールアドレスを抽出するためのセキュリティ設定を強化。(2026-01-28)

### Braintrust

**概要**: Braintrustは、プロキシベースのアーキテクチャを利用してAPIアクセス、キャッシュ、ロギングを統合する、エンタープライズグレードのAIオブザーバビリティおよび評価プラットフォームです。「評価ファースト」の手法を特徴とし、プログラム可能なスコアラーと広範なSDKサポートを通じて、オフライン実験（CI/CD）とオンライン本番モニタリングの両方に堅牢なツールを提供します。

**Weaveと比較した強み**:
- AIプロキシアーキテクチャ: キャッシュ、シークレット、レート制限のための中央集権型ゲートウェイ。
- 幅広いSDKエコシステム: Java, Go, Ruby, C#をネイティブサポート（Weaveは主にPython/TS）。
- BTQL (Braintrust Query Language): 複雑なトレース集計やカスタムメトリクスのためのSQLベースの柔軟性。
- オンラインスコアリング: プロキシを介して本番トラフィックに対して評価を実行するための成熟したフレームワーク。

**Weaveと比較した弱み**:
- トレーニングエコシステム: W&Bが提供するような、モデルトレーニングランやアーティファクトの系統との深い統合が欠けています。
- 可視化のカスタマイズ: 「Loop」はカスタムビューを提供しますが、Weave Boardsのようなプログラム可能なUIの深みには欠けます。
- 価格モデル: プロキシベースのスループット課金は、インジェクションのみのモデルと比較して、大量のロギングが発生する場合に複雑または高価になる可能性があります。

**最新のアップデート**:
- トレースレベル・スコアラー: カスタムコードスコアラーが実行トレース全体にアクセスし、マルチステップのワークフローを評価できるようになりました。(2026-02)
- LangSmith 統合: LangSmithのトレースをBraintrust、またはその両方に並行してルーティングするためのラッパー。(2026-02)
- 自動インスツルメンテーション (Python/Ruby/Go): 主要言語向けのコード記述不要なトレースサポート。(2026-01)
- Temporal 統合: Temporalのワークフローとアクティビティの自動トレース。(2026-01)
- Claude Code 統合: 自然言語によるログクエリのための、Anthropicのエージェント型コーディングツールとの統合。(2025-12)

### Langfuse

**概要**: Langfuseは、オブザーバビリティ、プロンプト管理、評価を組み合わせた人気のオープンソースLLMエンジニアリングプラットフォームです。強力なセルフホスティングオプション、専用のプロンプトCMS、および大規模な本番モニタリングのためのClickHouseベースの分析エンジンを特徴としています。

**Weaveと比較した強み**:
- オープンソースかつセルフホスト可能（MITライセンス）であり、厳格なデータプライバシーのニーズに対応。
- バージョニング、デプロイラベル、SDK取得機能を備えた専用のプロンプト管理CMS。
- 複雑でコンテキストに依存する価格モデルをサポートする、きめ細かなコスト追跡。
- 人手によるラベル付けワークフローを管理するための組み込みの「Annotation Queues」。

**Weaveと比較した弱み**:
- モデルのトレーニング/ファインチューニングパイプラインとの深い統合が欠けている（W&Bエコシステムとは対照的）。
- Weave Boardsに匹敵するようなコード駆動型のダッシュボード体験を提供していません。
- モデルレジストリ機能がW&B Artifactsと比較して限定的です。

**最新のアップデート**:
- トレースの修正出力: ファインチューニングデータセットを構築するために、トレースビュー内でLLM出力の改善版を直接取得。(2026-01-14)
- インラインコメント: トレースの入力/出力内の特定のテキスト選択箇所にコメントを固定。(2026-01-07)
- データセット項目のバージョニング: 更新または削除時にデータセット項目を自動的にバージョニング。(2025-12-15)
- ホスト型 MCP サーバー: エージェントがプロンプトを直接取得/更新できるようにするネイティブのModel Context Protocolサーバー。(2025-11-20)
- Langfuse for Agents: ツール呼び出しのレンダリングとエージェント固有の評価を備えた強化されたトレース。(2025-11-05)

### Humanloop

**概要**: Humanloopは、プロンプトのバージョニングとテストのための堅牢なUIを通じて、プロダクトマネージャーとエンジニアの間のギャップを埋めるように設計されたプロンプトエンジニアリングおよび評価プラットフォームです。しかし、2025年8月のAnthropicによる買収に伴い、プラットフォームは現在終了フェーズにあり、2025年9月8日にシャットダウンが予定されています。

**Weaveと比較した強み**:
- 非技術ユーザー（PM/SME）がプロンプトを編集・バージョニングするための優れたUI。
- プロンプトロジックをコードから分離する成熟した「Prompt CMS」機能。
- より強力な組み込みの人間によるフィードバックおよびアノテーションインターフェース。

**Weaveと比較した弱み**:
- プラットフォームが終了予定（2025年後半にサービス終了）。
- 任意のPythonオブジェクト/コードに対するトレースの柔軟性がWeaveと比較して低い。
- 歴史的にリクエストのプロキシに依存しており、レイテンシや複雑さが増す要因となっていました。

**最新のアップデート**:
- プラットフォーム終了: HumanloopはAnthropicに買収され、2025年9月8日にプラットフォームを終了します。(2025-08-13)
- LLMs.txt サポート: ドキュメントコンテキストのためのLLMs.txt標準のサポートを追加。(2025-05-01)

### Logfire

**概要**: LogfireはPydanticチームによって構築されたオブザーバビリティプラットフォームであり、OpenTelemetryを活用してPythonアプリケーションやAIエージェントの深い構造的デバッグを提供します。Pydanticモデルとの緊密な統合により、構造化されたログやトレースのSQLベースのクエリを可能にし、本番モニタリングと開発者体験に優れています。トレースとモニタリングには堅牢ですが、現時点ではWeaveにあるような専用のデータセット管理、評価ワークフロー、プロンプトエンジニアリングのプレイグラウンドが欠けています。

**Weaveと比較した強み**:
- ネイティブのOpenTelemetry基盤により、LLM以外の既存インフラともシームレスに統合可能。
- トレースのSQLベースのクエリにより、カスタムメトリクスやデバッグにおいて極めて高い柔軟性を提供。
- Pydanticモデルとの深い統合により、Python開発者にとって構造化ロギングとバリデーションが非常に使いやすくなっています。

**Weaveと比較した弱み**:
- 体系的なモデル改善のための専用の評価/データセット管理システムが欠けています。
- プロンプトエンジニアリングやデバッグのためのインタラクティブなプレイグラウンドやリプレイ機能がありません。
- 一部の領域において、Weaveのより広範な言語に依存しないアプローチと比較して、Python以外のエコシステムへのサポートが限定的です。

**最新のアップデート**:
- マルチトークンサポート: プロジェクトの移行を容易にするための複数トークンのサポートを追加。(2026-02-04)
- Pytest 統合: Pytest実行内でのトレースとデバッグのためのネイティブ統合。(2026-01-26)
- DSPy 統合: DSPyフレームワークの公式インスツルメンテーションを追加。(2026-01-16)
- フィードバックアノテーション: トレースに対するユーザーフィードバックを記録するための実験的機能。(2025-04-11)
- OpenAI Agents インスツルメンテーション: 新しいOpenAI Agents Frameworkのトレースをサポート。(2025-03-11)

### Helicone

**概要**: Heliconeは、LLMリクエストのログ記録、キャッシュ、ルーティングを行うプロキシミドルウェアとして主に機能する、オープンソースのAIゲートウェイおよびオブザーバビリティプラットフォームです。コスト追跡、キャッシュ、レート制限などの本番インフラタスクに優れていますが、Weaveを特徴づけるエージェント向けの深いコードレベルのトレースインスツルメンテーションよりも、APIレベルのトラフィックに重点を置いています。

**Weaveと比較した強み**:
- ゲートウェイ機能: ネイティブキャッシュ、レート制限、リクエストルーティング/フォールバック。
- ゼロコード統合: APIのベースURLを変更するだけで動作（プロキシモデル）。
- コスト管理: コスト追跡と属性割り当てにおける優れた粒度。
- オープンソース: 完全にセルフホスト可能なコアプラットフォーム。

**Weaveと比較した弱み**:
- 深いトレースの欠如: エージェントの内部ロジックやLLM以外のスパン（例：検索ステップ）を可視化できません。
- 限定的な評価: Weaveにあるような堅牢でプログラム可能なLLM-as-a-judgeフレームワークが欠けています。
- トレーニング統合なし: ファインチューニング/トレーニングのライフサイクル（W&Bの核心的な強み）から切り離されています。
- 可視化: マルチステップエージェントのための複雑なワークフローグラフがありません。

**最新のアップデート**:
- Experiments: プラットフォーム内で直接データセットに対してプロンプトをテスト・評価する機能。(2024-05)
- プロンプト管理: コードから分離されたプロンプトの作成、バージョニング、管理のためのUI。(2024-01)
- Sessions: 複数のリクエストを単一のインタラクションフローにリンクするためのグルーピングメカニズム。(2023-11)
