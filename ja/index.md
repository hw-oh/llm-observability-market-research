## 最新レポート

<!-- LATEST_REPORT_START -->

[📋 最新レポート (2026-02-11)](./reports/2026-02-11.md)


- Weave は2月1日の Audio Monitors のリリースにより、マルチモーダル・オブザーバビリティにおける先行者利益を確立しました。これにより、急成長するボイスエージェント分野において、LangSmith や MLflow といったテキスト中心の競合他社をリードしています。
- LangSmith は、LangGraph Cloud を通じて純粋なオブザーバビリティからインフラのロックインへと積極的にピボットしており、トレースレイヤーだけでなくデプロイレイヤーを掌握することで Weave を排除しようとしています。
- MLflow 3.9 でリリースされた「Judge Builder」と「MemAlign」は、当社の評価ワークフローを直接コモディティ化するものであり、Weave が優先している手動検査ツールへの依存を減らす自動 QA を企業に提供しています。
- Weave における成熟した「アノテーションキュー」の欠如は、大規模なヒューマン・イン・ザ・ループ（人間によるラベル付け）チーム向けの標準化されたワークフローを持つ LangSmith や Langfuse に対する、依然として重大な販売上の障害となっています。
- Braintrust は、ネイティブの Cursor IDE 統合をリリースすることで当社の開発者エクスペリエンス戦略の先手を打ち、開発者が Weave ダッシュボードに到達する前の「インナーループ」ワークフローを捕捉しています。
- Weave Playground への Serverless LoRA 推論の統合（1月16日）は、Arize Phoenix や Braintrust のようなスタンドアロンのプレーヤーでは技術的に複製できない、独自の「トレーニングから推論へ」のフライホイールを生み出しています。
- 必要なアクション：製品チームは第2四半期に OpenTelemetry (OTel) 互換性を優先する必要があります。MLflow と Arize Phoenix は、「ネイティブ OTel」サポートをより安全でベンダーニュートラルな選択肢として位置づけることで、企業のアーキテクチャ審査で勝利しています。

> Weave はマルチモーダルおよびトレーニング統合ワークフローにおいて明確な技術的リードを保っていますが、LangSmith のインフラロックインや MLflow の自動化されたエンタープライズ QA 機能による存亡の危機に直面しています。
