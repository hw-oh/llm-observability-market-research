# Weekly LLM Observability Market Research Report
**Date**: 2026-02-12 | **Model**: google/gemini-3-pro-preview | **Data Collected**: 2026-02-12

## 1. Executive Summary

- LangSmith released Self-Hosted v0.13 and introduced EU data residency support to meet stricter regional compliance demands.
- Langfuse launched an Organization Audit Log Viewer and continues to support full self-hosting for governance-focused teams.
- MLflow added Organization Support to enable multi-workspace management and PII masking within its Tracking Server.
- Braintrust expanded its agent observability with support for sub-agent nesting and deep integration with OpenAI agents.
- Arize Phoenix introduced a library of pre-built evaluators specifically for Tool Selection and Faithfulness metrics.
- W&B Weave continues to differentiate with deep nested function call tracing and dynamic leaderboards for LLM-as-a-judge scoring.
- The market saw a broad shift toward agentic support, with Weave, LangSmith, and Braintrust all explicitly tracking retrieval steps and tool calls.

**Market Insight by AI**:

> LangSmith's release of Self-Hosted v0.13 and Langfuse's new Audit Log Viewer directly challenge Weave's competitive advantage in secure, self-managed enterprise deployments.
> Arize Phoenix's addition of specialized Tool Selection evaluators targets the same agentic workflow analysis use case where Weave currently differentiates with deep trace visualization.
> Braintrust's new support for sub-agent nesting and OpenAI integration challenges Weave's positioning as the primary tool for debugging complex, hierarchical agent architectures.

## 3. New Features (Last 30 Days)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: Support for evaluating audio outputs (MP3/WAV) using LLM judges to assess conversation quality. (2026-02-01, Monitoring & Metrics)
- **Dynamic Leaderboards**: Auto-generated leaderboards from evaluations with persistent customization and CSV export. (2026-01-29, Evaluation Integration)
- **Custom LoRAs in Playground**: Support for using custom fine-tuned LoRA weights directly in the Weave Playground for inference and comparison. (2026-01-16, Experiment / Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **Client SDK v0.7.1**: Updates to the Python and JS client libraries, including OIDC fixes and dependency bumps. (2026-02-10, DevEx / Integration)
- **Customize trace previews**: New capability to customize how trace previews are rendered in the UI. (2026-02-06, Core Observability)
- **LangSmith Self-Hosted v0.13**: New version release for the self-hosted deployment option. (2026-01-16, Security & Governance)

### [Langfuse](https://langfuse.com/changelog)
- **Single Observation Evals**: Added support for running evaluations on single observations. (2026-02-12, Evaluation Integration)
- **Events-based Trace Table**: New UI view for traces based on events/observations. (2026-02-12, Core Observability)
- **Reasoning Trace Rendering**: Support for rendering thinking/reasoning parts in trace details (e.g., for reasoning models). (2026-02-12, Core Observability)
- **Org Audit Log Viewer**: New viewer for organization-level audit logs. (2026-02-12, Security & Governance)
- **Inline Trace Comments**: Ability to add comments inline on fractions of IO data within traces. (2026-02-12, DevEx / Integration)
- **Corrections in Trace Preview**: Added ability to view and manage corrections directly in trace and observation previews. (2026-02-12, Experiment / Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **OpenAI Agents Integration**: Updated SDK to handle all span types for OpenAI agents integration. (2026-02-05, DevEx / Integration)
- **Sub-agent Nesting**: Added support for sub-agent nesting specifically for the Claude Agent SDK wrapper. (2026-02-05, Agent / RAG Observability)
- **Review Span Type**: Introduced a new 'review' span type, likely to support human review workflows directly in traces. (2026-02-05, Evaluation Integration)
- **Classifications Field**: Added a classifications field to the SDK, enhancing structured data capture. (2026-01-31, Core Observability)
- **Eval Cache Control**: Added options to turn off caching during evaluations and after span export. (2026-01-29, Evaluation Integration)
- **Python Trace Scoring**: Introduced candidate support for trace scoring within the Python SDK. (2026-01-21, Evaluation Integration)
- **Workflow Renaming**: Renamed 'agents' to 'workflows' in the SDK to better reflect general execution graphs. (2026-01-15, DevEx / Integration)

### [MLflow](https://mlflow.org/releases)
- **Organization Support**: Support for multi-workspace environments in MLflow Tracking Server, allowing organization of experiments and resources across different workspaces. (2026-02-12, Security & Governance)
- **MLflow Assistant**: In-product chatbot backed by Claude Code to help identify, diagnose, and fix issues directly within the MLflow UI. (2026-01-29, DevEx / Integration)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 Support**: Added Claude Opus 4.6 model to the playground. (2026-02-09, DevEx / Integration)
- **Tool Selection Evaluator**: Added missing tool_selection evaluator to libraries. (2026-02-06, Evaluation Integration)
- **Faithfulness Evaluator**: Added FaithfulnessEvaluator and deprecated HallucinationEvaluator. (2026-02-02, Evaluation Integration)
- **Tool Invocation Accuracy Metric**: Added a specific metric to track tool invocation accuracy. (2026-01-27, Monitoring & Metrics)
- **LLM Classification Evaluators**: Added cursor rule for creating new built-in metrics for LLM classification. (2026-01-21, Evaluation Integration)
