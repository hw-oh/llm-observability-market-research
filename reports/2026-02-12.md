# Weekly LLM Observability Market Research Report
**Date**: 2026-02-12 | **Model**: google/gemini-3-pro-preview | **Data Collected**: 2026-02-12

## 1. Executive Summary

- LangSmith released Self-Hosted v0.13 and introduced EU data residency support to address strict enterprise compliance and data sovereignty requirements.
- Langfuse launched a new Organization Audit Log Viewer for security governance and added rendering capabilities for reasoning steps within its Agent Graphs.
- MLflow introduced Organization Support to enable multi-workspace management and expanded its auto-instrumentation to capture tool calling responses and retriever queries.
- Braintrust updated its platform terminology from 'agents' to 'workflows' to explicitly support sub-agent nesting and released updates to its Python trace scoring.
- Arize Phoenix added specific Faithfulness and Tool Selection evaluators to its open-source library and implemented configurable OAuth2 email extraction.
- W&B Weave expanded its evaluation suite with Audio Monitors and Dynamic Leaderboards while continuing to support SDK-level PII redaction for secure deployments.

**Market Insight by AI**:

> LangSmith's release of Self-Hosted v0.13 and EU data residency directly challenges Weave's enterprise advantage by offering comparable infrastructure control for regulated industries.
> Langfuse's introduction of Agent Graphs for visual workflow topology highlights a specific UI gap in Weave, which currently captures agent steps but lacks a dedicated DAG visualization.
> Arize Phoenix's addition of specialized Faithfulness and Tool Selection metrics increases competitive pressure on Weave to expand its library of pre-built, agent-specific evaluators.

## 3. New Features (Last 30 Days)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: Support for creating monitors that observe and judge audio outputs (MP3/WAV) alongside text using LLM judges. (2026-02-01, Monitoring & Metrics)
- **Dynamic Leaderboards**: Auto-generated leaderboards from evaluations with filtering, customization, and CSV export capabilities. (2026-01-29, Evaluation Integration)
- **Custom LoRAs in Playground**: Serverless LoRA Inference allows using custom fine-tuned LoRA weights directly in the Weave Playground. (2026-01-16, Experiment / Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **Customize Trace Previews**: Allows users to customize how traces are previewed in the LangSmith UI. (2026-02-06, Core Observability)
- **Non-Otel Google Vertex AI (ADK) Wrapper**: Added a non-OpenTelemetry wrapper for Google Vertex AI (ADK) in the Python SDK. (2026-02-02, DevEx / Integration)
- **LangSmith Self-Hosted v0.13**: Released version 0.13 of the self-hosted platform. (2026-01-16, Enterprise & Security)

### [Langfuse](https://langfuse.com/changelog)
- **Single Observation Evals**: Added support for running evaluations on single observations. (2026-02-09, Evaluation Integration)
- **Reasoning/Thinking Rendering**: Added support to render thinking/reasoning parts in trace details (e.g., for reasoning models). (2026-02-06, Core Observability)
- **Org Audit Log Viewer**: New UI for viewing organization-level audit logs. (2026-02-06, Enterprise & Security)
- **Inline Trace Comments**: Allows adding comments inline on fractions of IO data within traces. (2026-01-27, DevEx / Integration)
- **Corrections in Trace Preview**: Added ability to view and manage corrections directly in trace and observation previews. (2026-01-13, Experiment / Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Sub-agent nesting for Claude Agent**: Added support for sub-agent nesting within the Claude Agent SDK wrapper to improve hierarchical observability. (2026-02-05, Agent / RAG Observability)
- **Review Span Type**: Introduced a new 'review' span type to the SDK, likely to support manual or automated review steps in traces. (2026-02-05, Core Observability)
- **Classifications Field**: Added a classifications field to the SDK, enhancing data labeling and categorization capabilities. (2026-01-31, Evaluation Integration)
- **Trace Scoring Candidate**: Updates to Python trace scoring capabilities to refine automated evaluation logic. (2026-01-21, Evaluation Integration)
- **Workflows Renaming**: Renamed 'agents' to 'workflows' in the SDK to better reflect broader orchestration capabilities. (2026-01-15, DevEx / Integration)

### [MLflow](https://mlflow.org/releases)
- **Organization Support in MLflow Tracking Server**: Support for multi-workspace environments, allowing organization of experiments and resources across different workspaces. (2026-02-12, Enterprise & Security)
- **MLflow Assistant**: In-product chatbot backed by Claude Code to help identify, diagnose, and fix issues directly within the MLflow UI. (2026-01-29, DevEx / Integration)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 Support**: Added support for Claude Opus 4.6 model to the playground. (2026-02-09, DevEx / Integration)
- **Tool Selection Evaluator**: Added missing tool_selection evaluator to libraries. (2026-02-06, Evaluation Integration)
- **Faithfulness Evaluator**: Added FaithfulnessEvaluator and deprecated HallucinationEvaluator. (2026-02-02, Evaluation Integration)
- **Tool Invocation Accuracy Metric**: Added a new metric to track tool invocation accuracy. (2026-02-02, Monitoring & Metrics)
- **Configurable OAuth2 Email Extraction**: Added EMAIL_ATTRIBUTE_PATH for configurable email extraction in OAuth2. (2026-01-28, Enterprise & Security)
- **Cursor Rule for Built-in Metrics**: Added cursor rule for creating new built-in metrics (LLM classification evaluators). (2026-01-21, Evaluation Integration)
