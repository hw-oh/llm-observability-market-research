# Weekly LLM Observability Market Research Report
**Date**: 2026-02-12 | **Model**: google/gemini-3-pro-preview | **Data Collected**: 2026-02-12

## 1. Executive Summary

- LangSmith released Self-Hosted v0.13, reinforcing its on-premise capabilities with granular RBAC and audit logs for enterprise deployments.
- Langfuse introduced Organization Audit Log Viewers, expanding governance features within its open-source platform to support larger engineering teams.
- MLflow updated its Tracking Server with Organization Support, enabling centralized management of multi-workspace environments for MLOps practitioners.
- Arize Phoenix deployed specialized evaluators for tool selection and faithfulness, leveraging deep OpenTelemetry integration to target agentic workflow debugging.
- Braintrust continued to differentiate its infrastructure via a hybrid self-hosting model that retains data within customer VPCs while providing SSO and SOC 2 compliance.
- W&B Weave confirmed enterprise readiness through SOC 2 compliance and exportable audit logs, while maintaining robust support for nested span tracing via decorators.

**Market Insight by AI**:

> Arize Phoenix's introduction of specialized tool selection evaluators challenges Weave's agent observability differentiation by offering granular metrics for sub-agent decision-making.
> LangSmith's release of Self-Hosted v0.13 and Langfuse's new Audit Log Viewers increase competitive pressure on Weave's enterprise tier by closing the gap on governance and deployment flexibility.
> MLflow's addition of Organization Support to its Tracking Server strengthens its position as a unified MLOps alternative to Weave for teams already entrenched in the Databricks ecosystem.

## 3. New Features (Last 30 Days)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: Online evaluation monitors that observe and judge audio outputs (MP3/WAV) alongside text using LLM judges. (2026-02-01, Evaluation & Quality)
- **Dynamic Leaderboards**: Auto-generated leaderboards from evaluations with filters, persistent customization, and CSV export. (2026-01-29, Evaluation & Quality)
- **Custom LoRAs in Playground**: Support for bringing fine-tuned LoRA weights into the Playground for inference and comparison. (2026-01-16, Experiment & Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **Client SDK Updates (v0.7.x)**: Updates to Python and JS client libraries including bug fixes and dependency bumps. (2026-02-10, Developer Experience & Integration)
- **Customize Trace Previews**: New capability to customize how traces are previewed within the LangSmith UI. (2026-02-06, Core Tracing & Logging)
- **LangSmith Self-Hosted v0.13**: New version release for the self-hosted enterprise deployment option. (2026-01-16, Infrastructure & Enterprise)

### [Langfuse](https://langfuse.com/changelog)
- **Single Observation Evals**: Added support for running evaluations on single observations. (2026-02-12, Evaluation & Quality)
- **Events-based Trace Table**: Refactored trace and observation tables to be event-based for better visibility. (2026-02-12, Core Tracing & Logging)
- **Reasoning/Thinking Trace Rendering**: Added rendering support for thinking and reasoning parts in trace details (e.g., for reasoning models). (2026-02-12, Agent & RAG Observability)
- **Org Audit Log Viewer**: New viewer for organization-level audit logs. (2026-02-12, Infrastructure & Enterprise)
- **Inline Trace Comments**: Allows adding comments inline on fractions of IO data within traces. (2026-02-12, Developer Experience & Integration)
- **Corrections in Trace Preview**: Added support for viewing and managing corrections directly in the trace and observation preview. (2026-02-12, Experiment & Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Sub-agent nesting for Claude Agent SDK**: Added support for nested sub-agent tracing within the Claude Agent SDK wrapper. (2026-02-12, Agent & RAG Observability)
- **Review Span Type**: Introduced a new 'Review' span type in SDKs to support human review workflows. (2026-02-12, Evaluation & Quality)
- **Classifications Field**: Added a 'Classifications' field to traces to support structured categorization of spans. (2026-01-31, Core Tracing & Logging)
- **Disable Cache on Eval**: New option to turn off caching during evaluation runs to ensure fresh results. (2026-01-29, Evaluation & Quality)
- **Workflow Terminology**: Renamed 'agents' to 'workflows' in the SDK and UI to better reflect broader use cases. (2026-01-15, Developer Experience & Integration)

### [MLflow](https://mlflow.org/releases)
- **Organization Support in MLflow Tracking Server**: Support for multi-workspace environments allowing organization of experiments and resources across different workspaces. (2026-02-12, Infrastructure & Enterprise)
- **MLflow Assistant**: In-product chatbot backed by Claude Code to help identify, diagnose, and fix issues directly within the UI. (2026-01-29, Developer Experience & Integration)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 in Playground**: Added support for Claude Opus 4.6 model within the playground environment. (2026-02-09, Experiment & Improvement Loop)
- **Tool Selection Evaluator**: New evaluator added to assess the accuracy of tool selection in agents. (2026-02-06, Evaluation & Quality)
- **Faithfulness Evaluator**: Introduced FaithfulnessEvaluator to check response grounding, deprecating HallucinationEvaluator. (2026-02-02, Evaluation & Quality)
- **Tool Invocation Accuracy Metric**: New metric to measure the precision of tool invocations. (2026-02-02, Evaluation & Quality)
- **Configurable OAuth2 Email Extraction**: Added EMAIL_ATTRIBUTE_PATH to allow configurable email extraction for OAuth2. (2026-01-28, Infrastructure & Enterprise)
- **Cursor Rule for Metric Creation**: Added cursor rule to facilitate the creation of new built-in LLM classification evaluators. (2026-01-21, Evaluation & Quality)
