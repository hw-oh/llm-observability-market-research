# W&B Weave — 제품 상세 정보

### Weave

**개요**: Weave는 '평가 기반 개발(Evaluation-Driven Development)'을 위한 가볍고 개발자 중심적인 툴킷을 지향하며, W&B의 유산을 활용해 관측성(Observability)을 모델 학습 및 미세 조정(Fine-tuning)과 밀접하게 결합합니다. 무거운 게이트웨이 솔루션과 달리, Weave는 프롬프트와 모델을 추적, 평가 및 개선하는 반복적인 사이클에 집중하며, 현재는 멀티모달 에이전트 워크플로우 분야로 그 리드를 확장하고 있습니다.

**주요 강점**:
- 멀티모달 관측성: 오디오 모니터(Audio Monitors) 및 리치 미디어 추적을 기본 지원하여 음성 에이전트(Voice Agent) 개발 분야에서 텍스트 중심 도구들보다 앞서 나갑니다.
- 학습 통합: W&B Artifacts를 통해 평가 데이터를 모델 미세 조정 워크플로우에 직접 연결할 수 있는 독보적인 능력을 갖추고 있습니다.
- 동적 리더보드: 자동 생성 및 사용자 정의가 가능한 비교 뷰를 통해 경쟁사의 정적 보고 대시보드보다 더 높은 유연성을 제공합니다.
- 가벼운 도입: SDK 기반 통합 방식은 Braintrust나 Helicone이 요구하는 게이트웨이/프록시 설정에 비해 데이터 과학자들에게 진입 장벽이 낮습니다.

**개선 필요 사항**:
- 네이티브 프록시/게이트웨이 부재: Braintrust 및 Helicone에서 제공하는 능동적 트래픽 제어(캐싱, 속도 제한, 키 관리) 기능이 부족합니다.
- 휴먼 리뷰 워크플로우: LangSmith 및 Langfuse의 핵심 강점인 대규모 수동 라벨링을 위한 구조화된 '어노테이션 큐(Annotation Queue)' 워크플로우가 부족합니다.
- 에이전트 상태 시각화: LangSmith의 네이티브 LangGraph 통합은 Weave의 트레이스 트리(Trace tree)에 비해 순환적인 에이전트 상태 및 메모리에 대한 우수한 시각화를 제공합니다.
- 제한적인 엔터프라이즈 SDK: Braintrust와 같은 경쟁사가 Go, Java, C#을 네이티브로 지원하는 반면, Weave는 여전히 Python/TypeScript에 크게 치중되어 있습니다.

**최근 업데이트**:
- 오디오 모니터: 텍스트와 함께 오디오 출력을 관찰하고 판단하는 모니터 생성 지원으로 음성 에이전트 평가 가능. (2026-02-01)
- 동적 리더보드: 사용자 정의 필터 및 풍부한 시각화 옵션을 갖춘 평가 결과 기반 자동 생성 리더보드. (2026-01-29)
- 플레이그라운드 내 커스텀 LoRA: W&B Artifacts의 커스텀 미세 조정 LoRA 가중치를 Weave 플레이그라운드에서 직접 사용 가능. (2026-01-16)

### LangSmith

**개요**: LangSmith는 관측성, 평가, 배포를 긴밀하게 통합한 포괄적인 LLM 엔지니어링 플랫폼으로, LangChain/LangGraph 생태계의 기본 선택지 역할을 합니다. 디버깅 도구에서 '에이전트 DevOps' 플랫폼으로 진화하여, 복잡한 다단계 워크플로우에 대한 깊은 가시성과 함께 에이전트 실행을 위한 관리형 인프라를 제공합니다.

**Weave 대비 강점**:
- LangGraph와의 깊은 네이티브 통합으로 에이전트 상태, 메모리 및 순환 워크플로우에 대한 우수한 시각화 제공.
- 어노테이션 큐를 통해 휴먼 인 더 루프(Human-in-the-loop) 평가를 위한 더욱 견고하고 프로덕션 수준의 워크플로우 제공.
- 관리형 인프라('Deployment') 기능을 통해 사용자가 에이전트를 직접 호스팅할 수 있어 더 높은 벤더 락인(Lock-in) 형성.

**Weave 대비 약점**:
- LangChain을 사용하지 않는 개발자에게는 '무겁고' 복잡하게 느껴질 수 있는 반면, Weave는 가볍고 프레임워크에 구애받지 않는 것으로 인식됨.
- 모델 학습 및 미세 조정 워크플로우(W&B Core 유산)와의 깊은 통합이 부족함.
- Weave의 깔끔하고 차이점(diff) 중심인 인터페이스에 비해 UI 밀도가 압도적일 수 있음.

**최근 업데이트**:
- 트레이스 미리보기 사용자 정의: 관련 메타데이터나 입력/출력을 한눈에 볼 수 있도록 트레이스 미리보기 창을 커스터마이징하는 기능. (2026-02-06)
- Google Gen AI 래퍼: 수동 인스트루먼테이션 없이 Google Gen AI 모델을 추적할 수 있는 새로운 SDK 래퍼. (2026-01-31)
- LangSmith 셀프 호스팅 v0.13: 안정성 개선 및 클라우드 버전의 새로운 기능이 포함된 업데이트된 셀프 호스팅 릴리스. (2026-01-16)

### Arize Phoenix

**개요**: Arize Phoenix는 OpenTelemetry(OpenInference) 기반으로 구축된 오픈 소스 우선 LLM 관측성 및 평가 플랫폼입니다. 로컬에서 클라우드로 이어지는 워크플로우에 탁월하며, 견고한 추적, 깊이 있는 평가 기능(에이전트 전용 지표 포함), 그리고 광범위한 Arize AI 엔터프라이즈 플랫폼과 원활하게 통합되는 데이터셋 관리를 제공합니다.

**Weave 대비 강점**:
- 네이티브 OpenTelemetry 기반(OpenInference)으로 이미 APM에 OTLP를 사용하는 팀에 매력적임.
- 클라우드 기능과 동일한 강력한 로컬 우선(OSS) 경험으로 개별 개발자의 진입 장벽을 낮춤.
- 에이전트 도구 사용(선택/호출)을 위한 새로운 전문 평가 도구가 즉각적인 에이전트 지표를 제공함.
- 포괄적인 CLI를 통해 터미널 기반 워크플로우(프롬프트 파이핑, 실험 관리)가 가능하며, 이는 현재 Weave CLI에 부족한 부분임.

**Weave 대비 약점**:
- Weave가 W&B로부터 물려받은 광범위한 ML 학습 생태계(Artifacts, Sweeps)와의 깊은 통합이 부족함.
- Weave의 간소화된 인터페이스에 비해 엔지니어가 아닌 사용자에게는 UI가 더 조밀하고 복잡할 수 있음.
- 모델 버전 관리가 덜 중심적이며, 전용 모델 레지스트리보다는 설정 문자열 추적에 더 의존함.
- 전체 엔터프라이즈 기능(Arize AX)을 위한 설정이 Weave의 SaaS 우선 온보딩보다 무거움.

**최근 업데이트**:
- Claude Opus 4.6 지원: 확장된 사고(thinking) 파라미터 및 정확한 비용 추적을 포함한 Anthropic 최신 모델 플레이그라운드 지원. (2026-02-09) [[docs]](https://docs.arize.com/phoenix/release-notes)
- 도구 선택 및 호출 평가기: 에이전트가 올바른 도구를 선택하고 유효한 파라미터로 호출했는지 평가하는 새로운 전문 평가기. (2026-01-31) [[docs]](https://docs.arize.com/phoenix/release-notes)
- 프롬프트/데이터셋용 CLI 명령: 프롬프트 목록 조회, 확인 및 AI 어시스턴트로의 파이핑, 터미널에서의 데이터셋/실험 관리를 위한 포괄적인 CLI 지원. (2026-01-22) [[docs]](https://docs.arize.com/phoenix/release-notes)
- 스팬 링크를 통한 트레이스-데이터셋 변환: 원본 소스 스팬과의 양방향 링크를 유지하면서 프로덕션 트레이스로부터 큐레이션된 데이터셋을 생성하는 기능. (2026-01-21) [[docs]](https://docs.arize.com/phoenix/release-notes)
- 트레이스와 함께 어노테이션 내보내기: 오프라인 분석을 위해 수동 라벨 및 평가 점수와 함께 트레이스를 내보내는 CLI 지원. (2026-01-19) [[docs]](https://docs.arize.com/phoenix/release-notes)

### Braintrust

**개요**: Braintrust는 모델 액세스 관리를 위한 통합 'AI Proxy'와 개발 워크플로우로의 '왼쪽 이동(shifting left)'에 중점을 둔 차별화된 엔터프라이즈급 AI 관측성 및 평가 플랫폼입니다. 프롬프트 엔지니어링 플레이그라운드, 엄격한 CI/CD 평가, 그리고 자체 SQL 유사 쿼리 언어(BTQL) 기반의 프로덕션 모니터링을 결합한 포괄적인 제품군을 제공합니다.

**Weave 대비 강점**:
- AI Proxy: Weave에는 없는 캐싱, 속도 제한 및 키 관리를 위한 통합 게이트웨이 제공.
- 광범위한 SDK 지원: Go, Java, Ruby, C#용 네이티브 SDK를 통해 Weave의 Python/TS 중심 전략이 놓치는 엔터프라이즈 백엔드 트래픽을 캡처할 수 있음.
- BTQL/SQL 분석: 강력한 SQL 호환 쿼리 언어를 통해 Weave의 UI 중심 접근 방식보다 더 깊고 유연한 데이터 분석 가능.
- IDE 통합: 새로운 Cursor 통합을 통해 코드 에디터 내에 관측성을 직접 내장함.

**Weave 대비 약점**:
- 학습/미세 조정 연결 부재: Weave가 W&B로부터 상속받은 모델 학습 파이프라인 및 Artifact 관리와의 원활한 통합이 부족함.
- 복잡성: 프록시 설정, SQL/BTQL 및 구성의 조합은 Weave의 가벼운 SDK보다 도입하기에 더 무거울 수 있음.
- 시각적 커스터마이징: 대시보드는 기능적이지만 Weave Boards에서 볼 수 있는 높은 수준의 시각적 커스터마이징이 부족함.
- 가격 모델: 프록시 기반 가격 책정은 대량 사용자에게 Weave의 모델에 비해 복잡하고 잠재적으로 비쌀 수 있음.

### Langfuse

**개요**: Langfuse는 프로덕션 관측성, 프롬프트 관리 및 평가를 개발자 친화적인 패키지로 결합한 선도적인 오픈 소스 LLM 엔지니어링 플랫폼입니다. 강력한 셀프 호스팅 기능(Docker/Kubernetes)과 '프롬프트용 Git' 워크플로우로 차별화되어, 데이터 주권과 통합 유연성을 우선시하는 엔지니어링 팀에 매우 매력적입니다.

**Weave 대비 강점**:
- 셀프 호스팅 및 오픈 소스: 마찰 없는 로컬 설정(Docker)은 보안에 민감한 팀에 매력적이며 엔터프라이즈 계약 없이도 완전한 데이터 주권을 보장합니다.
- 프롬프트 CMS: 비기술적 이해관계자가 코드와 독립적으로 프롬프트를 관리, 버전 관리 및 배포할 수 있는 전용 UI 제공.
- 어노테이션 큐: 트레이스의 휴먼 리뷰 및 라벨링을 위한 구조화된 내장 워크플로우로, Weave의 임시 피드백 메커니즘보다 우수함.
- 비용 가시성: 사용자 및 모델별 토큰 소비를 추적하기 위한 성숙하고 사전 구성된 대시보드 제공.

**Weave 대비 약점**:
- 학습 생태계: 평가에서 모델 미세 조정으로 이어지는 루프를 닫기 위한 W&B Runs/Artifacts와의 원활한 통합이 부족함.
- 데이터 탐색: Weave의 'Board'와 표현 언어는 Langfuse의 정적 대시보드보다 임시(ad-hoc) 데이터 분석에 훨씬 더 강력한 기능을 제공합니다.
- 리치 미디어: Weave는 일반적으로 Langfuse의 텍스트 중심 인터페이스보다 복잡한 멀티모달(오디오/비디오) 트레이스 시각화를 더 잘 처리합니다.

**최근 업데이트**:
- 트레이스 출력 수정: 미세 조정 데이터셋 구축을 위해 트레이스 뷰에서 직접 개선된 버전의 LLM 출력을 캡처. (2026-01-14) [[docs]](https://langfuse.com/docs/observability/corrections)

### Logfire

**개요**: Logfire는 OpenTelemetry 기반으로 구축된 개발자 중심 관측성 플랫폼으로, Pydantic 생태계와의 깊은 통합을 통해 원활한 Python 자동 인스트루먼테이션을 제공합니다. 트레이스에 대한 고유한 SQL 기반 쿼리 엔진과 JSON 로그의 '동적 파쇄(dynamic shredding)'를 통해 차별화되어 프로덕션 애플리케이션을 위한 고성능 분석을 가능하게 합니다. 실시간 디버깅 및 모니터링에는 강력하지만, 현재 Weave에서 볼 수 있는 포괄적인 오프라인 평가 및 데이터셋 관리 워크플로우는 부족합니다.

**Weave 대비 강점**:
- SQL 기반 트레이스 쿼리: 표준 SQL을 사용하여 트레이스 데이터에 대한 임의의 복잡한 분석이 가능하며, Weave의 UI 필터가 가질 수 없는 유연성을 제공함.
- Pydantic 생태계 통합: Pydantic 모델 및 FastAPI에 대한 원활한 제로 설정 인스트루먼테이션은 마찰을 획기적으로 줄여주는 장점임.
- 프로덕션 성능: 대량의 로그 수집 및 빠른 쿼리를 위해 '동적 파쇄' 기능을 갖춘 Rust 백엔드 기반으로 구축됨.

**Weave 대비 약점**:
- 평가 워크플로우 부족: 데이터셋 관리, 체계적인 평가 실행 및 모델 버전의 나란히 비교를 위한 전용 UI가 없음.
- 프롬프트 관리 부재: 프롬프트 레지스트리나 버전 관리 시스템을 제공하지 않아 개발자가 코드 내에서 프롬프트를 관리해야 함.
- 제한적인 비 Python 집중: OTel과 호환되지만, 심층 기능 세트가 Python/Pydantic 스택에 크게 최적화되어 있는 반면 Weave는 더 넓은 다국어 지원을 목표로 함.

**최근 업데이트**:
- 프로젝트 마이그레이션을 위한 멀티 토큰 지원: 원활한 프로젝트 마이그레이션을 돕기 위해 여러 토큰을 처리하는 기능 추가. (2026-02-04) [[docs]](https://logfire.pydantic.dev/docs/release-notes)
- OTel Gen AI 시맨틱 컨벤션: OpenTelemetry Gen AI 시맨틱 컨벤션 스칼라 속성 지원 추가. (2026-01-28) [[docs]](https://logfire.pydantic.dev/docs/release-notes)
- Pytest 통합: pytest 테스트 실행 내에서 원활한 추적 및 관측성을 허용하는 새로운 통합. (2026-01-26) [[docs]](https://logfire.pydantic.dev/docs/release-notes)
- DSPy 통합: DSPy 애플리케이션을 추적하고 모니터링하기 위한 공식 통합. (2026-01-16) [[docs]](https://logfire.pydantic.dev/docs/release-notes)

### Helicone

**개요**: Helicone은 주로 LLM 애플리케이션을 위한 프록시 미들웨어 역할을 하는 오픈 소스 AI 게이트웨이 및 관측성 플랫폼입니다. 핵심 가치 제안은 베이스 URL 변경을 통한 '한 줄 통합'으로, 요청 로깅과 함께 즉각적인 비용 추적, 캐싱, 속도 제한 및 모델 라우팅을 제공하는 것입니다.

**Weave 대비 강점**:
- 제로 마찰 도입: 베이스 URL 변경만 필요하며 SDK 코드 래핑이 필요 없음.
- 게이트웨이 기능: 네이티브 캐싱, 속도 제한 및 모델 라우팅 제공 (Weave에는 이러한 능동적 트래픽 제어 기능이 없음).
- 오픈 소스 및 셀프 호스팅 가능: 완전한 VPC 제어를 원하는 보안 중심 팀에 높은 매력.

**Weave 대비 약점**:
- 얕은 추적: Weave의 SDK만큼 애플리케이션 내부(검색 단계, 내부 로직)를 상세히 볼 수 없음.
- 제한적인 평가 워크플로우: Weave의 성숙하고 깊이 있는 평가 및 데이터셋 비교 도구가 부족함.
- 리치 미디어 추적 부재: Weave에 비해 멀티모달 입력/출력(이미지, 오디오) 시각화에 대한 강조가 적음.

**최근 업데이트**:
- *보고된 데이터 없음*

| 핵심 관측성 | Weave 우위 | Helicone은 프록시 아키텍처 덕분에 에지 관측성(지연 시간, 비용, 입력/출력)에 뛰어나지만, Weave보다 애플리케이션의 내부 실행 로직에 대한 가시성은 낮습니다. |
| 에이전트 / RAG 관측성 | Weave 우위 | Helicone은 애플리케이션 코드 내부가 아닌 API 경계에 위치하므로, Weave에 비해 심층적인 RAG/에이전트 디버깅(검색, 내부 상태)에 덜 최적화되어 있습니다. |
| 평가 통합 | Weave 우위 | 평가 기능(데이터셋, 점수)이 존재하지만 게이트웨이 기능에 비해 부차적입니다. Weave는 더 포괄적인 평가 기반 개발 워크플로우를 제공합니다. |
| 모니터링 및 지표 | Weave 우위 | Helicone은 운영 지표(비용, 지연 시간, 오류)에 매우 강력하며, LLM 사용량에 대한 재무적 가시성 측면에서 종종 범용 도구보다 뛰어난 성능을 보입니다. |
| 실험 / 개선 루프 | Weave 우위 | Helicone은 오프라인 실험 및 반복적 개선보다는 프로덕션 A/B 테스트(라우팅)에 집중하며, 이 분야에서는 Weave가 더 강력합니다. |
| 개발자 경험(DevEx) / 통합 | 대등함 | Helicone은 코드 변경이 없는 프록시 통합으로 '가치 창출 시간(Time to Value)'에서 승리하지만, Weave의 SDK는 더 깊은 프로그래밍 방식의 제어를 제공합니다. |
| 엔터프라이즈 및 보안 | Weave 우위 | 오픈 소스 및 셀프 호스팅 특성 덕분에 완전한 데이터 주권을 허용하여 엔터프라이즈 측면에서 강력한 매력을 가집니다. |
