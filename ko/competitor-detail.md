### Weave

**개요**: Weave는 관측 가능성(observability)을 체계적인 평가 및 모델 개선과 밀접하게 결합하여 LLM 엔지니어링을 위한 '과학적' 선택지로 자리매김하고 있습니다. 모니터링(Helicone)이나 배포(LangSmith)에만 집중하는 경쟁사들과 달리, Weave는 Weights & Biases의 유산을 활용하여 프로덕션 로그를 학습 데이터셋 및 미세 조정(fine-tuning) 작업에 직접 연결하는 통합 '데이터 플라이휠(Data Flywheel)'을 제공합니다.

**주요 강점**:
- **네이티브 학습 통합:** 프로덕션 트레이스(trace)를 W&B 모델 학습 실행과 원활하게 연결하여 미세 조정을 위한 진정한 데이터 플라이휠을 구현하는 유일한 플랫폼입니다.
- **프로그래밍 방식의 평가:** 'eval-as-code' 철학을 바탕으로, 경직된 UI 기반 평가 도구보다 데이터 과학자들에게 더 매력적인 복잡하고 커스텀 가능한 스코어링 로직을 지원합니다.
- **탐색적 캔버스:** 노트북 스타일의 UI 패러다임을 통해 Helicone이나 Datadog의 정적인 대시보드보다 더 깊이 있는 임의 데이터 탐색 및 필터링이 가능합니다.

**개선 필요 사항**:
- **에이전트 시각화:** LangSmith(Studio)나 Langfuse가 뛰어난 성능을 보이는 복잡하고 상태 유지가 필요한 에이전트 루프를 위한 전용 '그래프 뷰'나 시각적 디버거가 부족합니다.
- **게이트웨이/프록시 레이어 부재:** Braintrust나 Helicone에서 제공하는 캐싱, 속도 제한(rate-limiting) 또는 장애 조치(failover) 라우팅과 같은 능동적인 트래픽 관리 기능을 제공하지 않습니다.
- **휴먼 어노테이션 워크플로우:** LangSmith와 비교했을 때 대규모 인간 검수 팀을 관리하기 위한 성숙한 UI 기능(예: 할당 대기열, 합의 스코어링)이 부족합니다.

**최근 업데이트**:
- Stats Clickhouse 쿼리 레이어: 통계 데이터 처리를 위한 백엔드 쿼리 레이어 개선으로 대시보드 성능이 향상되었습니다. (2026-01-20)
- OTel 내보내기 개선: 신뢰성과 표준 준수를 위해 OpenTelemetry 내보내기를 비동기 삽입(async inserts) 방식에서 변경했습니다. (2026-01-08)

### LangSmith

**개요**: LangSmith는 단순한 관측 도구에서 전체 스택 에이전트 개발, 배포 및 모니터링 솔루션으로 진화한 포괄적인 LLM 엔지니어링 플랫폼입니다. LangChain 및 LangGraph 생태계와 깊이 통합되어 복잡한 에이전트 워크플로우를 시각화하고 인간 참여형(human-in-the-loop) 평가 프로세스를 관리하는 데 탁월합니다. 프레임워크에 구애받지 않는다고 주장하지만, Studio 그래프 뷰 및 배포 기능과 같은 가장 강력한 기능들은 LangGraph 사용자에게 고도로 최적화되어 있습니다.

**Weave 대비 강점**:
- **에이전트 상태 시각화:** LangGraph Studio는 Weave의 트레이스 트리보다 복잡하고 상태 유지가 필요한 에이전트 루프에 대해 우수한 시각적 디버거를 제공합니다.
- **배포 인프라:** LangSmith는 이제 에이전트를 위한 관리형 배포('LangSmith Deployment')를 제공하여 서빙 영역까지 확장하고 있습니다.
- **휴먼 어노테이션 워크플로우:** '어노테이션 대기열' 및 쌍체 비교(pairwise comparison) UI는 Weave의 현재 피드백 메커니즘보다 더 성숙하고 워크플로우 중심적입니다.

**Weave 대비 약점**:
- **학습과의 단절:** LangSmith는 모델 학습이나 미세 조정 작업과 직접적인 통합이 없습니다. 반면 Weave는 관측 데이터를 W&B의 학습 추적 기능과 원활하게 연결합니다.
- **프레임워크 편향:** 기술적으로는 중립적이지만, 핵심 기능(Studio, 배포)은 LangGraph에 종속적인 반면 Weave는 진정한 Python 네이티브이며 프레임워크 중립적입니다.
- **모델 관리:** 모델 아티팩트의 버전 관리를 위한 전용 모델 레지스트리가 부족하며, 대신 메타데이터 문자열에 의존합니다.

**최근 업데이트**:
- 트레이스 미리보기 커스텀: 사용자가 트레이스 입력 및 출력의 특정 부분을 트레이싱 테이블에 직접 표시되도록 설정할 수 있어 커스텀 데이터 구조의 사용성이 개선되었습니다. (2026-02-06)
- LangSmith Self-Hosted v0.13: 클라우드 버전과의 기능 패리티(parity), 성능 및 운영 제어 능력을 향상시킨 자체 호스팅 고객용 주요 릴리스입니다. (2026-01-16)

### Arize Phoenix

**개요**: Arize Phoenix는 OpenTelemetry 및 OpenInference 표준을 활용하여 깊이 있는 트레이싱 및 평가 기능을 제공하는 오픈 소스 우선 LLM 관측 플랫폼입니다. 로컬 우선 개발 도구로 작동하면서 프로덕션 모니터링을 위한 광범위한 Arize AI 엔터프라이즈 플랫폼으로 확장 가능하며, 강력한 임베딩 시각화 및 RAG 문제 해결 기능으로 차별화됩니다.

**Weave 대비 강점**:
- 오픈 표준 네이티브: 전체가 OpenTelemetry/OpenInference 기반으로 구축되어 벤더 종속(lock-in)을 피하려는 팀에게 매력적입니다.
- 임베딩 시각화: Arize의 ML 배경을 바탕으로 벡터 검색을 시각화하고 디버깅하는 우수한 도구(3D 포인트 클라우드)를 제공합니다.
- 로컬 우선 워크플로우: 초기 개발 시 클라우드 연결이 필요 없는 강력한 로컬 디버깅 경험('Phoenix')을 제공합니다.

**Weave 대비 약점**:
- 학습 통합: 원활한 미세 조정 루프를 위한 학습 플랫폼(W&B)과의 네이티브 연결이 부족합니다.
- 모델 레지스트리: W&B와 비교할 만한 내장 모델 레지스트리가 없으며 메타데이터 태그에 의존합니다.
- 통합 플랫폼: Weave는 학습+평가+운영을 위한 단일 창구를 제공하는 반면, Arize는 별도의 ML 및 LLM 워크플로우를 연결해야 합니다.

**최근 업데이트**:
- Claude Opus 4.6 지원: 플레이그라운드에 Claude Opus 4.6 모델 지원이 추가되었습니다. (2026-02-09)
- 도구 선택 평가기: 도구 선택 로직을 평가하기 위한 전용 평가기가 새로 추가되었습니다. (2026-02-06)
- 충실도(Faithfulness) 평가기: 더 나은 RAG 평가를 위해 FaithfulnessEvaluator를 추가하고 HallucinationEvaluator를 지원 중단했습니다. (2026-02-02)
- 도구 호출 정확도 지표: 트레이스 내 도구 호출의 정확도를 추적하는 새로운 지표가 추가되었습니다. (2026-02-02)
- 커스텀 지표 커서 규칙: 커서 규칙을 통해 새로운 내장 지표(LLM 분류)를 생성할 수 있는 UI 업데이트가 진행되었습니다. (2026-01-21)

### Braintrust

**개요**: Braintrust는 평가, 프롬프트 엔지니어링(플레이그라운드) 및 관측 가능성을 밀접하게 결합한 성숙한 엔터프라이즈 중심 AI 엔터프라이즈 플랫폼입니다. 캐싱 및 속도 제한을 위한 강력한 'AI Proxy', 광범위한 SDK 지원(Java, Go, C# 포함), 그리고 데이터 분석을 위한 임베디드 AI 어시스턴트인 'Loop'로 차별화됩니다.

**Weave 대비 강점**:
- 더 넓은 SDK 생태계(Java, Go, Ruby, C#)로 다양한 엔터프라이즈 스택에 어필
- 캐싱, 속도 제한 및 비용 제어 미들웨어를 제공하는 통합 'AI Proxy'
- 강력한 프롬프트 버전 관리 및 CMS 기능을 갖춘 성숙한 '플레이그라운드'
- 내구성 있는 실행/에이전트 트레이싱을 위한 'Temporal'과의 깊은 통합

**Weave 대비 약점**:
- 모델 학습/미세 조정 파이프라인과의 깊은 통합 부족 (W&B Core와 대조적)
- UI가 더 밀집되고 복잡하여 데이터 과학자들이 접근하기에 다소 어려울 수 있음
- 대량의 트레이싱 발생 시 Weave의 유연한 옵션에 비해 가격 모델이 비쌀 수 있음

**최근 업데이트**:
- 트레이스 레벨 스코어러: 커스텀 코드 스코어러가 이제 전체 실행 트레이스에 접근하여 다단계 워크플로우 및 에이전트 동작을 평가할 수 있습니다. (2026-02)
- LangSmith 통합: LangSmith 트레이스를 Braintrust로 라우팅하는 래퍼를 통해 병행 사용 또는 마이그레이션을 지원합니다. (2026-02)
- Cursor 및 MCP 통합: Cursor 에디터 및 Model Context Protocol과의 통합으로 IDE에서 로그를 쿼리하고 평가를 실행할 수 있습니다. (2026-02)
- 자동 계측(Python/Ruby/Go): Python, Ruby, Go SDK에서 주요 제공업체에 대한 코드 수정 없는 트레이싱을 지원합니다. (2026-01)
- Temporal 통합: 분산 에이전트 관측을 위해 Temporal 워크플로우 및 액티비티의 자동 트레이싱을 지원합니다. (2026-01)

### Logfire

**개요**: Logfire(Pydantic 제작)는 OpenTelemetry 기반으로 구축된 개발자 중심 관측 플랫폼으로, Pydantic 생태계 및 Python 기반 AI 에이전트와 원활한 통합을 제공합니다. 프로덕션 모니터링, SQL 기반 트레이스 쿼리 및 심층 코드 레벨 계측을 통한 디버깅에 탁월하지만, Weave에서 볼 수 있는 체계적인 평가, 데이터셋 관리 및 실험 워크플로우는 부족합니다.

### Helicone

**개요**: Helicone은 애플리케이션과 LLM 제공업체 사이의 프록시 역할을 하는 오픈 소스 AI 게이트웨이 및 관측 플랫폼입니다. 주요 가치 제안은 인프라 레벨의 제어(캐싱, 라우팅, 장애 조치, 속도 제한 제공)와 입력, 출력 및 비용에 대한 고정밀 로깅의 결합입니다.

**Weave 대비 강점**:
- **게이트웨이 기능:** Weave(수동적 관찰자)가 제공하지 않는 능동적 트래픽 제어(캐싱, 속도 제한, 폴백)를 제공합니다.
- **코드 수정 없는 통합:** OpenAI SDK에서 `base_url`만 변경하면 통합되는 경우가 많아 코드 변경이나 데코레이터가 필요하지 않습니다.
- **비용 관리:** 지출을 추적하고 캐싱을 통해 능동적으로 비용을 절감하는 우수한 기능을 제공합니다.

**Weave 대비 약점**:
- **내부 트레이싱 부족:** Weave의 트레이스 트리처럼 애플리케이션 내부(함수 호출, 검색 단계, 로직 루프)를 볼 수 없으며 LLM I/O만 확인 가능합니다.
- **제한된 평가 프레임워크:** Weave에서 볼 수 있는 강력한 데이터셋 버전 관리, 회귀 테스트 및 CI/CD 통합이 부족합니다.
- **멀티모달/학습 이력 부재:** 광범위한 W&B 플랫폼처럼 관측 데이터를 모델 학습이나 미세 조정 실험으로 다시 연결하지 않습니다.
