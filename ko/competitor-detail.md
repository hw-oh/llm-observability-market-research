### Weave

**개요**: Weave는 'AI 엔지니어의 워크벤치'를 표방하며, 모델 학습 라이프사이클(W&B)과 깊게 통합된 코드 우선(code-first) 방식의 화이트박스 관측성(observability)을 우선시합니다. 프록시 기반의 경쟁사(Helicone)나 CMS 중심의 도구(Humanloop)와 달리, Weave는 임의의 코드를 추적하고 추론 데이터를 모델 아티팩트와 연결하는 데 탁월합니다. 따라서 단순히 API를 호출하는 수준을 넘어 복잡하고 커스텀화된 LLM 애플리케이션을 구축하는 팀에게 최적의 선택입니다.

**주요 강점**:
- 깊이 있는 학습 이력(Lineage): 프로덕션 트레이스(trace)를 학습 실행 및 모델 아티팩트(W&B 에코시스템)와 연결하는 독보적인 기능을 제공하며, 이는 독립형 관측 도구들이 따라올 수 없는 부분입니다.
- 화이트박스 가시성: 프록시 기반 경쟁사(Helicone)가 API 입출력만 볼 수 있는 것과 달리, 임의의 Python 코드와 내부 로직을 추적하는 뛰어난 능력을 갖추고 있습니다.
- 유연한 'Board' UI: 고도로 커스터마이징 가능한 노트북 스타일의 대시보드를 통해 LangSmith나 Langfuse의 경직된 사전 정의 대시보드보다 더 깊이 있는 탐색적 분석이 가능합니다.
- 모델 레지스트리 통합: 실제 모델 레지스트리와의 네이티브 통합을 통해 단순한 메타데이터 문자열이 아닌 모델 바이너리에 대한 엄격한 버전 관리가 가능합니다.

**개선 필요 사항**:
- 프롬프트 관리 CMS: LangSmith(Hub), Langfuse, 그리고 기존의 Humanloop는 코드 변경 없이 프롬프트를 버전 관리하고 편집할 수 있는 우수한 비기술적 UI를 제공합니다.
- 재무 운영(FinOps) 및 비용 세분화: Helicone과 Langfuse는 커스텀 가격 책정 계층 및 예산 알림을 포함하여 프로덕션 운영에 필수적인 훨씬 더 진보된 비용 추적 기능을 제공합니다.
- 휴먼 어노테이션 워크플로우: Weave에는 LangSmith와 Langfuse에서 표준이 된 사람 라벨러 관리 전용 '어노테이션 큐' 워크플로우가 부족합니다.
- 에이전트 그래프 시각화: 경쟁사들은 순환형 에이전트 워크플로우(예: LangGraph 통합)를 위한 특화된 시각화 기능을 빠르게 출시하고 있는 반면, Weave의 트레이스 뷰는 주로 선형적입니다.

### LangSmith

**개요**: LangSmith는 LLM 애플리케이션을 위한 포괄적인 DevOps 플랫폼으로, 엔드 투 엔드 관측성, 평가 및 프롬프트 엔지니어링 도구를 제공합니다. 프레임워크에 구애받지 않는다고 마케팅하지만, LangChain 및 LangGraph 에코시스템에 깊게 최적화되어 있어 복잡한 에이전트 워크플로우와 검색 파이프라인에 대해 업계 최고 수준의 시각화를 제공합니다. 최근 배포(Agent Servers) 및 엔터프라이즈급 모니터링으로 영역을 확장하며 완전한 라이프사이클 관리 솔루션으로 자리매김하고 있습니다.

**Weave 대비 강점**:
- 복잡하고 순환적인 에이전트 동작 시각화를 위한 LangGraph와의 네이티브 통합.
- 프롬프트 관리 및 버전 관리를 위한 CMS 역할을 하는 전용 'Prompt Hub'.
- 인간 참여형(human-in-the-loop) 평가를 위한 우수한 워크플로우를 제공하는 Pairwise 어노테이션 큐.
- 트레이스에서 즉시 '편집 후 실행' 디버깅이 가능한 Playground 기능.

**Weave 대비 약점**:
- 모델 학습/미세 조정 파이프라인(W&B의 핵심 강점)과의 깊은 통합이 부족함.
- 타 프레임워크 사용자에게는 UI가 복잡하고 'LangChain 중심적'으로 느껴질 수 있음.
- 커스텀 지표 로깅이 Weave의 일반 객체 로깅 기능보다 덜 유연함.
- 모델 버전 관리가 전체 모델 레지스트리가 아닌 메타데이터 문자열로 제한됨.

**최근 업데이트**:
- 트레이스 미리보기 커스텀: 트레이스 테이블 뷰에 표시될 입출력을 제어하는 구성 옵션 추가. (2026-02-06)
- LangSmith Fetch CLI: 터미널에서 직접 트레이스에 액세스하고 디버깅할 수 있는 도구. (2025-12-10)
- Polly AI 어시스턴트: 에이전트 디버깅 및 트레이스 분석을 위한 플랫폼 내 AI 어시스턴트(베타). (2025-12-10)
- 통합 비용 추적: LLM, 도구, 검색 단계 전반에 대한 풀스택 비용 모니터링. (2025-12-09)
- Pairwise 어노테이션 큐: 검토자가 두 에이전트의 출력을 나란히 비교할 수 있는 구조화된 워크플로우. (2025-12-17)

### Arize Phoenix

**개요**: Arize Phoenix는 로컬 개발과 프로덕션 모니터링 사이의 간극을 메우는 OpenTelemetry 네이티브 관측성 및 평가 플랫폼입니다. 에이전트 워크플로우 추적에 능하며, 오픈 소스 로컬 디버깅에서 Arize 플랫폼을 통한 엔터프라이즈급 모니터링으로의 원활한 경로를 제공합니다.

**Weave 대비 강점**:
- OpenTelemetry 네이티브: OTLP 표준을 기반으로 구축되어 벤더 종속 없이 광범위한 인프라(Grafana, Jaeger)와 더 쉽게 통합됩니다.
- 로컬 우선 경험: OSS 버전은 완전히 오프라인으로 실행되는 우수한 로컬 노트북 경험을 제공하여 개인정보 보호를 중시하는 개발자에게 매력적입니다.
- 프로덕션 유산: W&B의 개발 중심 포커스보다 역사적으로 강력한 Arize의 성숙한 플랫폼을 활용하여 드리프트 감지 및 프로덕션 모니터링을 수행합니다.

**Weave 대비 약점**:
- 학습 통합: 관측성을 모델 학습 실행 및 아티팩트 버전 관리와 연결하는 W&B의 매끄러운 이력 관리가 부족합니다.
- 커스텀 시각화: UI가 고도로 커스터마이징된 대화형 대시보드를 생성하는 Weave Boards에 비해 덜 유연합니다.
- 에코시스템 범위: W&B가 제공하는 전체 MLOps 도구 세트(Sweeps, Artifacts, Model Registry)를 제공하지 않으므로 전체 라이프사이클을 위해 다른 도구와의 통합이 필요합니다.

**최근 업데이트**:
- FaithfulnessEvaluator: 더 나은 RAG 정확도 측정을 위해 HallucinationEvaluator를 대체하는 새로운 평가자. (2026-02-02)
- 도구 호출 정확도 지표: 에이전트 도구 호출의 성공률을 측정하기 위해 특별히 설계된 새로운 지표. (2026-01-27)
- Claude Opus 4.6 지원: 플레이그라운드에 최신 Claude 모델 지원 추가. (2026-02-09)
- 지표를 위한 Cursor 규칙: Cursor 규칙을 사용하여 커스텀 분류 평가자를 생성하는 기능 강화. (2026-01-21)

### Braintrust

**개요**: Braintrust는 AI 제품 구축을 위한 '운영 체제'를 표방하는 엔터프라이즈급 AI 관측성 및 평가 플랫폼입니다. 프롬프트 엔지니어링(AIR 플레이그라운드), 평가(LLM-as-a-judge), 프로덕션 모니터링을 단일 워크플로우로 긴밀하게 통합하며, 캐싱 및 속도 제한을 위한 강력한 AI Proxy가 이를 뒷받침합니다. 강력한 쿼리 기능(BTQL/SQL)과 광범위한 다국어 SDK 지원이 돋보입니다.

**Weave 대비 강점**:
- 통합 프롬프트 플레이그라운드(AIR)가 프롬프트를 버전 관리되는 일급 자산으로 취급합니다.
- AI Proxy가 내장 캐싱, 속도 제한 및 키 관리를 제공합니다.
- Python/TS를 넘어 Java, Go, C#, Ruby 등 광범위한 다국어 지원을 제공합니다.
- 애드혹 데이터 분석을 위한 강력한 UI 내 쿼리 엔진(SQL/BTQL)을 갖추고 있습니다.

**Weave 대비 약점**:
- 모델 학습 파이프라인 및 아티팩트 이력과의 깊은 통합이 부족합니다.
- 시각화 기능이 Weave의 대화형 Board에 비해 더 실용주의적(utilitarian)입니다.
- 오픈 소스 친화적인 옵션에 비해 대량 트레이싱 시 가격 모델이 제한적일 수 있습니다.

**최근 업데이트**:
- 트레이스 레벨 스코어러: 커스텀 코드 스코어러가 이제 전체 실행 트레이스에 액세스하여 다단계 워크플로우와 에이전트 동작을 평가할 수 있습니다. (2026-02)
- SDK 지원 확대 (Java, Go, Ruby, C#): Java, Go, Ruby, C#/.NET을 위한 자동 인스트루멘테이션 기능이 포함된 네이티브 SDK. (2026-01)
- SQL 구문 지원: 트레이스 및 실험 쿼리를 위해 표준 SQL 구문(HAVING 및 암시적 앨리어싱 포함) 지원. (2026-01)
- Temporal 통합: Temporal 워크플로우 및 액티비티의 자동 트레이싱을 통해 워커 전반의 분산 트레이스 캡처. (2026-01)
- Claude Code 및 Cursor 통합: AI 코딩 어시스턴트가 자연어를 통해 로그 및 실험 결과를 쿼리할 수 있는 깊은 통합. (2025-12)

### Langfuse

**개요**: Langfuse는 관측성, 프롬프트 관리, 평가를 결합한 포괄적인 오픈 소스 LLM 엔지니어링 플랫폼입니다. 강력한 셀프 호스팅 기능, OpenTelemetry 네이티브 아키텍처, 그리고 인간 참여형 어노테이션 및 비용 추적을 위한 성숙한 도구 세트로 차별화됩니다.

**Weave 대비 강점**:
- Weave의 코드 우선 방식과 대비되는 성숙한 프롬프트 관리 CMS(UI 기반 버전 관리, 배포 라벨).
- 보안에 민감한 팀에게 매력적인 강력한 셀프 호스팅 스토리(Open Source/Docker).
- 인간 라벨링 팀 관리를 위한 전용 '어노테이션 큐' 워크플로우.
- 세분화된 비용 모델링(컨텍스트 기반 가격 계층) 및 지출 알림.

**Weave 대비 약점**:
- 모델 학습/미세 조정 파이프라인(W&B의 핵심 강점)과의 깊은 통합이 부족함.

- 프롬프트와 데이터셋 이외의 아티팩트 관리에 대한 강조가 덜함.

**최근 업데이트**:
- 트레이스 출력 수정: 트레이스 뷰에서 직접 개선된 버전의 LLM 출력을 캡처하여 미세 조정 데이터셋 구축. (2026-01-14)
- 데이터셋 아이템 버전 관리: 데이터셋 아이템의 모든 추가, 업데이트, 삭제 시 자동 버전 관리. (2025-12-15)

- LLM-as-a-Judge 실행 트레이싱: 모든 평가자 실행 시 트레이스를 생성하여 평가에 사용된 프롬프트와 토큰 사용량 조사. (2025-10-16)

### Humanloop

**개요**: 프롬프트 엔지니어링, 평가 및 관측성 분야의 선두 플랫폼이었던 Humanloop는 2025년 중반 Anthropic에 인수되었습니다. 현재 플랫폼은 일몰(sunset) 단계에 있으며 2025년 9월 8일에 완전히 종료될 예정으로, 새로운 기능 개발은 중단되었습니다. 기존 사용자들은 활발하게 대안 솔루션으로 마이그레이션 중입니다.

**Weave 대비 강점**:
- 프롬프트 관리 CMS: 비기술적 사용자가 프롬프트를 버전 관리하고 편집할 수 있는 우수한 UI.

- 데이터 탐색: Weave보다 덜 강력한 대화형 데이터 탐색 및 커스텀 보드 생성 기능.

### Logfire

**개요**: Logfire는 Pydantic 팀이 만든 관측성 플랫폼으로, Python 및 AI 애플리케이션을 위한 비편향적이고 오픈 소스 기반(OpenTelemetry)인 솔루션으로 설계되었습니다. 깊은 Pydantic/FastAPI 통합, 트레이스용 SQL 기반 쿼리 엔진, 그리고 LLM 상호작용을 표준 애플리케이션 로그로 취급하는 개발자 중심 워크플로우로 차별화됩니다.

**Weave 대비 강점**:
- SQL 기반 쿼리: 표준 SQL을 사용하여 트레이스 데이터에 대해 매우 유연한 애드혹 분석이 가능합니다.
- Pydantic 에코시스템: 이미 Pydantic과 FastAPI를 사용하는 사용자에게 매우 낮은 진입 장벽을 제공합니다.
- OpenTelemetry 네이티브: 완전히 OTel 기반으로 구축되어 데이터 수집 시 벤더 종속이 없습니다.
- 가격 모델: 넉넉한 무료 티어와 단순한 사용량 기반 가격 책정으로 개인 개발자와 스타트업에게 매력적입니다.

**Weave 대비 약점**:
- 평가 플랫폼 부재: 데이터셋 관리, 평가 실행 또는 모델 버전의 나란히 비교를 위한 전용 UI가 없습니다.
- 프롬프트 관리 없음: 프롬프트를 버전 관리, 관리 또는 플레이그라운드에서 테스트하는 도구를 제공하지 않습니다.
- 제한적인 비기술적 UI: 인터페이스가 데이터 집약적이고 개발자 중심적이어서 Weave의 협업용 'Board' 기능이 부족합니다.
- 실험 루프 누락: 트레이싱과 데이터셋 생성/미세 조정 사이의 루프를 닫아주지 못합니다.

**최근 업데이트**:
- Pytest 통합: pytest 실행을 스팬(span)으로 로깅하여 테스트 관측성을 가능하게 하는 직접 통합. (2026-01-26)
- DSPy 통합: LLM 프로그래밍 프레임워크인 DSPy를 위한 자동 인스트루멘테이션. (2026-01-16)
- MCP 지원: 컨텍스트 교환 표준화를 위한 Model Context Protocol (MCP) 지원. (2025-03-31)
- OpenAI Agents 프레임워크: OpenAI의 새로운 Agents SDK를 위한 인스트루멘테이션. (2025-03-11)
- SQL 기반 지표: 트레이스에 대해 임의의 SQL 쿼리를 실행하여 커스텀 지표를 생성하는 기능. (2025-09-05)

### Helicone

**개요**: Helicone은 오픈 소스 AI 게이트웨이 및 관측성 플랫폼으로, 주로 애플리케이션과 LLM 제공자 사이의 프록시 역할을 합니다. API 트래픽을 가로채서 견고한 로깅, 캐싱, 비용 분석을 제공하지만, Weave가 제공하는 깊은 코드 레벨 인스트루멘테이션과 내부 에이전트 로직에 대한 계층적 트레이스 시각화는 부족합니다.

**Weave 대비 강점**:
- 제로 코드 통합(프록시/게이트웨이 모델): SDK 인스트루멘테이션 변경 없이 URL 교체만으로 가능합니다.
- 내장 게이트웨이 기능: Weave가 제공하지 않는 캐싱, 속도 제한, 스마트 라우팅(폴백) 기능을 제공합니다.
- 수많은 제공자에 걸친 즉각적이고 우수한 비용 추적 및 분석.
- 완전한 오픈 소스이며 엄격한 데이터 프라이버시 요구 사항을 위해 쉽게 셀프 호스팅이 가능합니다.

**Weave 대비 약점**:
- 깊은 코드 가시성 부족: 내부 함수 호출, 검색 단계 또는 도구 실행 로직을 추적할 수 없습니다(블랙박스 vs 화이트박스).
- 취약한 평가 프레임워크: Weave의 프로그래밍 방식 'LLM-as-a-Judge' 깊이와 데이터셋 버전 관리가 부족합니다.
- 복잡한 에이전트 워크플로우 디버깅을 위한 계층적 플레임 그래프(flame graph) 부재.
- 시각화가 트레이스/작업 중심이 아닌 요청(request) 중심으로 이루어집니다.

**최근 업데이트**:
- 실험(Experiments): 데이터셋에 대해 프롬프트를 테스트하여 출력과 비용을 비교하는 UI. (2024-10)
- 프롬프트 관리(Assembly): 코드와 분리되어 프롬프트를 버전 관리하고 관리할 수 있는 강화된 UI. (2024-09)
- 세션 그룹화: 채팅 기록 추적을 위해 여러 요청을 단일 세션으로 그룹화하는 기능. (2024-08)
