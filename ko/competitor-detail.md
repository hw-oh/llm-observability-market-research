# W&B Weave — 제품 상세 정보

[← 홈](./) · [상세 비교](./comparison)

### Weave

**개요**: Weave는 모델 학습(W&B)과 프로덕션 추론 사이의 간극을 네이티브하게 연결하는 유일한 관측성(Observability) 플랫폼을 표방합니다. 아티팩트 관리, 서버리스 LoRA 추론, 멀티모달 평가를 통합함으로써, Weave는 단순한 프롬프트뿐만 아니라 모델 가중치를 반복적으로 개선해야 하는 AI 엔지니어를 타겟으로 합니다.

**주요 강점**:
- 네이티브 학습 통합: 프로덕션 트레이스에서 W&B 학습 실행 및 아티팩트로 이어지는 원활한 리니지(Lineage)를 제공하며, 이는 순수 관측성 경쟁사들이 갖추지 못한 점입니다.
- 멀티모달 관측성: 업계 최초로 네이티브 오디오 모니터(Audio Monitors)를 출시하여, 텍스트 중심의 LangSmith와 같은 경쟁사와 차별화됩니다.
- 서버리스 LoRA 추론: Playground에서 미세 조정된 어댑터를 핫스왑(hot-swap)할 수 있는 기능은 MLflow나 Langfuse가 따라올 수 없는 빠른 모델 반복 학습을 가능하게 합니다.
- 탐색적 분석: 'Board' 아키텍처는 Braintrust나 Arize의 정적인 대시보드보다 더 유연하고 코드 중심적인 분석을 제공합니다.

**개선 필요 사항**:
- 에이전트 시각화: LangSmith(LangGraph)나 Arize Phoenix에서 볼 수 있는 네이티브 순환 그래프 뷰 및 상태 관리 시각화 기능이 부족합니다.
- 비기술자용 CMS: Langfuse와 Braintrust는 제품 관리자를 위해 우수한 'CMS 스타일'의 프롬프트 관리 UI를 제공하는 반면, Weave는 여전히 코드 중심적입니다.
- SDK 생태계: Braintrust와 MLflow는 더 넓은 범위의 엔터프라이즈 언어(Java, C#, Go)를 지원하지만, Weave는 주로 Python/TypeScript에 집중하고 있습니다.
- Financial Ops: Langfuse는 즉시 사용 가능한 훨씬 더 세밀한 비용 추적 및 빌링 분석 기능을 제공합니다.

**최근 업데이트**:
- 오디오 모니터: 오디오 지원 LLM을 사용하여 텍스트와 함께 오디오 출력을 관찰하고 판단하는 모니터 생성 지원 (2026-02-01)
- 동적 리더보드: 지속적인 커스터마이징 및 CSV 내보내기가 가능한 평가 기반 자동 생성 리더보드 (2026-01-29)
- Playground 내 커스텀 LoRA: W&B 아티팩트에서 미세 조정된 커스텀 LoRA 가중치를 Weave Playground로 직접 로드하여 추론하는 기능 (2026-01-16)

### LangSmith

**개요**: LangSmith는 관측성, 평가, 프롬프트 관리를 결합한 포괄적인 LLM 엔지니어링 플랫폼으로, LangChain 생태계의 기본 선택지 역할을 합니다. 에이전트 워크플로우를 위해 LangGraph와 깊이 통합되어 있으면서도, 강력한 인간 평가 큐와 엔터프라이즈급 셀프 호스팅 기능을 갖춘 프레임워크 독립적 도구로 공격적으로 확장하고 있습니다.

**Weave 대비 강점**:
- 네이티브 LangGraph 통합: 순환 에이전트 워크플로우 및 상태 관리에 대한 우수한 시각화를 제공합니다.
- 프롬프트 허브(Prompt Hub): Playground와 긴밀하게 통합된 프롬프트 관리를 위한 별도의 성숙한 제품 모듈입니다.
- 주석 큐(Annotation Queues): 단순한 피드백을 넘어 인간 검토자를 위한 전문화되고 효율적인 UI 워크플로우(쌍체 비교, 리커트 척도 등)를 제공합니다.

**Weave 대비 약점**:
- 학습 통합: 원활한 미세 조정 루프를 위한 학습 플랫폼(W&B Runs 등)과의 네이티브 연결이 부족합니다.
- 아티팩트 관리: 바이너리 아티팩트의 버전을 관리하는 전체 모델 레지스트리를 제공하지 않으며 메타데이터만 관리합니다.
- 프레임워크 인식: 가볍고 독견적이지 않은(unopinionated) 트레이싱을 선호하는 개발자들에게 종종 '너무 복잡함' 또는 'LangChain 전용'으로 인식됩니다.

**최근 업데이트**:
- 트레이스 미리보기 커스텀: 대시보드에서 트레이스 미리보기가 표시되는 방식을 사용자가 설정할 수 있는 UI 업데이트 (2026-02-06)
- LangSmith 셀프 호스팅 v0.13: 셀프 호스팅 엔터프라이즈 인프라 구성 요소 업데이트 (2026-01-16)

| 개발자 경험 / 통합 | 대등함 | 특히 LangChain 생태계에서 개발자 경험이 강력합니다. CLI와 SDK가 성숙해졌으나, LangChain 미사용자는 생태계 편향으로 인해 마찰을 느낄 수 있습니다. |
| 엔터프라이즈 및 보안 | 대등함 | LangSmith는 강력한 셀프 호스팅(Kubernetes/Helm) 및 컴플라이언스 기능을 갖춘 완전한 엔터프라이즈용 플랫폼으로 성장하여 Weave의 엔터프라이즈 포지션과 대등해졌습니다. |

### Langfuse

**개요**: Langfuse는 관측성, 프롬프트 관리, 평가를 통합된 개발자 경험으로 결합한 선도적인 오픈 소스 LLM 엔지니어링 플랫폼입니다. 강력한 셀프 호스팅 지원, 대규모 처리를 위한 ClickHouse 기반 분석 엔진, 엔지니어와 제품 팀 모두에게 매력적인 성숙한 'CMS 스타일' 프롬프트 관리 시스템으로 차별화됩니다.

**Weave 대비 강점**:
- 오픈 소스 및 셀프 호스팅: 네이티브 OSS 모델은 보안에 민감한 기업에 매력적이며 제한 없는 커스터마이징을 허용합니다.
- 프롬프트 관리 CMS: Weave의 코드 중심 접근 방식에 비해 프롬프트 관리, 버전 관리 및 배포를 위한 우수한 비기술자용 UI를 제공합니다.
- 비용 및 빌링 분석: 커스텀 가격 책정 계층 및 지출 알림 지원을 통해 더 세밀한 재무 추적이 가능합니다.

**Weave 대비 약점**:
- 학습 통합: 학습/미세 조정 작업(W&B Runs)과의 네이티브 연결이 부족하며 데이터셋 내보내기만 제공합니다.
- 노트북 경험: Weave의 대화형 노트북 UI는 Langfuse의 대시보드 우선 UX보다 더 나은 탐색적 코딩 경험을 제공합니다.
- 모델 레지스트리: 전용 모델 레지스트리가 없으며 모델 반복을 추적하기 위해 프롬프트 버전에 의존합니다.

**최근 업데이트**:
- 트레이스 출력 수정: 미세 조정 데이터셋 구축을 위해 트레이스 뷰에서 직접 개선된 LLM 출력 버전을 캡처 (2026-01-14)
- 추론/생각 렌더링: 트레이스 상세 정보에서 모델 출력의 '생각(thinking)' 또는 '추론(reasoning)' 부분을 렌더링하는 새로운 UI 지원 (v3.148) (2026-01-20)
- 조직 감사 로그 뷰어: 보안 및 컴플라이언스 가시성을 높이기 위해 조직 수준의 감사 로그 뷰어 추가 (2026-01-20)

| 개발자 경험 / 통합 | 대등함 | Langfuse의 오픈 소스 및 셀프 호스팅 가능 특성은 개인정보를 중시하는 팀에게 큰 개발자 경험적 이점입니다. 두 플랫폼 모두 강력한 SDK와 프레임워크 통합을 제공합니다. |
| 엔터프라이즈 및 보안 | 대등함 | Langfuse는 오픈 소스 코어를 통해 팀이 셀프 호스팅으로 데이터 거주성 문제를 해결할 수 있게 함으로써 엔터프라이즈 딜에서 매우 경쟁력이 있습니다. RBAC 및 감사 로그와 같은 보안 기능도 성숙해졌습니다. |

### Braintrust

**개요**: Braintrust는 '프롬프트 우선' 엔지니어링 워크플로우와 광범위한 언어 지원(Java, Go, Ruby, C#)으로 차별화되는 엔터프라이즈급 AI 관측성 및 평가 플랫폼입니다. 게이트웨이 기능을 위한 내장 AI 프록시와 자연어 데이터 분석을 위한 AI 에이전트 'Loop'를 특징으로 하며, 대규모 엔지니어링 조직을 위한 포괄적인 솔루션으로 포지셔닝하고 있습니다.

**Weave 대비 강점**:
- 광범위한 SDK 생태계: Java, Go, Ruby, C#에 대한 네이티브 지원을 통해 Weave(Python/TS)가 덜 보급된 엔터프라이즈 스택을 공략합니다.
- 통합 AI 프록시: 캐싱, 속도 제한, 키 관리를 위한 내장 게이트웨이는 통합 전용 방식인 Weave에 비해 인프라를 단순화합니다.
- Loop AI 어시스턴트: 'Loop' 에이전트는 Weave의 현재 UI보다 데이터 쿼리를 위한 우수한 자연어 인터페이스를 제공합니다.
- 프롬프트 Playground 성숙도: 프롬프트 엔지니어링 및 버전 관리 워크플로우가 매우 정교하며 제품 경험의 핵심입니다.

**Weave 대비 약점**:
- 학습 리니지 부족: Braintrust는 모델 학습, 미세 조정 및 아티팩트 리니지 추적을 위한 W&B와의 통합 수준을 따라올 수 없습니다.
- 가격 책정 복잡성: 호스트당/좌석당 가격 모델은 사용량 기반 모델에 비해 확장 중인 팀에게 더 큰 마찰이 될 수 있습니다.
- 시각화 유연성: 대시보드는 갖추고 있지만, 심층적인 커스텀 분석을 위한 Weave의 'Board'와 동등한 수준의 완전 프로그래밍 가능 UI가 부족합니다.
- 하이퍼파라미터 최적화: 프롬프트/모델의 자동 하이퍼파라미터 튜닝을 위한 W&B Sweeps와 동등한 네이티브 기능이 없습니다.

**최근 업데이트**:
- 트레이스 수준 스코어러(Scorer): 커스텀 코드 스코어러가 이제 전체 실행 트레이스에 접근하여 다단계 워크플로우 및 에이전트 동작을 평가할 수 있습니다. (2026-02)
- LangSmith 통합: 트레이싱 및 평가 호출을 LangSmith와 Braintrust에 병렬로 보내거나 Braintrust로만 라우팅할 수 있는 래퍼(Wrapper) 제공 (2026-02)

### MLflow

**개요**: MLflow는 방대한 설치 기반을 활용하여 관측성을 범용화함으로써 고전적인 MLOps 플랫폼에서 포괄적인 LLMOps 스위트로 공격적으로 전환했습니다. 2026년 1월 출시된 MLflow 3.9를 통해 이제 네이티브 OpenTelemetry 트레이싱, 지속적인 프로덕션 모니터링, 'Judge Builder' UI를 제공하며 실험과 프로덕션 사이의 간극을 효과적으로 메우고 있습니다. UI는 Weave의 대화형 흐름에 비해 실용주의적이지만, 더 넓은 Databricks 생태계 및 OTel 표준과의 통합은 엔터프라이즈 엔지니어링 팀에게 강력한 '기본' 선택지가 되게 합니다.

**Weave 대비 강점**:
- 네이티브 OpenTelemetry 지원을 통해 Weave의 독자적인 포맷과 달리 기존 엔터프라이즈 APM 스택(Datadog 등)과 원활하게 통합됩니다.
- Databricks 사용 기업에서의 방대한 설치 기반과 '기본' 도구로서의 지위는 Weave와 같은 별도 도구 도입에 대한 높은 저항력을 형성합니다.
- 엔드 투 엔드 라이프사이클 관리(모델 레지스트리, 배포)는 Weave의 관측성 중심 접근보다 더 완전한 플랫폼을 제공합니다.
- 새로운 '지속적 온라인 모니터링' 기능은 피드백 루프를 효과적으로 자동화하며, 이는 Weave가 여전히 구축 중인 기능입니다.

**Weave 대비 약점**:
- UI/UX가 실용주의적이고 복잡합니다. 빠른 반복을 위한 Weave의 유연하고 대화형인 'Playground' 느낌이 부족합니다.
- 전체 스택을 셀프 호스팅하는 것은 Weave의 가벼운 통합에 비해 운영 부담이 큽니다.
- 기존의 실험 구조는 현대적인 에이전트 워크플로우의 그래프 기반, 비선형적 특성에 경직되어 있고 부적합하게 느껴질 수 있습니다.
- 탐색적 분석을 위해 Weave가 제공하는 깊이 있고 커뮤니티 중심적인 'Board' 시각화 유연성이 부족합니다.

**최근 업데이트**:
- 지속적 온라인 모니터링: 유입되는 프로덕션 트레이스에 대해 LLM Judge를 자동으로 실행하여 품질 문제를 실시간으로 감지 (2026-01-29)
- 에이전트 성능 대시보드: 에이전트 지연 시간, 요청 수, 도구 사용 요약을 모니터링하기 위한 사전 구축된 시각화 탭 (2026-01-29)
- Judge Builder UI: 배포 전 커스텀 LLM Judge를 생성, 테스트 및 검증할 수 있는 노코드 인터페이스 (2026-01-29)

### Arize Phoenix

**개요**: Arize Phoenix는 OpenInference 표준 및 OpenTelemetry를 기반으로 구축된 포괄적인 오픈 소스 우선 LLM 관측성 및 평가 플랫폼입니다. 전문화된 도구 사용 평가기를 통해 에이전트 워크플로우에서 탁월하며, 관리형 클라우드와 동기화하기 전에 완전한 오프라인 트레이싱 및 평가를 허용하는 독특한 '로컬 투 클라우드' 개발자 경험을 제공합니다.

**Weave 대비 강점**:
- OpenInference 표준: 벤더 중립적인 인스트루멘테이션은 특정 업체에 종속(lock-in)되는 것을 경계하는 엔지니어들에게 매력적입니다.
- 로컬 우선 워크플로우: 완전 기능의 로컬 UI를 통해 클라우드 의존성 없이 오프라인 개발 및 디버깅이 가능합니다.
- 에이전트 특화 지표: 새로운 '도구 선택(Tool Selection)' 및 '도구 호출(Tool Invocation)' 평가기는 일반적인 지표보다 에이전트 신뢰성에 대한 더 깊은 통찰력을 제공합니다.
- CLI 및 IDE 통합: 최근 CLI 업데이트를 통해 현대적인 AI 코딩 어시스턴트와 긴밀하게 통합되는 터미널 기반 워크플로우를 지원합니다.

**Weave 대비 약점**:
- 학습 통합: W&B가 제공하는 더 넓은 모델 학습/미세 조정 생태계와의 원활한 연결이 부족합니다.
- UI 정교함: 기능적이지만 비기술적 이해관계자들에게는 Weave에 비해 UI가 덜 정제되어 보일 수 있습니다.
- 설정 복잡성: OpenTelemetry/OpenInference에 대한 의존성은 '즉시 도입' 가능한 솔루션을 원하는 팀에게 더 가파른 학습 곡선을 제공할 수 있습니다.
- 엔터프라이즈 거버넌스: 감사 로깅 및 고급 컴플라이언스 기능이 Weave의 엔터프라이즈 오퍼링에 비해 덜 성숙해 보입니다.

**최근 업데이트**:
- Claude Opus 4.6 지원: 확장된 생각(thinking) 파라미터 지원 및 비용 추적 기능을 갖춘 Anthropic의 최신 모델 지원 추가 (2026-02-09)
- 도구 선택 및 호출 평가기: 에이전트가 올바른 도구를 선택하고 유효한 파라미터로 호출했는지 판단하는 전문 평가기 (2026-01-31)
- 프롬프트 및 데이터셋용 CLI: AI 코딩 어시스턴트에 최적화되어 터미널에서 직접 프롬프트, 데이터셋, 실험을 관리하는 새로운 CLI 명령 (2026-01-22)
- Span 링크를 통한 트레이스 투 데이터셋: 원본 소스 Span과의 양방향 링크를 유지하면서 프로덕션 트레이스에서 데이터셋을 생성하는 기능 (2026-01-21)
- 트레이스와 함께 주석 내보내기: 오프라인 분석을 위해 트레이스와 함께 인간 및 LLM 주석을 내보내는 CLI 지원 (2026-01-19)
