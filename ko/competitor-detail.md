# LLM Observability — 제품 상세 정보
**날짜**: 2026-02-13 | **모델**: google/gemini-3-pro-preview

### W&B Weave

**개요**: W&B Weave는 Weights & Biases 에코시스템에 깊이 통합된 개발자 중심의 Observability 및 Eval 툴킷으로, 엄격한 LLM 애플리케이션 구축을 위해 설계되었습니다. 코드 우선(code-first) Eval 워크플로우, 복잡한 에이전트 Tracing(네이티브 MCP 지원 포함), 엔터프라이즈급 인프라에 강점을 가지고 있으며, 오프라인 실험과 온라인 프로덕션 모니터링 사이의 간극을 메워줍니다.

**강점**:
- 전체 라이프사이클 관리를 위한 W&B 에코시스템(Experiments, Models)과의 깊은 통합.
- 네이티브 MCP 통합 및 복잡한 Trace 트리를 포함한 에이전트 패턴에 대한 강력한 지원.
- 코드 기반 Scoring 및 GUI 위저드를 모두 갖춘 포괄적인 Eval 기능.
- 다양한 배포 옵션(SaaS, VPC, Self-hosted) 및 컴플라이언스를 갖춘 엔터프라이즈 준비성.

**약점**:
- 비기술적 이해관계자를 위한 노코드 'Prompt CMS' 부재 (프로그래밍 방식 생성에 의존).
- 제로 코드 인터셉션을 위한 네이티브 Gateway/Proxy 모드 없음.
- 대규모 팀을 위한 고급 큐 관리 기능이 부족한 어노테이션 워크플로우.

**최근 업데이트**:
- Audio Monitors: 텍스트와 함께 오디오 출력을 관찰하고 판단하는 모니터를 생성하여 보이스 에이전트 Eval 가능. (2026-02-01)
- Dynamic Leaderboards: 수동 설정 대신 필터링 및 커스터마이징 기능이 포함된 Eval 결과 기반 자동 생성 리더보드. (2026-01-29)
- Custom LoRAs in Playground: Weave Playground에서 직접 커스텀 Fine-tuning된 LoRA 가중치를 테스트하고 Eval할 수 있도록 지원. (2026-01-16)

### LangSmith

**개요**: LangSmith는 LLM 애플리케이션을 위한 포괄적인 DevOps 플랫폼으로, 다른 프레임워크를 지원하면서도 LangChain 에코시스템의 사실상 표준 Observability 역할을 합니다. 복잡한 에이전트 워크플로우의 심층 Tracing, Eval 파이프라인, Self-hosting을 포함한 엔터프라이즈급 배포 옵션에 탁월합니다.

**강점**:
- LangChain 및 에이전트 워크플로우(중첩된 Trace, 스레드 재생)와의 깊은 네이티브 통합.
- 온라인/오프라인 Scoring 및 어노테이션 큐를 갖춘 견고한 Eval 프레임워크.
- 멀티 테넌트, 전용(Dedicated), Self-hosted Kubernetes를 포함한 유연한 엔터프라이즈 배포 옵션.
- 터미널 기반 'Fetch' 및 커스터마이징 가능한 Trace 뷰를 포함한 포괄적인 디버깅 도구.

**약점**:
- 실시간 방지를 위한 네이티브 액티브 Guardrails 부족 (패시브 로깅에 집중).
- 제로 코드 Gateway/Proxy 모드 없음; SDK 계측(instrumentation) 필요.
- 텍스트 기능에 비해 텍스트 이외의 멀티모달 Trace 시각화 지원이 제한적임.
- 소스 비공개 핵심 플랫폼으로 인해 인프라 자체에 대한 커뮤니티 기여가 제한됨.

**최근 업데이트**:
- Trace 미리보기 커스텀: LangSmith UI에서 Trace가 미리 보이는 방식을 커스터마이징하는 기능. (2026-02-06)
- Non-otel Google ADK 래퍼: OpenTelemetry 의존성 없는 Google ADK 통합을 위한 새로운 래퍼. (2026-02-02)
- Google Gen AI 래퍼: Google Generative AI 통합을 위해 내보내기 된 래퍼. (2026-01-31)
- Gemini TS 래퍼: Gemini 모델을 위한 베타 TypeScript 래퍼. (2026-01-26)
- LangSmith Self-Hosted v0.13: 플랫폼의 Self-hosted 버전 업데이트. (2026-01-16)

### Langfuse

**개요**: Langfuse는 견고한 Observability와 포괄적인 Eval 및 프롬프트 관리 기능을 결합한 오픈 소스 LLM 엔지니어링 플랫폼입니다. 복잡한 에이전트 워크플로우와 RAG 파이프라인 Tracing에 탁월하며, LangChain 및 LlamaIndex와 같은 프레임워크와 강력한 통합을 제공하는 동시에 Self-hosted 솔루션을 포함한 유연한 배포 옵션을 제공합니다.

**강점**:
- 전체 Self-hosting 기능이 포함된 오픈 소스 (MIT 라이선스).
- 인기 프레임워크(LangChain, LlamaIndex)와의 강력한 통합.
- 포괄적인 프롬프트 관리 및 버전 관리 시스템 (CMS).
- 견고한 LLM-as-a-Judge 및 온라인 Eval 기능.
- 상세한 비용 및 토큰 사용량 분석.

**약점**:
- 네이티브 멀티모달 Tracing 지원 부족.
- 내장된 PII 마스킹 또는 민감 데이터 탐지 기능 없음.
- 에이전트 그래프를 위한 시각적 디버깅 도구 제한 (DAG 뷰 없음).
- 프롬프트 출력에 대한 나란히 보기(side-by-side) 비교 뷰 없음.
- Fine-tuning 파이프라인 통합 누락.

**최근 업데이트**:
- LLM-as-a-Judge on Observations: 더 세밀한 품질 관리를 위해 특정 관찰 데이터에 대해 직접 LLM-as-a-judge Eval을 실행하는 기능 추가. (2026-02-13)
- Thinking/Reasoning Trace Rendering: DeepSeek와 같은 Chain-of-Thought 모델을 지원하기 위해 'thinking' 및 'reasoning' 부분에 대한 새로운 Trace 상세 렌더링. (2026-02-05)
- Inline Trace Comments: Trace 내의 IO 데이터 일부에 인라인 주석을 추가하여 협업 개선. (2026-01-25)
- Single Observation Evals: 전체 Trace뿐만 아니라 단일 관찰 데이터에 대해서도 Eval 실행 가능. (2026-02-08)

### Braintrust

**개요**: Braintrust는 Eval, 프롬프트 관리, Observability를 지속적 개선 루프에 긴밀하게 통합하는 엔터프라이즈급 AI 개발 플랫폼입니다. 회귀 테스트, 커스텀 Scoring, 데이터셋 관리에 탁월하여 엄격한 엔지니어링 표준에 집중하는 팀에 매우 효과적입니다. 강력한 Tracing 및 SDK 지원을 제공하지만, 현재는 실행 그래프나 세션 재생과 같은 시각적 디버깅 도구보다 정량적 Eval 및 리스트 기반 로깅을 우선시합니다.

**강점**:
- 프롬프트 엔지니어링, Eval, 로깅을 연결하는 통합 워크플로우.
- 견고한 회귀 테스트 및 실험 추적 기능.
- RBAC, SSO, 하이브리드 배포를 포함한 강력한 엔터프라이즈 보안 기능.
- 우수한 자동 계측 지원을 갖춘 고품질 SDK.

**약점**:
- 에이전트 실행 그래프와 같은 고급 시각적 디버깅 도구 부족.
- 멀티 턴 대화를 검토하기 위한 네이티브 세션 재생 UI 없음.
- 전문 보안 도구에 비해 제한적인 기본 Guardrails (PII, 탈옥).
- 완전한 데이터 주권을 위한 전체 Self-hosting 또는 오픈 소스 옵션 없음.

**최근 업데이트**:
- Thread Retrieval API: Python SDK에서 프로그래밍 방식으로 스레드를 가져오는 기능 추가. (2026-02-12)
- Sub-agent Nesting: Claude Agent SDK 래퍼에 서브 에이전트 중첩 지원 추가. (2026-02-12)
- Review Span Type: 휴먼 리뷰 워크플로우를 지원하기 위해 특정 'Review' Span 유형 도입. (2026-02-05)

- Trace Scoring Candidate: Python SDK 내에서 직접 Trace를 Scoring하는 새로운 기능. (2026-01-21)

### MLflow

**개요**: MLflow는 견고한 Tracing, Eval, 라이프사이클 관리 기능을 통해 GenAI Observability 분야로 성공적으로 확장한 성숙한 오픈 소스 MLOps 플랫폼입니다. 핵심 실험 추적, 커스텀 Eval(LLM-as-a-Judge), Autolog를 통한 광범위한 라이브러리 지원에는 뛰어나지만, 전문화된 에이전트 시각화 및 대화형 프롬프트 엔지니어링 워크플로우에서는 틈새 경쟁자들에 비해 뒤처집니다.

**강점**:
- 업계 표준 실험 추적 및 모델 레지스트리.
- UI 빌더를 갖춘 강력한 LLM-as-a-Judge 기능.
- LangChain과 같은 라이브러리에 대한 광범위한 자동 계측.
- 유연한 오픈 소스 및 Self-hosted 배포 옵션.
- 견고한 커스텀 지표 및 Dashboard 시스템.

**약점**:
- 전문화된 에이전트 시각화(예: 실행 그래프) 부족.
- 프롬프트 테스트를 위한 대화형 Playground 없음.
- 제한적인 네이티브 안전 Guardrails (PII, 환각).
- 실시간 Streaming Observability 부재.
- 휴먼 리뷰를 위한 내장된 어노테이션 큐 없음.

**최근 업데이트**:
- Organization Support: 서로 다른 워크스페이스 간에 실험과 리소스를 구성할 수 있는 멀티 워크스페이스 환경 지원. (2026-02-12)

| 핵심 Tracing & Logging | ●●● | Phoenix는 네이티브 OpenTelemetry 아키텍처를 통해 핵심 Tracing에서 탁월하며, 견고한 자동 계측 및 계층적 뷰를 통해 LLM 호출에 대한 심층적인 가시성을 제공합니다. 멀티모달 지원은 부족하지만, 텍스트 기반 Tracing 및 메타데이터 필터링은 엔지니어링 워크플로우에 있어 업계 최고 수준입니다. |
| 에이전트 & RAG 특화 | ●●○ | 특히 검색 및 도구 호출을 시각화하는 데 있어 에이전트 RAG 디버깅을 위한 강력한 도구를 제공합니다. 세션 재생 및 추론 검사를 잘 지원하지만, 복잡한 에이전트 그래프의 시각화 및 명시적인 실패 강조 기능은 더 발전할 여지가 있습니다. |
| Eval & 품질 | ●●● | Phoenix는 코드 중심 Eval의 강자로, 커스텀 Scoring, 데이터셋 관리, 회귀 테스트를 위한 강력한 도구를 제공합니다. 평가자를 위한 GUI 위저드와 오픈 소스 버전의 고급 어노테이션 워크플로우가 부족하여 비기술 사용자에게는 덜 적합합니다. |
| Guardrails & 안전 | ●●● | Guardrails AI와의 통합을 활용하여 PII, 환각, 탈옥에 대한 포괄적인 탐지를 제공하는 등 안전 기능이 견고합니다. Policy-as-code 접근 방식은 엔지니어링 워크플로우와 잘 맞지만 개발자의 구현이 필요합니다. |
| Analytics & Dashboard | ●●○ | 분석 기능은 토큰 사용량 및 커스텀 성능 지표와 같은 운영 지표에 집중되어 있으며 Dashboard 유연성이 뛰어납니다. 그러나 비용 할당과 같은 재무 운영(FinOps) 기능과 임베딩 또는 지연 시간 분포에 대한 고급 시각화는 부족합니다. |
| 개발 라이프사이클 | ●●○ | 개발의 실험 단계에서 강점을 보이며 엔지니어가 데이터로 프롬프트와 모델을 반복 개선할 수 있게 해줍니다. 하지만 Prompt CMS는 아니며 버전 제어, 롤백 또는 Fine-tuning 내보내기와 같은 라이프사이클 관리 기능은 부족합니다. |
| 통합 & DX | ●●○ | 심층적인 프레임워크 통합과 견고한 API를 갖추어 Python 및 TypeScript 사용자에게 개발자 경험이 훌륭합니다. Go SDK 및 Proxy 모드의 부재는 일부 인프라 설정에서의 유용성을 제한하지만, 코드 우선 환경에는 잘 맞습니다. |
| 엔터프라이즈 & 인프라 | ●●○ | 오픈 소스 Self-hosting부터 관리형 SaaS까지 뛰어난 배포 유연성을 제공하여 다양한 엔터프라이즈 요구 사항에 적응할 수 있습니다. Self-hosting을 통해 데이터 주권을 지원하지만, 상세한 감사 로그 및 자동화된 데이터 내보내기와 같은 일부 엔터프라이즈급 관리 기능은 부족합니다. |
