# W&B Weave — 제품 상세 정보
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview

[상세 비교](./comparison)

### Weave

**개요**: Weave는 LLM 애플리케이션 개발과 모델 학습 라이프사이클 사이의 간극을 독보적으로 메워주는 가볍고 파이썬다운(pythonic) 관측성(observability) 솔루션입니다. LangSmith의 프레임워크 중심 접근 방식이나 MLflow의 무거운 인프라와 달리, Weave는 광범위한 W&B 생태계를 활용하여 프로덕션 트레이스(trace)에서 파인튜닝 및 모델 반복 학습으로 이어지는 원활한 루프를 제공합니다.

**주요 강점**:
- 학습 이력(Lineage) 통합: Weave는 프로덕션 트레이스를 W&B 모델 아티팩트, 학습 실행(runs) 및 스윕(sweeps)과 네이티브하게 연결하여 진정한 데이터 플라이휠을 가능하게 하는 유일한 플랫폼입니다.
- 멀티모달 평가: 최근 출시된 Audio Monitors(2026년 2월)는 음성 에이전트 개발자들에게 LangSmith나 MLflow 같은 텍스트 중심 경쟁사 대비 뚜렷한 우위를 제공합니다.
- 대화형 디버깅: Weave의 Playground는 정적인 트레이스 보기 중심인 MLflow나 Arize Phoenix에 비해 빠른 반복 작업을 위한 우수한 '편집 후 실행(edit-and-run)' 경험을 제공합니다.
- 프레임워크 불가지론(Agnosticism): Weave는 LangSmith보다 가볍고 특정 프레임워크를 강요하지 않아, LangChain 생태계 외부에서 커스텀 스택을 구축하는 개발자들에게 매력적입니다.

**개선 필요 영역**:
- Human-in-the-Loop 워크플로우: LangSmith와 Langfuse는 대규모 인간 라벨링 팀을 관리하기 위한 훨씬 더 성숙한 '주석 큐(Annotation Queues)'를 제공합니다.
- 에이전트 상태 시각화: LangSmith는 LangGraph와의 깊은 통합을 통해 복잡한 상태 머신과 순환형 에이전트 워크플로우에 대한 우수한 시각화를 제공합니다.
- 트래픽 관리: Weave에는 Braintrust가 제공하는 속도 제한, 캐싱 및 트래픽 제어를 위한 활성 AI 프록시/게이트웨이 아키텍처가 부족합니다.
- OpenTelemetry 표준화: MLflow와 Arize Phoenix는 '네이티브 OTel' 방식을 채택하여, Weave의 SDK보다 개방형 표준을 우선시하는 기업들에게 더 안전한 선택지가 됩니다.

**최근 업데이트**:
- Audio Monitors: 텍스트와 함께 오디오 출력을 관찰하고 판단하는 모니터 생성 지원으로 음성 에이전트 평가 가능. (2026-02-01)
- 동적 리더보드: 영구적인 커스터마이징 및 CSV 내보내기 기능을 갖춘 평가 기반 자동 생성 리더보드. (2026-01-29)
- Playground 내 커스텀 LoRA: 비교를 위해 Weave Playground에서 직접 커스텀 파인튜닝된 LoRA 가중치를 로드하고 테스트하는 기능. (2026-01-16)

### LangSmith

**개요**: LangSmith는 LLM 애플리케이션을 위한 포괄적인 DevOps 플랫폼으로, LangChain 및 LangGraph 생태계와 깊이 통합된 최고 수준의 관측성, 평가 및 프롬프트 엔지니어링 도구를 제공합니다. 프레임워크에 구애받지 않는다고 주장하지만, 주요 강점은 복잡한 에이전트 워크플로우 처리, Human-in-the-loop 주석 큐, 그리고 최근 LangGraph Cloud를 통한 배포 확장성에 있습니다.

**Weave 대비 강점**:
- LangGraph와의 네이티브 통합으로 복잡한 에이전트 상태 머신에 대한 우수한 시각화 제공.
- 성숙한 '주석 큐' 워크플로우는 Human-in-the-loop 데이터 큐레이션 측면에서 Weave를 크게 앞섬.
- 통합된 프롬프트 허브(Prompt Hub)를 통해 플랫폼을 떠나지 않고도 프롬프트의 원활한 버전 관리 및 테스트 가능.
- 이제 배포(LangGraph Cloud)를 포함한 엔드투엔드 라이프사이클 관리가 가능해져 외부 인프라 도구의 필요성 감소.

**Weave 대비 약점**:
- Weave가 가볍고 파이썬답다고 여겨지는 반면, LangChain 프레임워크에 대한 종속성(lock-in)이 강하다는 인식이 있음.
- 모델 학습/파인튜닝 파이프라인과의 깊은 통합 부족 (W&B의 핵심 강점).
- 트레이스/사용자 수 기반의 가격 모델은 대규모 소비자 애플리케이션의 경우 Weave에 비해 비용 부담이 커질 수 있음.
- 에이전트 전용 기능의 밀도가 높아 단순한 사용 사례의 경우 UI가 복잡하게 느껴질 수 있음.

**최근 업데이트**:
- 트레이스 미리보기 커스텀: 빠른 디버깅을 위해 트레이스 목록 보기에서 표시할 필드 구성 기능. (2026-02-06)
- Google Gen AI 래퍼: OpenTelemetry 없이 Google의 생성형 AI 모델을 네이티브하게 트레이싱할 수 있는 새로운 SDK 래퍼. (2026-01-31)
- LangSmith 셀프 호스팅 v0.13: 성능 개선 및 새로운 구성 옵션이 포함된 업데이트된 셀프 호스팅 버전 출시. (2026-01-16)

| 개발자 경험 / 통합 | 대등 | LangSmith의 개발자 경험은 LangChain 사용자에게 탁월하지만, Weave의 가볍고 프레임워크에 구애받지 않는 방식은 순수 API나 커스텀 스택으로 구축하는 개발자들에게 더 매력적입니다. |
| 엔터프라이즈 및 보안 | 경쟁사 우위 | 두 플랫폼 모두 강력한 엔터프라이즈 기능을 제공하지만, LangSmith의 셀프 호스팅 옵션은 매우 성숙하며 규제 산업에서 널리 채택되고 있습니다. |

### Langfuse

**개요**: Langfuse는 관측성, 프롬프트 관리 및 평가를 통합된 개발자 경험으로 결합한 선도적인 오픈 소스 LLM 엔지니어링 플랫폼입니다. 최근 분석 엔진 강화를 위해 ClickHouse와 파트너십을 맺었으며(또는 인수됨), 에이전트 워크플로우 및 Human-in-the-loop 주석에 대한 깊은 지원을 바탕으로 SaaS 전용 솔루션에 대한 고성능 셀프 호스팅 대안으로 자리매김하고 있습니다.

**Weave 대비 강점**:
- 오픈 소스 및 셀프 호스팅: MIT 라이선스를 통해 벤더 종속 없이 온프레미스 배포가 가능하여 규제 산업에 적합합니다.
- 프롬프트 CMS: 코드와 독립적으로 프롬프트를 관리, 버전 관리 및 배포할 수 있는 비기술자용 전용 UI를 제공합니다.
- 주석 큐: 인간 라벨링 팀을 위한 구조화된 내장 워크플로우를 제공하며, 이는 임시 피드백 메커니즘보다 우수합니다.
- ClickHouse 백엔드: 대규모 트레이스 볼륨에 대한 고카디널리티(high-cardinality) 지표 및 실시간 분석에서 우수한 쿼리 성능을 제공합니다.

**Weave 대비 약점**:
- 학습 통합 부족: Weave가 W&B로부터 물려받은 모델 학습, 스윕 및 아티팩트 버전 관리와의 깊은 연결성이 없습니다.
- 객체 중심 로깅: 임의의 Python 객체/타입을 네이티브하게 로깅하고 시각화하는 Weave의 유연성에 비해 다소 제한적입니다.
- 데이터 과학 워크플로우: Weave가 Boards를 통해 가능하게 하는 탐색적 '데이터 과학' 워크플로우보다는 '엔지니어링/DevOps'(지연 시간, 비용, 트레이스)에 더 집중합니다.

**최근 업데이트**:
- 트레이스 출력 수정: 파인튜닝 데이터셋 구축을 위해 트레이스 뷰에서 직접 개선된 버전의 LLM 출력을 캡처. (2026-01-14)
- Python SDK v3.14.1: Langfuse 기능 접근을 위한 클라이언트 라이브러리 업데이트. (2026-02-09)

### Braintrust

**개요**: Braintrust는 강력한 '평가 우선(eval-first)' 워크플로우와 트래픽 관리를 위한 독점 AI 프록시로 차별화되는 엔터프라이즈급 AI 관측성 및 평가 플랫폼입니다. 6개 언어(Java, Go, C# 포함)에 대한 광범위한 SDK 지원을 제공하며 Cursor 및 VS Code와 같은 도구를 통해 개발자 루프에 깊이 통합되어, 단순한 추적 도구를 넘어 AI 개발을 위한 완전한 '운영 체제'를 지향합니다.

**Weave 대비 강점**:
- 더 넓은 SDK 생태계: Java, Go, Ruby 및 C#에 대한 네이티브 지원을 통해 Weave(Python/TS 중심)가 놓치는 엔터프라이즈 백엔드 팀을 확보합니다.
- AI 프록시 아키텍처: 내장된 게이트웨이는 Weave의 수동적 관측성 모델이 완전히 복제하지 못하는 활성 트래픽 관리(캐싱, 속도 제한)를 제공합니다.
- 통합 프롬프트 Playground: 데이터셋 및 실험과 긴밀하게 결합된 매우 성숙한 '평가 우선' 플레이그라운드를 제공하여 프롬프트 엔지니어에게 더 매끄러운 반복 루프를 제공합니다.

**Weave 대비 약점**:
- 학습 통합 부족: Braintrust는 모델 학습/파인튜닝 레이어와 분리되어 있는 반면, Weave는 전체 라이프사이클 ML Ops를 위해 거대한 W&B 생태계의 이점을 누립니다.
- 시각화 유연성: Braintrust에 차트용 'Loop'가 있지만, Weave의 커스텀 Boards와 표현 언어는 더 세밀하고 엔지니어 중심적인 데이터 탐색 기능을 제공합니다.
- 가격 모델: Braintrust의 사용자당 비용 + 사용량 기반 가격 책정은 W&B의 번들형 또는 볼륨 기반 엔터프라이즈 계약에 비해 진입 장벽이 될 수 있습니다.

**최근 업데이트**:

### MLflow

**개요**: MLflow는 3.x 시리즈를 통해 생성형 AI로 공격적으로 전환하며, 네이티브 OpenTelemetry 지원을 바탕으로 전통적인 MLOps 도구에서 포괄적인 LLM 라이프사이클 플랫폼으로 변모했습니다. 2026년 1월 출시된 MLflow 3.9를 통해 이제 고급 'Judge Builder' 워크플로우, 지속적인 온라인 평가 및 에이전트 전용 성능 대시보드를 제공하며, 방대한 기존 사용자 기반과 Databricks 통합을 통해 전문 관측성 도구들에 큰 위협이 되고 있습니다.

**Weave 대비 강점**:
- 네이티브 OpenTelemetry 지원으로 Weave의 SDK에 비해 벤더 종속 우려를 줄여줌.
- Databricks 생태계(Unity Catalog, Model Serving)와의 깊은 통합으로 데이터 팀에게 원활한 프로덕션 경로 제공.
- MemAlign 및 Judge Builder UI는 Weave에 현재 부족한 평가용 고급 '자동 최적화' 기능을 제공함.
- 방대한 오픈 소스 커뮤니티 및 플러그인 생태계 (30개 이상의 프레임워크).

**Weave 대비 약점**:
- UI/UX가 여전히 '실험 추적'에 뿌리를 두고 있어 Weave의 현대적이고 대화형인 'Playground' 중심 UI보다 무겁고 투박하게 느껴짐.
- 재생/디버그 루프가 덜 유연함. Weave는 트레이스에서 즉시 '편집 및 실행'이 가능한 반면, MLflow는 실행 기록을 불변의 레코드로 취급하는 경향이 있음.
- 전체 스택(Tracking Server + UI + Artifacts)을 셀프 호스팅하는 것이 Weave의 가벼운 시작보다 운영상 더 무거움.
- Weave의 전용 주석 기능에 비해 'Human-in-the-loop' 주석 워크플로우가 덜 전문화되어 있음.

**최근 업데이트**:
- MLflow Assistant: UI의 컨텍스트를 사용하여 문제를 진단하고 테스트를 설정하며 코드를 수정하는 Claude Code 기반 인프로덕트 챗봇. (2026-01-29)
- 에이전트 성능 대시보드: 설정 없이 지연 시간, 요청 수 및 품질 점수를 보여주는 생성형 AI 실험용 사전 구축 '개요' 탭. (2026-01-29)
- MemAlign Judge Optimizer: 과거 피드백으로부터 평가 가이드라인을 학습하여 LLM 판정(judge) 정확도를 자동으로 개선하는 알고리즘. (2026-01-29)
- Judge Builder UI: 코드 작성 없이 커스텀 LLM 판정기를 생성, 테스트 및 검증할 수 있는 시각적 인터페이스. (2026-01-29)
- 지속적인 온라인 모니터링: 유입되는 프로덕션 트레이스에 대해 LLM 판정기를 자동으로 실행하여 실시간으로 품질 문제 감지. (2026-01-29)

### Arize Phoenix

**개요**: Arize Phoenix는 OpenTelemetry 기반의 OpenInference 표준 위에 구축된 강력한 오픈 소스 LLM 관측성 및 평가 플랫폼입니다. 강력한 로컬 디버깅 도구, 노트북 통합 및 성숙한 평가 라이브러리를 통해 엔지니어링 '이너 루프(inner loop)'에서 탁월하며, 프로덕션 모니터링을 위해 Arize 엔터프라이즈 플랫폼으로의 원활한 업그레이드 경로를 제공합니다.

**Weave 대비 강점**:
- OpenTelemetry 네이티브: OpenInference를 기반으로 구축되어 벤더 중립적인 인스트루멘테이션 표준을 원하는 팀에게 매력적입니다.
- 로컬 우선 디버깅: 클라우드 동기화 없이도 빠른 반복 작업을 수행할 수 있는 뛰어난 노트북 통합 및 로컬 UI를 제공합니다.
- 특화된 에이전트 평가: RAG(검색) 및 에이전트(도구 선택/호출)를 위한 성숙한 전용 평가기 라이브러리를 보유하고 있습니다.

**Weave 대비 약점**:
- 광범위한 ML 생태계: W&B와 같은 전체 학습/모델 레지스트리 플랫폼과의 네이티브 통합이 부족합니다.
- 설정 복잡성: OTLP 인스트루멘테이션은 Weave의 '한 줄' 자동 인스트루멘테이션보다 구성이 더 장황할 수 있습니다.
- UI 세련미: 인터페이스는 기능적이고 실용적이지만, Weave의 디자인에 비해 덜 현대적이거나 유연하다는 평가를 받기도 합니다.

**최근 업데이트**:
- Claude Opus 4.6 지원: 확장된 사고(thinking) 파라미터 지원과 함께 Playground에서 Anthropic의 Claude Opus 4.6 모델 지원 추가. (2026-02-09)
- FaithfulnessEvaluator: 지원 중단된 HallucinationEvaluator를 대체하여 성실성(faithfulness)을 측정하는 새로운 평가기. (2026-02-02)
- 도구 선택 및 호출 평가기: 에이전트가 올바른 도구를 선택하고 유효한 파라미터로 호출했는지 평가하는 특화된 평가기. (2026-01-31)
- 프롬프트 및 데이터셋용 CLI: 터미널에서 프롬프트, 데이터셋 및 실험을 관리할 수 있는 포괄적인 CLI 명령. (2026-01-22)
- 스팬(Span) 연관을 통한 트레이스-데이터셋 변환: 소스 스팬에 대한 양방향 링크를 유지하면서 프로덕션 트레이스에서 데이터셋을 생성하는 기능. (2026-01-21)
