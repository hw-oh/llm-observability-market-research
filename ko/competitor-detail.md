# W&B Weave — 제품 상세 정보
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview

[상세 비교](./comparison)

### Weave

**개요**: Weave는 W&B의 유산을 활용하여 모델 개발(LoRA, 미세 조정)과 프로덕션 모니터링을 원활하게 연결하는 '학습 인지형(training-aware)' 관측성 플랫폼으로 자리매김하고 있습니다. 최근 멀티모달 지원(Audio Monitors) 및 프로그래밍 가능한 보고 기능(Dynamic Leaderboards)으로의 전환은 고급 AI 팀을 타겟으로 하며, Langfuse와 같은 순수 운영 도구나 Braintrust와 같은 프록시 기반 솔루션과 차별화됩니다.

**주요 강점**:
- 멀티모달 평가: Audio Monitors(MP3/WAV)를 기본 지원하여 음성 에이전트를 평가할 수 있으며, 이는 현재 대부분의 경쟁사에는 없는 기능입니다.
- 학습-추론 계보: W&B Artifacts의 커스텀 LoRA를 Weave Playground에 원활하게 통합하여 즉각적인 테스트가 가능합니다.
- 프로그래밍 가능한 보고: Dynamic Leaderboards는 자동 업데이트 및 고도의 커스터마이징 뷰를 제공하여 Langfuse나 Braintrust의 정적 대시보드보다 뛰어난 성능을 보여줍니다.
- 데이터 탐색: 'Boards'는 MLflow의 경직된 대시보드보다 강력하고 유연한 애드혹(ad-hoc) 분석 캔버스를 제공합니다.

**개선 필요 영역**:
- 시각적 에이전트 디버깅: LangSmith의 LangGraph Studio나 Langfuse의 Agent Graphs에 필적하는 에이전트 토폴로지 전용 대화형 그래프 뷰가 부족합니다.
- 구조화된 어노테이션 워크플로우: 대규모 휴먼 라벨링 팀을 관리하기 위한 전용 '어노테이션 큐(Annotation Queue)' 기능이 없으며, 이는 LangSmith와 Braintrust의 핵심 강점입니다.
- SDK 생태계: Python/TypeScript로 제한되어 있는 반면, Braintrust와 MLflow는 광범위한 기업 도입을 위해 Java, Go, C#을 지원합니다.
- 평가자(Judge) 최적화: 평가자 프롬프트를 자동으로 개선하는 MLflow의 MemAlign과 같은 자동화된 'Judge Optimizer' 기능이 부족합니다.

**최근 업데이트**:
- Audio Monitors: 텍스트와 함께 오디오 입력/출력(MP3/WAV)을 관찰하고 판별하는 모니터 생성 지원. (2026-02-01)
- Dynamic Leaderboards: 새로운 실행 결과가 도착하면 즉시 업데이트되는 자동 생성 및 커스터마이징 가능 평가 리더보드. (2026-01-29)
- Playground 내 커스텀 LoRA: W&B Artifacts에서 미세 조정된 LoRA 가중치를 Weave Playground로 직접 로드하여 추론하는 기능. (2026-01-16)

### LangSmith

**개요**: LangSmith는 LangChain 생태계를 위한 네이티브 관측성 및 평가 플랫폼으로, 에이전트 워크플로우를 위한 LangGraph와의 깊은 통합을 제공합니다. 성숙한 어노테이션 큐, 프롬프트 엔지니어링 허브, 디버깅을 위한 시각적 플레이그라운드를 갖춘 '휴먼 인 더 루프(human-in-the-loop)' 개발에 탁월하지만, LangChain 프레임워크 내에서 사용할 때 가장 강력합니다.

**Weave 대비 강점**:
- 어노테이션 큐: Weave에는 없는 휴먼 리뷰 및 데이터 라벨링을 위한 전용의 성숙한 워크플로우.
- LangGraph Studio: 복잡한 에이전트에 대해 우수한 이해도를 제공하는 에이전트 그래프용 시각적 대화형 디버거.
- 프롬프트 허브: 프롬프트 버전 관리 및 공유를 위해 널리 채택된 통합 저장소.
- 플레이그라운드 통합: 대화형 플레이그라운드 환경에서 모든 트레이스(trace)를 '재생'할 수 있는 원활한 기능.

**Weave 대비 약점**:
- 프레임워크 편향: LangChain에 과도하게 최적화되어 있어, 순수 OpenAI/Anthropic SDK 사용자에게는 무겁거나 덜 직관적으로 느껴질 수 있습니다.
- 모델 레지스트리: Weave가 W&B로부터 상속받은 깊이 있는 모델 아티팩트 관리 및 계보 추적 기능이 부족합니다.
- 복잡성: LangChain 추상화 계층을 사용하지 않는 개발자에게는 UI와 개념이 압도적일 수 있습니다.
- 데이터 탐색: Weave의 'Boards'는 LangSmith의 다소 경직된 대시보드에 비해 더 유연한 애드혹 데이터 탐색을 제공합니다.

**최근 업데이트**:
- 클라이언트 라이브러리 v0.7.1: 안정성 향상 및 OIDC 지원을 위한 Python/JS 클라이언트 라이브러리 업데이트. (2026-02-10)
- 트레이스 미리보기 커스터마이징: 트레이스 리스트 뷰에서 표시할 열과 데이터를 사용자가 직접 설정할 수 있는 UI 업데이트. (2026-02-06)
- 셀프 호스팅 v0.13: 엔터프라이즈 배포를 위한 셀프 호스팅 인프라 구성 요소의 새 버전. (2026-01-16)

### Langfuse

**개요**: Langfuse는 관측성, 프롬프트 관리, 평가를 단일 제품군으로 결합한 오픈 소스 LLM 엔지니어링 플랫폼입니다. 견고한 셀프 호스팅 기능(Docker/Kubernetes)과 대규모 프로덕션 분석을 위해 설계된 ClickHouse 기반 아키텍처로 차별화됩니다.

**Weave 대비 강점**:
- 오픈 소스 셀프 호스팅(MIT 라이선스)은 완전한 제어가 필요한 보안 중심 팀에 매력적입니다.
- 전용 프롬프트 관리 CMS를 통해 비기술 사용자도 프롬프트를 쉽게 버전 관리하고 배포할 수 있습니다.
- 어노테이션 큐는 휴먼 인 더 루프 리뷰를 위한 구조화된 내장 워크플로우를 제공합니다.
- ClickHouse 백엔드는 방대한 프로덕션 트레이스 볼륨에 대해 고성능 분석을 가능하게 합니다.

**Weave 대비 약점**:
- 모델 학습 및 미세 조정 워크플로우(W&B의 핵심 강점)와의 깊은 통합이 부족합니다.
- 분석이 대시보드 중심적이며, Weave Boards와 같은 프로그래밍 가능한 유연성이 부족합니다.
- 프롬프트 관리가 UI 우선 방식이어서, Weave의 객체 중심 방식에 비해 코드가 많은 엔지니어링 팀에게는 덜 유연할 수 있습니다.

**최근 업데이트**:
- 트레이스 결과 수정: 트레이스 뷰에서 직접 LLM 출력의 개선된 버전을 캡처하여 미세 조정 데이터셋 구축. (2026-01-14)

### Braintrust

**개요**: Braintrust는 관측성을 평가 및 프롬프트 엔지니어링과 긴밀하게 결합한 엔터프라이즈급 'AI 운영 체제'를 표방합니다. 아키텍처는 캐싱 및 키 관리를 위해 AI Proxy에 크게 의존하며, 이는 순수 SDK 기반 로거와 차별화되는 점입니다. 2026년 현재 언어 지원(Go, Java, Ruby, C#)을 공격적으로 확장했으며 트레이스 레벨 스코어러와 자동화된 피드백 루프를 통해 에이전트 평가 기능을 심화했습니다.

**Weave 대비 강점**:
- 광범위한 SDK 생태계: Java, Go, Ruby, C# 기본 지원 (Weave의 Python/JS 중심과 대비).
- AI Proxy 아키텍처: 캐싱, 속도 제한, 키 관리를 위한 내장 게이트웨이로 인프라 단순화.
- Loop AI 어시스턴트: 'Loop' 기능을 통해 트레이스 및 데이터를 자연어로 쿼리할 수 있는 독특한 UX 장점 제공.
- 에이전트 평가: 고급 '트레이스 레벨 스코어러(Trace-level scorers)'를 통해 다단계 에이전트 동작에 대한 정교한 채점 가능.

**Weave 대비 약점**:
- 미세 조정 통합: W&B의 모델 학습 및 미세 조정 계보와의 깊은 연결이 부족합니다.
- 설정 복잡성: 프록시 기반 아키텍처는 Weave의 가벼운 SDK 전용 방식보다 배포가 무겁습니다.
- 시각화 유연성: BTQL/SQL을 통한 커스텀 차트 작성은 강력하지만 Weave의 드래그 앤 드롭 보드 탐색보다는 덜 직관적입니다.

**최근 업데이트**:
- 트레이스 레벨 스코어러: 커스텀 코드 스코어러가 이제 전체 실행 트레이스에 접근하여 다단계 워크플로우와 에이전트 동작을 평가할 수 있음. (2026-02)
- 트레이스 기원 탐색: 프로덕션 트레이스에서 해당 트레이스를 생성한 프롬프트 버전이나 데이터셋 행으로 직접 이동. (2026-02)
- LangSmith 통합: 트레이스를 LangSmith와 Braintrust 양쪽으로 라우팅하거나 트래픽을 Braintrust로 마이그레이션하는 래퍼(wrapper). (2026-02)
- 자동 인스트루멘테이션(Python/Ruby/Go): Python, Ruby, Go SDK에서 주요 제공업체에 대한 코드 없는 트레이싱 지원. (2026-01)
- Temporal 통합: 내구성 있는 에이전트 실행을 위한 Temporal 워크플로우 및 액티비티 자동 트레이싱. (2026-01)

### MLflow

**개요**: MLflow는 포괄적인 '학습에서 프로덕션까지' 제품군을 통해 GenAI 분야로 공격적으로 확장한 성숙한 오픈 소스 MLOps 플랫폼입니다. 클래식 모델 관리에서의 지배력을 활용하여 견고한 트레이싱, 로우코드 Judge Builder, 네이티브 OpenTelemetry 지원을 제공하며 강력한 '올인원' 경쟁자로 부상했습니다.

**Weave 대비 강점**:
- 클래식 ML과 GenAI(학습 + 운영)를 위한 통합 플랫폼으로 도구 파편화 감소.
- 네이티브 OpenTelemetry 지원으로 벤더 종속성을 방지하고 통합 용이.
- 고급 'Judge Builder' 및 'MemAlign' 기능으로 평가 설정의 마찰 감소.
- Databricks를 통한 거대한 오픈 소스 커뮤니티 및 엔터프라이즈 지원.

**Weave 대비 약점**:
- Weave의 가벼운 SDK 방식에 비해 무거운 인프라 발자국(Tracking Server, DB).
- GenAI뿐만 아니라 모든 ML 모달리티를 지원하므로 UI가 복잡하고 어수선하게 느껴질 수 있음.
- 전체 기능 세트를 셀프 호스팅하려면 Weave의 SaaS 편의성에 비해 상당한 DevOps 노력이 필요함.

**최근 업데이트**:
- MLflow Assistant: 오류 디버깅 및 코드 생성을 돕는 AI 기반 인프로덕트 챗봇(Claude Code 기반). (2026-01-29)
- 에이전트 성능 대시보드: 지연 시간, 요청 수, 도구 사용 요약을 시각화하는 사전 구축된 'Overview' 탭. (2026-01-29)
- MemAlign Judge Optimizer: 피드백으로부터 학습하여 LLM 평가자 프롬프트를 자동으로 최적화하는 알고리즘. (2026-01-29)
- Judge Builder UI: 코드 없이 커스텀 LLM 평가자를 생성, 테스트 및 검증할 수 있는 시각적 인터페이스. (2026-01-29)
- 지속적 온라인 모니터링: 유입되는 프로덕션 트레이스에 대해 LLM 평가자를 자동으로 실행. (2026-01-29)

### Arize Phoenix

**개요**: Arize Phoenix는 OpenTelemetry(OpenInference) 기반으로 구축된 오픈 소스 우선 LLM 관측성 및 평가 플랫폼입니다. 강력한 로컬 디버깅 도구, 트레이스-투-데이터셋 워크플로우, 엄격한 LLM-as-a-judge 기능을 통해 엔지니어링의 '이너 루프(inner loop)'를 타겟으로 하며, 더 넓은 Arize AX 엔터프라이즈 플랫폼으로 가는 개발자 중심의 관문 역할을 합니다.

**Weave 대비 강점**:
- 네이티브 OpenTelemetry 기반(OpenInference)은 표준을 중시하는 엔지니어링 팀에 매력적입니다.
- 견고한 CLI와 쉬운 Docker 셀프 호스팅을 통한 우수한 로컬 개발 환경 제공.
- 에이전트 디버깅을 위해 특별히 설계된 더 세밀한 '도구 사용(Tool Usage)' 평가 지표.

**Weave 대비 약점**:
- 성숙한 모델 학습/미세 조정 생태계(W&B Training/Sweeps)와의 원활한 통합이 부족합니다.
- UI 경험이 오픈 소스 'Phoenix'와 엔터프라이즈 'Arize AX'로 나뉘어 있어 마찰이 발생할 수 있습니다.
- 비기술 이해관계자를 위한 협업 중심의 캔버스 스타일 구성('Boards')에 대한 강조가 적습니다.

**최근 업데이트**:
- Claude Opus 4.6 지원: 자동 비용 추적 및 'thinking' 파라미터 지원을 포함한 Anthropic의 최신 모델 지원 추가. (2026-02-09)
- 도구 선택 및 호출 평가자: 에이전트가 올바른 도구를 선택했는지, 정확한 파라미터로 호출했는지 판별하는 새로운 전문 평가자. (2026-01-31)
- 구성 가능한 OAuth2 이메일 추출: 비표준 IDP 클레임(예: Azure AD)에서 사용자 ID를 추출하기 위한 커스텀 JMESPath 표현식 지원. (2026-01-28)
- 프롬프트 및 데이터셋용 CLI: 터미널에서 직접 프롬프트, 데이터셋, 실험을 관리할 수 있는 포괄적인 CLI 명령어로 AI 어시스턴트 통합 가능. (2026-01-22)
- 스팬 링크를 포함한 트레이스-투-데이터셋: 계보 유지를 위해 소스 스팬으로의 양방향 링크를 유지하면서 트레이스에서 데이터셋을 생성하는 기능. (2026-01-21)
