# 주간 LLM Observability 시장 조사 보고서
**날짜**: 2026-02-12 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-12

## 1. 요약 (Executive Summary)

- LangSmith는 더 엄격한 지역별 컴플라이언스 요구 사항을 충족하기 위해 Self-Hosted v0.13을 출시하고 EU 데이터 레지던시 지원을 도입했습니다.
- Langfuse는 Organization Audit Log Viewer를 출시했으며, 거버넌스 중심 팀을 위한 완전한 self-hosting 지원을 지속하고 있습니다.
- MLflow는 Tracking Server 내에서 멀티 워크스페이스 관리 및 PII 마스킹을 가능하게 하는 Organization Support를 추가했습니다.
- Braintrust는 서브 에이전트 중첩(nesting) 지원 및 OpenAI 에이전트와의 깊은 연동을 통해 에이전트 Observability를 확장했습니다.
- Arize Phoenix는 Tool Selection 및 Faithfulness 메트릭을 위해 특별히 제작된 사전 빌드된 Eval 라이브러리를 도입했습니다.
- W&B Weave는 심층적인 중첩 함수 호출 Tracing과 LLM-as-a-judge Scoring을 위한 동적 리더보드를 통해 차별화를 지속하고 있습니다.
- 시장 전반이 에이전트 지원으로 크게 이동하고 있으며, Weave, LangSmith, Braintrust 모두 검색(retrieval) 단계와 Tool 호출을 명시적으로 추적하고 있습니다.

**AI 시장 인사이트**:

> LangSmith의 Self-Hosted v0.13 출시와 Langfuse의 새로운 Audit Log Viewer는 보안 및 자체 관리형 엔터프라이즈 배포 분야에서 Weave가 가졌던 경쟁 우위에 직접적인 도전이 됩니다.
> Arize Phoenix의 특화된 Tool Selection Eval 도구 추가는 현재 Weave가 심층 Tracing 시각화로 차별화하고 있는 에이전트 워크플로우 분석 유스케이스를 정조준하고 있습니다.
> Braintrust의 새로운 서브 에이전트 중첩 지원 및 OpenAI 연동은 복잡하고 계층적인 에이전트 아키텍처 디버깅을 위한 주요 도구로서의 Weave의 포지셔닝을 위협합니다.

## 3. 신규 기능 (최근 30일)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: 대화 품질 평가를 위해 LLM judge를 사용하여 오디오 출력(MP3/WAV)을 평가하는 기능을 지원합니다. (2026-02-01, Monitoring & Metrics)
- **Dynamic Leaderboards**: 영구적인 커스터마이징 및 CSV 내보내기 기능이 포함된 Eval 기반 자동 생성 리더보드입니다. (2026-01-29, Evaluation Integration)
- **Playground 내 커스텀 LoRA**: 추론 및 비교를 위해 Weave Playground에서 직접 커스텀 Fine-tuning된 LoRA 가중치를 사용할 수 있도록 지원합니다. (2026-01-16, Experiment / Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **Client SDK v0.7.1**: OIDC 수정 및 종속성 업데이트를 포함한 Python 및 JS 클라이언트 라이브러리 업데이트입니다. (2026-02-10, DevEx / Integration)
- **Tracing 미리보기 커스터마이징**: UI에서 Tracing 미리보기가 렌더링되는 방식을 커스터마이징할 수 있는 새로운 기능입니다. (2026-02-06, Core Observability)
- **LangSmith Self-Hosted v0.13**: 자체 호스팅 배포 옵션을 위한 신규 버전 릴리스입니다. (2026-01-16, Security & Governance)

### [Langfuse](https://langfuse.com/changelog)
- **단일 관찰(Observation) Eval**: 개별 관찰 데이터에 대해 Eval을 실행하는 기능이 추가되었습니다. (2026-02-12, Evaluation Integration)
- **이벤트 기반 Tracing 테이블**: 이벤트/관찰 데이터를 기반으로 한 새로운 Tracing UI 뷰입니다. (2026-02-12, Core Observability)
- **추론(Reasoning) Tracing 렌더링**: Tracing 상세 정보에서 사고/추론 과정(예: 추론 모델용)을 렌더링하는 기능을 지원합니다. (2026-02-12, Core Observability)
- **Org Audit Log Viewer**: 조직 수준의 감사 로그를 위한 새로운 뷰어입니다. (2026-02-12, Security & Governance)
- **인라인 Tracing 코멘트**: Tracing 내 IO 데이터의 특정 부분에 인라인으로 코멘트를 추가하는 기능입니다. (2026-02-12, DevEx / Integration)
- **Tracing 미리보기 내 수정 사항**: Tracing 및 관찰 미리보기에서 직접 수정 사항을 확인하고 관리하는 기능이 추가되었습니다. (2026-02-12, Experiment / Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **OpenAI 에이전트 연동**: OpenAI 에이전트 연동을 위해 모든 span 유형을 처리하도록 SDK를 업데이트했습니다. (2026-02-05, DevEx / Integration)
- **서브 에이전트 중첩(Nesting)**: Claude 에이전트 SDK 래퍼를 위한 서브 에이전트 중첩 지원을 추가했습니다. (2026-02-05, Agent / RAG Observability)
- **Review Span 유형**: Tracing에서 직접 휴먼 리뷰 워크플로우를 지원하기 위한 새로운 'review' span 유형을 도입했습니다. (2026-02-05, Evaluation Integration)
- **Classifications 필드**: SDK에 classifications 필드를 추가하여 구조화된 데이터 캡처를 강화했습니다. (2026-01-31, Core Observability)
- **Eval 캐시 제어**: Eval 도중 및 span 내보내기 후 캐싱을 끌 수 있는 옵션을 추가했습니다. (2026-01-29, Evaluation Integration)
- **Python Tracing Scoring**: Python SDK 내에서 Tracing Scoring을 위한 후보 지원을 도입했습니다. (2026-01-21, Evaluation Integration)
- **워크플로우 이름 변경**: 일반적인 실행 그래프를 더 잘 반영하기 위해 SDK에서 'agents'를 'workflows'로 변경했습니다. (2026-01-15, DevEx / Integration)

### [MLflow](https://mlflow.org/releases)
- **Organization Support**: MLflow Tracking Server에서 멀티 워크스페이스 환경을 지원하여, 서로 다른 워크스페이스 간에 실험과 리소스를 조직화할 수 있습니다. (2026-02-12, Security & Governance)
- **MLflow Assistant**: MLflow UI 내에서 직접 문제를 식별, 진단 및 수정할 수 있도록 돕는 Claude Code 기반의 인프로덕트 챗봇입니다. (2026-01-29, DevEx / Integration)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 지원**: Playground에 Claude Opus 4.6 모델을 추가했습니다. (2026-02-09, DevEx / Integration)
- **Tool Selection Evaluator**: 라이브러리에 누락되었던 tool_selection evaluator를 추가했습니다. (2026-02-06, Evaluation Integration)
- **Faithfulness Evaluator**: FaithfulnessEvaluator를 추가하고 HallucinationEvaluator를 지원 중단(deprecated)했습니다. (2026-02-02, Evaluation Integration)
- **Tool 호출 정확도 메트릭**: Tool 호출 정확도를 추적하기 위한 특정 메트릭을 추가했습니다. (2026-01-27, Monitoring & Metrics)
- **LLM 분류(Classification) Evaluator**: LLM 분류를 위한 새로운 빌트인 메트릭 생성을 위한 cursor rule을 추가했습니다. (2026-01-21, Evaluation Integration)

## 5. 엔터프라이즈 신호

- LangSmith는 배포 유연성을 높이기 위해 Self-Hosted 솔루션의 v0.13을 출시했습니다.
- Langfuse는 컴플라이언스 추적을 개선하기 위해 Organization Audit Log Viewer를 도입했습니다.
- MLflow는 Tracking Server에서 멀티 워크스페이스 관리를 가능하게 하는 Organization Support를 추가했습니다.
- Braintrust는 민감한 데이터를 고객 VPC 내에 유지하는 self-hosted 데이터 플레인을 통해 차별화를 지속하고 있습니다.
