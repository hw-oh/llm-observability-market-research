# 주간 LLM 관측성(Observability) 시장 조사 보고서
**날짜**: 2026-02-12 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-12

## 1. 요약 (Executive Summary)

- Weave는 2월 1일 Audio Monitors 출시를 통해 음성 에이전트 관측성 분야에서 선점 우위를 확보했으며, 멀티모달 평가 경쟁에서 Langfuse 및 Arize Phoenix와 같은 텍스트 중심 경쟁사들을 앞질렀습니다.
- Braintrust는 Java, Go, C# SDK를 추가하며 기존 엔터프라이즈 스택을 공격적으로 공략하고 있으며, 이는 Python/TS 이외의 환경에서 Weave에 대한 강력한 진입 장벽을 형성하고 있습니다.
- Weave의 '데이터 플라이휠(Data Flywheel)' 차별화가 강화되고 있습니다. Playground에서 Custom LoRA를 핫스왑(Hot-swap)할 수 있는 기능(1월 16일)은 네이티브 학습 레지스트리 없이는 MLflow나 LangSmith가 복제할 수 없는 실험 루프를 제공합니다.
- 거버넌스 압박이 거세지고 있습니다. MLflow의 '조직 지원(Organization Support)' 출시(2026년 2월)는 다중 팀 테넌시에 대한 새로운 기준을 제시했으며, Weave가 대규모 플랫폼 계약을 확보하기 위해 반드시 충족해야 할 요건이 되었습니다.
- Weave는 컨트롤 플레인(Control Plane) 측면에서 여전히 취약합니다. 캐싱 및 속도 제한을 위한 네이티브 AI Proxy의 부재로 인해, 운영 비용 제어가 주요 KPI인 아키텍처 결정에서 Braintrust가 승리하는 상황이 발생하고 있습니다.
- Dynamic Leaderboards(1월 29일)의 출시로 평가 보고서 작성 시의 수동 번거로움을 성공적으로 제거했으며, 이는 이전에 LangSmith의 랭킹 뷰가 가졌던 사용성 우위를 상쇄했습니다.
- 전략적 감시 사항: 엔지니어링 팀은 '자동화된 평가자 최적화(Automated Judge Optimization)' 기능을 우선순위에 두어야 합니다. MLflow의 새로운 'Judge Builder'는 평가 튜닝을 자동화하여, Weave의 수동 평가자 설정이 구식으로 보이게 만들 위험이 있습니다.

> **한 줄 평**: Weave는 멀티모달 및 학습 루프 관측성 분야에서 시장을 선도하고 있으나, Braintrust의 우수한 프록시 아키텍처와 광범위한 SDK 지원으로 인해 고부가가치 엔터프라이즈 플랫폼 계약을 놓칠 위험이 있습니다.

### Weave 주요 강점

- **멀티모달 관측성:** 최근 출시된 Audio Monitors(2026년 2월)는 Langfuse와 같은 텍스트 중심 경쟁사에 비해 음성 에이전트 평가에서 뚜렷한 우위를 제공합니다.
- **학습-추론 계보(Lineage):** Arize Phoenix나 LangSmith와 달리, Weave는 운영 트레이스를 W&B Registry의 학습 실행 및 아티팩트와 네이티브하게 연결하여 진정한 반복 개선 루프를 가능하게 합니다.
- **서버리스 LoRA 통합:** Playground에서 커스텀 미세 조정 어댑터를 핫스왑하는 기능(2026년 1월)은 MLflow나 Braintrust가 갖추지 못한 신속한 실험 역량을 제공합니다.

### Weave 개선 필요 사항

- **AI Proxy 부재:** Weave는 캐싱, 속도 제한, 키 관리를 위한 중간 프록시 레이어가 부족하며, 이는 현재 Braintrust가 주도하고 있는 핵심 가치 제안입니다.
- **휴먼 어노테이션 워크플로우:** LangSmith와 Braintrust는 수동 데이터 라벨링을 위한 전용 '어노테이션 큐'와 칸반 스타일 뷰를 제공합니다. Weave의 UI는 대규모 휴먼 리뷰에 덜 최적화되어 있습니다.
- **자동화된 평가자 최적화:** MLflow의 새로운 'MemAlign' 및 'Judge Builder' 기능은 LLM 평가자(Judge)의 튜닝을 자동화하는 반면, Weave는 평가 신뢰성을 위한 설정 과정이 더 수동적입니다.
- **엔터프라이즈 SDK 범위:** Braintrust의 Java, Go, Ruby, C# 지원은 Python/TypeScript 전용이 아닌 전통적인 엔터프라이즈 환경에서 Weave의 진입 장벽이 됩니다.

## 3. 신규 기능 (최근 30일)

### [Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: LLM 평가자를 사용하여 오디오 입출력을 관찰하고 판단하는 모니터 생성 지원, 음성 에이전트 평가 가능. (2026-02-01, 에이전트 관측성)
- **Dynamic Leaderboards**: 평가 결과로부터 자동 생성되는 리더보드. 영구적인 커스터마이징 및 CSV 내보내기 지원으로 수동 설정 제거. (2026-01-29, 평가 통합)
- **Playground 내 Custom LoRAs**: 사용자가 W&B Artifacts의 커스텀 LoRA 가중치를 Weave Playground에서 직접 로드하고 테스트할 수 있는 통합 기능. (2026-01-16, 실험 / 개선 루프)

### [LangSmith](https://changelog.langchain.com)
- **트레이스 미리보기 커스터마이징**: 트레이스 미리보기 테이블에 표시되는 열과 데이터를 사용자 정의하는 기능. (2026-02-06, 개발자 경험 / 통합)
- **Google Gen AI 래퍼**: Google Gen AI 모델의 트레이싱을 용이하게 하는 새로운 래퍼 내보내기. (2026-01-31, 개발자 경험 / 통합)
- **Self-Hosted v0.13**: 안정성이 향상된 업데이트된 셀프 호스팅 릴리스. (2026-01-16, 엔터프라이즈 및 보안)

### [Langfuse](https://langfuse.com/changelog)
- **트레이스 출력 수정(Corrected Outputs)**: 트레이스 뷰에서 직접 개선된 버전의 LLM 출력을 캡처하여 미세 조정 데이터셋 구축. (2026-01-14, 실험 / 개선 루프)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **LangSmith 통합**: LangSmith 트레이스를 Braintrust로 라우팅하는 실험적 래퍼로, 병행 사용 또는 마이그레이션 지원. (2026-02, 개발자 경험 / 통합)
- **트레이스 레벨 스코어러(Scorer)**: 커스텀 코드 스코어러가 전체 실행 트레이스에 접근하여 다단계 워크플로우를 평가할 수 있음. (2026-02, 평가 통합)
- **자동 인스트루멘테이션 (Py/Ruby/Go)**: Python, Ruby, Go 애플리케이션을 위한 코드 수정 없는 트레이싱 지원 추가. (2026-01, 개발자 경험 / 통합)
- **Temporal 통합**: 부모-자식 관계 매핑을 통한 Temporal 워크플로우 및 액티비티 자동 트레이싱. (2026-01, 에이전트 / RAG 관측성)
- **리뷰용 칸반 레이아웃**: 플래그가 지정된 스팬(span) 및 휴먼 리뷰 큐 관리를 위한 새로운 드래그 앤 드롭 인터페이스. (2026-01, 평가 통합)

### [MLflow](https://mlflow.org/releases)
- **조직 지원(Organization Support)**: 다중 워크스페이스 환경 및 조직 수준의 리소스 관리 지원. (2026-02-12, 엔터프라이즈 및 보안)
- **MLflow Assistant**: 트레이스 디버깅, 테스트 작성, 에이전트 최적화를 돕는 Claude Code 기반의 인제품 AI 챗봇. (2026-01-29, 개발자 경험 / 통합)
- **MemAlign 평가자 최적화 도구**: 피드백을 통해 학습하여 LLM 평가자 프롬프트와 기준을 자동으로 최적화하는 알고리즘. (2026-01-29, 평가 통합)
- **Judge Builder UI**: 커스텀 LLM 평가자를 생성, 테스트 및 검증하기 위한 노코드 시각적 인터페이스. (2026-01-29, 평가 통합)
- **에이전트 성능 대시보드**: 실험 UI 내 지연 시간, 요청 수, 품질 점수에 대한 사전 구축된 시각화. (2026-01-29, 모니터링 및 메트릭)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **데이터셋 평가자**: 데이터셋에 평가자를 직접 연결하여 실험 중에 서버 측에서 자동으로 실행. (2026-02-12, 평가 통합)
- **Playground용 커스텀 프로바이더**: Playground 내 커스텀 모델 프로바이더 및 라우팅을 위한 중앙 집중식 구성. (2026-02-11, 개발자 경험 / 통합)
- **도구 선택 및 호출 평가자**: 에이전트가 올바른 도구를 선택하고 정확한 파라미터로 호출했는지 판단하는 전문 평가자. (2026-01-31, 에이전트 / RAG 관측성)
- **프롬프트/데이터셋용 CLI 명령**: 프롬프트/데이터셋을 나열, 조회하고 다른 도구로 파이프 연결하는 터미널 명령. (2026-01-22, 개발자 경험 / 통합)
- **스팬 연관성을 포함한 트레이스-데이터셋 변환**: 원본 소스 스팬에 대한 양방향 링크를 유지하면서 트레이스에서 데이터셋 생성. (2026-01-21, 평가 통합)

## 5. 엔터프라이즈 신호

- MLflow의 '조직 지원' 출시(2026년 2월)는 다중 팀 거버넌스를 직접 겨냥하며 테넌시 관리의 기준을 높였습니다.
- Braintrust의 Java, Go, C# SDK 확장은 Python AI 사일로를 넘어 기존 엔터프라이즈 애플리케이션 스택을 확보하려는 전략적 움직임입니다.
- LangSmith의 'Self-Hosted v0.13'에 대한 지속적인 투자는 폐쇄망(Air-gapped)/VPC 배포가 고부가가치 엔터프라이즈 계약의 핵심 요구 사항임을 확인시켜 줍니다.
