# 주간 LLM Observability 시장 조사 보고서
**날짜**: 2026-02-12 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-12

## 1. 요약 (Executive Summary)

- MLflow는 서비스 간 컨텍스트 전파를 위한 분산 Tracing 기능과 Eval 설정을 간소화하는 시각적 Judge Builder UI를 출시했습니다.
- LangSmith는 플랫폼을 셀프 호스팅 버전 v0.13으로 업데이트하고 LangGraph와의 긴밀한 통합을 통해 에이전트 가시성을 강화했습니다.
- W&B Weave는 멀티모달 Eval을 위한 Audio Monitors와 비용 및 성능 지표를 시각화하는 Dynamic Leaderboards를 도입했습니다.
- Braintrust는 장기 실행 워크플로우 Tracing을 위해 Temporal과의 통합을 추가하고, Trace 내 미디어 첨부 파일 렌더링 기능을 활성화했습니다.
- Langfuse는 엔터프라이즈 거버넌스를 위한 Organization Audit Log Viewer를 출시하고, reasoning/thinking 단계에 대한 전용 렌더링 지원을 추가했습니다.
- Arize Phoenix는 자동화된 서버 측 실험을 위한 Dataset Evaluators와 Tool Selection 신뢰성 평가를 위한 전문 Evaluator를 출시했습니다.
- MLflow는 지연 시간과 도구 효율성을 모니터링하기 위한 전용 Agent Performance Dashboard를 배포했으며, Langfuse는 상세한 도구 호출 필터링 기능을 추가했습니다.

**AI 시장 인사이트**:

> MLflow의 시각적 Judge Builder와 Arize Phoenix의 Dataset Evaluators는 자동화된 지표 설정을 간소화함으로써 Weave의 Eval 차별화 요소를 위협하고 있습니다.
> LangSmith의 LangGraph 시각화와 Langfuse의 reasoning 단계 렌더링은 Weave의 에이전트 워크플로우 Tracing 및 도구 호출 분석 기능과 직접적인 경쟁 구도를 형성합니다.
> Langfuse의 Audit Log Viewer와 LangSmith의 셀프 호스팅 v0.13은 Weave의 현재 RBAC 및 배포 사양을 넘어서는 엄격한 엔터프라이즈 거버넌스로의 시장 변화를 강조합니다.

## 3. 신규 기능 (최근 30일)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: 오디오 지원 LLM judge를 사용하여 텍스트와 함께 오디오 출력(MP3/WAV)을 관찰하고 판정하는 모니터링 기능. (2026-02-01, Evaluation Integration)
- **Dynamic Leaderboards**: 영구적인 커스터마이징, 필터링 및 CSV 내보내기 기능이 포함된 Eval 결과 기반 자동 생성 리더보드. (2026-01-29, Evaluation Integration)
- **Playground 내 커스텀 LoRA**: W&B Artifacts의 커스텀 Fine-tuning LoRA 가중치를 Weave Playground에서 직접 추론 및 Eval에 사용 가능. (2026-01-16, DevEx / Integration)

### [LangSmith](https://changelog.langchain.com)
- **Trace 미리보기 커스터마이징**: UI에서 Trace 미리보기가 표시되는 방식을 사용자 정의하는 기능. (2026-02-06, Core Observability)
- **Google Gen AI Wrapper**: SDK에서 Google Gen AI를 위한 새로운 래퍼 지원. (2026-01-31, DevEx / Integration)
- **LangSmith Self-Hosted v0.13**: 업데이트된 셀프 호스팅 버전 출시. (2026-01-16, Enterprise & Security)

### [Langfuse](https://langfuse.com/changelog)
- **버전 관리된 데이터셋 기반 실험 실행**: 특정 타임스탬프의 데이터셋을 가져와 과거 버전에서 실험을 실행하여 재현성 확보. (2026-02-11, Experiment / Improvement Loop)
- **Single Observation Evals**: Trace 내의 개별 관찰 항목에 Eval을 추가하는 기능 지원. (2026-02-09, Evaluation Integration)
- **이벤트 기반 Trace 테이블**: 이벤트/관찰 항목을 기반으로 한 새로운 Trace 테이블 뷰. (2026-02-09, Core Observability)
- **Reasoning/Thinking Trace 렌더링**: Trace 상세 정보에서 사고(thinking) 및 추론(reasoning) 부분의 시각적 렌더링. (2026-02-05, Agent / RAG Observability)
- **Org Audit Log Viewer**: 조직 수준의 감사 로그를 확인하기 위한 UI. (2026-02-05, Enterprise & Security)
- **Corrected Outputs**: Fine-tuning 데이터셋 구축을 위해 Trace 뷰에서 직접 개선된 LLM 출력 버전을 캡처. (2026-01-14, Experiment / Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Trace 수준 Scorers**: 커스텀 코드 Scorer가 전체 실행 Trace에 접근하여 다단계 워크플로우를 평가할 수 있는 기능. (2026-02-01, Evaluation Integration)
- **LangSmith 통합**: LangSmith Trace를 Braintrust로 라우팅하는 실험적 래퍼. (2026-02-01, DevEx / Integration)
- **Cursor 통합**: Cursor가 로그를 쿼리하고 실험 결과를 가져올 수 있도록 하는 MCP 서버 통합. (2026-02-01, DevEx / Integration)
- **첨부 파일 렌더링**: 커스텀 Trace 뷰에서 서명된 URL의 이미지, 비디오, 오디오를 렌더링 가능. (2026-02-01, Core Observability)
- **Trace 기원 탐색**: Trace에서 해당 Trace를 생성한 프롬프트나 데이터셋 행으로 직접 이동. (2026-02-01, Evaluation Integration)
- **집계 기능이 포함된 단일 Span 필터**: 단일 Span 필터와 GROUP BY를 결합하여 집계된 Trace 분석 수행. (2026-02-01, Monitoring & Metrics)
- **자동 Instrumentation (Python, Ruby, Go)**: 주요 언어에 대한 코드 수정 없는 Tracing 지원. (2026-01-21, DevEx / Integration)
- **Temporal 통합**: Temporal 워크플로우 및 액티비티의 자동 Tracing. (2026-01-21, Agent / RAG Observability)
- **TrueFoundry 통합**: OpenTelemetry를 통해 TrueFoundry AI Gateway에서 LLM Trace 내보내기. (2026-01-21, DevEx / Integration)
- **리뷰용 칸반 레이아웃**: 플래그가 지정된 Span 및 리뷰 관리를 위한 드래그 앤 드롭 인터페이스. (2026-01-21, Evaluation Integration)

### [MLflow](https://mlflow.org/releases)
- **조직 지원 (Organization Support)**: 실험 및 리소스 정리를 위한 멀티 워크스페이스 환경 지원. (2026-02-12, Enterprise & Security)
- **MLflow Assistant**: 문제 진단 및 해결을 돕는 Claude Code 기반의 제품 내 챗봇. (2026-01-29, DevEx / Integration)
- **Agent Performance Dashboards**: 에이전트 지연 시간, 요청 수 및 품질 Scoring 모니터링을 위한 사전 구축된 차트. (2026-01-29, Monitoring & Metrics)
- **MemAlign Judge Optimizer**: 과거 피드백으로부터 Eval 가이드라인을 학습하여 Judge 정확도를 향상시키는 알고리즘. (2026-01-29, Evaluation Integration)
- **Judge Builder UI**: 커스텀 LLM judge 프롬프트를 생성, 테스트 및 내보낼 수 있는 시각적 인터페이스. (2026-01-29, Evaluation Integration)
- **지속적 온라인 모니터링**: 프로덕션 환경의 유입 Trace에 대해 LLM judge를 자동으로 실행. (2026-01-29, Monitoring & Metrics)
- **분산 Tracing**: 컨텍스트 전파를 통해 여러 서비스에 걸친 요청 추적. (2026-01-29, Core Observability)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **OpenAI Responses API 타입 지원**: Playground 및 커스텀 프로바이더에서 OpenAI API 타입(Chat Completions vs Responses) 선택 지원. (2026-02-12, DevEx / Integration)
- **Dataset Evaluators**: 데이터셋에 Evaluator를 직접 연결하여 실험 중 서버 측에서 자동으로 실행. (2026-02-12, Evaluation Integration)
- **Playground용 커스텀 프로바이더**: Playground와 프롬프트 전체에서 재사용 가능한 커스텀 모델 프로바이더 중앙 설정. (2026-02-11, DevEx / Integration)
- **Claude Opus 4.6 지원**: 확장된 thinking 파라미터를 포함한 Anthropic의 Claude Opus 4.6 모델 지원. (2026-02-09, Core Observability)
- **Tool Selection & Invocation Evaluators**: 에이전트의 도구 선택 정확도 및 파라미터 포맷팅을 평가하는 전문 Evaluator. (2026-01-31, Agent / RAG Observability)
- **프롬프트/데이터셋용 CLI 명령**: 터미널에서 프롬프트, 데이터셋 및 실험을 관리하기 위한 새로운 CLI 명령. (2026-01-22, DevEx / Integration)
- **Trace 기반 데이터셋 생성**: Span 연관성을 유지하면서 프로덕션 Trace를 데이터셋으로 변환. (2026-01-21, Evaluation Integration)
- **Trace와 함께 어노테이션 내보내기**: 오프라인 분석을 위해 Trace와 해당 어노테이션을 함께 내보내는 CLI 지원. (2026-01-19, DevEx / Integration)

| MLflow | 선도적인 오픈 소스 MLOps 플랫폼에서 포괄적인 GenAI 및 에이전트 Observability 제품군으로 완전히 확장. | 실험과 프로덕션 모니터링 사이의 간극을 메우는 AI 에이전트용 기본 '올인원' OS로 진화. | v3.9.0에서 분산 Tracing, MemAlign, 지속적 온라인 모니터링과 같은 전문 에이전트 기능 출시. |
| Arize Phoenix | 오프라인 실험과 프로덕션 모니터링을 연결하는 포괄적인 오픈 소스 우선 Observability 및 Eval 플랫폼. | CLI 도구 및 고급 에이전트 Eval 기능을 통해 개발자 워크플로우 통합을 심화하는 방향으로 이동. | 전문화된 Tool Evaluator, 광범위한 CLI 명령 및 Dataset Evaluators의 최근 출시는 에이전트 신뢰성과 개발자 효율성에 대한 집중을 나타냄. |

## 5. 엔터프라이즈 신호

- Langfuse는 Organization Audit Log Viewer를 도입하여 엔터프라이즈 팀을 위한 거버넌스를 강화했습니다.
- MLflow는 멀티 워크스페이스 환경 및 인증을 처리하기 위한 Organization Support를 출시했습니다.
- LangSmith는 셀프 호스팅 버전(v0.13)을 업데이트하여 온프레미스 엔터프라이즈 요구 사항에 대한 의지를 공고히 했습니다.
- Braintrust는 Temporal과의 통합을 추가하여 복잡한 엔터프라이즈급 워크플로우 오케스트레이션 지원 신호를 보냈습니다.
