# 주간 LLM Observability 시장 조사 보고서
**날짜**: 2026-02-12 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-12

## 1. 요약 (Executive Summary)

- LangSmith는 엄격한 엔터프라이즈 컴플라이언스 및 데이터 주권 요구 사항을 해결하기 위해 Self-Hosted v0.13을 출시하고 EU 데이터 거주(data residency) 지원을 도입했습니다.
- Langfuse는 보안 거버넌스를 위한 새로운 Organization Audit Log Viewer를 출시하고, Agent Graphs 내에서 reasoning 단계를 렌더링하는 기능을 추가했습니다.
- MLflow는 멀티 워크스페이스 관리를 가능하게 하는 Organization Support를 도입하고, tool calling 응답 및 retriever 쿼리를 캡처하도록 auto-instrumentation을 확장했습니다.
- Braintrust는 서브 에이전트 중첩을 명시적으로 지원하기 위해 플랫폼 용어를 'agents'에서 'workflows'로 업데이트했으며, Python trace Scoring 업데이트를 출시했습니다.
- Arize Phoenix는 오픈 소스 라이브러리에 특정 Faithfulness 및 Tool Selection Eval 도구를 추가하고 구성 가능한 OAuth2 이메일 추출 기능을 구현했습니다.
- W&B Weave는 Audio Monitors 및 Dynamic Leaderboards로 Eval 스위트를 확장하는 동시에, 안전한 배포를 위한 SDK 수준의 PII 비식별화 지원을 지속하고 있습니다.

**AI 시장 인사이트**:

> LangSmith의 Self-Hosted v0.13 출시 및 EU 데이터 거주 지원은 규제 산업에 대해 유사한 인프라 제어 권한을 제공함으로써 Weave의 엔터프라이즈 강점에 직접적으로 도전하고 있습니다.
> 시각적 워크플로우 토폴로지를 위한 Langfuse의 Agent Graphs 도입은 현재 에이전트 단계를 캡처하지만 전용 DAG 시각화 기능이 부족한 Weave의 특정 UI 격차를 부각시킵니다.
> Arize Phoenix의 전문화된 Faithfulness 및 Tool Selection 메트릭 추가는 Weave가 사전 구축된 에이전트 전용 Eval 도구 라이브러리를 확장해야 한다는 경쟁 압력을 가중시킵니다.

## 3. 신규 기능 (최근 30일)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: LLM judge를 사용하여 텍스트와 함께 오디오 출력(MP3/WAV)을 관찰하고 판정하는 모니터 생성 지원. (2026-02-01, Monitoring & Metrics)
- **Dynamic Leaderboards**: 필터링, 커스터마이징 및 CSV 내보내기 기능이 포함된 Eval 결과 기반 자동 생성 리더보드. (2026-01-29, Evaluation Integration)
- **Playground 내 커스텀 LoRA**: Weave Playground에서 직접 커스텀 Fine-tuning된 LoRA 가중치를 사용할 수 있는 Serverless LoRA Inference 기능. (2026-01-16, Experiment / Improvement Loop)

### [LangSmith](https://changelog.langchain.com)
- **Trace Preview 커스터마이징**: LangSmith UI에서 Tracing 결과가 미리 보이는 방식을 사용자가 직접 설정 가능. (2026-02-06, Core Observability)
- **Non-Otel Google Vertex AI (ADK) 래퍼**: Python SDK에 Google Vertex AI(ADK)를 위한 비 OpenTelemetry 래퍼 추가. (2026-02-02, DevEx / Integration)
- **LangSmith Self-Hosted v0.13**: 셀프 호스팅 플랫폼의 0.13 버전 출시. (2026-01-16, Enterprise & Security)

### [Langfuse](https://langfuse.com/changelog)
- **단일 Observation Eval**: 개별 observation에 대해 Eval을 실행하는 기능 추가. (2026-02-09, Evaluation Integration)
- **Reasoning/Thinking 렌더링**: Trace 상세 정보에서 사고(thinking/reasoning) 프로세스 부분을 렌더링하는 기능 추가 (예: reasoning 모델용). (2026-02-06, Core Observability)
- **Org Audit Log Viewer**: 조직 수준의 감사 로그를 확인하기 위한 새로운 UI. (2026-02-06, Enterprise & Security)
- **인라인 Trace 코멘트**: Trace 내 IO 데이터의 일부에 인라인으로 코멘트를 추가하는 기능. (2026-01-27, DevEx / Integration)
- **Trace Preview 내 수정 사항**: Trace 및 observation 미리보기에서 직접 수정 사항을 확인하고 관리하는 기능 추가. (2026-01-13, Experiment / Improvement Loop)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Claude Agent를 위한 서브 에이전트 중첩**: 계층적 Observability를 개선하기 위해 Claude Agent SDK 래퍼 내 서브 에이전트 중첩 지원 추가. (2026-02-05, Agent / RAG Observability)
- **Review Span 유형**: Trace 내 수동 또는 자동 리뷰 단계를 지원하기 위해 새로운 'review' span 유형 도입. (2026-02-05, Core Observability)
- **Classifications 필드**: 데이터 라벨링 및 분류 기능을 강화하기 위해 SDK에 classifications 필드 추가. (2026-01-31, Evaluation Integration)
- **Trace Scoring 후보**: 자동화된 Eval 로직을 정교화하기 위한 Python Trace Scoring 기능 업데이트. (2026-01-21, Evaluation Integration)
- **Workflows 이름 변경**: 광범위한 오케스트레이션 기능을 더 잘 반영하기 위해 SDK에서 'agents'를 'workflows'로 변경. (2026-01-15, DevEx / Integration)

### [MLflow](https://mlflow.org/releases)
- **MLflow Tracking Server 내 Organization 지원**: 여러 워크스페이스에 걸쳐 실험과 리소스를 관리할 수 있는 멀티 워크스페이스 환경 지원. (2026-02-12, Enterprise & Security)
- **MLflow Assistant**: MLflow UI 내에서 직접 문제를 식별, 진단 및 수정할 수 있도록 돕는 Claude Code 기반의 인프로덕트 챗봇. (2026-01-29, DevEx / Integration)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 지원**: Playground에 Claude Opus 4.6 모델 지원 추가. (2026-02-09, DevEx / Integration)
- **Tool Selection Eval 도구**: 라이브러리에 누락되었던 tool_selection Eval 도구 추가. (2026-02-06, Evaluation Integration)
- **Faithfulness Eval 도구**: FaithfulnessEvaluator를 추가하고 HallucinationEvaluator를 지원 중단(deprecated). (2026-02-02, Evaluation Integration)
- **Tool Invocation Accuracy 메트릭**: 도구 호출 정확도를 추적하기 위한 새로운 메트릭 추가. (2026-02-02, Monitoring & Metrics)
- **구성 가능한 OAuth2 이메일 추출**: OAuth2에서 구성 가능한 이메일 추출을 위한 EMAIL_ATTRIBUTE_PATH 추가. (2026-01-28, Enterprise & Security)
- **내장 메트릭을 위한 Cursor Rule**: 새로운 내장 메트릭(LLM 분류 Eval 도구) 생성을 위한 cursor rule 추가. (2026-01-21, Evaluation Integration)

## 5. 엔터프라이즈 신호

- LangSmith는 Self-Hosted v0.13을 출시하여 엔터프라이즈 인프라 제어에 대한 의지를 강화했습니다.
- MLflow는 대규모 팀을 위한 멀티 워크스페이스 관리를 가능하게 하는 Organization Support를 도입했습니다.
- Langfuse는 보안 및 컴플라이언스 가시성을 높이기 위해 새로운 Organization Audit Log Viewer를 출시했습니다.
- Braintrust는 데이터가 고객 VPC 내에 머물도록 보장하는 셀프 호스팅 데이터 플레인 아키텍처로 차별화를 지속하고 있습니다.
- Arize Phoenix는 구성 가능한 OAuth2 이메일 추출 기능을 추가하여 엔터프라이즈 ID 제공업체와의 통합을 개선했습니다.
