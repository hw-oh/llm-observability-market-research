# W&B Weave — 주간 경쟁사 인텔리전스 보고서
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-11

[← 홈](../) · [상세 비교](../comparison) · [제품 상세](../competitor-detail)

## 1. 요약 (Executive Summary)

- LangSmith는 'LangGraph Cloud'를 통해 수동적 관측(Observability)에서 능동적 런타임 관리로 공격적으로 이동하고 있으며, Weave가 데이터를 캡처하기 전에 전체 에이전트 라이프사이클을 선점하려 위협하고 있습니다.
- Braintrust는 IDE(Cursor)에 직접 통합하고 Weave가 현재 소홀히 하고 있는 백엔드 언어(Java, Go, C#)를 지원함으로써 '엔터프라이즈 엔지니어링' 측면에서 성공적으로 측면 공격을 수행하고 있습니다.
- Humanloop의 인수 및 서비스 종료는 이탈 고객을 확보할 수 있는 즉각적인 전술적 기회를 창출하지만, Langfuse의 우수한 Prompt CMS가 이 마이그레이션의 강력한 경쟁자로 부상하고 있습니다.
- Arize Phoenix와 Langfuse는 강력한 '로컬 우선(Local-First)' 및 셀프 호스팅 스토리를 통해 상향식(Bottom-up) 도입 퍼널을 압박하며, SaaS 도입을 주저하는 보안 중심 엔지니어들에게 어필하고 있습니다.
- Logfire는 Pydantic과의 깊은 통합을 통해 '풀스택' 트레이스(데이터베이스 + API + LLM)를 소유할 수 있게 되었으며, 이는 LLM 이외의 인프라 구성 요소에 대한 Weave의 가시성 부족을 드러냅니다.
- 에이전트 관측성(Agentic observability)이 주요 격전지가 되고 있습니다. LangSmith와 Langfuse는 순환 워크플로우(Cyclic workflows)에 대해 현재 Weave의 표준 트레이스 뷰보다 뛰어난 전용 그래프/추론 시각화 기능을 출시했습니다.

> **한 줄 평**: Weave는 모델 학습에서 프로덕션에 이르는 리니지(Lineage)에서 해자를 유지하고 있으나, 에이전트 오케스트레이션에서는 LangSmith로부터, 엔터프라이즈 개발자 워크플로우에서는 Braintrust로부터 극심한 압박을 받고 있습니다.

### Weave 주요 강점

- W&B Model Registry 및 학습 파이프라인과의 독점적 통합으로 경쟁사가 복제할 수 없는 고유한 '데이터 플라이휠'을 생성합니다.
- Helicone이나 Logfire의 경직된 UI 기반 워크플로우에 비해 프로그래밍 방식 평가(Evaluation)에서 우수한 유연성을 제공합니다.
- Arize Phoenix 및 Langfuse의 실용주의적 인터페이스를 능가하는, 다중 이해관계자 리뷰를 위한 세련되고 협업적인 UI를 갖추고 있습니다.

### Weave 개선 필요 사항

- Langfuse 및 기존 Humanloop 서비스와 비교했을 때 성숙한 비기술자용 'Prompt CMS' 및 Playground가 부족합니다.
- 제한된 SDK 생태계(Python/JS 전용)로 인해 엔터프라이즈 백엔드 팀(Java/Go/C#)이 Braintrust로 이탈할 위험이 있습니다.
- 심층 인프라(데이터베이스 쿼리, API 내부)를 추적할 수 없는 점은 Logfire 및 Arize와 같은 OTel 네이티브 도구에 비해 열세에 놓이게 합니다.

## 3. 이번 주 신규 기능

### LangSmith
- **트레이스 미리보기 사용자 정의**: 사용자가 트레이싱 테이블에 직접 표시될 입력 및 출력 부분을 구성할 수 있게 되어, 커스텀 데이터 구조에 대한 사용성이 개선되었습니다. (2026-02-06, DevEx / 통합)
- **LangSmith 셀프 호스팅 v0.13**: 'Insights' 및 성능 개선을 포함하여 클라우드 버전과 기능 패리티(Parity)를 확장한 셀프 호스팅 고객용 주요 릴리스입니다. (2026-01-16, 엔터프라이즈 & 보안)
- **SDK v0.7.1**: 최신 LangChain 버전과의 호환성 보장 및 안정성이 향상된 Python/JS 클라이언트 라이브러리 업데이트입니다. (2026-02-10, DevEx / 통합)

### Arize Phoenix
- **Claude Opus 4.6 지원**: 나란히 비교하기(Side-by-side comparison)를 위해 Playground에 Claude Opus 4.6 지원이 추가되었습니다. (2026-02-09, 실험 / 개선 루프)
- **도구 선택 평가기 (Tool Selection Evaluator)**: 에이전트의 도구 선택 로직 정확도를 측정하기 위해 특별히 설계된 새로운 평가기입니다. (2026-02-06, 에이전트 / RAG 관측성)
- **FaithfulnessEvaluator**: 더 미묘한 RAG 검사를 위해 새로운 FaithfulnessEvaluator를 도입했습니다 (HallucinationEvaluator는 지원 중단 예정). (2026-02-02, 평가 통합)
- **도구 호출 정확도 지표**: 에이전트가 도구를 얼마나 정확하게 호출하는지 추적하기 위한 특정 지표가 추가되었습니다. (2026-01-27, 모니터링 & 지표)

### Braintrust
- **트레이스 기원 탐색**: 트레이스를 생성한 특정 프롬프트 버전이나 데이터셋 행으로 다시 연결하는 기능입니다. (2026-02, 핵심 관측성)
- **트레이스 레벨 스코어러 (Scorers)**: 커스텀 스코어러가 전체 실행 트레이스에 접근하여 다단계 워크플로우와 에이전트 동작을 평가할 수 있습니다. (2026-02, 평가 통합)
- **LangSmith 통합**: LangSmith 트레이싱 및 평가 호출을 Braintrust로 병렬 또는 독점적으로 라우팅하는 래퍼(Wrapper)입니다. (2026-02, DevEx / 통합)
- **Cursor 통합**: 자연어를 통해 로그를 쿼리하고 실험 결과를 가져올 수 있는 Cursor 에디터용 Braintrust 확장 프로그램입니다. (2026-02, DevEx / 통합)
- **자동 인스트루멘테이션 (Python/Ruby/Go)**: 주요 언어에 대한 제로 코드 트레이싱 지원으로 초기 설정을 간소화했습니다. (2026-01, DevEx / 통합)

### Langfuse
- **트레이스 출력 수정**: 파인튜닝 데이터셋 구축을 위해 트레이스 뷰에서 직접 개선된 버전의 LLM 출력을 캡처합니다. (2026-01-14, 실험 / 개선 루프)
- **관측 I/O 인라인 주석**: 트레이스 입력 및 출력 내의 특정 텍스트 선택 영역에 주석을 고정할 수 있습니다. (2026-01-07, DevEx / 통합)

### Logfire
- **멀티 토큰 프로젝트 마이그레이션**: 여러 토큰을 사용하는 프로젝트 마이그레이션 지원이 추가되었습니다. (2026-02-04, 엔터프라이즈 & 보안)
- **OTel Gen AI 시맨틱 컨벤션**: OpenTelemetry Gen AI 시맨틱 컨벤션 스칼라 속성에 대한 지원이 추가되었습니다. (2026-01-28, 핵심 관측성)
- **Pytest 통합**: pytest 실행 트레이싱을 위한 네이티브 통합입니다. (2026-01-26, DevEx / 통합)
- **DSPy 통합**: DSPy 프레임워크에 대한 인스트루멘테이션 지원이 추가되었습니다. (2026-01-16, DevEx / 통합)
- **Claude SDK 인스트루멘테이션**: Anthropic Claude SDK를 위한 특정 인스트루멘테이션이 추가되었습니다. (2026-01-12, DevEx / 통합)

## 5. 엔터프라이즈 신호

- LangSmith 셀프 호스팅 v0.13이 클라우드와 기능 패리티를 달성함에 따라 규제 산업의 주요 거부 사유가 제거되었습니다.
- Braintrust의 Java 및 C# SDK 출시는 전통적인 엔터프라이즈 금융/보험 스택에 대한 직접적인 공격 신호입니다.
- Langfuse의 ClickHouse 채택 및 감사 로그(Audit Logs)는 고용량 엔터프라이즈 계정을 타겟팅하기 위해 상위 시장으로 이동하고 있음을 나타냅니다.

## 6. 감시 목록 (Watchlist)

- LangSmith의 'Deployment' 확장 및 LangGraph Cloud를 통한 능동적 런타임 관리.
- Braintrust가 CI/CD 단계에 도달하기 전의 개발자를 선점하기 위해 IDE(Cursor/MCP)와의 통합을 심화하는 움직임.
- 서비스가 종료된 Humanloop 플랫폼 고객의 마이그레이션 패턴 — 특히 그들이 Weave의 평가 깊이와 Langfuse의 프롬프트 관리 중 무엇을 우선시하는지 여부.
