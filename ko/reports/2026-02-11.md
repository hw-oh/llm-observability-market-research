# W&B Weave — 주간 경쟁사 인텔리전스 보고서
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-11

[상세 비교](../comparison) · [제품 세부 정보](../competitor-detail)

## 1. 요약 보고 (Executive Summary)

- Weave는 2월 1일 Audio Monitors 출시를 통해 멀티모달 관측성(observability) 분야에서 선점 효과를 확보했으며, 급성장하는 음성 에이전트 섹터에서 LangSmith 및 MLflow와 같은 텍스트 중심 경쟁사들을 앞서 나갔습니다.
- LangSmith는 LangGraph Cloud를 통해 단순 관측성 도구에서 인프라 종속(lock-in) 모델로 공격적으로 전환하고 있으며, 트레이스 계층이 아닌 배포 계층을 점유함으로써 Weave를 대체하려 위협하고 있습니다.
- MLflow 3.9의 'Judge Builder' 및 'MemAlign' 출시는 우리의 평가 워크플로우를 직접적으로 범용화(commoditize)하고 있으며, Weave가 우선시하는 수동 검사 도구에 대한 의존도를 낮추는 기업용 자동 QA 기능을 제공합니다.
- Weave의 미성숙한 'Annotation Queues'는 대규모 휴먼 인더루프(human-in-the-loop) 라벨링 팀을 위한 표준화된 워크플로우를 보유한 LangSmith 및 Langfuse에 대항하는 데 있어 핵심적인 영업 장애 요소로 남아 있습니다.
- Braintrust는 네이티브 Cursor IDE 통합을 출시하여 우리의 개발자 경험(DevEx) 전략을 앞질렀으며, 개발자가 Weave 대시보드에 도달하기 전인 '이너 루프(inner loop)' 워크플로우를 선점했습니다.
- Weave Playground에 Serverless LoRA 추론 통합(1월 16일)은 Arize Phoenix나 Braintrust와 같은 단독 플레이어들이 기술적으로 복제할 수 없는 고유한 '학습-추론' 플라이휠을 생성합니다.
- 조치 필요 사항: 제품 팀은 2분기에 OpenTelemetry(OTel) 호환성을 최우선 과제로 삼아야 합니다. MLflow와 Arize Phoenix는 '네이티브 OTel' 지원을 보다 안전하고 벤더 중립적인 선택지로 포지셔닝하여 기업 아키텍처 검토에서 승리하고 있습니다.

> **한 줄 평**: Weave는 멀티모달 및 학습 통합 워크플로우에서 뚜렷한 기술적 우위를 점하고 있으나, LangSmith의 인프라 종속 전략과 MLflow의 자동화된 기업용 QA 기능으로부터 실존적 위협에 직면해 있습니다.

### Weave 주요 강점

- 학습 이력(Training Lineage) 통합: Weave는 프로덕션 트레이스를 W&B 모델 아티팩트, 학습 실행(runs) 및 스윕(sweeps)과 네이티브하게 연결하여 진정한 데이터 플라이휠을 구현하는 유일한 플랫폼입니다.
- 멀티모달 평가: 최근 출시된 Audio Monitors(2026년 2월)는 음성 에이전트 빌더들에게 LangSmith 및 MLflow와 같은 텍스트 중심 경쟁사 대비 뚜렷한 이점을 제공합니다.
- 대화형 디버깅: Weave의 Playground는 정적 트레이스 보기에 집중하는 MLflow 및 Arize Phoenix에 비해 신속한 반복 작업을 위한 우수한 '편집 후 실행(edit-and-run)' 경험을 제공합니다.
- 프레임워크 불가지론(Agnosticism): Weave는 LangSmith보다 가볍고 특정 방식에 얽매이지 않아, LangChain 생태계 외부에서 커스텀 스택을 구축하는 개발자들에게 매력적입니다.

### Weave 개선 필요 사항

- 휴먼 인더루프(Human-in-the-Loop) 워크플로우: LangSmith와 Langfuse는 대규모 인력 라벨링 팀을 관리하기 위한 훨씬 성숙한 'Annotation Queues'를 제공합니다.
- 에이전트 상태 시각화: LangSmith의 LangGraph 심층 통합은 복잡한 상태 머신 및 순환 에이전트 워크플로우에 대한 우수한 시각화를 제공합니다.
- 트래픽 관리: Weave는 Braintrust가 제공하는 속도 제한(rate limiting), 캐싱 및 트래픽 제어를 위한 활성 AI Proxy/Gateway 아키텍처가 부족합니다.
- OpenTelemetry 표준화: MLflow와 Arize Phoenix는 '네이티브 OTel' 방식을 채택하여, Weave의 SDK보다 개방형 표준을 우선시하는 기업들에게 더 안전한 선택지가 되고 있습니다.

## 3. 신규 기능 (최근 30일)

### [Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: 텍스트와 함께 오디오 출력을 관찰하고 판단하는 모니터 생성 지원으로 음성 에이전트 평가 가능. (2026-02-01, 핵심 관측성)
- **Dynamic Leaderboards**: 영구적인 커스터마이징 및 CSV 내보내기 기능이 포함된 평가 결과 기반 자동 생성 리더보드. (2026-01-29, 평가 통합)
- **Playground 내 커스텀 LoRA**: 비교를 위해 Weave Playground에서 직접 커스텀 파인튜닝된 LoRA 가중치를 로드하고 테스트하는 기능. (2026-01-16, 실험 / 개선 루프)

### [LangSmith](https://changelog.langchain.com)
- **트레이스 미리보기 커스터마이징**: 빠른 디버깅을 위해 트레이스 목록 보기에서 표시할 필드를 설정하는 기능. (2026-02-06, DevEx / 통합)
- **Google Gen AI 래퍼**: OpenTelemetry 없이 Google Generative AI 모델의 네이티브 트레이싱을 지원하는 신규 SDK 래퍼. (2026-01-31, DevEx / 통합)
- **LangSmith Self-Hosted v0.13**: 성능 개선 및 신규 설정 옵션이 포함된 업데이트된 셀프 호스팅 릴리스. (2026-01-16, 엔터프라이즈 및 보안)

### [Langfuse](https://langfuse.com/changelog)
- **트레이스용 교정 출력(Corrected Outputs)**: 파인튜닝 데이터셋 구축을 위해 트레이스 보기에서 직접 개선된 버전의 LLM 출력을 캡처. (2026-01-14, 핵심 관측성)
- **Python SDK v3.14.1**: Langfuse 기능 액세스를 위한 클라이언트 라이브러리 업데이트. (2026-02-09, DevEx / 통합)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **트레이스 레벨 스코어러(Scorers)**: 커스텀 코드 스코어러가 전체 실행 트레이스에 접근하여 다단계 워크플로우 및 에이전트 동작을 평가 가능. (2026-02, 평가 통합)
- **LangSmith 통합**: LangSmith 트레이싱 및 평가 호출을 Braintrust로 라우팅하여 도구를 통합할 수 있는 래퍼 제공. (2026-02, DevEx / 통합)
- **Cursor 통합**: Braintrust MCP 서버를 자동 설정하고 자연어로 로그를 쿼리할 수 있는 Cursor 에디터용 확장 프로그램. (2026-02, DevEx / 통합)
- **자동 인스트루멘테이션 (Python/Ruby/Go)**: Python, Ruby, Go 애플리케이션에 대한 제로 코드 트레이싱 지원 추가. (2026-01, DevEx / 통합)
- **Temporal 통합**: 부모-자식 관계 매핑을 통한 Temporal 워크플로우 및 액티비티 자동 트레이싱. (2026-01, DevEx / 통합)

### [MLflow](https://mlflow.org/releases)
- **MLflow Assistant**: UI의 컨텍스트를 사용하여 문제를 진단하고 테스트를 설정하며 코드를 수정하는 Claude Code 기반 인프로덕트 챗봇. (2026-01-29, DevEx / 통합)
- **에이전트 성능 대시보드**: 설정 없이 지연 시간, 요청 수, 품질 점수를 보여주는 GenAI 실험용 사전 구축 '개요' 탭. (2026-01-29, 모니터링 및 메트릭)
- **MemAlign Judge Optimizer**: 과거 피드백으로부터 평가 가이드라인을 학습하여 LLM 평가자(judge)의 정확도를 자동으로 개선하는 알고리즘. (2026-01-29, 평가 통합)
- **Judge Builder UI**: 코드 작성 없이 커스텀 LLM 평가자를 생성, 테스트 및 검증할 수 있는 시각적 인터페이스. (2026-01-29, 평가 통합)
- **지속적 온라인 모니터링**: 유입되는 프로덕션 트레이스에 대해 LLM 평가자를 자동으로 실행하여 실시간으로 품질 문제를 감지. (2026-01-29, 모니터링 및 메트릭)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 지원**: 확장된 사고(thinking) 파라미터 지원과 함께 Playground에 Anthropic의 Claude Opus 4.6 모델 지원 추가. (2026-02-09, DevEx / 통합)
- **FaithfulnessEvaluator**: 기존 HallucinationEvaluator를 대체하여 성실성(faithfulness)을 측정하는 신규 평가자. (2026-02-02, 평가 통합)
- **도구 선택 및 호출 평가자**: 에이전트가 올바른 도구를 선택하고 유효한 파라미터로 호출했는지 평가하는 전문 평가자. (2026-01-31, 에이전트 / RAG 관측성)
- **프롬프트 및 데이터셋용 CLI**: 터미널에서 프롬프트, 데이터셋 및 실험을 관리할 수 있는 포괄적인 CLI 명령. (2026-01-22, DevEx / 통합)
- **Span 연동을 통한 트레이스-데이터셋 변환**: 소스 span에 대한 양방향 링크를 유지하면서 프로덕션 트레이스에서 데이터셋을 생성하는 기능. (2026-01-21, 평가 통합)

## 5. 엔터프라이즈 신호

- MLflow 3.9의 'Judge Builder' 및 'MemAlign' 출시는 기업 QA 자동화로의 이동을 의미하며, 수동 평가 팀의 필요성을 줄이고 있습니다.
- LangSmith가 LangGraph Cloud를 통해 배포 영역으로 확장하는 것은 전체 인프라 계층을 점유하여 벤더 종속성을 높이려는 전략을 나타냅니다.
- Langfuse가 ClickHouse 백엔드로 전환한 것은 대규모 트레이스 데이터에 대한 실시간 분석이 필요한 대용량, 비용 민감형 기업에 집중하고 있음을 보여줍니다.
- Braintrust의 새로운 Cursor 통합 및 LangSmith 래퍼는 IDE 수준에서 개발자 워크플로우를 선점하려는 공격적인 전략을 보여줍니다.
