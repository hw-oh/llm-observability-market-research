# W&B Weave — 주간 경쟁사 인텔리전스 보고서
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-11

## 1. 핵심 요약 (Executive Summary)

- 시장 통합 기회: Anthropic의 Humanloop 인수 및 서비스 종료 예정(2025년 9월)으로 인해, 프롬프트 엔지니어링 및 평가를 위한 새로운 플랫폼을 찾는 기업 고객을 확보할 수 있는 즉각적이고 우선순위가 높은 기회가 창출되었습니다.
- 'Agent Ops'로의 전환: 경쟁사들은 단순한 트레이싱을 넘어 전체 라이프사이클 관리로 공격적으로 이동하고 있습니다. LangSmith의 'Agent Servers' 및 '통합 비용 추적' 출시는 호스팅 및 재무 운영(FinOps)으로의 피벗을 의미하며, 이는 Weave가 프로덕션 배포 스토리를 정의해야 한다는 과제를 던져줍니다.
- 시각화 군비 경쟁: 순환형/그래프 기반 에이전트 워크플로우를 위한 전문 시각화 추세가 뚜렷합니다. LangSmith (LangGraph), Langfuse (Agent Graphs), Arize Phoenix는 전용 그래프 뷰로 차별화하고 있으며, 이는 복잡한 에이전트에서 선형 트레이스 리스트를 구식처럼 보이게 만듭니다.
- OpenTelemetry 및 MCP를 통한 표준화: Arize Phoenix, Langfuse, Logfire는 OpenTelemetry (OTel)와 Model Context Protocol (MCP)을 표준으로 삼아 결집하고 있습니다. 이러한 압박은 독점 SDK에 대항하는 핵심 판매 포인트로서 상호운용성과 '벤더 종속성 없음'을 강조합니다.
- 평가 성숙도: 시장은 '관리형 평가기(Managed Evaluators)'와 '어노테이션 큐(Annotation Queues)'로 이동했습니다. LangSmith와 Langfuse는 이제 인간 참여형(human-in-the-loop) 검토를 위한 전용 워크플로우를 제공하여, 평가를 단순한 개발자 유틸리티가 아닌 팀 스포츠로 포지셔닝하고 있습니다.
- 다국어 확장: Braintrust는 Java, Go, Ruby, C#에 대한 네이티브 SDK 지원을 확장하여 기업 백엔드 시장을 공격적으로 공략하고 있으며, 이는 대부분의 관측 도구가 가진 Python/JS 지배력에 도전하고 있습니다.
- 차별화 요소로서의 SQL: Logfire와 Braintrust는 트레이스에 대한 SQL 기반 쿼리 기능을 통해 파워 유저에게 권한을 부여하고 있으며, 사전 구축된 대시보드를 넘어 세밀하고 임시적인 데이터 분석 기능을 요구하는 엔지니어들의 요구를 충족시키고 있습니다.

> **한 줄 평**: Weave는 '학습에서 프로덕션까지'의 계보(lineage) 관리에서 선두를 유지하고 있으나, 에이전트 배포 측면에서는 LangSmith로부터, 개방형 표준 및 비용 가시성 측면에서는 Langfuse/Phoenix로부터 거센 압박을 받고 있습니다.

### Weave 주요 강점

- 깊이 있는 학습 계보: 프로덕션 트레이스를 학습 실행 및 모델 아티팩트(W&B 에코시스템)와 연결하는 독보적인 능력으로, 독립형 관측 도구는 이를 따라올 수 없습니다.
- 화이트박스 가시성: 프록시 기반 경쟁사(Helicone)가 API 입출력만 보는 것과 달리, 임의의 Python 코드와 내부 로직을 추적하는 뛰어난 기능을 보유하고 있습니다.
- 유연한 'Board' UI: 고도로 맞춤화 가능한 노트북 스타일의 대시보드를 통해 LangSmith나 Langfuse의 경직된 사전 정의 대시보드보다 더 깊은 탐색적 분석이 가능합니다.
- 모델 레지스트리 통합: 실제 모델 레지스트리와의 네이티브 통합을 통해 메타데이터 문자열뿐만 아니라 모델 바이너리에 대한 엄격한 버전 관리가 가능합니다.

### Weave 개선 필요 영역

- 프롬프트 관리 CMS: LangSmith (Hub), Langfuse 및 기존 Humanloop와 같은 경쟁사들은 코드 변경 없이 프롬프트를 버전 관리하고 편집할 수 있는 우수한 비기술적 UI를 제공합니다.
- 재무 운영(FinOps) 및 비용 세분화: Helicone과 Langfuse는 맞춤형 가격 책정 계층 및 예산 알림을 포함하여 프로덕션 운영에 필수적인 훨씬 더 발전된 비용 추적 기능을 제공합니다.
- 휴먼 어노테이션 워크플로우: Weave에는 인간 라벨러를 관리하기 위한 전용 '어노테이션 큐' 워크플로우가 부족하며, 이는 현재 LangSmith와 Langfuse에서 표준 기능이 되었습니다.
- 에이전트 그래프 시각화: 경쟁사들은 순환형 에이전트 워크플로우(예: LangGraph 통합)를 위한 전문 시각화 기능을 더 빠르게 시장에 내놓고 있는 반면, Weave의 트레이스 뷰는 주로 선형적입니다.

## 3. 이번 주 신규 기능

### LangSmith
- **트레이스 미리보기 맞춤 설정**: 트레이스 테이블 뷰에 표시될 입력/출력을 제어하는 구성 옵션. (2026-02-06, DevEx / 통합)
- **LangSmith Fetch CLI**: 커맨드 라인에서 직접 트레이스에 액세스하고 디버깅할 수 있는 터미널 기반 도구. (2025-12-10, DevEx / 통합)
- **Polly AI 어시스턴트**: 에이전트 디버깅 및 트레이스 분석을 위한 플랫폼 내 AI 어시스턴트 (Beta). (2025-12-10, 에이전트 / RAG 관측성)
- **통합 비용 추적**: LLM, 도구 및 검색(retrieval) 단계 전반에 걸친 풀스택 비용 모니터링. (2025-12-09, 모니터링 및 메트릭)
- **Pairwise 어노테이션 큐**: 검토자가 두 개의 에이전트 출력을 나란히 비교할 수 있는 구조화된 워크플로우. (2025-12-17, 평가 통합)

### Arize Phoenix
- **FaithfulnessEvaluator**: 더 나은 RAG 정확도 측정을 위해 HallucinationEvaluator를 대체하는 새로운 평가기. (2026-02-02, 평가 통합)
- **도구 호출 정확도 메트릭**: 에이전트 도구 호출의 성공률을 측정하기 위해 특별히 설계된 새로운 메트릭. (2026-01-27, 에이전트 / RAG 관측성)
- **Claude Opus 4.6 지원**: Playground에 최신 Claude 모델 지원 추가. (2026-02-09, 실험 / 개선 루프)
- **메트릭을 위한 Cursor 규칙**: Cursor 규칙을 사용하여 맞춤형 분류 평가기를 생성하는 기능 강화. (2026-01-21, 모니터링 및 메트릭)

### Braintrust
- **트레이스 레벨 스코어러(Scorer)**: 맞춤형 코드 스코어러가 이제 전체 실행 트레이스에 액세스하여 다단계 워크플로우와 에이전트 동작을 평가할 수 있습니다. (2026-02, 평가 통합)
- **SDK 지원 확장 (Java, Go, Ruby, C#)**: Java, Go, Ruby, C#/.NET을 위한 자동 계측(auto-instrumentation) 기능이 포함된 네이티브 SDK. (2026-01, DevEx / 통합)
- **SQL 구문 지원**: 트레이스 및 실험 쿼리를 위한 표준 SQL 구문(HAVING 및 암시적 에일리어싱 포함) 지원. (2026-01, 모니터링 및 메트릭)
- **Temporal 통합**: Temporal 워크플로우 및 액티비티의 자동 트레이싱을 통해 워커 간 분산 트레이스 캡처. (2026-01, DevEx / 통합)
- **Claude Code 및 Cursor 통합**: AI 코딩 어시스턴트가 자연어를 통해 로그 및 실험 결과를 쿼리할 수 있는 심층 통합. (2025-12, DevEx / 통합)

### Langfuse
- **트레이스 수정 출력**: 파인튜닝 데이터셋 구축을 위해 트레이스 뷰에서 직접 개선된 버전의 LLM 출력을 캡처. (2026-01-14, 실험 / 개선 루프)
- **데이터셋 아이템 버전 관리**: 데이터셋 아이템의 모든 추가, 업데이트 또는 삭제 시 자동 버전 관리. (2025-12-15, 실험 / 개선 루프)
- **호스팅형 MCP 서버**: AI 에이전트가 프롬프트를 직접 가져오고 업데이트할 수 있는 네이티브 Model Context Protocol (MCP) 서버. (2025-11-20, DevEx / 통합)
- **LLM-as-a-Judge 실행 트레이싱**: 모든 평가기 실행 시 트레이스를 생성하여 평가에 사용된 프롬프트와 토큰 사용량 조사. (2025-10-16, 평가 통합)
- **실험 러너 SDK**: 자동 트레이싱 및 동시 실행 기능을 갖춘 데이터셋 실험 실행용 고수준 SDK. (2025-09-17, 실험 / 개선 루프)

### Humanloop
- **플랫폼 서비스 종료**: Humanloop가 Anthropic에 인수되었으며 2025년 9월 8일에 서비스를 종료할 예정입니다. (2025-08, 일반)

### Logfire
- **Pytest 통합**: pytest 실행을 스팬(span)으로 기록하여 테스트 관측성을 확보하는 직접 통합. (2026-01-26, DevEx / 통합)
- **DSPy 통합**: LLM 프로그래밍 프레임워크인 DSPy를 위한 자동 계측. (2026-01-16, 프레임워크 통합)
- **MCP 지원**: 컨텍스트 교환 표준화를 위한 Model Context Protocol (MCP) 지원. (2025-03-31, 에이전트 / RAG 관측성)
- **OpenAI Agents 프레임워크**: OpenAI의 새로운 Agents SDK를 위한 계측. (2025-03-11, 에이전트 / RAG 관측성)
- **SQL 기반 메트릭**: 트레이스에 대해 임의의 SQL 쿼리를 실행하여 맞춤형 메트릭 생성 기능. (2025-09-05, 모니터링 및 메트릭)

### Helicone
- **실험(Experiments)**: 데이터셋에 대해 프롬프트를 테스트하여 출력과 비용을 비교할 수 있는 UI. (2024-10, 실험 / 개선 루프)
- **프롬프트 관리 (Assembly)**: 코드와 분리된 프롬프트 버전 관리 및 관리를 위한 강화된 UI. (2024-09, 실험 / 개선 루프)
- **세션 그룹화**: 채팅 기록 추적을 위해 여러 요청을 단일 세션으로 그룹화하는 기능. (2024-08, 핵심 관측성)

## 5. 엔터프라이즈 신호

- Humanloop 서비스 종료: Humanloop의 기업 고객들은 2025년 9월까지 강제로 마이그레이션해야 하며, 이는 특정 '교체' 영업 기회를 창출합니다.
- LangSmith 온프레미스: LangSmith는 강력한 셀프 호스팅 옵션과 SSO/RBAC 업데이트를 통해 엔터프라이즈 포지션을 공고히 하며 규제 산업을 직접 공략하고 있습니다.
- Braintrust 다국어 SDK: Braintrust의 Java 및 C# 지원 추가는 Python 기반 AI 연구 팀뿐만 아니라 전통적인 기업 백엔드 팀을 확보하려는 움직임을 나타냅니다.
- SOC2/컴플라이언스 집중: 모든 업체(Langfuse, Arize, LangSmith)에서 PII 마스킹과 감사 로그를 강조하고 있으며, 이는 '최소 가용 엔터프라이즈 기능'의 기준을 높이고 있습니다.
