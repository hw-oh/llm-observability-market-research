# W&B Weave — 주간 경쟁사 인텔리전스 보고서
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-11

[상세 비교](../comparison) · [제품 세부 정보](../competitor-detail)

## 1. 요약 보고서 (Executive Summary)

- Weave는 **Audio Monitors**(2월 1일) 출시를 통해 멀티모달 평가 분야에서 선점 효과를 구축한 반면, **LangSmith**와 **Braintrust**는 여전히 텍스트 중심에 머물러 있습니다.
- **Dynamic Leaderboards**(1월 29일)의 출시는 **MLflow**의 새로운 'Judge Builder'에 대응하여 모델 비교를 위한 우수한 자동 업데이트 시각화 기능을 제공합니다. 다만, MLflow의 'MemAlign'은 평가자(judge) 개선에 있어 더 깊은 자동화 기능을 제공합니다.
- **Weave**의 Playground 내 **Custom LoRAs** 지원(1월 16일)은 학습(W&B)과 추론을 연결하는 독보적인 위치를 강화합니다. 이는 학습 인프라가 부족한 **Langfuse**나 **Braintrust**가 모방할 수 없는 역량입니다.
- **Weave**가 백엔드 트레이싱을 개선하는 동안, **LangSmith**(LangGraph Studio)와 **Arize Phoenix**(Tool Selection Evaluators)는 에이전트 워크플로우를 위한 *시각적* 디버깅 및 특정 지표 분야에서 더 빠르게 발전하고 있습니다.
- **Braintrust**는 SDK 지원(Go, Java, C#, Ruby)을 공격적으로 확장하며, Python/JS 이외의 엔터프라이즈 환경에서 **Weave**의 채택을 위협하고 있습니다.
- **LangSmith**, **Langfuse**, **Braintrust** 모두 구조화된 휴먼 리뷰를 위한 성숙한 'Annotation Queues' 기능을 갖추고 있습니다. **Weave**는 대규모 수동 라벨링을 위한 전용 워크플로우가 아직 부족합니다.

> **한 줄 평**: Weave는 멀티모달 평가 및 학습-추론 통합 분야를 선도하고 있으나, 시각적 에이전트 디버깅에서는 LangSmith, 엔터프라이즈 언어 지원에서는 Braintrust로부터 거센 압박을 받고 있습니다.

### Weave 주요 강점

- 멀티모달 평가: Audio Monitors(MP3/WAV)에 대한 네이티브 지원을 통해 음성 에이전트 평가가 가능하며, 이는 현재 대부분의 경쟁사에 없는 기능입니다.
- 학습-추론 계보(Lineage): W&B Artifacts의 Custom LoRA 가중치를 Weave Playground로 직접 가져와 즉시 테스트할 수 있는 원활한 통합을 제공합니다.
- 프로그래밍 가능한 보고서: Dynamic Leaderboards는 평가 결과로부터 자동 생성 및 커스텀이 가능한 뷰를 제공하며, Langfuse나 Braintrust의 정적 대시보드보다 뛰어난 성능을 보입니다.
- 데이터 탐색: 'Boards'는 MLflow의 경직된 대시보드보다 강력한 애드혹(ad-hoc) 분석용 유연한 캔버스를 제공합니다.

### Weave 개선 필요 사항

- 시각적 에이전트 디버깅: LangSmith의 LangGraph Studio나 Langfuse의 Agent Graphs와 비교할 때, 에이전트 토폴로지를 위한 전용 대화형 그래프 뷰가 부족합니다.
- 구조화된 어노테이션 워크플로우: 대규모 휴먼 라벨링 팀을 관리하기 위한 전용 'Annotation Queue' 기능이 부재하며, 이는 LangSmith와 Braintrust의 핵심 강점입니다.
- SDK 생태계: Python/TypeScript로 제한되어 있는 반면, Braintrust와 MLflow는 더 넓은 엔터프라이즈 채택을 위해 Java, Go, C#을 지원합니다.
- 평가자(Judge) 최적화: 평가자 프롬프트를 자동으로 개선하는 MLflow의 MemAlign과 같은 자동 'Judge Optimizer' 기능이 부족합니다.

## 3. 신규 기능 (최근 30일)

### [Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: 텍스트와 함께 오디오 입력/출력(MP3/WAV)을 관찰하고 판단하는 모니터 생성 지원. (2026-02-01, 핵심 관측성)
- **Dynamic Leaderboards**: 새로운 실행 결과가 도착하는 즉시 업데이트되는 평가 기반 자동 생성 커스텀 리더보드. (2026-01-29, 평가 통합)
- **Custom LoRAs in Playground**: W&B Artifacts에서 미세 조정된 LoRA 가중치를 Weave Playground로 직접 로드하여 추론하는 기능. (2026-01-16, 실험 / 개선 루프)

### [LangSmith](https://changelog.langchain.com)
- **Client Library v0.7.1**: 안정성 향상 및 OIDC 지원을 위한 Python/JS 클라이언트 라이브러리 업데이트. (2026-02-10, 개발자 경험 / 통합)
- **Customize Trace Previews**: 트레이스 리스트 뷰에서 표시할 컬럼과 데이터를 사용자가 직접 설정할 수 있는 UI 업데이트. (2026-02-06, 핵심 관측성)
- **Self-Hosted v0.13**: 엔터프라이즈 배포를 위한 셀프 호스팅 인프라 구성 요소의 신규 버전. (2026-01-16, 엔터프라이즈 및 보안)

### [Langfuse](https://langfuse.com/changelog)
- **Corrected Outputs for Traces**: 미세 조정 데이터셋 구축을 위해 트레이스 뷰에서 직접 개선된 버전의 LLM 출력을 캡처. (2026-01-14, 핵심 관측성)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Trace-level Scorers**: 커스텀 코드 스코어러가 전체 실행 트레이스에 접근하여 다단계 워크플로우 및 에이전트 동작을 평가할 수 있음. (2026-02, 평가 통합)
- **Navigate to Trace Origins**: 프로덕션 트레이스에서 해당 트레이스를 생성한 프롬프트 버전이나 데이터셋 행으로 직접 이동. (2026-02, 핵심 관측성)
- **LangSmith Integration**: 트레이스를 LangSmith와 Braintrust 양쪽으로 라우팅하거나 Braintrust로 트래픽을 마이그레이션하는 래퍼(Wrapper). (2026-02, 개발자 경험 / 통합)
- **Auto-instrumentation (Python/Ruby/Go)**: Python, Ruby, Go SDK에서 주요 제공업체에 대한 코드 없는(Zero-code) 트레이싱 지원. (2026-01, 개발자 경험 / 통합)
- **Temporal Integration**: 내구성이 뛰어난 에이전트 실행을 위한 Temporal 워크플로우 및 액티비티 자동 트레이싱. (2026-01, 개발자 경험 / 통합)

### [MLflow](https://mlflow.org/releases)
- **MLflow Assistant**: 오류 디버깅 및 코드 생성을 돕는 AI 기반 인제품 챗봇(Claude Code 기반). (2026-01-29, 개발자 경험 / 통합)
- **Agent Performance Dashboards**: 지연 시간, 요청 수, 도구 사용 요약을 시각화하는 사전 구축된 '개요' 탭. (2026-01-29, 모니터링 및 지표)
- **MemAlign Judge Optimizer**: 피드백을 통해 학습하여 LLM 평가자 프롬프트를 자동으로 최적화하는 알고리즘. (2026-01-29, 평가 통합)
- **Judge Builder UI**: 코드 없이 커스텀 LLM 평가자를 생성, 테스트 및 검증할 수 있는 시각적 인터페이스. (2026-01-29, 평가 통합)
- **Continuous Online Monitoring**: 유입되는 프로덕션 트레이스에 대해 LLM 평가자를 자동으로 실행. (2026-01-29, 모니터링 및 지표)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 Support**: 자동 비용 추적 및 'thinking' 파라미터 지원을 포함한 Anthropic의 최신 모델 지원 추가. (2026-02-09, 핵심 관측성)
- **Tool Selection & Invocation Evaluators**: 에이전트가 적절한 도구를 선택했는지, 올바른 파라미터로 호출했는지 판단하는 새로운 전문 평가자. (2026-01-31, 에이전트 / RAG 관측성)
- **Configurable OAuth2 Email Extraction**: 비표준 IDP 클레임(예: Azure AD)에서 사용자 식별 정보를 추출하기 위한 커스텀 JMESPath 표현식 지원. (2026-01-28, 엔터프라이즈 및 보안)
- **CLI for Prompts & Datasets**: 터미널에서 직접 프롬프트, 데이터셋, 실험을 관리할 수 있는 포괄적인 CLI 명령어를 제공하여 AI 어시스턴트 통합 가능. (2026-01-22, 개발자 경험 / 통합)
- **Trace-to-Dataset with Span Links**: 계보 추적을 위해 소스 스팬(span)으로의 양방향 링크를 유지하면서 트레이스로부터 데이터셋을 생성하는 기능. (2026-01-21, 평가 통합)

## 5. 엔터프라이즈 신호

- Braintrust의 공격적인 SDK 확장(Java/Go/C#)은 기존 엔터프라이즈 스택을 타겟으로 합니다.
- MLflow의 'Judge Builder'와 'MemAlign'은 심도 있는 프롬프트 엔지니어링 기술이 부족한 엔터프라이즈 팀의 진입 장벽을 낮춥니다.
- Langfuse의 ClickHouse 마이그레이션은 대규모 엔터프라이즈 데이터 수집을 지원하기 위한 움직임입니다.
