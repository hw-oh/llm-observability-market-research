# W&B Weave — 주간 경쟁사 인텔리전스 보고서
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-11

## 1. 핵심 요약 (Executive Summary)

- 주요 시장 변화: Humanloop가 Anthropic에 인수됨에 따라 2025년 9월 서비스를 종료합니다. 이는 안정적인 프롬프트 관리 및 평가 대안을 찾는 기업 고객을 확보할 수 있는 Weave의 즉각적이고 우선순위가 높은 기회입니다.
- '프록시(Proxy) vs 비동기(Async)'의 대립: Braintrust 및 Helicone과 같은 경쟁사들은 비용 제어, 캐싱, 속도 제한(rate limiting)에서 우위를 점하기 위해 게이트웨이/프록시 아키텍처에 집중하고 있습니다. 반면 Weave, LangSmith, Phoenix는 개발자 편의성을 위해 비동기 로깅을 우선시하며 시장에서 뚜렷한 아키텍처적 분리를 형성하고 있습니다.
- 평가 성숙도 곡선: 선두 경쟁사(LangSmith, Braintrust)들은 단순한 'LLM-as-a-Judge'를 넘어 정교한 Human-in-the-Loop(HITL) 워크플로우로 이동하고 있으며, 수동 검토를 운영화하기 위해 전용 '어노테이션 큐(Annotation Queues)' 및 '쌍체 비교(Pairwise Comparison)' UI를 도입하고 있습니다.
- SQL 기반 관측성(Observability)의 부상: Logfire와 Braintrust(BTQL)는 트레이스(trace)를 위한 SQL 유사 쿼리 언어로 차별화하고 있으며, 경직된 대시보드보다 유연하고 코드 중심적인 분석이 필요한 파워 유저들을 공략하고 있습니다.
- 에이전트 특화 도구: 시장이 일반적인 LLM 트레이싱에서 '에이전트' 관측성으로 이동하고 있습니다. LangSmith, Langfuse, Logfire는 모두 도구 호출(tool calling), 다단계 추론 루프, 서브 에이전트 중첩을 위한 전문화된 시각화 기능을 출시했습니다.
- OpenTelemetry 표준화: Arize Phoenix, Langfuse, Logfire는 OpenTelemetry(OTel) 및 OpenInference를 표준으로 적극 추진하며, 벤더 중립적인 인스트루멘테이션(instrumentation)을 약속함으로써 독점 SDK에 도전하고 있습니다.
- 언어 생태계 확장: Braintrust는 Java, Go, Ruby, C#용 네이티브 SDK를 출시하며 언어 지원 측면에서 경쟁사들을 앞지르고 있으며, 다중 언어(polyglot) 기업 환경을 위한 최적의 선택지로 자리매김하고 있습니다.

> **한 줄 평**: Weave는 전체 라이프사이클 ML 팀을 위한 최고의 선택지로 남아 있으나, 비용 제어 측면의 프록시 기반 게이트웨이와 표준화 측면의 OTel 네이티브 도구들로부터 거센 압박을 받고 있습니다.

### Weave 주요 강점

- 전체 라이프사이클 리니지(Lineage): 프로덕션 트레이스를 모델 학습 실행 및 아티팩트와 직접 연결하는 고유한 능력(W&B 생태계 통합).
- 프로그래밍 가능한 대시보드: Weave Boards는 LangSmith나 Phoenix의 경직된 UI보다 우수한 코드 중심의 고도로 맞춤화된 시각화 경험을 제공합니다.
- 낮은 통합 마찰: 비동기 로깅 방식은 Braintrust 및 Helicone이 사용하는 프록시/게이트웨이 아키텍처의 지연 시간 및 보안 복잡성 문제를 피할 수 있습니다.
- 데이터 프라이버시: 데이터 흐름의 핵심 경로에 위치하는 프록시 미들웨어를 요구하지 않으므로 보안에 민감한 팀에 강력한 매력을 제공합니다.

### Weave 개선 필요 사항

- 게이트웨이 기능 부재: Helicone 및 Braintrust와 같은 경쟁사들은 네이티브 캐싱, 속도 제한, 비밀 키 관리와 같은 인프라 기능에서 앞서 있습니다.
- 휴먼 어노테이션 워크플로우: LangSmith와 Braintrust는 대규모 수동 어노테이션 큐 및 쌍체 비교를 위한 더 성숙하고 전용화된 UI를 제공합니다.
- 언어 지원: Braintrust와 Langfuse는 Python과 TypeScript에 집중하는 Weave에 비해 더 넓은 SDK 지원(Java, Go, C#)을 제공합니다.
- 특화된 RAG 시각화: Arize Phoenix는 깊이 있는 LlamaIndex 통합을 통해 벡터 검색 내부(임베딩 클러스터, 청크 랭킹) 시각화 분야를 선도하고 있습니다.

## 3. 이번 주 신규 기능

### LangSmith
- **LangSmith Fetch CLI**: 터미널에서 직접 트레이스를 가져오고 에이전트를 디버깅할 수 있는 CLI 도구. (2025-12, DevEx / 통합)
- **쌍체 어노테이션 큐(Pairwise Annotation Queues)**: 두 에이전트의 출력을 나란히 비교하여 승자를 선택하는 구조화된 워크플로우. (2025-12, 평가 통합)
- **통합 비용 추적**: LLM, 도구, 검색 단계 전반에 걸친 풀스택 비용 모니터링. (2025-12, 모니터링 및 메트릭)
- **Polly (Beta)**: 에이전트 디버깅 및 분석을 돕는 LangSmith 내 AI 기반 어시스턴트. (2025-12, DevEx / 통합)
- **Self-Hosted v0.13**: Insights를 포함하여 Cloud 버전과 기능적 동등성을 맞춘 업데이트된 셀프 호스팅 릴리스. (2026-01, 엔터프라이즈 및 보안)

### Arize Phoenix
- **Claude Opus 4.6 지원**: Prompt Playground에 Claude Opus 4.6 모델 지원 추가. (2026-02-09, 실험 / 개선 루프)
- **도구 선택 평가기(Tool Selection Evaluator)**: 에이전트 워크플로우에서 도구 선택의 정확도를 측정하는 새로운 평가기. (2026-02-06, 평가 통합)
- **성실성 평가기(Faithfulness Evaluator)**: 정확도 향상을 위해 기존 HallucinationEvaluator를 대체하는 FaithfulnessEvaluator 도입. (2026-02-02, 평가 통합)
- **OAuth2 이메일 추출**: OAuth2 속성을 통해 사용자 이메일을 추출하기 위한 강화된 보안 구성. (2026-01-28, 엔터프라이즈 및 보안)

### Braintrust
- **트레이스 레벨 스코어러(Trace-level Scorers)**: 커스텀 코드 스코어러가 다단계 워크플로우 평가를 위해 전체 실행 트레이스에 접근 가능. (2026-02, 평가 통합)
- **LangSmith 통합**: LangSmith 트레이스를 Braintrust로 라우팅하거나 양쪽으로 병렬 전송하는 래퍼(Wrapper). (2026-02, DevEx / 통합)
- **자동 인스트루멘테이션 (Python/Ruby/Go)**: 주요 언어에 대한 코드 수정 없는 트레이싱 지원. (2026-01, DevEx / 통합)
- **Temporal 통합**: Temporal 워크플로우 및 액티비티의 자동 트레이싱. (2026-01, DevEx / 통합)
- **Claude Code 통합**: 자연어 로그 쿼리를 위한 Anthropic의 에이전트 코딩 도구와의 통합. (2025-12, DevEx / 통합)

### Langfuse
- **트레이스 수정 출력(Corrected Outputs for Traces)**: 파인튜닝 데이터셋 구축을 위해 트레이스 뷰에서 직접 개선된 LLM 출력 버전을 캡처. (2026-01-14, 실험 / 개선 루프)
- **관측 I/O 인라인 코멘트**: 트레이스 입력/출력 내 특정 텍스트 선택 영역에 코멘트 고정. (2026-01-07, DevEx / 통합)
- **데이터셋 아이템 버전 관리**: 업데이트 또는 삭제 시 데이터셋 아이템의 자동 버전 관리. (2025-12-15, 실험 / 개선 루프)
- **호스팅된 MCP 서버**: 에이전트가 프롬프트를 직접 가져오거나 업데이트할 수 있는 네이티브 Model Context Protocol 서버. (2025-11-20, DevEx / 통합)
- **에이전트용 Langfuse**: 도구 호출 렌더링 및 에이전트 특화 평가 기능이 강화된 트레이싱. (2025-11-05, 에이전트 / RAG 관측성)

### Humanloop
- **플랫폼 서비스 종료**: Humanloop가 Anthropic에 인수되었으며 2025년 9월 8일에 플랫폼 서비스를 종료할 예정입니다. (2025-08-13, 엔터프라이즈 및 보안)
- **LLMs.txt 지원**: 문서 컨텍스트를 위한 LLMs.txt 표준 지원 추가. (2025-05-01, DevEx / 통합)

### Logfire
- **멀티 토큰 지원**: 프로젝트 마이그레이션을 용이하게 하기 위해 여러 토큰 지원 추가. (2026-02-04, 엔터프라이즈 및 보안)
- **Pytest 통합**: Pytest 실행 내에서 트레이싱 및 디버깅을 위한 네이티브 통합. (2026-01-26, DevEx / 통합)
- **DSPy 통합**: DSPy 프레임워크를 위한 공식 인스트루멘테이션 추가. (2026-01-16, 프레임워크 통합)
- **피드백 어노테이션**: 트레이스에 대한 사용자 피드백을 기록하기 위한 실험적 함수. (2025-04-11, 평가 통합)
- **OpenAI Agents 인스트루멘테이션**: 새로운 OpenAI Agents 프레임워크 트레이싱 지원. (2025-03-11, 에이전트 / RAG 관측성)

### Helicone
- **실험(Experiments)**: 플랫폼 내에서 직접 데이터셋에 대해 프롬프트를 테스트하고 평가하는 기능. (2024-05, 실험 / 개선 루프)
- **프롬프트 관리**: 코드와 분리된 프롬프트 생성, 버전 관리 및 관리를 위한 UI. (2024-01, 실험 / 개선 루프)
- **세션(Sessions)**: 여러 요청을 단일 상호작용 흐름으로 연결하는 그룹화 메커니즘. (2023-11, 핵심 관측성)

## 5. 엔터프라이즈 신호

- Humanloop의 서비스 종료로 인해 기업 고객들이 마이그레이션을 강요받고 있으며, 이는 시급한 고객 확보 기회를 창출합니다.
- LangSmith의 'Self-Hosted v0.13' 릴리스는 엄격한 기업 규정 준수를 위해 클라우드와 온프레미스 간의 기능 동등성을 확보하려는 움직임을 보여줍니다.
- Braintrust의 Java/C# SDK 확장은 Python AI 스택을 넘어 기존 기업 환경을 장악하려는 전략적 행보입니다.
- '어노테이션 큐'(LangSmith, Braintrust)의 부상은 기업들이 수동 검토 프로세스를 확장하고 있으며 이를 위한 전용 도구가 필요함을 강조합니다.
