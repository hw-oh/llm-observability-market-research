# W&B Weave — 주간 경쟁사 인텔리전스 보고서
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-11

[← 홈](../) · [상세 비교](../comparison) · [제품 상세](../competitor-detail)

## 1. 요약 (Executive Summary)

- Weave는 Audio Monitors(2026년 2월) 출시를 통해 멀티모달 관측성(observability) 분야에서 선점 효과를 확보했으며, 이는 텍스트 중심의 경쟁사인 LangSmith 및 Langfuse와 차별화되는 지점입니다.
- LangSmith는 복잡한 에이전트 워크플로우 분야에서 여전히 주요 위협으로 남아 있습니다. 특히 Weave의 현재 트레이스 트리(trace tree) 뷰가 아직 따라잡지 못한 순환 그래프(cyclic graphs) 및 상태 머신(state machines)에 대해 우수한 시각화를 제공합니다.
- Braintrust는 '데이터 플레인(Data Plane)' 아키텍처와 광범위한 SDK 지원(Java, Go, C#)을 통해 엔터프라이즈 측면을 성공적으로 공략하고 있으며, Weave의 Python/TS 중심 SDK가 놓치고 있는 백엔드 엔지니어링 팀을 포섭하고 있습니다.
- '휴먼 인 더 루프(Human-in-the-Loop)' 격차가 벌어지고 있습니다. LangSmith, Langfuse, Braintrust는 모두 전용 '어노테이션 큐(Annotation Queues)' 또는 칸반 워크플로우를 출시한 반면, Weave는 리뷰를 위해 보다 일반적인 보드/테이블 인터페이스에 의존하고 있습니다.
- MLflow(v3.9)는 'Judge Builder' 및 'MemAlign'을 통해 평가(evaluation) 격차를 공격적으로 좁혔으며, Databricks 네이티브 팀을 위한 평가 기반 개발 루프에서 Weave의 지배력을 위협하고 있습니다.
- Arize Phoenix의 특화된 '도구 선택 평가기(Tool Selection Evaluators)' 출시는 시장이 세분화된 에이전트 구성 요소 테스트로 이동하고 있음을 시사합니다. Weave는 에이전트 신뢰성 분야에서 동등한 수준을 유지하기 위해 이 영역을 우선시해야 합니다.

> **한 줄 평**: Weave는 멀티모달 지원과 학습-추론 연속성 측면에서 앞서 나가고 있으나, 에이전트 워크플로우 시각화에서는 LangSmith로부터, 엔터프라이즈 데이터 프라이버시 아키텍처에서는 Braintrust로부터 상당한 압박을 받고 있습니다.

### Weave 주요 강점

- 멀티모달 관측성: Audio Monitors 및 재생에 대한 네이티브 지원을 통해 음성 에이전트 개발 분야에서 텍스트 전용 경쟁사들보다 앞서 있습니다.
- 학습 통합: 프로덕션 트레이스를 W&B Model Registry 및 파인튜닝 작업(Serverless LoRAs)에 직접 연결할 수 있는 독보적인 능력을 갖추고 있습니다.
- 동적 리더보드: 새로운 자동 생성 평가 리더보드는 MLflow에서 요구되는 수동 설정 없이 즉각적인 모델 비교를 제공합니다.
- 데이터 탐색: 표현식(Expression) 기반 보드 시스템은 Langfuse나 Arize의 경직된 대시보드보다 더 유연한 애드혹(ad-hoc) 분석을 제공합니다.

### Weave 개선 필요 사항

- 휴먼 어노테이션 워크플로우: LangSmith와 Langfuse에서 이제 표준이 된 체계적인 휴먼 리뷰를 위한 전용 '어노테이션 큐' 또는 칸반 인터페이스가 부족합니다.
- 에이전트 그래프 시각화: LangGraph 에이전트의 순환 종속성이나 상태 머신을 LangSmith만큼 효과적으로 시각화하지 못합니다.
- SDK 언어 지원: Python/TypeScript로 제한되어 있어, 엔터프라이즈 Java/C#/Go 팀들이 Braintrust나 OpenTelemetry 기반의 경쟁사로 이탈하고 있습니다.
- 프라이버시 아키텍처: 트레이스 데이터를 고객의 클라우드에 완전히 유지하면서 SaaS UI를 사용할 수 있게 해주는 '데이터 플레인' 방식(Braintrust)이 부재합니다.

## 3. 이번 주 신규 기능

### [Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: 오디오 기능이 있는 LLM 심사위원을 사용하여 텍스트와 함께 오디오 출력을 관찰하고 판단하는 모니터 생성 지원. (2026-02-01, 핵심 관측성)
- **동적 리더보드**: 모델/평가 필터에 따라 즉시 채워지는 평가 내 자동 생성 리더보드. (2026-01-29, 평가 통합)
- **Playground 내 커스텀 LoRA**: W&B Artifacts에서 커스텀 파인튜닝된 LoRA 가중치를 Weave Playground로 직접 로드하고 테스트하는 기능. (2026-01-16, 실험 / 개선 루프)

### [LangSmith](https://changelog.langchain.com)
- **트레이스 미리보기 커스텀**: 리스트 뷰에서 트레이스 데이터가 미리 보이는 방식을 사용자가 설정할 수 있는 UI 업데이트. (2026-02-06, DevEx / 통합)
- **SDK v0.7.1**: LangSmith 관측성 및 평가 플랫폼 연결을 위한 클라이언트 라이브러리 업데이트. (2026-02-10, DevEx / 통합)

### [Langfuse](https://langfuse.com/changelog)
- **트레이스 출력 교정**: 파인튜닝 데이터셋 구축을 위해 트레이스 뷰에서 직접 개선된 버전의 LLM 출력을 캡처. (2026-01-14, 실험 / 개선 루프)
- **관찰 I/O 인라인 코멘트**: 트레이스 입력 및 출력 내의 특정 텍스트 선택 영역에 코멘트를 고정. (2026-01-07, DevEx / 통합)
- **추론/생각 렌더링**: 트레이스 내 추론 모델(예: O1, R1)의 '생각(thinking)' 부분에 대한 특화된 UI 렌더링. (2026-02-01, 핵심 관측성)
- **조직 감사 로그 뷰어**: 보안 및 컴플라이언스를 위한 조직 수준의 감사 로그를 확인하는 신규 UI. (2026-02-01, 엔터프라이즈 및 보안)
- **단일 관찰 평가**: 전체 트레이스가 아닌 개별 관찰(observation) 단위로 평가를 실행하는 기능. (2026-02-01, 평가 통합)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **트레이스 레벨 스코어러**: 커스텀 코드 스코어러가 이제 전체 실행 트레이스에 접근하여 다단계 워크플로우 및 에이전트 동작을 평가 가능. (2026-02, 평가 통합)
- **LangSmith 통합**: 트레이스를 LangSmith와 Braintrust 양쪽으로 라우팅하거나 트래픽을 완전히 마이그레이션하는 래퍼(wrapper). (2026-02, DevEx / 통합)
- **Cursor 통합**: Cursor IDE에서 직접 로그 쿼리 및 실험 가져오기를 가능하게 하는 Braintrust MCP 서버 설정 확장 프로그램. (2026-02, DevEx / 통합)
- **자동 계측 (Python/Ruby/Go)**: Python, Ruby, Go SDK에 대한 코드 수정 없는(zero-code) 트레이싱 지원 추가. (2026-01, DevEx / 통합)
- **Temporal 통합**: Temporal 워크플로우 및 액티비티의 자동 트레이싱을 통해 워커 간 분산 트레이스 캡처. (2026-01, 에이전트 / RAG 관측성)

### [MLflow](https://mlflow.org/releases)
- **MLflow Assistant**: UI 내에서 문제를 진단하고 수정을 제안하는 Claude Code 기반의 인프로덕트 챗봇. (2026-01-29, DevEx / 통합)
- **에이전트 성능 대시보드**: 에이전트 지연 시간, 요청 수, 품질 점수 모니터링을 위한 사전 구축된 차트. (2026-01-29, 모니터링 및 메트릭)
- **MemAlign 심사위원 최적화 도구**: 과거 피드백으로부터 평가 가이드라인을 학습하여 심사위원 정확도를 향상시키는 알고리즘. (2026-01-29, 평가 통합)
- **Judge Builder UI**: 코드 없이 커스텀 LLM 심사위원 프롬프트를 생성, 테스트 및 검증하는 시각적 인터페이스. (2026-01-29, 평가 통합)
- **지속적 온라인 모니터링**: 유입되는 프로덕션 트레이스에 대해 LLM 심사위원을 자동으로 실행하여 품질 문제를 실시간으로 감지. (2026-01-29, 평가 통합)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 지원**: Playground에서 정확한 비용 추적과 함께 Anthropic의 Claude Opus 4.6 모델 지원 추가. (2026-02-09, 실험 / 개선 루프)
- **도구 선택 및 호출 평가기**: 에이전트가 올바른 도구를 선택하고 유효한 파라미터로 호출했는지 평가하는 신규 특화 평가기. (2026-01-31, 에이전트 / RAG 관측성)
- **Phoenix CLI 확장**: 터미널에서 직접 프롬프트, 데이터셋, 실험을 관리하는 포괄적인 CLI 명령. (2026-01-22, DevEx / 통합)
- **스팬 링크를 포함한 트레이스-데이터셋 변환**: 계보(lineage) 추적을 위해 소스 스팬에 대한 양방향 링크를 유지하면서 트레이스에서 데이터셋을 생성하는 기능. (2026-01-21, 평가 통합)
- **트레이스와 함께 어노테이션 내보내기**: 오프라인 분석을 위해 트레이스와 함께 사람 및 AI 어노테이션을 내보내는 CLI 지원. (2026-01-19, 평가 통합)

## 5. 엔터프라이즈 신호

- Braintrust의 '데이터 플레인' 아키텍처는 UI와 데이터 스토리지를 분리함으로써 보안 검토에서 승리하고 있습니다.
- LangSmith의 'Deployment'(인프라 관리) 확장 소식은 완전한 런타임 환경이 되어 벤더 락인을 강화하려는 움직임으로 보입니다.
- MLflow의 'MemAlign' 기능은 LLM 심사위원을 인간의 정책 가이드라인에 맞게 정렬하는 과정을 자동화함으로써 엔터프라이즈 거버넌스를 타겟팅합니다.
- Langfuse의 '조직 감사 로그' 집중(2026년 2월)은 오픈 소스 대안에서도 컴플라이언스 기능에 대한 수요가 커지고 있음을 강조합니다.

## 6. 주시 사항 (Watchlist)
