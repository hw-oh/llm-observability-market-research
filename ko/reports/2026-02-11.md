# W&B Weave — 주간 경쟁사 인텔리전스 보고서
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-11

[← 홈](../) · [상세 비교](../comparison) · [제품 상세](../competitor-detail)

## 1. 요약 (Executive Summary)

- Weave의 오디오 모니터(Audio Monitors) 도입(2026년 2월)은 LangSmith 및 Langfuse와 같은 경쟁사들이 주로 텍스트/도구 중심에 머물러 있는 상황에서 뚜렷한 멀티모달 우위를 점하게 합니다.
- MLflow의 '지속적 온라인 모니터링(Continuous Online Monitoring)' 및 '저지 빌더(Judge Builder)' 출시(2026년 1월)는 프로덕션-평가 루프를 공격적으로 겨냥하고 있으며, Databricks 고객들에게 '기본' 솔루션을 제공함으로써 Weave의 자동화 역량에 도전하고 있습니다.
- LangSmith는 네이티브 LangGraph 통합을 통해 에이전트 시각화 분야를 계속해서 장악하고 있습니다. Weave의 선형 트레이스 뷰는 LangSmith의 순환 그래프 뷰와 Arize Phoenix의 새로운 도구 전용 평가 도구들로부터 압박을 받고 있습니다.
- Braintrust는 '범용' 엔터프라이즈 지원(Java, Go, C# SDK)과 깊은 IDE 통합(Cursor)을 통해 성공적으로 차별화하고 있으며, 대규모 다국어 엔지니어링 조직 내 Weave의 입지를 위협하고 있습니다.
- Langfuse의 '프롬프트 CMS'와 세분화된 비용 분석(ClickHouse 기반)은 비기술적 이해관계자와 비용에 민감한 팀을 지원하는 측면에서 Weave를 계속해서 앞서고 있습니다.
- Weave의 Playground 내 커스텀 LoRA 통합(2026년 1월)은 '학습에서 추론까지' 이어지는 고유한 해자를 강화하며, 이는 순수 관측성 벤더(LangSmith, Arize Phoenix)가 따라올 수 없는 역량입니다.

> **한 줄 평**: Weave는 멀티모달 평가 및 학습-추론 워크플로우에서 앞서 나가고 있으나, 에이전트 시각화에서는 LangSmith, 자동화된 프로덕션 모니터링에서는 MLflow로부터 거센 압박을 받고 있습니다.

### Weave 주요 강점

- 네이티브 학습 통합: 프로덕션 트레이스에서 W&B 학습 실행 및 아티팩트로 이어지는 원활한 리니지(Lineage)를 제공하며, 이는 순수 관측성 경쟁사들에게는 없는 기능입니다.
- 멀티모달 관측성: 오디오 모니터를 업계 최초로 출시하여 LangSmith와 같은 텍스트 중심 경쟁사들과 차별화됩니다.
- 서버리스 LoRA 추론: Playground에서 미세 조정된 어댑터를 즉시 교체할 수 있어 MLflow나 Langfuse가 따라올 수 없는 빠른 모델 반복이 가능합니다.
- 탐색적 분석: 'Board' 아키텍처는 Braintrust나 Arize의 정적 대시보드보다 더 유연하고 코드 중심적인 분석을 제공합니다.

### Weave 개선 필요 사항

- 에이전트 시각화: LangSmith(LangGraph)나 Arize Phoenix에서 볼 수 있는 네이티브 순환 그래프 뷰 및 상태 관리 시각화 기능이 부족합니다.
- 비기술직용 CMS: Langfuse와 Braintrust는 제품 관리자를 위해 우수한 'CMS 스타일' 프롬프트 관리 UI를 제공하는 반면, Weave는 여전히 코드 중심적입니다.
- SDK 생태계: Braintrust와 MLflow는 더 넓은 범위의 엔터프라이즈 언어(Java, C#, Go)를 지원하지만, Weave는 주로 Python/TypeScript에 집중하고 있습니다.
- 재무 운영(FinOps): Langfuse는 즉시 사용 가능한 훨씬 더 세분화된 비용 추적 및 빌링 분석을 제공합니다.

## 3. 이번 주 신규 기능

### [Weave](https://app.getbeamer.com/wandb/en)
- **오디오 모니터(Audio Monitors)**: 오디오 지원 LLM을 사용하여 텍스트와 함께 오디오 출력을 관찰하고 판단하는 모니터 생성 지원. (2026-02-01, 핵심 관측성)
- **동적 리더보드(Dynamic Leaderboards)**: 영구적인 커스터마이징 및 CSV 내보내기 기능이 포함된 평가 기반 자동 생성 리더보드. (2026-01-29, 평가 통합)
- **Playground 내 커스텀 LoRA**: W&B Artifacts에서 미세 조정된 커스텀 LoRA 가중치를 Weave Playground로 직접 로드하여 추론하는 기능. (2026-01-16, 실험 / 개선 루프)

### [LangSmith](https://changelog.langchain.com)
- **트레이스 미리보기 커스터마이징**: 대시보드에서 트레이스 미리보기가 표시되는 방식을 사용자가 설정할 수 있는 UI 업데이트. (2026-02-06, 핵심 관측성)
- **LangSmith 셀프 호스팅 v0.13**: 셀프 호스팅 엔터프라이즈 인프라 구성 요소 업데이트. (2026-01-16, 엔터프라이즈 및 보안)

### [Langfuse](https://langfuse.com/changelog)
- **트레이스용 수정 출력(Corrected Outputs)**: 미세 조정 데이터셋 구축을 위해 트레이스 뷰에서 직접 개선된 버전의 LLM 출력을 캡처. (2026-01-14, 핵심 관측성)
- **추론/생각 렌더링(Reasoning/Thinking Rendering)**: 트레이스 상세 정보에서 모델 출력의 '생각' 또는 '추론' 부분을 렌더링하는 새로운 UI 지원(v3.148). (2026-01-20, 에이전트 / RAG 관측성)
- **조직 감사 로그 뷰어**: 보안 및 컴플라이언스 가시성 강화를 위해 조직 수준의 감사 로그 뷰어 추가. (2026-01-20, 엔터프라이즈 및 보안)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **트레이스 수준 스코어러(Trace-level Scorers)**: 커스텀 코드 스코어러가 전체 실행 트레이스에 접근하여 다단계 워크플로우 및 에이전트 행동을 평가 가능. (2026-02, 평가 통합)
- **LangSmith 통합**: 트레이싱 및 평가 호출을 LangSmith와 Braintrust에 병렬로 전송하거나 Braintrust로만 라우팅하는 래퍼(Wrapper). (2026-02, 개발자 경험 / 통합)
- **Cursor 통합**: MCP를 통해 Cursor 에디터와 통합하여 자연어로 로그를 쿼리하고 실험 결과를 가져오는 기능. (2026-02, 개발자 경험 / 통합)
- **자동 인스트루멘테이션 (Py/Ruby/Go)**: Python, Ruby, Go 애플리케이션을 위한 코드 수정 없는 트레이싱 지원 추가. (2026-01, 개발자 경험 / 통합)
- **Temporal 통합**: 부모-자식 관계 매핑을 포함한 Temporal 워크플로우 및 액티비티 자동 트레이싱. (2026-01, 개발자 경험 / 통합)

### [MLflow](https://mlflow.org/releases)
- **지속적 온라인 모니터링**: 프로덕션 트레이스에 대해 LLM 저지(Judge)를 자동으로 실행하여 품질 문제를 실시간으로 감지. (2026-01-29, 평가 통합)
- **에이전트 성능 대시보드**: 에이전트 지연 시간, 요청 수, 도구 사용 요약을 모니터링하기 위한 사전 구축된 시각화 탭. (2026-01-29, 모니터링 및 지표)
- **저지 빌더(Judge Builder) UI**: 배포 전 커스텀 LLM 저지를 생성, 테스트 및 검증할 수 있는 노코드 인터페이스. (2026-01-29, 평가 통합)
- **MemAlign 저지 최적화 도구**: 과거 인간 피드백으로부터 평가 가이드라인을 학습하여 저지 정확도를 향상시키는 알고리즘. (2026-01-29, 평가 통합)
- **MLflow Assistant**: 트레이스 디버깅 및 수정 제안을 돕는 Claude Code 기반의 인제품 AI 챗봇. (2026-01-29, 개발자 경험 / 통합)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Claude Opus 4.6 지원**: 확장된 생각(thinking) 파라미터 지원 및 비용 추적 기능이 포함된 Anthropic의 최신 모델 지원 추가. (2026-02-09, 핵심 관측성)
- **도구 선택 및 호출 평가 도구**: 에이전트가 올바른 도구를 선택하고 유효한 파라미터로 호출했는지 판단하는 전문 평가 도구. (2026-01-31, 에이전트 / RAG 관측성)
- **프롬프트 및 데이터셋용 CLI**: 터미널에서 직접 프롬프트, 데이터셋, 실험을 관리할 수 있는 새로운 CLI 명령어로, AI 코딩 어시스턴트에 최적화됨. (2026-01-22, 개발자 경험 / 통합)
- **스팬 링크를 포함한 트레이스-데이터셋 변환**: 원본 소스 스팬에 대한 양방향 링크를 유지하면서 프로덕션 트레이스로부터 데이터셋을 생성하는 기능. (2026-01-21, 평가 통합)
- **트레이스와 함께 어노테이션 내보내기**: 오프라인 분석을 위해 트레이스와 함께 인간 및 LLM 어노테이션을 내보내는 CLI 지원. (2026-01-19, 평가 통합)

## 5. 엔터프라이즈 신호

- MLflow의 v3.9 업데이트(2026년 1월)는 Databricks 사용자들에게 '지속적 모니터링'을 사실상 범용화하여, 단독 도구들에 대한 기준을 높였습니다.
- Braintrust의 Cursor 및 MCP 통합은 개발의 '내부 루프(Inner Loop)'를 장악하려는 움직임으로, CI/CD의 상류 단계로 이동하고 있음을 보여줍니다.
- Langfuse의 조직 수준 감사 로그 추가(2026년 1월)는 규제가 엄격한 기업들의 도입 장벽을 제거했습니다.
- Arize Phoenix의 '로컬-투-클라우드(Local-to-Cloud)' 워크플로우는 동기화 전 오프라인에서 트레이스를 디버깅하고자 하는 보안 민감형 팀들 사이에서 인기를 얻고 있습니다.

## 6. 주시 사항

- MLflow의 '저지 빌더' UI 채택 여부를 모니터링해야 합니다. 성공할 경우 Weave의 코드 우선 평가 방식의 가치가 하락할 수 있습니다.
- LangSmith의 'Deployment' 플랫폼 진화를 주시해야 합니다. 이들은 서빙 레이어를 소유하려 하며, 이는 Weave의 관측성 도구 채택률을 위협합니다.
- 광범위한 SDK 지원(Java/C#)을 통한 Braintrust의 비 LLM 소프트웨어 팀으로의 확장을 추적해야 합니다.
- Weave 오디오 모니터의 영향력을 평가해야 합니다. 이것이 틈새 기능인지, 아니면 멀티모달 에이전트 구축자들을 위한 주요 동인인지 파악이 필요합니다.
