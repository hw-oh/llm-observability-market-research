# 주간 LLM Observability 시장 조사 보고서
**날짜**: 2026-02-23 | **모델**: google/gemini-3-pro-preview | **데이터 수집일**: 2026-02-23

## 1. Executive Summary

- MLflow는 멀티 워크스페이스 환경과 리소스 격리를 지원하기 위해 Organization Support(v2026-02-20)를 출시했습니다.
- Langfuse는 거버넌스 강화를 위해 single-span judging 기능과 Organization Audit Log Viewer(v2026-02-06)를 도입했습니다.
- W&B Weave는 보이스 에이전트를 위한 새로운 오디오 모니터와 Model Context Protocol(MCP)과의 네이티브 연동 기능을 배포했습니다.
- Braintrust는 하이브리드 배포 모델을 유지하면서 자사 Eval 플랫폼에 OpenAI Agents 지원을 추가했습니다.
- LangSmith는 복잡한 에이전트 실행 그래프를 시각화하기 위해 전체 세션 리플레이와 계층적 run tree를 제공합니다.
- Arize Phoenix는 네이티브 OpenTelemetry 지원을 통해 retrieval 청크 및 에이전트 실행 경로에 특화된 시각화 기능을 제공합니다.

> **시장 인사이트**: Weave가 새로운 오디오 모니터로 멀티모달 Tracing 분야에서 선두를 유지하고 있는 반면, Langfuse와 MLflow가 세밀한 거버넌스 기능을 출시함에 따라 엔터프라이즈 관리 기능에 대한 경쟁 압력이 거세지고 있습니다.


## 2. 신규 기능 (최근 30일)

### [W&B Weave](https://app.getbeamer.com/wandb/en)
- **Audio Monitors**: 텍스트와 함께 오디오 출력을 관찰하고 판단하는 모니터 생성 기능을 지원하여 보이스 에이전트의 평가를 가능하게 함. (2026-02-01, Guardrails & Safety)
- **Dynamic Leaderboards**: 수동 설정 대신 필터링 및 커스터마이징이 가능한 Eval 기반 리더보드 자동 생성 기능. (2026-01-29, Evaluation & Quality)

### [LangSmith](https://changelog.langchain.com)
- **Trace 미리보기 커스텀**: LangSmith UI에서 Trace 미리보기를 커스터마이징할 수 있는 기능. (2026-02-06, Core Tracing & Logging)
- **Python async용 Sandbox 엔드포인트**: Python 비동기 실행을 위한 Sandbox 엔드포인트 지원 추가. (2026-02-05, Development Lifecycle)
- **Non-otel Google ADK 래퍼**: Google ADK를 위한 비 OpenTelemetry 래퍼 추가. (2026-02-02, Integration & DX)
- **Google Gen AI 래퍼 export**: SDK에서 Google Gen AI 래퍼를 export함. (2026-01-31, Integration & DX)

### [Langfuse](https://langfuse.com/changelog)
- **Single Span Evals (Open Beta)**: 전체 Trace가 아닌 개별 Span 단위로 Eval을 실행할 수 있는 기능. (2026-02-23, Evaluation & Quality)
- **Observations 대상 LLM-as-a-Judge**: 특정 observation에 대해 직접 LLM-as-a-judge 평가자를 실행하는 기능 지원. (2026-02-23, Evaluation & Quality)
- **Reasoning/Thinking Trace 렌더링**: 모델 출력의 'thinking' 및 추론 부분을 렌더링하기 위한 UI 지원 (예: 추론 모델용). (2026-02-06, Core Tracing & Logging)
- **Org Audit Log Viewer**: 조직 수준의 감사 로그를 확인하기 위한 새로운 UI. (2026-02-06, Enterprise & Infrastructure)

### [Braintrust](https://braintrust.dev/docs/changelog)
- **Experiment Tags**: SDK를 통해 실험 생성 시 태그를 전달할 수 있는 기능. (2026-02-23, Development Lifecycle)
- **Thread Retrieval**: Python SDK에서 스레드 상세 정보를 가져오는 기능 추가. (2026-02-12, Core Tracing & Logging)
- **OpenAI Agents 연동**: OpenAI 에이전트 연동을 위해 모든 Span 유형을 처리하도록 Python SDK 업데이트. (2026-02-05, Integration & DX)
- **Review Span 유형**: 어노테이션 워크플로우 등을 위해 SDK에 특정 'review' Span 유형 추가. (2026-02-05, Core Tracing & Logging)
- **Classifications 필드**: Python SDK에 classifications 필드 추가. (2026-01-31, Core Tracing & Logging)
- **Eval 캐시 제어**: Eval 진행 중 캐싱을 끌 수 있는 옵션 추가. (2026-01-29, Evaluation & Quality)

### [MLflow](https://mlflow.org/releases)
- **Organization Support**: 실험과 리소스를 논리적으로 격리하기 위한 멀티 워크스페이스 환경 지원. (2026-02-20, Enterprise & Infrastructure)
- **MLflow Assistant**: UI에서 직접 문제를 진단하고 해결할 수 있도록 돕는 Claude Code 기반의 인프로덕트 챗봇. (2026-01-29, Integration & DX)

### [Arize Phoenix](https://arize.com/docs/phoenix/release-notes)
- **Conciseness Classification Evaluator**: 응답의 간결성을 측정하기 위한 새로운 평가자 추가. (2026-02-20, Evaluation & Quality)
- **AWS Bedrock 교차 리전 지원**: AWS Bedrock 교차 리전 추론 모델 접두사에 대한 기본 설정 지원 추가. (2026-02-19, Integration & DX)
- **Eval 프롬프트 자동 완성**: LLM Eval 프롬프트 에디터에 자동 완성 기능 추가. (2026-02-13, Evaluation & Quality)
- **Tool Response Evaluator 템플릿**: 도구 응답 처리에 특화된 새로운 평가자 템플릿 도입. (2026-02-13, Evaluation & Quality)
