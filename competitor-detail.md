# W&B Weave — 경쟁사 상세 분석
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview

### LangSmith

**개요**: LangSmith는 LangChain 생태계의 사실상 표준 LLM 엔지니어링 플랫폼으로, 단순한 추적을 넘어 에이전트 워크플로우 디버깅, 평가, 배포까지 포괄하는 올인원 솔루션으로 진화했습니다. 특히 LangGraph와의 긴밀한 통합을 통해 복잡한 에이전트 시스템의 가시성을 확보하는 데 강점이 있으며, 최근 엔터프라이즈 기능과 비용 추적 기능을 대폭 강화하고 있습니다.

**Weave 대비 강점**:
- LangChain 및 LangGraph와의 완벽한 통합으로 에이전트 워크플로우 시각화가 탁월함
- Annotation Queues 등 휴먼 피드백(Human-in-the-loop)을 위한 UI/UX가 매우 성숙함
- Prompt Hub와 연동된 프롬프트 관리 및 플레이그라운드 기능이 강력함
- 방대한 커뮤니티와 문서화를 바탕으로 초기 진입 장벽이 낮음

**Weave 대비 약점**:
- LangChain을 사용하지 않는 순수 Python/Custom 스택에서는 기능 활용도가 다소 떨어질 수 있음
- UI가 기능이 많아 복잡하게 느껴질 수 있으며, 데이터 중심(Data-centric)보다는 워크플로우 중심임
- 모델 학습(Training/Fine-tuning) 파이프라인과의 직접적인 연동은 W&B에 비해 약함
- 트레이스 양이 많아질 경우 비용 예측이 어려울 수 있음

**주요 업데이트**:
- LangSmith Fetch (CLI): 터미널에서 직접 LangSmith 트레이스를 조회하고 디버깅할 수 있는 CLI 도구 (2025-12-10)
- Pairwise Annotation Queues: 두 에이전트의 출력을 나란히 비교하여 승자를 선택하는 방식의 구조화된 평가 큐 (2025-12-17)
- Unified Cost Tracking: LLM, 도구, 검색 등을 포함한 에이전트 애플리케이션 전체 스택에 대한 통합 비용 추적 (2025-12-02)
- Polly (AI Assistant Beta): LangSmith 내에서 에이전트 디버깅과 분석을 돕는 AI 어시스턴트 (2025-12-10)

### Arize Phoenix

**개요**: Arize Phoenix는 오픈소스(OpenInference) 표준을 주도하며 로컬 개발 환경과 프로덕션 모니터링을 아우르는 강력한 LLM 옵저버빌리티 플랫폼입니다. 특히 RAG 파이프라인의 검색(Retrieval) 품질 평가와 임베딩 분석에 강점을 가지며, 최근 에이전트 도구 사용(Tool Use) 평가 기능을 대폭 강화하고 있습니다.

**Weave 대비 강점**:
- 완전한 오픈소스 및 자체 호스팅(Self-hosting) 지원으로 데이터 보안 민감 기업에 유리
- OpenInference 표준 주도로 벤더 종속성(Lock-in) 우려가 적음
- Arize의 유산인 임베딩 시각화 및 검색(Retrieval) 품질 분석 기능이 매우 성숙함
- 로컬 환경에서 도커 컨테이너 하나로 즉시 실행 가능한 가벼운 시작 경험

**Weave 대비 약점**:
- W&B의 강력한 학습(Training) 파이프라인 및 모델 레지스트리와의 연동성 부족
- Weave의 'Board'와 같은 고도로 유연하고 인터랙티브한 데이터 시각화/리포팅 기능 다소 부족
- 대규모 실험 결과에 대한 심층적인 탐색적 데이터 분석(EDA) 기능은 Weave가 더 우수함

**주요 업데이트**:
- Claude Opus 지원 (Playground): 프롬프트 플레이그라운드에 Claude Opus 4.6 모델 지원 추가 (2026-02-09)
- 도구 선택(Tool Selection) Evaluator: 에이전트가 적절한 도구를 선택했는지 평가하는 새로운 메트릭 추가 (2026-02-06)
- Faithfulness Evaluator: 기존 Hallucination 평가를 대체/개선한 신뢰성(Faithfulness) 평가 모듈 도입 (2026-02-02)
- 커스텀 메트릭 커서 규칙: LLM 분류 평가를 기반으로 새로운 내장 메트릭을 생성하는 규칙 엔진 추가 (2026-01-21)

### Braintrust

**개요**: Braintrust는 평가(Evaluation) 중심의 엔터프라이즈급 AI 옵저버빌리티 플랫폼으로, 단순한 로깅을 넘어 개발-평가-배포의 순환 루프를 강조합니다. 강력한 AI Proxy(게이트웨이) 기능을 내장하여 모델 접근을 통합 관리하며, 최근 다양한 언어(Java, Go, C# 등)의 SDK 지원과 AI 코딩 도구(Cursor, Claude Code)와의 통합을 통해 개발자 생태계를 확장하고 있습니다.

**Weave 대비 강점**:
- AI Proxy(Gateway) 기능 내장으로 모델 접근 제어 및 캐싱 용이
- 다양한 언어(Java, Go, Ruby, C#)에 대한 네이티브 SDK 지원
- 평가(Evaluation) 워크플로우의 성숙도 및 'Online Scoring' 기능
- SQL 기반(BTQL)의 강력하고 유연한 데이터 쿼리 및 분석 능력
- 자체 호스팅(Self-hosted) 옵션을 통한 데이터 주권 보장

**Weave 대비 약점**:
- Weave의 'Board'와 같은 유연한 탐색적 데이터 시각화 UI는 상대적으로 부족할 수 있음
- W&B의 기존 ML 학습(Training) 생태계와의 깊은 연동 부재 (Weave는 W&B Runs/Artifacts와 통합됨)
- 초기 설정 및 개념(Proxy, Project, Experiment 구조)이 단순 로깅 도구보다 복잡할 수 있음

**주요 업데이트**:
- Trace-level Scorers: 단일 스팬이 아닌 전체 트레이스 실행 흐름을 분석하여 다단계 워크플로우를 평가하는 기능 (2026-02)
- LangSmith Integration: LangSmith와 Braintrust로 트레이스를 동시에 전송하거나 라우팅할 수 있는 래퍼 제공 (2026-02)
- Auto-instrumentation (Python, Ruby, Go): 코드 변경 없이 주요 라이브러리를 자동으로 추적하는 기능 추가 (2026-01)
- Temporal Integration: Temporal 워크플로우 및 액티비티를 자동으로 추적하여 분산 시스템 가시성 확보 (2026-01)
- Claude Code Integration: Anthropic의 에이전트 코딩 도구인 Claude Code와의 통합 지원 (2025-12)

### Langfuse

**개요**: Langfuse는 오픈소스 기반의 LLM 엔지니어링 플랫폼으로, 상세한 트레이싱, 프롬프트 관리(CMS), 그리고 평가(Evaluation) 기능이 긴밀하게 통합되어 있습니다. 최근 ClickHouse 기반의 분석 엔진을 도입하여 대규모 데이터 처리에 강점을 보이며, 자체 호스팅(Self-hosting)이 가능해 데이터 보안을 중시하는 기업에게 강력한 대안으로 자리 잡았습니다.

**Weave 대비 강점**:
- 강력한 오픈소스 셀프 호스팅 옵션 (데이터 보안 민감 기업 선호)
- CMS 스타일의 직관적인 프롬프트 관리 및 배포 UI
- 휴먼 라벨링을 위한 전용 'Annotation Queue' 워크플로우
- ClickHouse 기반의 고성능 쿼리 및 분석 속도

**Weave 대비 약점**:
- Weave의 'Board'와 같은 유연한 탐색적 데이터 분석 UI 부족
- W&B 생태계(Sweeps, Artifacts)와의 깊은 통합 부재
- Python 코드 중심의 워크플로우보다는 UI 중심의 관리가 강조됨 (취향 차이)

**주요 업데이트**:
- Corrected Outputs (수정된 출력): 트레이스 뷰에서 LLM의 출력을 직접 수정하고 저장하여 파인튜닝 데이터셋으로 활용 가능 (2026-01)
- Inline Comments (인라인 코멘트): 트레이스 내 특정 텍스트 구간에 코멘트를 달아 협업 및 피드백 강화 (2026-01)
- Hosted MCP Server: Model Context Protocol 서버를 내장하여 에이전트가 Langfuse의 프롬프트를 직접 가져오고 업데이트 가능 (2025-11)
- Dataset Item Versioning: 데이터셋 아이템의 생성, 수정, 삭제 등 모든 변경 사항을 자동으로 버전 관리 (2025-12)

**Weave 대비 강점**:

- 직관적인 Human-in-the-loop 피드백 수집 워크플로우

### Logfire

**개요**: Pydantic 팀이 개발한 개발자 중심의 AI 및 애플리케이션 옵저버빌리티 플랫폼입니다. OpenTelemetry를 기반으로 하며, Pydantic 및 Python 생태계와의 강력한 통합, SQL 기반의 유연한 데이터 쿼리 기능을 핵심 강점으로 내세웁니다. 평가나 실험 관리보다는 프로덕션 모니터링과 디버깅에 집중하고 있습니다.

**Weave 대비 강점**:
- SQL을 사용한 극도로 유연한 데이터 쿼리 및 대시보드 구성 능력
- Pydantic 및 Python 생태계와의 완벽에 가까운 통합 (코드 한 줄로 설정)
- OpenTelemetry 표준 준수로 인한 벤더 종속성 최소화 및 호환성
- 가볍고 빠른 UI와 개발자 친화적인 CLI 도구

**Weave 대비 약점**:
- 체계적인 평가(Evaluation) 파이프라인 및 데이터셋 관리 기능 부재
- 프롬프트 엔지니어링 및 실험 관리를 위한 전용 UI(Playground, Registry) 부족
- 비개발자(PM, 도메인 전문가)를 위한 협업 기능 및 휴먼 피드백 도구 미흡
- 모델 비교 및 회귀 테스트와 같은 QA 중심 기능 부족

**주요 업데이트**:
- Pytest 통합: 테스트 실행 시 자동으로 로그를 캡처하고 실패 원인을 분석할 수 있는 Pytest 플러그인 지원 (2026-01)
- Pydantic AI 통합: Pydantic AI 프레임워크를 위한 전용 계측 및 시각화 지원 강화 (2025-03)

- SQL 기반 분석: 수집된 모든 트레이스 및 로그 데이터를 SQL로 자유롭게 쿼리하고 시각화하는 기능 (2024-05)

### Helicone

**개요**: Helicone은 오픈소스 AI 게이트웨이 및 옵저버빌리티 플랫폼으로, 애플리케이션과 LLM 공급자 사이의 프록시 역할을 수행하는 데 중점을 둡니다. Weave가 평가와 개발 워크플로우에 집중하는 반면, Helicone은 비용 추적, 캐싱, 속도 제한, 라우팅과 같은 운영 및 인프라 제어에 강점을 가집니다.

**Weave 대비 강점**:
- AI 게이트웨이 기능(캐싱, 속도 제한, 자동 재시도) 내장으로 인프라 제어 가능
- 코드 변경 없이 Base URL 변경만으로 도입 가능한 매우 낮은 진입 장벽
- 강력한 비용 분석 및 모델 간 가격 비교 도구
- 오픈소스 기반으로 완전한 자체 호스팅(Self-hosting) 가능

**Weave 대비 약점**:
- 복잡한 에이전트/RAG 워크플로우의 내부 단계(Step)에 대한 가시성 부족
- 체계적인 오프라인 평가(Evaluation) 및 데이터셋 관리 기능이 Weave보다 약함
- 리치 미디어(이미지, 오디오 등) 및 복잡한 트레이스 시각화 기능 부족
- Python/JS 외의 다양한 ML 프레임워크와의 깊은 코드 레벨 통합 부족

**주요 업데이트**:
- Requests 페이지 리디자인: 사용자 경험과 효율성을 높이기 위해 요청 로그 조회 페이지 UI 전면 개편 (2026-01)
- 비용 패키지(Cost Package) 업데이트: 300개 이상의 모델 및 10개 이상의 공급자에 대한 최신 가격 계산 로직 지원 (2026-01)
