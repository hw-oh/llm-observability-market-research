# W&B Weave — 제품 상세분석
**날짜**: 2026-02-11 | **모델**: google/gemini-3-pro-preview

### Weave

**개요**: Weave는 W&B의 강력한 MLOps 유산을 바탕으로, 모델 학습부터 LLM 애플리케이션 평가까지 이어지는 'End-to-End AI 개발'을 지원하는 개발자 중심 플랫폼입니다. 경쟁사들이 단순 모니터링이나 프록시 기능에 집중할 때, Weave는 코드 중심의 유연한 실험 추적과 데이터셋 관리를 통해 엔지니어링 루프를 가속화하는 데 핵심 가치를 둡니다.

**핵심 강점**:
- W&B Training(학습) 및 Model Registry와 연동되어 모델 개발부터 배포 후 평가까지 끊김 없는 워크플로우 제공
- 특정 프레임워크(LangChain 등)에 종속되지 않는 유연한 '코드 중심(Code-first)' 트레이싱 및 객체 버전 관리
- 노트북 환경 친화적인 인터랙티브 데이터 탐색 및 시각화(Weave Board) 경험
- 강력한 커스텀 평가 로직 구현 능력과 이를 실험 추적(Sweeps)과 연결하는 확장성

**개선 영역**:
- Braintrust나 Helicone이 제공하는 AI Gateway(Proxy) 기능 부재로 인한 비용 제어 및 인프라 레벨의 락인 효과 부족
- Langfuse나 Humanloop 대비 비개발자(PM/도메인 전문가)를 위한 CMS 스타일의 프롬프트 관리 UI 편의성 부족
- LangSmith의 'Annotation Queues'와 같은 체계적이고 고도화된 휴먼 피드백(RLHF) 워크플로우 UI 미흡
- Arize Phoenix 대비 RAG 파이프라인 특화(검색 품질, 환각 탐지 등) 사전 구축 평가 지표의 다양성 부족

### LangSmith

**개요**: LangSmith는 LangChain 생태계의 핵심 축으로, 프로토타이핑부터 배포 후 모니터링까지 LLM 애플리케이션의 전 주기를 관리하는 올인원 플랫폼입니다. 특히 복잡한 에이전트 워크플로우(LangGraph)의 디버깅과 평가에 강점이 있으며, 최근 CLI 도구와 비용 추적 기능을 강화하며 엔터프라이즈 준비성을 높이고 있습니다.

**Weave 대비 강점**:
- LangChain/LangGraph 프레임워크와의 완벽한 네이티브 통합 및 시각화
- Annotation Queues를 통한 체계적이고 사용자 친화적인 휴먼 피드백 워크플로우
- Playground 기능을 통해 트레이스에서 즉시 프롬프트를 수정하고 재실행하는 빠른 반복 루프
- 에이전트 워크플로우(Graph)의 직관적인 시각화 및 디버깅

**Weave 대비 약점**:
- LangChain을 사용하지 않는 순수 Python/타 프레임워크 프로젝트에서는 통합의 이점이 감소함
- Weave(W&B)에 비해 범용적인 ML 실험 추적보다는 LLM 애플리케이션에 특화되어 있어 유연성이 다소 낮을 수 있음
- 초기 설정 및 개념이 LangChain의 추상화에 익숙하지 않은 개발자에게는 복잡하게 느껴질 수 있음

**주요 업데이트**:
- LangSmith Self-Hosted v0.13: 클라우드 버전과의 기능 동등성을 확보하고 성능 및 보안이 강화된 셀프 호스팅 버전 (2026-01-16)
- Pairwise Annotation Queues: 두 에이전트의 출력을 나란히 비교하여 승자를 선택하는 방식의 빠른 휴먼 피드백 인터페이스 (2025-12-17)
- LangSmith Fetch (CLI): 터미널에서 직접 트레이스에 접근하고 디버깅할 수 있는 CLI 도구 (2025-12-10)
- Unified Cost Tracking: LLM, 도구, 검색 등 전체 스택에 걸친 비용을 통합하여 추적하는 기능 (2025-12-02)
- Polly (AI Assistant Beta): 에이전트 디버깅 및 분석을 돕는 플랫폼 내장 AI 어시스턴트 (2025-12-10)

### Arize Phoenix

**개요**: Arize Phoenix는 오픈소스 우선(Open-source first) 전략을 취하는 LLM 옵저버빌리티 및 평가 플랫폼으로, OpenTelemetry 및 OpenInference 표준을 기반으로 합니다. 로컬 개발 환경에서의 디버깅부터 프로덕션 모니터링까지 연결되는 워크플로우를 제공하며, 특히 RAG 파이프라인 평가와 심층적인 트레이스 분석에 강점을 보입니다.

**Weave 대비 강점**:
- OpenTelemetry/OpenInference 표준 준수로 인한 높은 호환성 및 벤더 중립성
- RAG 파이프라인(검색 품질, 환각 탐지)에 특화된 깊이 있는 사전 구축 평가기
- 완전한 자체 호스팅(Self-hosting)이 가능한 오픈소스 버전 제공 (데이터 보안 민감 기업에 유리)
- Arize의 기존 MLOps 역량을 상속받아 프로덕션 모니터링 및 드리프트 감지에 강력함

**Weave 대비 약점**:
- W&B 생태계(기존 ML 실험 추적)와의 통합 깊이는 Weave가 우세함
- UI/UX가 다소 복잡하고 엔지니어링 중심적일 수 있음 (Weave의 직관적인 노트북 경험 대비)
- 완전한 엔터프라이즈 기능을 위해서는 Arize 상용 플랫폼으로의 전환이 필요할 수 있음

**주요 업데이트**:
- Claude Opus 4.6 지원: 프롬프트 플레이그라운드에 Claude Opus 4.6 모델 지원 추가 (2026-02-09)
- 도구 선택 평가기 (Tool Selection Evaluator): 에이전트의 도구 호출 정확도를 측정하기 위한 새로운 평가 로직 추가 (2026-02-06)
- 충실도 평가기 (FaithfulnessEvaluator): 기존 HallucinationEvaluator를 대체/보완하여 RAG 응답의 충실도를 측정하는 평가기 도입 (2026-02-02)

### Braintrust

**개요**: Braintrust는 엔터프라이즈급 AI 제품 구축을 위한 통합 플랫폼으로, 평가(Evaluation), 프롬프트 관리, 그리고 AI Proxy를 통한 관측성(Observability)에 강점을 가집니다. 특히 'Instrument-Eval-Improve'의 반복 루프를 강조하며, BTQL을 통한 강력한 데이터 쿼리 및 분석 기능과 다양한 언어의 SDK를 제공합니다.

**Weave 대비 강점**:
- 강력한 프롬프트 관리 시스템(버전 관리, 배포, 테스트 통합) 내장
- AI Proxy(Gateway) 기능을 통한 캐싱, 비용 제어, 통합 연결성 제공
- BTQL(SQL-like)을 통한 매우 유연하고 깊이 있는 로그 데이터 분석
- Go, Java, C#, Ruby 등 다양한 엔터프라이즈 백엔드 언어 SDK 지원

**Weave 대비 약점**:
- W&B Training(학습) 파이프라인과의 네이티브 통합 및 모델 레지스트리 연동 부재
- Weave Board와 같은 높은 자유도의 인터랙티브 데이터 탐색/시각화 UI 부족
- 초기 설정 및 개념(Proxy, Data Plane 등)이 소규모 팀이나 개인 연구자에게는 다소 무거울 수 있음

**주요 업데이트**:
- Trace-level Scorers: 단일 스팬이 아닌 전체 트레이스 실행 흐름을 분석하여 다단계 워크플로우 및 에이전트 행동을 평가하는 기능 (2026-02)
- Auto-instrumentation (Python, Ruby, Go): 코드 변경 없이 라이브러리 호출을 자동으로 감지하여 트레이스를 생성하는 기능 확장 (2026-01)
- Temporal Integration: 내구성 있는 실행(Durable Execution) 플랫폼인 Temporal 워크플로우의 자동 트레이싱 지원 (2026-01)
- Claude Code Integration: Anthropic의 에이전트 코딩 툴인 Claude Code 세션 추적 및 자연어 로그 쿼리 지원 (2025-12)
- BTQL SQL Syntax Support: 자체 쿼리 언어인 BTQL에서 표준 SQL 문법을 지원하여 데이터 분석 편의성 증대 (2025-11)

### Langfuse

**개요**: Langfuse는 오픈소스 기반의 LLM 엔지니어링 플랫폼으로, 상세한 트레이싱, 프롬프트 관리(CMS), 그리고 평가(Evaluation) 기능을 통합하여 제공합니다. 최근 ClickHouse로 백엔드를 전환하여 대규모 데이터 처리를 강화했으며, 에이전트 워크플로우와 'Human-in-the-loop' 어노테이션 기능에서 강점을 보입니다.

**Weave 대비 강점**:

### Humanloop

**개요**: Humanloop은 Anthropic에 인수되었으며, 2025년 9월 8일부로 플랫폼 운영이 완전히 종료될 예정입니다. 기존에는 프롬프트 엔지니어링, 버전 관리, 비개발자(PM) 친화적인 협업 UI에 강점을 가진 LLM Ops 플랫폼이었으나, 현재는 신규 기능을 중단하고 데이터 마이그레이션을 안내하는 단계입니다.

**Weave 대비 강점**:
- 직관적인 프롬프트 관리 및 버전 제어 UI (비개발자 친화적)

- 로그에서 데이터셋으로 이어지는 매끄러운 워크플로우 (Human-in-the-loop 최적화)

**Weave 대비 약점**:

- Weave 대비 복잡한 에이전트/체인 트레이싱의 유연성 부족
- 코드 중심(Code-first)의 실험 추적보다는 UI 중심의 관리가 강조됨

**주요 업데이트**:
- 서비스 종료 (Sunset) 발표: Anthropic 인수로 인해 2025년 9월 8일 플랫폼 운영 종료 및 데이터 마이그레이션 안내 (2025-08-13)

### Logfire

**개요**: Pydantic 팀이 개발한 엔지니어링 중심의 옵저버빌리티 플랫폼으로, OpenTelemetry를 기반으로 구축되었습니다. Pydantic 및 Python 생태계와의 강력한 통합을 통해 LLM 애플리케이션뿐만 아니라 일반적인 백엔드(DB, API)까지 아우르는 풀스택 가시성을 제공합니다. 평가나 실험 관리보다는 프로덕션 모니터링과 디버깅 경험(DevEx)에 최적화되어 있습니다.

**Weave 대비 강점**:
- Pydantic 모델과의 완벽한 통합으로 데이터 구조 검증 및 로깅이 자연스러움
- OpenTelemetry 표준을 기반으로 하여 벤더 종속성이 낮고 확장성이 높음
- LLM뿐만 아니라 DB(SQLAlchemy), API(FastAPI) 등 전체 백엔드 스택에 대한 가시성 제공
- 설정이 매우 간편하며(Magic Auto-instrumentation), Python 개발자 친화적인 UX

**Weave 대비 약점**:
- 데이터셋 관리, 모델 비교, 휴먼 피드백 등 '평가(Evaluation)' 워크플로우 기능이 매우 부족함
- 프롬프트 엔지니어링을 위한 Playground나 버전 관리 UI가 없음
- 실험 추적(Experiment Tracking) 기능이 W&B에 비해 체계적이지 않음
- 비개발자(PM, 도메인 전문가)가 사용하기에는 UI가 다소 기술적임

**주요 업데이트**:
- Pytest 통합: 테스트 실행 결과를 Logfire로 전송하여 테스트 실패 원인 분석 강화 (2026-01)
- DSPy 통합: DSPy 프레임워크의 컴파일 및 실행 과정을 추적하는 기능 추가 (2026-01)
- OTel Gen AI Semantic Conventions: 최신 OpenTelemetry Gen AI 표준 속성 지원 강화 (2026-01)
- Pydantic AI 통합: Pydantic AI 에이전트 프레임워크에 대한 심층적인 계측 지원 (2025-03)

### Helicone

**개요**: Helicone은 오픈소스 AI Gateway 및 옵저버빌리티 플랫폼으로, 애플리케이션과 LLM 제공자 사이의 프록시 역할을 수행하며 로깅, 캐싱, 속도 제한 기능을 제공합니다. Weave가 코드 레벨의 심층적인 트레이싱과 평가(Evaluation)에 집중하는 반면, Helicone은 프로덕션 트래픽의 비용 제어, 모델 라우팅, API 레벨의 모니터링에 강점을 가집니다.

**Weave 대비 강점**:
- AI Gateway 기능(캐싱, 속도 제한, 라우팅, 폴백) 내장으로 운영 효율성 증대
- 코드 변경 없이 Base URL 변경만으로 도입 가능한 매우 낮은 진입 장벽
- 상세하고 직관적인 비용 분석 및 예산 관리 기능
- 완전한 오픈소스 및 자체 호스팅(Self-hosting) 용이성

**Weave 대비 약점**:
- 프록시 기반이라 코드 내부 로직(함수 호출, 리트리버 등)에 대한 가시성 부족
- 복잡한 RAG/에이전트 워크플로우의 시각화 및 디버깅 기능 미흡
- 체계적인 데이터셋 기반 오프라인 평가(Evaluation) 워크플로우 부족
- Python/JS 코드와 긴밀하게 결합된 실험 추적 기능 부재

**주요 업데이트**:
- Prompt Management (Prompts as Code): 프롬프트를 버전 관리하고 API를 통해 동적으로 가져올 수 있는 기능 강화 (2024-Q4)
- Sessions: 여러 요청을 하나의 논리적 세션으로 그룹화하여 대화 흐름 추적 기능 추가 (2024-Q3)
- Cost Tracking Updates: 300개 이상의 모델에 대한 최신 가격 정책 반영 및 비용 계산 로직 업데이트 (2025-01)
